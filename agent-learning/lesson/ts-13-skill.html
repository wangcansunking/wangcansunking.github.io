第13课：Skill + Tool Runner

学习目标

  1. 理解 Skill 的概念——可复用的工具单元
  2. 用 Zod 定义类型安全的工具
  3. 实现一个通用 Tool Runner，自动处理工具调用循环
  4. 组合多个 Skill 构建强大的 Agent

什么是 Skill？

Skill 是封装了具体能力的可复用工具单元，与普通 Tool 的区别：

普通 Tool：
  - 每次用完就丢弃
  - 紧耦合在某个 Agent 里
  - 无法跨项目复用

Skill：
  - 独立封装，可导入任意 Agent
  - 有明确的输入/输出类型定义
  - 像乐高积木一样组合使用

用 Zod 定义类型安全的 Skill

我们自己写一个 30 行的 defineSkill() helper，用 Zod 做类型校验，自动转 JSON Schema：

// shared/skill.ts
import { z } from "zod";
import { zodToJsonSchema } from "zod-to-json-schema";

export type Skill<I extends z.ZodObject<any>, O> = {
  name: string;
  description: string;
  inputSchema: I;
  run: (input: z.infer<I>) => Promise<O>;
};

export function defineSkill<I extends z.ZodObject<any>, O>(s: Skill<I, O>): Skill<I, O> {
  return s;
}

/** 把多个 Skill 转成 OpenAI tools 格式 */
export function skillsToTools(skills: Skill<any, any>[]) {
  return skills.map((s) => ({
    type: "function" as const,
    function: {
      name: s.name,
      description: s.description,
      parameters: zodToJsonSchema(s.inputSchema, { target: "openApi3" }) as any,
    },
  }));
}

定义 Skills

import { defineSkill } from "../shared/skill";
import { z } from "zod";

// ── Skill 1：天气查询 ──────────────────────────────────────────
const weatherSkill = defineSkill({
  name: "get_weather",
  description: "获取指定城市的实时天气信息",
  inputSchema: z.object({
    city: z.string().describe("城市名称，如 '北京'、'上海'"),
    unit: z.enum(["celsius", "fahrenheit"]).default("celsius").describe("温度单位"),
  }),
  run: async (input) => {
    // 实际项目中调用天气 API
    const temps: Record<string, number> = { 北京: 22, 上海: 28, 广州: 32 };
    const temp = temps[input.city] ?? 25;
    const display = input.unit === "fahrenheit"
      ? `${(temp * 9 / 5 + 32).toFixed(1)}°F`
      : `${temp}°C`;
    return `${input.city}：晴天，${display}，湿度 65%，微风`;
  },
});

// ── Skill 2：网页搜索 ──────────────────────────────────────────
const searchSkill = defineSkill({
  name: "web_search",
  description: "搜索互联网获取最新信息",
  inputSchema: z.object({
    query: z.string().describe("搜索关键词"),
    maxResults: z.number().default(5).describe("返回结果数量"),
  }),
  run: async (input) => {
    return `搜索"${input.query}"的结果：
1. [示例结果1] - 相关描述...
2. [示例结果2] - 相关描述...
（共返回 ${input.maxResults} 条）`;
  },
});

// ── Skill 3：数据计算 ──────────────────────────────────────────
const calculatorSkill = defineSkill({
  name: "calculate",
  description: "执行数学计算",
  inputSchema: z.object({
    expression: z.string().describe("数学表达式，如 '(15 + 27) * 2'"),
  }),
  run: async (input) => {
    try {
      const result = Function(`"use strict"; return (${input.expression})`)();
      return `${input.expression} = ${result}`;
    } catch {
      return `计算错误：${input.expression} 不是有效表达式`;
    }
  },
});

// ── Skill 4：批量数据收集 ─────────────────────────────────────
const dataCollectorSkill = defineSkill({
  name: "collect_city_data",
  description: "并行收集多个城市的天气信息",
  inputSchema: z.object({
    cities: z.array(z.string()).describe("城市列表"),
  }),
  run: async (input) => {
    const results = await Promise.all(
      input.cities.map(async (city) => `${city}：${Math.floor(20 + Math.random() * 15)}°C`)
    );
    return results.join("\n");
  },
});

Tool Runner：自动处理调用循环

第 6 课的 Agent Loop 我们手写了 30 行循环。把它抽成一个通用 Runner，所有 Agent 都能复用：

// shared/toolRunner.ts
import OpenAI from "openai";
import { client, model } from "./llmClient";
import type { Skill } from "./skill";
import { skillsToTools } from "./skill";

type Msg = OpenAI.ChatCompletionMessageParam;

export async function runWithSkills(opts: {
  messages: Msg[];
  skills: Skill<any, any>[];
  maxIterations?: number;
  onIteration?: (msg: OpenAI.ChatCompletionMessage, iteration: number) => void;
}): Promise<string> {
  const { messages, skills, maxIterations = 10, onIteration } = opts;
  const skillByName = new Map(skills.map((s) => [s.name, s]));
  const tools = skillsToTools(skills);

  for (let i = 0; i < maxIterations; i++) {
    const resp = await client.chat.completions.create({
      model,
      max_tokens: 4096,
      tools,
      messages,
    });

    const msg = resp.choices[0].message;
    messages.push(msg);
    onIteration?.(msg, i);

    // 没调工具 → 任务完成
    if (resp.choices[0].finish_reason !== "tool_calls" || !msg.tool_calls?.length) {
      return msg.content ?? "";
    }

    // 并行执行所有工具
    const results = await Promise.all(
      msg.tool_calls.map(async (call) => {
        const skill = skillByName.get(call.function.name);
        if (!skill) return { id: call.id, content: `Unknown skill: ${call.function.name}` };
        try {
          const args = skill.inputSchema.parse(JSON.parse(call.function.arguments));
          const out = await skill.run(args);
          return { id: call.id, content: typeof out === "string" ? out : JSON.stringify(out) };
        } catch (e: any) {
          return { id: call.id, content: `Skill error: ${e.message}` };
        }
      })
    );

    for (const r of results) {
      messages.push({ role: "tool", tool_call_id: r.id, content: r.content });
    }
  }

  return "达到最大迭代次数";
}

用法：3 行就跑起来

import { runWithSkills } from "../shared/toolRunner";

const answer = await runWithSkills({
  skills: [weatherSkill, searchSkill, calculatorSkill, dataCollectorSkill],
  messages: [
    { role: "user", content: "北京和上海今天天气怎样？另外 (88+12)*3 等于多少？" },
  ],
});

console.log(answer);

关于 Anthropic 原生 betaZodTool

如果你切到 Anthropic SDK，可以直接用官方 betaZodTool + toolRunner，省去自己实现：

import Anthropic from "@anthropic-ai/sdk";
import { betaZodTool } from "@anthropic-ai/sdk/helpers/beta/zod";

const ant = new Anthropic();
const skill = betaZodTool({ name, description, inputSchema, run });

// 一行跑完整循环
const final = await ant.beta.messages.toolRunner({
  model: "claude-opus-4-7",
  max_tokens: 4096,
  tools: [skill, /* ... */],
  messages: [{ role: "user", content: "..." }],
});

差异：

通用版（本课） Anthropic 原生
defineSkill() 自己实现 betaZodTool() SDK 提供
runWithSkills() 30 行 client.beta.messages.toolRunner()
跑 DeepSeek / Kimi / 任意 OpenAI 兼容 仅 Anthropic
