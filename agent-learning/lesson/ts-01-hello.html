第1课：Hello Claude

学习目标

  1. 理解 messages 的结构（role + content）
  2. 掌握 OpenAI SDK 的基本用法（DeepSeek / Kimi / Anthropic 三家通用）
  3. 读懂响应结构（choices、finish_reason、usage）
  4. 理解 TypeScript 类型守卫在 API 响应中的作用

技术背景

主流 LLM 都对外暴露无状态的 HTTP 端点 POST /v1/chat/completions。无论你用什么框架，底层都是往这个地址发 JSON、收 JSON。

你的代码
  └── OpenAI SDK
        └── POST <baseURL>/chat/completions
              └── DeepSeek / Kimi / Anthropic / ...

为什么用 OpenAI SDK？因为 DeepSeek、Kimi、Anthropic 都提供 OpenAI 兼容协议——一套代码切换 baseURL 就能跑任意一家。本教程默认用 DeepSeek：

Provider 模型 特点
DeepSeek（推荐） deepseek-chat 最便宜，1 元/百万 token，国内 5 分钟拿 key
DeepSeek deepseek-reasoner 推理（R1），思考链可见
Kimi (Moonshot) moonshot-v1-32k 128K 上下文，文档分析最强
Anthropic claude-opus-4-7 行业最强 Tool Use，需海外信用卡

第一步：注册 + 拿 API Key

打开 platform.deepseek.com，手机号/微信登录，到 API keys 页面创建一个 key（sk-... 开头）。

想换 Kimi？去 platform.moonshot.cn 拿 key。 想用 Anthropic？去 console.anthropic.com。 三家都跟着本教程跑，唯一差异是环境变量与默认模型，下面有切换说明。

第二步：安装 SDK

npm install openai

设置环境变量：

export DEEPSEEK_API_KEY="sk-xxxxxxxxxxxx"

第三步：发送第一个请求

import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://api.deepseek.com/v1",
  apiKey: process.env.DEEPSEEK_API_KEY,
});

const response = await client.chat.completions.create({
  model: "deepseek-chat",
  max_tokens: 1024,            // 控制最大输出长度
  messages: [
    {
      role: "user",            // 角色：system | user | assistant | tool
      content: "用一句话解释什么是 AI Agent？",
    },
  ],
});

max_tokens 为什么重要？ 防止模型无限输出导致费用失控。通常设 1024~4096 即可，复杂任务设 16000。

第四步：读取响应

OpenAI 兼容协议下读取很直观：

const message = response.choices[0].message;
console.log("回答:", message.content);

为什么是 choices 数组？

理论上 n 参数可以让模型一次返回多个候选回答。99% 的场景只用第一条 choices[0]。

// 完整可能的响应结构
response = {
  choices: [
    {
      index: 0,
      message: {
        role: "assistant",
        content: "AI Agent 是...",
        tool_calls: [...],     // 工具调用（第4课）
        reasoning_content: ... // R1 模型才有的思考过程
      },
      finish_reason: "stop",   // stop | length | tool_calls | content_filter
    },
  ],
  usage: { prompt_tokens: 20, completion_tokens: 35, total_tokens: 55 },
};

第五步：读取响应元数据

console.log(`finish_reason: ${response.choices[0].finish_reason}`);
// stop          → 正常完成
// length        → 超出 max_tokens，需要增大或改用流式
// tool_calls    → 模型要调用工具（第4课）
// content_filter → 触发安全过滤

console.log(`输入 tokens: ${response.usage?.prompt_tokens}`);
console.log(`输出 tokens: ${response.usage?.completion_tokens}`);

完整代码

import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://api.deepseek.com/v1",
  apiKey: process.env.DEEPSEEK_API_KEY,
});

const response = await client.chat.completions.create({
  model: "deepseek-chat",
  max_tokens: 1024,
  messages: [
    { role: "user", content: "用一句话解释什么是 AI Agent？" },
  ],
});

console.log("回答:", response.choices[0].message.content);
console.log(`finish_reason: ${response.choices[0].finish_reason}`);
console.log(`tokens: ${response.usage?.prompt_tokens} in / ${response.usage?.completion_tokens} out`);

跑一下：

npx tsx 01_hello_claude.ts

切换 Provider — 一行环境变量

本仓库 typescript/shared/llmClient.ts 提供了三 provider 适配层。所有后续课程都直接 import { client, model } from '../shared/llmClient'，环境变量切换：

# 默认：DeepSeek
export LLM_PROVIDER=deepseek
export DEEPSEEK_API_KEY=sk-xxxxxxxxxxxx

# 切到 Kimi
export LLM_PROVIDER=kimi
export MOONSHOT_API_KEY=sk-xxxxxxxxxxxx

# 切到 Anthropic
export LLM_PROVIDER=anthropic
export ANTHROPIC_API_KEY=sk-ant-xxxxxxxx

代码本身不变：

import { client, model } from "../shared/llmClient";

const response = await client.chat.completions.create({
  model,
  max_tokens: 1024,
  messages: [{ role: "user", content: "..." }],
});

响应结构速查

{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "model": "deepseek-chat",
  "choices": [
    {
      "index": 0,
      "message": { "role": "assistant", "content": "AI Agent 是能自主感知环境..." },
      "finish_reason": "stop"
    }
  ],
  "usage": { "prompt_tokens": 20, "completion_tokens": 35, "total_tokens": 55 }
}

小结

概念 要点
messages 数组，每条有 role（system/user/assistant/tool）和 content
choices[0].message.content 模型回复正文
finish_reason stop = 正常结束；tool_calls = 需要执行工具（第4课）
usage 记录 token 消耗，用于计费和监控
baseURL 切换 一套 SDK 跑 DeepSeek / Kimi / Anthropic
