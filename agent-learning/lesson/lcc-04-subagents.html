第4课：Subagents — 干净上下文隔离子任务

介绍

长时间运行的 Agent 会积累大量对话历史，导致模型注意力分散、效率下降。Subagent 模式让父 Agent 保持高层规划，将具体子任务分派给拥有空白上下文的子 Agent 执行。子任务完成后，只有最终摘要回到父 Agent——父 Agent 的历史保持简洁。

源码参考：github.com/shareAI-lab/learn-claude-code — agents/s04_subagent.py

遇到什么问题

一个 Agent 处理"重构代码 + 写测试 + 更新文档"时：

[Round 1]  读取源文件 (2000 tokens)
[Round 5]  修改代码 (3000 tokens 累积)
[Round 10] 写测试 (5000 tokens 历史)
[Round 15] 更新文档 (7000 tokens — 模型开始"遗忘"早期决策)
flowchart TB
    subgraph Parent["父 Agent（保持简洁）"]
        P1[接收任务] --> P2[todo_write 规划]
        P2 --> P3[dispatch task: 重构代码]
        P3 --> P4[dispatch task: 写测试]
        P4 --> P5[dispatch task: 更新文档]
        P5 --> P6[汇总完成]
    end
    subgraph Child1["子 Agent 1（空白历史）"]
        C1[只做：重构代码] --> C2[返回摘要]
    end
    subgraph Child2["子 Agent 2（空白历史）"]
        C3[只做：写测试] --> C4[返回摘要]
    end
    P3 --> C1
    C2 --> P4
    P4 --> C3
    C4 --> P5

有哪些方案

方案 A：单 Agent 处理所有任务

✅ 实现简单 ❌ 上下文爆炸，长任务质量严重下降

方案 B：多进程子 Agent

每个子任务在独立进程中运行，通过文件系统通信。

✅ 进程级隔离，天然获得上下文隔离 ✅ 共享文件系统，子 Agent 完成的工作立即可见 ❌ 实现稍复杂

方案 C：函数调用子 Agent（推荐）

父 Agent 通过 task 工具调用子 Agent 函数，子 Agent 在同进程内以新的 messages=[] 运行。

✅ 实现简单（无需多进程） ✅ 上下文完全隔离（新的 messages 列表） ✅ 只有最终文本摘要回到父 Agent

怎么选择

方案 C。同进程函数调用是最简单的实现方式。关键在于：子 Agent 使用新的 messages = [] 启动，父 Agent 的历史不传入子 Agent，子 Agent 的历史也不保存到父 Agent。

怎么执行

子 Agent 实现

# agents/s04_subagent.py

# 子 Agent 只有文件操作工具，没有 task 工具（防止递归）
CHILD_TOOLS = [bash_tool, read_tool, write_tool, edit_tool]
CHILD_HANDLERS = {
    "bash": lambda **kw: run_bash(kw["command"]),
    "read_file":  lambda **kw: run_read(kw["path"]),
    "write_file": lambda **kw: run_write(kw["path"], kw["content"]),
    "edit_file":  lambda **kw: run_edit(kw["path"], kw["old_text"], kw["new_text"]),
}


def run_subagent(task_description: str) -> str:
    """
    以空白历史运行子 Agent，返回最终文本摘要。
    父 Agent 永远不会看到子 Agent 的中间步骤。
    """
    messages = []  # ← 关键：空白历史，与父 Agent 完全隔离
    system = f"""You are a focused coding agent at {WORKDIR}.
Complete ONLY this task: {task_description}
Use tools to act. When done, provide a brief summary of what you did."""

    max_iterations = 30  # 安全限制：防止子 Agent 无限循环
    for _ in range(max_iterations):
        response = client.messages.create(
            model=MODEL, system=system,
            messages=messages, tools=CHILD_TOOLS, max_tokens=8000,
        )
        messages.append({"role": "assistant", "content": response.content})

        if response.stop_reason != "tool_use":
            # 返回最终文本回复作为摘要
            for block in response.content:
                if hasattr(block, "text"):
                    return block.text
            return "Task completed (no summary)"

        results = []
        for block in response.content:
            if block.type == "tool_use":
                handler = CHILD_HANDLERS.get(block.name)
                output = handler(**block.input) if handler else f"Unknown: {block.name}"
                print(f"  [subagent] {block.name}: {output[:100]}")
                results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": output,
                })
        messages.append({"role": "user", "content": results})

    return "Task incomplete: reached iteration limit"

父 Agent：Task 工具

def dispatch_task(description: str) -> str:
    """父 Agent 调用此工具，触发子 Agent 执行"""
    print(f"\n\033[35m[Spawning subagent] {description[:80]}\033[0m")
    result = run_subagent(description)
    print(f"\033[35m[Subagent done] {result[:100]}\033[0m\n")
    return result


# 父 Agent 拥有所有工具 + task 工具
PARENT_TOOLS = [bash_tool, read_tool, write_tool, edit_tool, {
    "name": "task",
    "description": "Delegate a self-contained subtask to a subagent with fresh context.",
    "input_schema": {
        "type": "object",
        "properties": {
            "description": {
                "type": "string",
                "description": "Complete, self-contained description of what the subagent should do",
            }
        },
        "required": ["description"],
    },
}]

PARENT_HANDLERS = {
    **CHILD_HANDLERS,
    "task": lambda **kw: dispatch_task(kw["description"]),
}

父 Agent Loop

def parent_agent_loop(messages: list):
    """父 Agent 只负责规划和分派，不直接执行具体工作"""
    while True:
        response = client.messages.create(
            model=MODEL,
            system=f"You are an orchestrator agent at {WORKDIR}. "
                   "Break complex tasks into subtasks and delegate via task().",
            messages=messages,
            tools=PARENT_TOOLS,
            max_tokens=8000,
        )
        messages.append({"role": "assistant", "content": response.content})
        if response.stop_reason != "tool_use":
            return
        results = []
        for block in response.content:
            if block.type == "tool_use":
                handler = PARENT_HANDLERS.get(block.name)
                output = handler(**block.input) if handler else f"Unknown: {block.name}"
                results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": output,
                })
        messages.append({"role": "user", "content": results})
