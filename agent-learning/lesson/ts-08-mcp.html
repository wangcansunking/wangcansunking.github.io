第8课：MCP 集成

学习目标

  1. 理解 MCP（Model Context Protocol）是什么
  2. 掌握三种接入方式：API 直连、Agent SDK、自建 Server
  3. 了解 MCP 的核心优势

什么是 MCP？

MCP（Model Context Protocol）是 Anthropic 推出的开放协议，让 AI 模型能够以标准化方式访问外部工具和资源。

没有 MCP：每个应用自己实现工具  → 重复开发，无法复用
有了 MCP：工具按标准发布        → 任何 AI 应用都能复用

MCP 生态：GitHub、Slack、数据库、浏览器自动化……数千个现成的 MCP Server 可直接使用。

你的 Agent
    ↓ MCP 协议
MCP Server（工具库）
    ↓
外部服务（GitHub API / 数据库 / 浏览器...）

国内用户须知：MCP 协议本身与 LLM provider 无关——MCP Server 暴露的是 JSON-RPC 接口。本课的"方式一"（Anthropic API 内置 mcp_server tool 类型）是 Anthropic 专属。但方式二（Agent SDK）和方式三（自建）都能让 DeepSeek / Kimi 用上 MCP——你只需要在自己的代码里：

  1. 启动 MCP Server（npx @playwright/mcp@latest）
  2. 用 MCP client SDK 拿到 server 暴露的工具列表
  3. 把工具列表转成第 4-6 课的 OpenAI 兼容 function calling 格式
  4. 收到 tool_calls 后转发给 MCP Server 执行

末尾有"DeepSeek/Kimi 桥接 MCP" 完整代码。

方式一：Anthropic API 直连 MCP Server（Anthropic 专属）

适合生产环境，Claude API 直接连接 MCP 服务器。仅 Anthropic 原生 SDK 支持，DeepSeek/Kimi 走方式二或方式三：

import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

// claude-opus-4-7 支持直接连接 MCP Server
const response = await (client.beta.messages.create as Function)({
  model: "claude-opus-4-7",
  max_tokens: 4096,
  tools: [
    // 声明 MCP Server（远程 SSE 方式）
    {
      type: "mcp_server",
      server_url: "https://mcp.example.com/sse",
      server_name: "filesystem",
      allowed_tools: ["read_file", "write_file", "list_directory"],
    },
  ],
  messages: [
    { role: "user", content: "读取 ./src 目录下的所有 TypeScript 文件" },
  ],
  betas: ["mcp-client-2025-11-20"],
});

方式二：Agent SDK（最简单）

Claude Code Agent SDK 内置 MCP 支持，配置即用：

import { query } from "@anthropic-ai/claude-agent-sdk";

// 使用 Playwright MCP 进行浏览器自动化
for await (const message of query({
  prompt: "打开 https://example.com，截图并总结页面内容",
  options: {
    mcpServers: {
      playwright: {
        command: "npx",
        args: ["@playwright/mcp@latest"],
      },
    },
  },
})) {
  if ("result" in message) {
    console.log(message.result);
  }
}

常用 MCP Server 列表：

// 浏览器自动化
{ command: "npx", args: ["@playwright/mcp@latest"] }

// 文件系统
{ command: "npx", args: ["-y", "@modelcontextprotocol/server-filesystem", "/path"] }

// PostgreSQL 数据库
{ command: "npx", args: ["-y", "@modelcontextprotocol/server-postgres"],
  env: { DATABASE_URL: "postgresql://..." } }

// GitHub
{ command: "npx", args: ["-y", "@modelcontextprotocol/server-github"],
  env: { GITHUB_TOKEN: "ghp_..." } }

方式三：自建 MCP Server

当你有私有 API 或内部服务时，自己实现 MCP Server：

import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import {
  CallToolRequestSchema,
  ListToolsRequestSchema,
} from "@modelcontextprotocol/sdk/types.js";

// 创建 MCP Server
const server = new Server(
  { name: "my-tools", version: "1.0.0" },
  { capabilities: { tools: {} } }
);

// 声明可用工具
server.setRequestHandler(ListToolsRequestSchema, async () => ({
  tools: [
    {
      name: "query_database",
      description: "查询公司内部数据库",
      inputSchema: {
        type: "object",
        properties: {
          sql: { type: "string", description: "SQL 查询语句" },
        },
        required: ["sql"],
      },
    },
  ],
}));

// 实现工具逻辑
server.setRequestHandler(CallToolRequestSchema, async (request) => {
  const { name, arguments: args } = request.params;

  if (name === "query_database") {
    // 实际连接数据库执行查询
    const result = await executeSQL(args.sql as string);
    return {
      content: [{ type: "text", text: JSON.stringify(result) }],
    };
  }

  return { content: [{ type: "text", text: "未知工具" }], isError: true };
});

// 启动 Server（通过标准输入输出通信）
const transport = new StdioServerTransport();
await server.connect(transport);

使用自建 Server：

// 在 Agent SDK 中引用自建 Server

  options: {
    mcpServers: {
      myDB: {
        command: "node",
        args: ["./my-mcp-server.js"],
      },
    },
  },

MCP vs 手动工具定义

对比项 手动 Tool Use MCP
工具定义 每个应用自己写 一次定义，到处复用
工具生态 自己实现 数千个现成 Server
适用场景 内部私有工具 通用工具（文件/数据库/浏览器）
协议标准 无 开放标准

小结

简单场景   → Agent SDK + 现成 MCP Server（npx 一行命令）
生产环境   → API 直连远程 MCP Server
内部工具   → 自建 MCP Server

DeepSeek / Kimi 桥接 MCP（兼容方案）

把 MCP Server 暴露的工具列表转成 OpenAI 兼容 function calling 即可：

import { client, model } from "../shared/llmClient";
import { Client as MCPClient } from "@modelcontextprotocol/sdk/client/index.js";
import { StdioClientTransport } from "@modelcontextprotocol/sdk/client/stdio.js";

// 1. 连接 MCP Server（这里以 Playwright 为例）
const transport = new StdioClientTransport({
  command: "npx",
  args: ["@playwright/mcp@latest"],
});
const mcp = new MCPClient({ name: "bridge", version: "1.0.0" }, { capabilities: {} });
await mcp.connect(transport);

// 2. 拿 MCP Server 的工具列表，转成 OpenAI function calling 格式
const { tools: mcpTools } = await mcp.listTools();
const tools = mcpTools.map(t => ({
  type: "function" as const,
  function: {
    name: t.name,
    description: t.description ?? "",
    parameters: t.inputSchema,
  },
}));

// 3. 跑 Agent Loop，遇到 tool_calls 就转发给 MCP Server
const messages: any[] = [
  { role: "user", content: "打开 example.com 并截图" },
];

for (let i = 0; i < 10; i++) {
  const resp = await client.chat.completions.create({ model, max_tokens: 4096, messages, tools });
  const msg = resp.choices[0].message;
  messages.push(msg);

  if (resp.choices[0].finish_reason !== "tool_calls" || !msg.tool_calls?.length) {
    console.log(msg.content);
    break;
  }

  // 调用 MCP Server 执行工具
  for (const call of msg.tool_calls) {
    const args = JSON.parse(call.function.arguments);
    const result = await mcp.callTool({ name: call.function.name, arguments: args });
    messages.push({
      role: "tool",
      tool_call_id: call.id,
      content: JSON.stringify(result.content),
    });
  }
}

await mcp.close();

这段代码就是一个通用 MCP-to-OpenAI-API 桥接器，让任何兼容 OpenAI 协议的 provider 都能接入 MCP 生态。
