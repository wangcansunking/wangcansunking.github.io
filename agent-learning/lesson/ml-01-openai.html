第1课：OpenAI API — GPT-4o + Tool Use Agent

介绍

OpenAI 是目前最广泛使用的 LLM 平台，GPT-4o 是其旗舰模型，支持文本、图像、音频的多模态输入。本课将从零开始掌握 OpenAI SDK 的核心用法：消息 API、流式输出、函数调用（Function Calling / Tool Use），并构建一个完整的工具调用 Agent 循环。学完本课你将理解 OpenAI SDK 与 Anthropic SDK 的异同，为后续多模型统一接口打下基础。

整体架构：本课知识地图

遇到什么问题

你已经学会了用 Anthropic SDK 构建 Agent，现在需要接入 OpenAI GPT-4o。直接上手时会遇到以下困惑：

// ❌ 把 Anthropic 的用法照搬到 OpenAI，会出错
const response = await anthropic.messages.create({
  model: "claude-opus-4-6",
  max_tokens: 1024,
  messages: [{ role: "user", content: "Hello" }]
});
// OpenAI 的字段名不同！response.content[0].text 在 OpenAI 里是 response.choices[0].message.content

主要差异点：

方面 Anthropic OpenAI
响应路径 response.content[0].text response.choices[0].message.content
Tool 定义 input_schema 字段 parameters 字段
Tool 结果 tool_result type tool role message
System prompt 独立 system 参数 messages 数组中 role=system
Token 统计 response.usage.input_tokens response.usage.prompt_tokens

请求-响应流对比（时序图）：

sequenceDiagram
    participant App as 你的应用
    participant OAI as OpenAI API
    participant ANT as Anthropic API

    Note over App,OAI: OpenAI 调用方式
    App->>OAI: POST /chat/completions<br/>{messages: [{role:"system",...},{role:"user",...}]}
    OAI-->>App: {choices:[{message:{content:"..."}, finish_reason:"stop"}]}

    Note over App,ANT: Anthropic 调用方式
    App->>ANT: POST /messages<br/>{system:"...", messages:[{role:"user",...}]}
    ANT-->>App: {content:[{type:"text", text:"..."}], stop_reason:"end_turn"}

关键差异：OpenAI 把 system 放在 messages 数组里，响应包在 choices[0] 中；Anthropic 的 system 是顶层参数，响应在 content 数组中。

有哪些方案

方案 A：直接使用 OpenAI SDK

import OpenAI from "openai";

const client = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

const response = await client.chat.completions.create({
  model: "gpt-4o",
  messages: [
    { role: "system", content: "You are a helpful assistant." },
    { role: "user", content: "What is 2+2?" }
  ]
});

console.log(response.choices[0].message.content);

✅ 原生支持，类型完整 ✅ 所有新功能第一时间可用 ❌ 每个模型 SDK 不同，多模型时需维护多套代码

方案 B：使用 LangChain 抽象层

import { ChatOpenAI } from "@langchain/openai";

const model = new ChatOpenAI({ model: "gpt-4o" });
const response = await model.invoke("What is 2+2?");

✅ 统一接口，可切换模型 ❌ 引入大量依赖，调试困难 ❌ 抽象层掩盖底层行为，学习成本高

方案 C：自己封装统一接口（推荐学习路径）

先掌握各 SDK 原生用法，后续课程中自己封装统一接口。

✅ 深刻理解各 SDK 差异 ✅ 封装符合自己需求 ❌ 需要投入时间设计

怎么选择

本课使用方案 A：直接用 OpenAI SDK 原生 API，彻底理解其行为模式。第6课我们会手动设计统一接口。

核心概念详解

消息格式设计

OpenAI 的消息以数组形式传入，每条消息都有 role 和 content。理解消息结构是构建 Agent 的基础。

flowchart TD
    A["messages 数组"] --> B["role: system\n（可选，设定角色/规则）"]
    A --> C["role: user\n（用户输入）"]
    A --> D["role: assistant\n（模型上一轮回复）"]
    A --> E["role: tool\n（工具执行结果）"]

    B --> F["放在数组第一位\n定义模型行为准则"]
    C --> G["每轮用户问题\n内容可以是字符串或数组"]
    D --> H["模型历史回复\n包含 tool_calls 时有调用信息"]
    E --> I["需携带 tool_call_id\n匹配对应的工具调用"]

消息角色说明：

Role 作用 使用时机 注意事项
system 设置模型的行为、角色、规则 对话开始前，放数组第一位 不计入对话轮次；OpenAI 放在 messages 内，Anthropic 是独立参数
user 用户的每一次输入 每轮用户发言时 内容可以是字符串或内容块数组（图文混合）
assistant 模型的历史回复 多轮对话时，把模型上一轮结果存入 包含 tool_calls 时不能省略，否则工具结果无法关联
tool 工具执行的返回结果 调用工具后，把结果以此角色加入 必须携带 tool_call_id，与 tool_calls 一一对应

多轮对话消息数组示意：

flowchart LR
    S["system\n你是数学老师"] --> U1["user\n什么是勾股定理？"]
    U1 --> A1["assistant\n勾股定理是..."]
    A1 --> U2["user\n举个实际例子"]
    U2 --> A2["assistant\n比如建筑施工中..."]

每轮请求都要把完整历史发送给 API，模型本身无状态记忆。

Tool Use 机制

Tool Use（工具调用）是 Agent 的核心能力，让模型能够调用外部函数获取实时信息或执行操作。

flowchart TD
    A["用户输入"] --> B["LLM 处理\n携带 tools 定义"]
    B --> C{finish_reason?}
    C -->|tool_calls| D["解析 tool_calls\n获取工具名+参数"]
    D --> E["执行对应工具函数\n获取真实结果"]
    E --> F["将结果以 role=tool 加入历史"]
    F --> B
    C -->|stop| G["返回最终文本回复给用户"]
    C -->|length| H["警告：回复被截断\n增大 max_tokens"]

工具调用的消息流：

sequenceDiagram
    participant App as 应用代码
    participant GPT as GPT-4o

    App->>GPT: 消息 + tools 定义
    GPT-->>App: finish_reason="tool_calls"<br/>tool_calls=[{id, name, arguments}]
    Note over App: 执行工具，获得结果
    App->>GPT: 原消息历史 + assistant(tool_calls) + tool(结果)
    GPT-->>App: finish_reason="stop"<br/>message.content="最终回答"

finish_reason 对照表

finish_reason 告诉你模型为什么停止生成，必须根据它决定下一步操作。

finish_reason 值 含义 下一步操作
stop 模型正常完成回复 提取 message.content 返回给用户
tool_calls 模型需要调用工具 解析 message.tool_calls，执行工具，继续循环
length 达到 max_tokens 限制被截断 警告用户，或增大 max_tokens 重试
content_filter 内容被安全过滤器拦截 向用户提示内容不合规，不要重试相同内容
null 流式输出中间 chunk，尚未完成 继续迭代 stream，等待最终 chunk

2. 基础消息 API

// src/basic-chat.ts
import client from "./openai-client";

async function basicChat() {
  const response = await client.chat.completions.create({
    model: "gpt-4o",
    max_tokens: 1024,
    messages: [
      {
        role: "system",
        // 为什么要有 system：定义模型的角色和行为规则，影响所有后续回复的风格
        content: "你是一位专业的技术顾问，回答简洁准确。"
      },
      {
        role: "user",
        content: "解释一下什么是 REST API？"
      }
    ]
  });

  // 注意：OpenAI 的响应路径与 Anthropic 不同
  // Anthropic: response.content[0].text
  // OpenAI:    response.choices[0].message.content
  const message = response.choices[0].message;
  console.log("回复:", message.content);

  // finish_reason 决定下一步操作（见上方对照表）
  console.log("停止原因:", response.choices[0].finish_reason);

  // Token 字段名与 Anthropic 不同
  console.log("Token 使用:", {
    input: response.usage?.prompt_tokens,      // 注意：不是 input_tokens（Anthropic 的命名）
    output: response.usage?.completion_tokens, // 注意：不是 output_tokens（Anthropic 的命名）
    total: response.usage?.total_tokens
  });
}

多轮对话：

// src/multi-turn.ts
import client from "./openai-client";

interface Message {
  role: "system" | "user" | "assistant";
  content: string;
}

async function multiTurnChat() {
  // 维护完整历史是多轮对话的关键
  // 因为 OpenAI API 是无状态的，每次请求都必须带完整上下文
  const history: Message[] = [
    { role: "system", content: "你是一个数学老师，擅长用简单的例子解释概念。" }
  ];

  // 第一轮
  history.push({ role: "user", content: "什么是勾股定理？" });

  let response = await client.chat.completions.create({
    model: "gpt-4o",
    messages: history  // 发送完整历史
  });

  const reply1 = response.choices[0].message.content!;
  console.log("Assistant:", reply1);

  // 把模型的回复存入历史，下一轮才有上下文
  history.push({ role: "assistant", content: reply1 });

  // 第二轮（上下文延续）
  history.push({ role: "user", content: "给我一个实际应用的例子" });

  response = await client.chat.completions.create({
    model: "gpt-4o",
    messages: history  // 包含了第一轮的问答
  });

  const reply2 = response.choices[0].message.content!;
  console.log("Assistant:", reply2);
}

multiTurnChat();

3. 流式输出（Streaming）

// src/streaming.ts
import client from "./openai-client";

async function streamingChat() {
  console.log("开始流式输出...\n");

  // 为什么用流式：
  // 非流式：等待模型生成完所有文字后一次性返回，用户等待时间长
  // 流式：每生成一个 token 就立即发给客户端，用户体验好（像 ChatGPT 网页）
  const stream = await client.chat.completions.create({
    model: "gpt-4o",
    max_tokens: 500,
    stream: true,  // 关键：启用流式，返回值变为 AsyncIterable
    messages: [
      { role: "user", content: "用100字介绍量子计算" }
    ]
  });

  let fullContent = "";

  // for await 遍历每个 chunk（通常每个 chunk 包含 1-5 个 token）
  for await (const chunk of stream) {
    // delta 是这个 chunk 新增的内容，不是完整内容
    const delta = chunk.choices[0]?.delta?.content ?? "";
    if (delta) {
      process.stdout.write(delta);  // 实时打印，不换行
      fullContent += delta;
    }

    // 流结束信号：finish_reason 变为 "stop"
    if (chunk.choices[0]?.finish_reason === "stop") {
      console.log("\n\n[流式结束]");
    }
  }

  return fullContent;
}

streamingChat();

流式输出 + Token 统计（需要 stream_options）：

async function streamingWithUsage() {
  // 默认流式模式不返回 usage，需要显式开启 include_usage
  // 这是因为 usage 只有在全部生成完成后才能统计，会附在最后一个 chunk
  const stream = await client.chat.completions.create({
    model: "gpt-4o",
    stream: true,
    stream_options: { include_usage: true },  // 在流末尾包含 usage 信息
    messages: [{ role: "user", content: "Hello!" }]
  });

  for await (const chunk of stream) {
    const delta = chunk.choices[0]?.delta?.content ?? "";
    if (delta) process.stdout.write(delta);

    // 最后一个 chunk（choices 为空时）包含 usage
    if (chunk.usage) {
      console.log("\nToken 统计:", chunk.usage);
    }
  }
}

4. Function Calling / Tool Use

这是最重要的部分。OpenAI 的工具格式与 Anthropic 有所不同：

// src/tool-definition.ts
import OpenAI from "openai";

// OpenAI 工具定义格式
// 为什么要这样定义：模型需要知道每个工具的名称、用途、参数格式
// 才能在合适的时机选择调用，并生成正确格式的参数
const tools: OpenAI.Chat.Completions.ChatCompletionTool[] = [
  {
    type: "function",       // 目前只有 "function" 类型
    function: {
      name: "get_weather",
      description: "获取指定城市的当前天气信息",
      // OpenAI 用 parameters（Anthropic 用 input_schema）
      // 两者内部的 JSON Schema 结构相同，只是字段名不同
      parameters: {
        type: "object",
        properties: {
          city: {
            type: "string",
            description: "城市名称，如 '北京', '上海'"
          },
          unit: {
            type: "string",
            enum: ["celsius", "fahrenheit"],
            description: "温度单位"
          }
        },
        required: ["city"]  // city 必填，unit 可选
      }
    }
  },
  {
    type: "function",
    function: {
      name: "calculate",
      description: "执行数学计算",
      parameters: {
        type: "object",
        properties: {
          expression: {
            type: "string",
            description: "数学表达式，如 '2 + 3 * 4'"
          }
        },
        required: ["expression"]
      }
    }
  }
];

模拟工具实现：

// src/tools.ts

interface WeatherResult {
  city: string;
  temperature: number;
  unit: string;
  condition: string;
  humidity: number;
}

// 模拟天气 API
// 实际项目中替换为真实 API 调用（如 OpenWeatherMap）
function getWeather(city: string, unit: string = "celsius"): WeatherResult {
  const mockData: Record<string, Omit<WeatherResult, "city" | "unit">> = {
    "北京": { temperature: 22, condition: "晴朗", humidity: 45 },
    "上海": { temperature: 26, condition: "多云", humidity: 72 },
    "深圳": { temperature: 30, condition: "小雨", humidity: 85 },
  };

  const data = mockData[city] || { temperature: 20, condition: "未知", humidity: 60 };
  const temp = unit === "fahrenheit"
    ? Math.round(data.temperature * 9/5 + 32)
    : data.temperature;

  return { city, temperature: temp, unit, ...data, temperature: temp };
}

// 安全的数学计算器
function calculate(expression: string): number | string {
  try {
    // 注意：生产环境不要用 eval，使用 mathjs 等库
    const sanitized = expression.replace(/[^0-9+\-*/().\s]/g, "");
    return Function(`"use strict"; return (${sanitized})`)();
  } catch (e) {
    return `计算错误: ${e}`;
  }
}

// 工具执行分发器：根据工具名路由到对应函数
// 这是 Agent 架构中的"工具注册表"模式
export function executeTool(name: string, args: Record<string, unknown>): string {
  switch (name) {
    case "get_weather":
      const weather = getWeather(args.city as string, args.unit as string);
      // 工具结果必须返回字符串，模型只能接收文本格式的结果
      return JSON.stringify(weather);

    case "calculate":
      const result = calculate(args.expression as string);
      return JSON.stringify({ expression: args.expression, result });

    default:
      return JSON.stringify({ error: `未知工具: ${name}` });
  }
}

5. 完整的 Agentic Loop

// src/agent-loop.ts
import OpenAI from "openai";
import { executeTool } from "./tools";

const client = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

const tools: OpenAI.Chat.Completions.ChatCompletionTool[] = [
  {
    type: "function",
    function: {
      name: "get_weather",
      description: "获取指定城市的当前天气信息",
      parameters: {
        type: "object",
        properties: {
          city: { type: "string", description: "城市名称" },
          unit: { type: "string", enum: ["celsius", "fahrenheit"] }
        },
        required: ["city"]
      }
    }
  },
  {
    type: "function",
    function: {
      name: "calculate",
      description: "执行数学计算",
      parameters: {
        type: "object",
        properties: {
          expression: { type: "string", description: "数学表达式" }
        },
        required: ["expression"]
      }
    }
  }
];

type Message = OpenAI.Chat.Completions.ChatCompletionMessageParam;

async function runAgent(userMessage: string): Promise<string> {
  const messages: Message[] = [
    {
      role: "system",
      content: "你是一个有用的助手，可以查天气和进行数学计算。"
    },
    {
      role: "user",
      content: userMessage
    }
  ];

  console.log(`\n用户: ${userMessage}\n`);

  // Agent 循环：最多10轮防止无限循环
  // 正常情况下 1-3 轮就能完成，设上限是安全防护
  for (let i = 0; i < 10; i++) {
    const response = await client.chat.completions.create({
      model: "gpt-4o",
      tools,
      tool_choice: "auto",  // auto：模型自己决定要不要调用工具；也可强制 "required"
      messages
    });

    const choice = response.choices[0];
    const assistantMessage = choice.message;

    // 必须把 assistant 消息加入历史
    // 原因：下一轮发送工具结果时，模型需要知道它之前请求了哪些工具调用
    messages.push(assistantMessage);

    // 分支1：模型请求调用工具
    if (choice.finish_reason === "tool_calls" && assistantMessage.tool_calls) {
      console.log(`[第 ${i+1} 轮] 模型调用工具...`);

      // OpenAI 支持并行调用多个工具（一次返回多个 tool_calls）
      // 所以要遍历处理所有工具调用，然后一次性把所有结果加入历史
      const toolResults: Message[] = [];

      for (const toolCall of assistantMessage.tool_calls) {
        const toolName = toolCall.function.name;
        const toolArgs = JSON.parse(toolCall.function.arguments);  // arguments 是 JSON 字符串，需要解析

        console.log(`  → 工具: ${toolName}`);
        console.log(`    参数: ${JSON.stringify(toolArgs)}`);

        const result = executeTool(toolName, toolArgs);
        console.log(`    结果: ${result}`);

        // OpenAI 的工具结果格式：role = "tool"
        // 与 Anthropic 的 role = "user" + type = "tool_result" 完全不同
        // tool_call_id 是关键：模型用它来匹配请求和结果
        toolResults.push({
          role: "tool",                    // OpenAI 特有的 role（Anthropic 没有这个 role）
          tool_call_id: toolCall.id,       // 必须与对应 tool_call 的 id 匹配
          content: result                  // 工具执行结果（字符串）
        });
      }

      // 将所有工具结果加入消息历史，继续下一轮
      messages.push(...toolResults);
      continue;
    }

    // 分支2：模型完成回复
    if (choice.finish_reason === "stop") {
      const finalContent = assistantMessage.content ?? "";
      console.log(`\n助手: ${finalContent}`);
      return finalContent;
    }

    // 分支3：被截断（max_tokens 不够用）
    if (choice.finish_reason === "length") {
      console.warn("回复被截断（max_tokens 限制）");
      return assistantMessage.content ?? "";
    }
  }

  return "Agent 循环次数超限";
}

// 测试
async function main() {
  // 测试1：只需计算
  await runAgent("25 乘以 36 等于多少？再加上 144 的平方根是多少？");

  // 测试2：只需查天气
  await runAgent("北京现在的天气怎么样？");

  // 测试3：综合使用两个工具
  await runAgent("上海的温度是多少摄氏度？如果换算成华氏度是多少？");
}

7. 错误处理最佳实践

// src/error-handling.ts
import OpenAI from "openai";

const client = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

async function robustChat(userMessage: string) {
  try {
    const response = await client.chat.completions.create({
      model: "gpt-4o",
      messages: [{ role: "user", content: userMessage }]
    });
    return response.choices[0].message.content;

  } catch (error) {
    if (error instanceof OpenAI.APIError) {
      // OpenAI 特定错误，包含 HTTP 状态码
      console.error(`OpenAI API 错误: ${error.status} - ${error.message}`);

      switch (error.status) {
        case 401:
          // API Key 问题，不应重试（重试也会失败）
          throw new Error("API Key 无效");
        case 429:
          // 速率限制：等待后重试（指数退避更好，这里简化为固定等待）
          console.log("达到速率限制，等待 60 秒...");
          await new Promise(r => setTimeout(r, 60000));
          return robustChat(userMessage);  // 重试
        case 500:
          // 服务器错误：OpenAI 侧的问题，短暂等待后可重试
          throw new Error("OpenAI 服务器错误，请稍后重试");
        default:
          throw error;
      }
    }
    throw error;
  }
}

API 差异速查表

构建多模型应用时，最容易混淆的是各 SDK 的字段命名差异。本表可作为快速参考：

字段 / 功能 OpenAI Anthropic 备注
客户端初始化 new OpenAI({apiKey}) new Anthropic({apiKey}) 包名不同：openai vs @anthropic-ai/sdk
发起请求 client.chat.completions.create(...) client.messages.create(...) 方法路径不同
System prompt messages 数组第一个元素，role: "system" 顶层 system 参数（独立于 messages） Anthropic 不允许 system 在 messages 里
获取文本回复 response.choices[0].message.content response.content[0].text 层级结构完全不同
停止原因字段 response.choices[0].finish_reason response.stop_reason 值也不同：stop vs end_turn
Tool 定义容器 tools[].function.parameters tools[].input_schema 内部 JSON Schema 结构相同
输入 Token 数 response.usage.prompt_tokens response.usage.input_tokens 命名风格不同
输出 Token 数 response.usage.completion_tokens response.usage.output_tokens 命名风格不同
工具结果消息 {role: "tool", tool_call_id, content} {role: "user", content: [{type: "tool_result", tool_use_id, content}]} OpenAI 有 tool role；Anthropic 用 user role 包装
流式输出字段 chunk.choices[0].delta.content event.delta.text（事件流） 数据结构不同
并行工具调用 原生支持（一次返回多个 tool_calls） 原生支持（content 数组包含多个 tool_use） 两者都支持，但格式不同

常见坑与解决方案

问题 原因 解决方法
response.content 是 null 模型调用了工具，此时 content 为 null，tool_calls 有值 先检查 finish_reason，为 tool_calls 时不读 content
工具调用后模型"忘了"上下文 把 assistant 消息（含 tool_calls）加入历史时漏掉了 调用工具前必须先 messages.push(assistantMessage)
tool_call_id 不匹配报错 工具结果的 tool_call_id 与请求时的 id 不一致 从 toolCall.id 复制，不要手写
多轮对话越来越慢且贵 messages 历史无限增长，每次都传所有内容 实现滑动窗口或摘要压缩，控制历史长度
流式输出丢失最后几个字 没有处理 finish_reason === "stop" 后的最后一个 chunk 确保循环在 finish_reason 出现后仍处理最后一个 delta
arguments 解析失败 toolCall.function.arguments 是字符串，忘记 JSON.parse 始终用 JSON.parse(toolCall.function.arguments)
429 速率限制频繁触发 请求频率超过 API 配额 实现指数退避重试；升级 API 使用层级
TypeScript 类型报错 某些字段可能为 null（如 message.content） 使用 ?? 空值合并或 ! 非空断言（确认安全时）
