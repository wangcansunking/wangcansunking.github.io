第5课：Skill Loading — 按需注入专业知识

介绍

把所有领域知识（API 文档、编码规范、操作手册）预加载到系统 prompt 会大量浪费 tokens。本课实现两层 Skill 架构：层一只注入技能名称+描述（~100 tokens/条），层二在模型请求时才加载完整技能内容。这是 Claude Code Skill 系统的核心设计。

源码参考：github.com/shareAI-lab/learn-claude-code — agents/s05_skill_loading.py

遇到什么问题

一个具备 10 个领域技能的 Agent，每个技能文档约 2000 tokens：

策略 每轮 token 消耗 问题
全部预加载 20000+ tokens 贵、慢、超出上下文
不加载 0 tokens 模型不知道如何操作
按需加载 100 (索引) + 2000 (单次) ✅ 最优 
flowchart LR
    A[系统 Prompt] -->|层一：索引| B["pdf: Process PDF files\ndocker: Build containers\ngit: Git operations..."]
    B --> C[模型决定需要 pdf 知识]
    C --> D[load_skill('pdf')]
    D --> E[层二：完整技能内容注入]
    E --> F[执行 PDF 相关操作]

有哪些方案

方案 A：全部预加载到系统 Prompt

SYSTEM = "\n\n".join([skill.full_content for skill in all_skills])

✅ 模型始终有完整知识 ❌ Token 消耗爆炸，每轮都要付出高昂成本

方案 B：RAG 检索注入

根据用户问题语义检索最相关的技能片段。

✅ 精准匹配 ❌ 需要 embedding 模型，增加延迟和复杂度

方案 C：两层按需加载（推荐）

系统 Prompt → 技能摘要（名称 + 1行描述）
模型判断后 → load_skill(name) → 完整内容作为 tool_result 注入

✅ 零 RAG 依赖，模型自己判断需要哪个技能 ✅ Token 消耗最优：只加载当前任务需要的技能

怎么选择

方案 C。信任模型的判断力——给它索引，它会自己决定何时需要完整文档。load_skill 工具的 tool_result 把技能内容注入到当前上下文，效果等同于事先放在系统 prompt 里。

### SkillLoader 实现

```python
# agents/s05_skill_loading.py
import yaml
from pathlib import Path


class SkillLoader:
    def __init__(self, skills_dir: str = "skills"):
        self.skills_dir = Path(skills_dir)
        self._cache: dict[str, dict] = {}
        self._scan()

    def _scan(self):
        """扫描所有 skills/*/SKILL.md 文件，解析 YAML frontmatter"""
        for skill_file in self.skills_dir.glob("*/SKILL.md"):
            content = skill_file.read_text()
            # 解析 --- frontmatter ---
            if content.startswith("---"):
                _, frontmatter, body = content.split("---", 2)
                meta = yaml.safe_load(frontmatter)
                self._cache[meta["name"]] = {
                    "meta": meta,
                    "body": body.strip(),
                }

    def get_descriptions(self) -> str:
        """层一：紧凑摘要，注入系统 Prompt（~100 tokens/条）"""
        if not self._cache:
            return "No skills available."
        lines = ["Available skills (use load_skill to get full content):"]
        for name, skill in self._cache.items():
            desc = skill["meta"].get("description", "No description")
            lines.append(f"  - {name}: {desc}")
        return "\n".join(lines)

    def get_content(self, name: str) -> str:
        """层二：完整技能内容，作为 tool_result 返回"""
        if name not in self._cache:
            available = ", ".join(self._cache.keys())
            return f"Skill '{name}' not found. Available: {available}"
        skill = self._cache[name]
        return f"<skill name='{name}'>\n{skill['body']}\n</skill>"


# 全局 loader
skill_loader = SkillLoader("skills")

系统 Prompt 注入层一

def build_system_prompt() -> str:
    return f"""You are a coding agent at {WORKDIR}.

{skill_loader.get_descriptions()}

When you need domain knowledge, call load_skill(name) BEFORE attempting the task.
The skill content will be injected directly into your context.

Act, don't explain."""

load_skill 工具

TOOL_HANDLERS["load_skill"] = lambda **kw: skill_loader.get_content(kw["name"])

TOOLS.append({
    "name": "load_skill",
    "description": "Load full content of a skill into context. Call before starting domain-specific tasks.",
    "input_schema": {
        "type": "object",
        "properties": {
            "name": {
                "type": "string",
                "description": "Skill name as listed in the available skills",
            }
        },
        "required": ["name"],
    },
})
