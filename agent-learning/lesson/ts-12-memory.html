第12课：记忆 Agent

学习目标

  1. 理解短期记忆 vs 长期记忆的区别
  2. 实现基于文件的长期记忆系统
  3. 用 Claude 自动从对话中提取重要信息存储
  4. 了解官方 Memory Tool 的用法

两种记忆类型

短期记忆（对话历史）       长期记忆（持久化存储）
─────────────────────     ─────────────────────
存在：messages 数组       存在：文件/数据库
生命周期：本次对话        生命周期：永久
访问方式：自动携带        访问方式：显式加载
容量限制：Context Window  容量限制：存储大小

场景：

  • 短期：当前对话的问答内容
  • 长期：用户名字、偏好、过往决策、项目背景

文件记忆系统

import { readFileSync, writeFileSync, existsSync } from "fs";
import OpenAI from "openai";
import { client, model } from "../shared/llmClient";

type Msg = OpenAI.ChatCompletionMessageParam;

// 记忆文件路径
const MEMORY_FILE = "./agent_memory.json";

interface Memory {
  facts: string[];          // 用户事实（姓名、职业等）
  preferences: string[];    // 用户偏好
  decisions: string[];      // 重要决策记录
  lastUpdated: string;
}

// 加载记忆
function loadMemory(): Memory {
  if (!existsSync(MEMORY_FILE)) {
    return { facts: [], preferences: [], decisions: [], lastUpdated: "" };
  }
  return JSON.parse(readFileSync(MEMORY_FILE, "utf-8"));
}

// 保存记忆
function saveMemory(memory: Memory): void {
  memory.lastUpdated = new Date().toISOString();
  writeFileSync(MEMORY_FILE, JSON.stringify(memory, null, 2), "utf-8");
}

// 格式化记忆为文本（注入 system prompt）
function formatMemoryForPrompt(memory: Memory): string {
  const parts: string[] = [];
  if (memory.facts.length > 0)
    parts.push(`用户信息：\n${memory.facts.map(f => `- ${f}`).join("\n")}`);
  if (memory.preferences.length > 0)
    parts.push(`用户偏好：\n${memory.preferences.map(p => `- ${p}`).join("\n")}`);
  if (memory.decisions.length > 0)
    parts.push(`历史决策：\n${memory.decisions.map(d => `- ${d}`).join("\n")}`);
  return parts.join("\n\n");
}

自动提取记忆

对话结束后，让模型分析并提取值得记住的信息：

async function extractAndUpdateMemory(
  conversation: Msg[],
  currentMemory: Memory
): Promise<Memory> {
  const extractionResponse = await client.chat.completions.create({
    model,                              // 简单提取任务用便宜模型
    max_tokens: 1024,
    response_format: { type: "json_object" },
    messages: [
      {
        role: "system",
        content: `你是记忆提取助手。分析对话，提取值得长期记住的信息。
只提取明确提到的、有用的信息，不要推测。
返回 JSON：
{
  "new_facts": ["..."],        // 新的用户事实
  "new_preferences": ["..."],  // 新的偏好
  "new_decisions": ["..."]     // 重要决策
}
如果没有新信息，返回空数组。`,
      },
      {
        role: "user",
        content: `当前记忆：\n${JSON.stringify(currentMemory)}\n\n对话内容：\n${
          conversation
            .map((m) => `${m.role}: ${typeof m.content === "string" ? m.content : JSON.stringify(m.content)}`)
            .join("\n")
        }\n\n请提取新的值得记忆的信息：`,
      },
    ],
  });

  const extractedText = extractionResponse.choices[0].message.content ?? "{}";
  const extracted = JSON.parse(extractedText) as {
    new_facts?: string[];
    new_preferences?: string[];
    new_decisions?: string[];
  };

  // 合并新旧记忆（去重）
  const updated: Memory = {
    ...currentMemory,
    facts: [...new Set([...currentMemory.facts, ...(extracted.new_facts ?? [])])],
    preferences: [...new Set([...currentMemory.preferences, ...(extracted.new_preferences ?? [])])],
    decisions: [...new Set([...currentMemory.decisions, ...(extracted.new_decisions ?? [])])],
  };

  return updated;
}

带记忆的对话 Agent

class MemoryAgent {
  private memory = loadMemory();
  private history: Msg[] = [];

  async chat(userMessage: string): Promise<string> {
    this.history.push({ role: "user", content: userMessage });

    // 把长期记忆注入 system prompt
    const memoryText = formatMemoryForPrompt(this.memory);
    const systemPrompt = memoryText
      ? `你是有记忆能力的助手。以下是关于用户的已知信息，请在对话中自然地运用它：\n\n${memoryText}`
      : "你是有记忆能力的助手。";

    const response = await client.chat.completions.create({
      model,
      max_tokens: 2048,
      messages: [
        { role: "system", content: systemPrompt },
        ...this.history,
      ],
    });

    const assistantText = response.choices[0].message.content ?? "";

    this.history.push({ role: "assistant", content: assistantText });

    // 每 5 轮自动更新记忆
    if (this.history.length % 10 === 0) {
      this.memory = await extractAndUpdateMemory(this.history, this.memory);
      saveMemory(this.memory);
      console.log("💾 记忆已更新");
    }

    return assistantText;
  }

  // 手动触发记忆更新
  async saveCurrentMemory(): Promise<void> {
    this.memory = await extractAndUpdateMemory(this.history, this.memory);
    saveMemory(this.memory);
  }
}

Anthropic 原生 Memory Tool（仅 Anthropic）

Anthropic 原生 SDK 内置 Memory Tool，无需自己实现文件读写：

import Anthropic from "@anthropic-ai/sdk";
const ant = new Anthropic();

const response = await ant.messages.create({
  model: "claude-opus-4-7",
  max_tokens: 4096,
  messages: [
    { role: "user", content: "记住我喜欢简洁的代码风格，不喜欢过度注释" },
  ],
  tools: [
    {
      type: "memory_20250818",  // Anthropic 内置工具类型
      name: "memory",
    },
  ],
});

// Claude 会自动调用 memory 工具存储这个偏好
// 在后续对话中，Claude 会读取并应用这个记忆

DeepSeek / Kimi 没有等价的内置 memory tool——但本课的"自建文件记忆系统"完全够用，且更可控。

Memory Tool vs 自建记忆：Anthropic 工具操作更简单，但需要你实现文件存储 handler。自建记忆系统更灵活、跨 provider，控制存储格式和检索策略。

使用示例

const agent = new MemoryAgent();

// 第1次对话（全新开始）
await agent.chat("你好，我叫小明，是一名 TypeScript 开发者");
// Agent 介绍自己，开始记录

await agent.chat("我在做一个 AI Agent 项目，主要用 Claude API");
// 记住：项目信息

// 退出程序后，下次启动...
const agent2 = new MemoryAgent(); // 加载已保存的记忆

await agent2.chat("继续帮我完善之前的项目");
// Agent："好的，小明，继续你的 Claude API AI Agent 项目..."
// 它记得！

小结

方法 优点 缺点
文件记忆系统 完全可控，灵活 需自行实现
官方 Memory Tool 简单，Claude 自己管理 需实现存储 handler
数据库存储 结构化，可搜索 需要向量搜索（RAG）
