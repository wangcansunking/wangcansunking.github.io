第6课：Agentic Loop（自主循环 Agent）

学习目标

  1. 理解 Agentic Loop 的核心逻辑
  2. 实现能自主多步执行任务的 Agent
  3. 提供文件系统工具：读文件、列目录、写文件、搜索
  4. 控制最大迭代次数，防止无限循环

什么是 Agentic Loop？

前面的工具调用是"一问一答"，而 Agentic Loop 是"持续循环直到任务完成"：

用户任务
  ↓
┌─────────────────────────────────┐
│  模型思考 → 调用工具            │
│      ↓                          │  循环
│  执行工具 → 结果回传            │  直到
│      ↓                          │  finish_reason !== 'tool_calls'
│  模型思考 → 调用工具/直接回答   │
└─────────────────────────────────┘
  ↓
最终答案

这就是 AI Agent 能完成复杂任务的核心机制。

核心循环结构

import OpenAI from "openai";
import { client, model } from "../shared/llmClient";

type Msg = OpenAI.ChatCompletionMessageParam;

async function runAgent(userTask: string): Promise<string> {
  const messages: Msg[] = [
    { role: "system", content: AGENT_SYSTEM_PROMPT },
    { role: "user",   content: userTask },
  ];

  const MAX_ITERATIONS = 10; // 防止无限循环

  for (let i = 0; i < MAX_ITERATIONS; i++) {
    const response = await client.chat.completions.create({
      model,                              // 默认 deepseek-chat
      max_tokens: 4096,
      tools,
      messages,
    });

    const msg = response.choices[0].message;
    messages.push(msg);                   // 把模型响应（含 tool_calls）原样追加历史

    // 任务完成 → 退出循环
    if (response.choices[0].finish_reason !== "tool_calls" || !msg.tool_calls?.length) {
      return msg.content ?? "";
    }

    // 需要调用工具 → 执行并回传
    for (const call of msg.tool_calls) {
      const result = executeTool(call.function.name, JSON.parse(call.function.arguments));
      messages.push({
        role: "tool",
        tool_call_id: call.id,
        content: result,
      });
    }
    // 继续下一轮循环
  }

  return "达到最大迭代次数，任务未完成";
}

const tools = [
  {
    type: "function" as const,
    function: {
      name: "read_file",
      description: "读取文件内容",
      parameters: {
        type: "object",
        properties: { path: { type: "string", description: "文件路径" } },
        required: ["path"],
      },
    },
  },
  {
    type: "function" as const,

      name: "list_directory",
      description: "列出目录内容",

        properties: { path: { type: "string", description: "目录路径，默认为当前目录" } },
        required: [],
      },
    },
  },
  {
    type: "function" as const,
    function: {
      name: "write_file",
      description: "写入内容到文件",
      parameters: {
        type: "object",

          path:    { type: "string", description: "文件路径" },
          content: { type: "string", description: "要写入的内容" },

  {
    type: "function" as const,

        type: "object",

          directory:   { type: "string", description: "搜索目录" },

工具执行函数

import { readFileSync, writeFileSync, readdirSync, statSync } from "fs";
import { join } from "path";

function executeTool(name: string, input: Record<string, unknown>): string {
  try {
    switch (name) {
      case "read_file": {
        const content = readFileSync(input.path as string, "utf-8");
        return content.length > 3000
          ? content.slice(0, 3000) + "\n...(内容过长，已截断)"
          : content;
      }

      case "list_directory": {
        const dir = (input.path as string) ?? ".";
        const entries = readdirSync(dir);
        return entries
          .map((e) => {
            const stat = statSync(join(dir, e));
            return `${stat.isDirectory() ? "📁" : "📄"} ${e}`;
          })
          .join("\n");
      }

      case "write_file": {
        writeFileSync(input.path as string, input.content as string, "utf-8");
        return `✅ 文件已写入：${input.path}`;
      }

      case "search_in_files": {
        const results: string[] = [];
        function search(dir: string) {
          for (const entry of readdirSync(dir)) {
            const fullPath = join(dir, entry);
            const stat = statSync(fullPath);
            if (stat.isDirectory()) {
              search(fullPath);
            } else if (stat.isFile()) {
              const content = readFileSync(fullPath, "utf-8");
              if (content.includes(input.search_text as string)) {
                results.push(fullPath);
              }
            }
          }
        }
        search(input.directory as string);
        return results.length > 0
          ? `找到 ${results.length} 个文件：\n${results.join("\n")}`
          : "未找到匹配文件";
      }

      default:
        return `未知工具：${name}`;
    }
  } catch (e: unknown) {
    return `工具执行错误：${(e as Error).message}`;
  }
}

实际效果

任务: "帮我列出当前目录的文件，然后读取 README.md 并总结其内容"

第1轮：模型调用 list_directory(".")
  → 返回文件列表

第2轮：模型调用 read_file("README.md")
  → 返回文件内容

第3轮：模型综合信息，给出总结
  → finish_reason: "stop"

Agent 完成！总共 3 轮迭代。

关键设计原则

  1. 迭代上限：始终设置 MAX_ITERATIONS，防止无限循环和费用失控
  2. 完整历史：每轮 assistant 消息（包含 tool_calls）原样追加到 messages，保证上下文连贯
  3. 错误处理：工具执行失败时返回错误信息（而非抛出异常），让模型自己决策
  4. 工具粒度：工具越原子化，模型越能灵活组合

关于 Anthropic 原生协议

如果你切到 Anthropic 原生 messages.create，循环条件和字段名要相应调整：

OpenAI 兼容（本课） Anthropic 原生
finish_reason === 'tool_calls' stop_reason === 'tool_use'
msg.tool_calls[] content[].type === 'tool_use'
{ role: 'tool', tool_call_id, content } { type: 'tool_result', tool_use_id, content }
messages: [{ role: 'system', content }, ...] API 顶层 system 参数
