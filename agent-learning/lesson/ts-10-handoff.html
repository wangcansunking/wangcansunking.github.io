第10课：Agent Handoff（智能路由）

学习目标

  1. 理解 Agent Handoff 的适用场景
  2. 实现智能路由器（Router）自动判断分配
  3. 实现专家 Agent 的注册与调用机制
  4. 处理 Handoff 链（多级转交）

什么是 Agent Handoff？

Handoff 是"接力棒"模式——当一个 Agent 遇到超出其专业范围的问题时，主动将用户转交给更合适的 Agent：

用户问题
    ↓
Router（路由器）—— 判断意图
    ├── 技术问题   → TechAgent
    ├── 账单问题   → BillingAgent
    ├── 销售问题   → SalesAgent
    └── 其他问题   → GeneralAgent

与 Orchestrator 的区别：

  • Orchestrator：主动分解任务，统筹多个 Worker
  • Handoff：被动路由，用户问题决定去哪个 Agent

专家 Agent 定义

路由器实现

async function routeToAgent(userMessage: string): Promise<ExpertAgent> {
  // 构建路由选项列表
  const agentOptions = expertAgents
    .map((a) => `- ${a.name}: ${a.description}`)
    .join("\n");

  // 路由判断用便宜小模型即可（DeepSeek-Chat 已经够便宜，Kimi-v1-8k 更适合短判断）
  const routingResponse = await client.chat.completions.create({
    model,
    max_tokens: 50,
    messages: [
      { role: "system", content: `你是智能路由器。根据用户消息，选择最合适的专家 Agent。
只返回 Agent 名称，不要其他内容。

可用 Agents：
${agentOptions}` },
      { role: "user", content: userMessage },
    ],
  });

  const selectedName = (routingResponse.choices[0].message.content ?? "").trim();

  const agent = expertAgents.find((a) => a.name === selectedName)
    ?? expertAgents.find((a) => a.name === "GeneralSupport")!;

  console.log(`  🔀 路由到: ${agent.name}`);
  return agent;
}

带 Handoff 的对话系统

专家 Agent 可以主动触发 Handoff，将对话转交给另一个 Agent：

// 定义 handoff 工具（OpenAI function calling 格式）
const handoffTool = {
  type: "function" as const,
  function: {
    name: "handoff_to_agent",
    description: "当问题超出当前专业范围时，将用户转交给更合适的 Agent",
    parameters: {
      type: "object",
      properties: {
        target_agent: {
          type: "string",
          enum: expertAgents.map((a) => a.name),
          description: "目标 Agent 名称",
        },
        reason: {
          type: "string",
          description: "转交原因",
        },
        context_summary: {
          type: "string",
          description: "为目标 Agent 提供的上下文摘要",
        },
      },
      required: ["target_agent", "reason"],
    },
  },
};

async function handleWithAgent(
  agent: ExpertAgent,
  userMessage: string,
  context: string = "",
  depth = 0  // 防止无限转交
): Promise<string> {
  if (depth > 3) return "抱歉，无法处理此请求，请联系人工客服。";

  console.log(`  💬 ${agent.name} 正在处理...`);

  const response = await client.chat.completions.create({
    model,
    max_tokens: 2048,
    tools: [handoffTool],
    messages: [
      { role: "system", content: agent.systemPrompt },
      ...(context
        ? [
            { role: "user" as const,      content: `上下文：${context}` },
            { role: "assistant" as const, content: "我已了解背景，请告诉我你的问题。" },
          ]
        : []),
      { role: "user", content: userMessage },
    ],
  });

  const msg = response.choices[0].message;

  // 检查是否触发了 Handoff
  if (response.choices[0].finish_reason === "tool_calls" && msg.tool_calls?.length) {
    const call = msg.tool_calls[0]!;

    if (call.function.name === "handoff_to_agent") {
      const { target_agent, reason, context_summary } = JSON.parse(call.function.arguments) as {
        target_agent: string;
        reason: string;
        context_summary?: string;
      };

      console.log(`  ↗️  Handoff: ${agent.name} → ${target_agent}（原因：${reason}）`);

      const targetAgent = expertAgents.find((a) => a.name === target_agent)!;
      return handleWithAgent(targetAgent, userMessage, context_summary, depth + 1);
    }
  }

  // 正常返回答案
  return msg.content ?? "";
}

完整使用示例

async function customerService(userMessage: string): Promise<string> {
  console.log(`\n用户: ${userMessage}`);

  // 1. 路由器判断
  const agent = await routeToAgent(userMessage);

  // 2. 专家 Agent 处理（可能触发 Handoff）
  const answer = await handleWithAgent(agent, userMessage);

  console.log(`\n回答: ${answer}`);
  return answer;
}

// 测试
await customerService("我的 API 调用一直报 401 错误，如何解决？");
// → 路由到 TechSupport → 提供技术解决方案

await customerService("为什么这个月的账单比上个月多了 200 元？");
// → 路由到 BillingSupport → 解释计费规则

小结

模式 核心价值
路由器（Router） 用最快模型做意图判断，降低成本
专家 Agent 专注单一领域，回答质量更高
Handoff 工具 主动转交，无缝衔接对话上下文
深度限制 防止循环转交，最多 3 层
