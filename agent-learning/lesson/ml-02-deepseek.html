第2课：DeepSeek — 推理模型 + 低成本 Agent

介绍

DeepSeek 是中国深度求索公司推出的高性价比大模型系列，其 API 完全兼容 OpenAI 格式，只需修改 baseURL 即可接入，无需学习新 SDK。本课重点介绍两个核心模型：

  • DeepSeek-V3（deepseek-chat）：通用对话、代码、翻译等日常任务，价格仅为 GPT-4o 的约 1/10
  • DeepSeek-R1（deepseek-reasoner）：专用推理模型，内置思维链（Chain-of-Thought），擅长数学证明、逻辑分析、复杂推理，并能将推理过程完整输出

学完本课你将掌握：如何用最小代价迁移现有 OpenAI 代码到 DeepSeek、如何利用 R1 的思维链解决 GPT-4o 不稳定的推理问题、以及如何设计按场景路由的智能模型选择器。

DeepSeek 的 API 完全实现了 OpenAI Chat Completions 接口规范，同一套代码只需修改两行配置即可切换。

遇到什么问题

你在用 GPT-4o 构建 Agent 时发现成本很高——每次复杂任务消耗数千 tokens，月底账单触目惊心。同时，某些推理任务（数学证明、逻辑分析）GPT-4o 表现不稳定。

GPT-4o 价格（2024年）：
  输入：$2.50 / 1M tokens
  输出：$10.00 / 1M tokens

一个典型 Agent 任务：~5000 tokens 输入 + ~1000 tokens 输出
成本：$0.0125 + $0.010 = $0.0225 / 次
月跑 10万次：≈ $2,250 / 月

核心问题：

  1. 日常任务用 GPT-4o 过于昂贵
  2. 复杂推理任务需要更强的推理模型
  3. 两种需求需要两套不同的模型策略

有哪些方案

方案 A：全用 GPT-4o

const response = await openai.chat.completions.create({
  model: "gpt-4o",
  messages: [...]
});

✅ 质量稳定，英文任务优秀 ❌ 成本高，日常任务大材小用 ❌ 推理能力不如专用推理模型

方案 B：DeepSeek-V3 替换日常任务

// 只需修改 baseURL，其余代码完全不变
const deepseek = new OpenAI({
  apiKey: process.env.DEEPSEEK_API_KEY,
  baseURL: "https://api.deepseek.com"
});

const response = await deepseek.chat.completions.create({
  model: "deepseek-chat",  // DeepSeek-V3
  messages: [...]
});

✅ 代码改动极小（只改 client 初始化） ✅ 成本降低 10-20 倍 ❌ 英文长文写作质量略逊于 GPT-4o

方案 C：R1 + V3 分场景路由（推荐）

// 根据任务类型选择模型
function selectModel(taskType: "reasoning" | "chat" | "code"): string {
  switch (taskType) {
    case "reasoning": return "deepseek-reasoner";  // R1：推理任务
    case "code":      return "deepseek-chat";       // V3：代码能力强
    case "chat":      return "deepseek-chat";       // V3：日常对话
  }
}

✅ 推理任务用 R1，成本效益最优 ✅ 日常任务用 V3，极低成本 ❌ 需要任务分类逻辑

怎么选择

使用方案 C：R1 用于数学/逻辑/推理任务（能看到思考过程），V3 用于对话/代码/文本任务。成本对比：

模型 模型 ID 输入价格 输出价格 适合场景 相对成本
GPT-4o gpt-4o $2.50/M $10.00/M 英文任务，需要可靠性 $$$$$
GPT-4o-mini gpt-4o-mini $0.15/M $0.60/M 简单英文任务，低成本 $$
Claude Sonnet 4.6 claude-sonnet-4-6 $3.00/M $15.00/M 编程，长文档，复杂指令 $$$$$
DeepSeek-V3 deepseek-chat $0.27/M $1.10/M 日常对话，中文任务 $

DeepSeek-V3 是日常任务性价比最高的选择；R1 在推理任务上超越 GPT-4o 且成本更低。

核心概念：R1 的思维链输出

R1 的独特之处在于它会将推理过程以 reasoning_content 字段单独输出，让你看到模型是如何一步步思考的。

sequenceDiagram
    participant U as 用户
    participant R1 as DeepSeek-R1

    U->>R1: 发送复杂问题<br/>（无需 system message）

    Note over R1: 开始推理阶段
    R1-->>R1: 内部思考：拆解问题<br/>建立推理框架<br/>逐步推导...

    Note over R1: 推理完成，生成回答
    R1-->>U: 响应包含两个字段：<br/>reasoning_content: "让我分析这道题..."<br/>content: "最终答案是..."

响应结构对比：

字段 普通模型（V3/GPT-4o） R1 推理模型
message.content 直接回复内容 最终答案（推理完成后的结论）
message.reasoning_content 不存在此字段 完整推理过程（可能很长）
流式输出 delta.content delta.reasoning_content（思考中）+ delta.content（回答时）

2. DeepSeek-V3 基础用法

// src/v3-basic.ts
import { deepseekClient, DEEPSEEK_V3 } from "./deepseek-client";

async function chatWithV3(message: string) {
  // 与 OpenAI 调用方式完全相同，只是 client 不同
  const response = await deepseekClient.chat.completions.create({
    model: DEEPSEEK_V3,
    max_tokens: 2048,
    messages: [
      {
        role: "system",
        // V3 支持 system message，可以正常设定角色和规则
        content: "你是一个专业的 TypeScript 开发助手，回答简洁准确。"
      },
      {
        role: "user",
        content: message
      }
    ]
  });

  // 响应格式与 OpenAI 完全一致
  return response.choices[0].message.content;
}

}

3. DeepSeek-R1 推理模型（核心特性）

R1 的特别之处在于它会输出思考过程（reasoning content），你可以看到模型一步步推理的内容：

// src/r1-reasoning.ts
import { deepseekClient, DEEPSEEK_R1 } from "./deepseek-client";

interface ReasoningResponse {
  thinking: string;   // 推理过程（reasoning_content 字段的内容）
  answer: string;     // 最终答案（content 字段）
  usage: {
    reasoning_tokens: number;  // 思考过程消耗的 tokens（计费）
    answer_tokens: number;     // 最终回答消耗的 tokens
  };
}

async function reasonWithR1(problem: string): Promise<ReasoningResponse> {
  const response = await deepseekClient.chat.completions.create({
    model: DEEPSEEK_R1,
    messages: [
      {
        // 注意：R1 不支持 system message，这里直接用 user role
        // 如果传入 system message，会被忽略（不报错，但不生效）
        role: "user",
        content: problem
      }
    ]
  });

  const message = response.choices[0].message;

  // R1 特有字段：reasoning_content 包含完整推理过程
  // OpenAI SDK 的 TypeScript 类型定义中没有这个字段，需要类型断言
  const extendedMessage = message as typeof message & {
    reasoning_content?: string;
  };

  return {
    thinking: extendedMessage.reasoning_content ?? "",
    answer: message.content ?? "",
    usage: {
      // reasoning_tokens 统计推理过程本身的 token 消耗
      reasoning_tokens: (response.usage as any)?.completion_tokens_details?.reasoning_tokens ?? 0,
      answer_tokens: response.usage?.completion_tokens ?? 0
    }
  };
}

// 测试：经典数学推理题
async function testMathReasoning() {
  const problem = `
    一个农场有鸡和兔子共 100 只，腿共 280 条。
    请问鸡有多少只？兔子有多少只？
    请详细展示你的推理过程。
  `;

  console.log("问题:", problem);
  console.log("\n开始推理...\n");

  const result = await reasonWithR1(problem);

  console.log("=== 思考过程 ===");
  console.log(result.thinking);
  console.log("\n=== 最终答案 ===");
  console.log(result.answer);
  console.log("\n=== Token 消耗 ===");
  console.log(`推理 tokens: ${result.usage.reasoning_tokens}`);
  console.log(`回答 tokens: ${result.usage.answer_tokens}`);
}

testMathReasoning();

4. R1 流式输出（展示实时思考）

// src/r1-streaming.ts
import { deepseekClient, DEEPSEEK_R1 } from "./deepseek-client";

async function streamR1Reasoning(problem: string) {
  const stream = await deepseekClient.chat.completions.create({
    model: DEEPSEEK_R1,
    stream: true,
    messages: [{ role: "user", content: problem }]
  });

  let isThinking = false;
  let isAnswering = false;

  for await (const chunk of stream) {
    // 流式时，R1 的思维链在 delta.reasoning_content 中
    // 注意：OpenAI SDK 类型不包含此字段，需要类型断言
    const delta = chunk.choices[0]?.delta as any;

    // 思考阶段：reasoning_content 有增量内容
    if (delta?.reasoning_content) {
      if (!isThinking) {
        console.log("\n💭 [思考中...]\n");
        isThinking = true;
      }
      process.stdout.write(delta.reasoning_content);
    }

    // 回答阶段：content 有增量内容（reasoning_content 结束后开始）
    if (delta?.content) {
      if (!isAnswering) {
        console.log("\n\n✅ [最终回答]\n");
        isAnswering = true;
      }
      process.stdout.write(delta.content);
    }
  }

  console.log("\n");
}

5. 智能路由：根据任务复杂度选择模型

// src/smart-router.ts
import OpenAI from "openai";

const deepseekClient = new OpenAI({
  apiKey: process.env.DEEPSEEK_API_KEY,
  baseURL: "https://api.deepseek.com"
});

type TaskComplexity = "simple" | "moderate" | "complex";

// 任务复杂度评估（简单关键词规则，实际生产中可用 LLM 分类）
function assessComplexity(task: string): TaskComplexity {
  const complexKeywords = [
    "证明", "推导", "分析", "为什么", "逻辑", "数学", "算法",
    "prove", "derive", "reasoning", "logic", "math"
  ];

  const simpleKeywords = [
    "翻译", "总结", "改写", "格式化",
    "translate", "summarize", "format"
  ];

  const lowerTask = task.toLowerCase();

  if (complexKeywords.some(kw => lowerTask.includes(kw))) {
    return "complex";  // 复杂推理 → R1
  }
  if (simpleKeywords.some(kw => lowerTask.includes(kw))) {
    return "simple";   // 简单任务 → V3
  }
  return "moderate";   // 中等任务 → V3
}

interface RoutedResponse {
  model: string;
  complexity: TaskComplexity;
  content: string;
  cost_estimate: number;  // 美元
}

async function smartQuery(task: string): Promise<RoutedResponse> {
  const complexity = assessComplexity(task);

  // 根据复杂度选择模型：只有明确的推理任务才用 R1（因为 R1 更贵且不支持工具）
  const model = complexity === "complex"
    ? "deepseek-reasoner"  // R1
    : "deepseek-chat";     // V3

  console.log(`任务复杂度: ${complexity} → 使用模型: ${model}`);

  const response = await deepseekClient.chat.completions.create({
    model,
    messages: [{ role: "user", content: task }]
  });

  const usage = response.usage!;
  const inputTokens = usage.prompt_tokens;
  const outputTokens = usage.completion_tokens;

  // 实时成本计算
  const prices = {
    "deepseek-chat":     { input: 0.27, output: 1.10 },
    "deepseek-reasoner": { input: 0.55, output: 2.19 }
  };

  const price = prices[model as keyof typeof prices];
  const cost = (inputTokens * price.input + outputTokens * price.output) / 1_000_000;

  return {
    model,
    complexity,
    content: response.choices[0].message.content ?? "",
    cost_estimate: cost
  };
}

// 测试路由决策
async function testRouting() {
  const tasks = [
    "把这段话翻译成英文：今天天气很好",  // simple → V3
    "解释 React hooks 的工作原理",          // moderate → V3
    "证明：如果 n 是完全平方数，那么 n+1 不是完全平方数（假设 n > 0）" // complex → R1
  ];

  for (const task of tasks) {
    console.log(`\n任务: ${task.substring(0, 50)}...`);
    const result = await smartQuery(task);
    console.log(`模型: ${result.model}`);
    console.log(`成本估算: $${result.cost_estimate.toFixed(6)}`);
    console.log(`回答: ${result.content.substring(0, 100)}...`);
  }
}

6. V3 工具调用示例

// src/v3-tools.ts
// DeepSeek-V3 完全支持工具调用，格式与 OpenAI 相同
// 注意：R1 不支持工具调用，只有 V3 支持

import { deepseekClient, DEEPSEEK_V3 } from "./deepseek-client";

const tools: any[] = [
  {
    type: "function",
    function: {
      name: "search_web",
      description: "搜索互联网获取最新信息",
      parameters: {
        type: "object",
        properties: {
          query: { type: "string", description: "搜索关键词" }
        },
        required: ["query"]
      }
    }
  }
];

// 模拟搜索工具（实际项目中接入真实搜索 API）
function searchWeb(query: string): string {
  return JSON.stringify({
    results: [
      { title: `${query} 相关结果1`, snippet: "这是搜索结果摘要..." },
      { title: `${query} 相关结果2`, snippet: "另一个搜索结果..." }
    ]
  });
}

async function agentWithV3(question: string) {
  const messages: any[] = [
    { role: "user", content: question }
  ];

  // Agent 循环（与 OpenAI 版本完全相同的逻辑，因为格式兼容）
  while (true) {
    const response = await deepseekClient.chat.completions.create({
      model: DEEPSEEK_V3,  // 只有 V3 支持工具调用
      tools,
      tool_choice: "auto",
      messages
    });

    const choice = response.choices[0];
    messages.push(choice.message);

    if (choice.finish_reason === "tool_calls") {
      for (const toolCall of choice.message.tool_calls!) {
        const args = JSON.parse(toolCall.function.arguments);
        const result = searchWeb(args.query);
        messages.push({
          role: "tool",
          tool_call_id: toolCall.id,
          content: result
        });
      }
    } else {
      return choice.message.content;
    }
  }
}

agentWithV3("DeepSeek-R1 最新进展是什么？").then(console.log);

7. 成本对比计算器

// src/cost-calculator.ts

interface ModelPricing {
  input: number;  // 美元 / 1M tokens
  output: number;
  name: string;
}

const PRICING: Record<string, ModelPricing> = {
  "gpt-4o":             { name: "GPT-4o",             input: 2.50, output: 10.00 },
  "gpt-4o-mini":        { name: "GPT-4o Mini",         input: 0.15, output: 0.60  },
  "claude-sonnet-4-6":  { name: "Claude Sonnet 4.6",   input: 3.00, output: 15.00 },
  "deepseek-chat":      { name: "DeepSeek-V3",         input: 0.27, output: 1.10  },
  "deepseek-reasoner":  { name: "DeepSeek-R1",         input: 0.55, output: 2.19  },
  "gemini-2.0-flash":   { name: "Gemini 2.0 Flash",    input: 0.10, output: 0.40  },
};

function calculateMonthlyCost(
  modelId: string,
  dailyRequests: number,
  avgInputTokens: number,
  avgOutputTokens: number
): void {
  const model = PRICING[modelId];
  if (!model) throw new Error(`未知模型: ${modelId}`);

  const monthlyRequests = dailyRequests * 30;
  const totalInput = (monthlyRequests * avgInputTokens) / 1_000_000;
  const totalOutput = (monthlyRequests * avgOutputTokens) / 1_000_000;

  const inputCost = totalInput * model.input;
  const outputCost = totalOutput * model.output;
  const totalCost = inputCost + outputCost;

  console.log(`${model.name}:`);
  console.log(`  月请求数: ${monthlyRequests.toLocaleString()}`);
  console.log(`  输入成本: $${inputCost.toFixed(2)}`);
  console.log(`  输出成本: $${outputCost.toFixed(2)}`);
  console.log(`  月总成本: $${totalCost.toFixed(2)}`);
  console.log();
}

// 场景：日均 1000 次请求，平均 2000 输入 + 500 输出 tokens
console.log("=== 月成本对比（1000次/天，2000输入+500输出tokens）===\n");
for (const modelId of Object.keys(PRICING)) {
  calculateMonthlyCost(modelId, 1000, 2000, 500);
}

注意事项

注意点 说明 示例 / 建议
R1 不支持 system message 向 R1 发送 role: "system" 的消息会被忽略，不会报错但不生效 R1 任务直接用 role: "user" 传递指令，把规则写在用户消息中
R1 不支持 Function Calling 工具调用只在 V3 上可用；R1 专注于推理，不支持工具 需要工具调用的任务必须用 V3；需要推理的任务用 R1
reasoning_content 只在非流式时直接可取 非流式响应中 message.reasoning_content 是完整字符串；流式时在 delta.reasoning_content 增量输出 流式场景需要拼接所有 delta.reasoning_content 片段
TypeScript 类型缺失 reasoning_content OpenAI SDK 的类型定义没有 reasoning_content，直接访问会报类型错误 使用类型断言：(message as any).reasoning_content 或扩展类型接口
缓存价格与非缓存价格不同 DeepSeek 对命中缓存的 tokens 有折扣价格（约半价） 重复前缀的请求（如固定 system prompt）可大幅节省成本
R1 响应较慢 推理过程本身需要生成大量 tokens，响应时间比 V3 长 对延迟敏感的场景用 V3；批量异步任务可用 R1
并发限制 DeepSeek 免费层有并发和速率限制，付费层限制更宽松 高并发场景需要申请付费 API 并实现限流重试逻辑
