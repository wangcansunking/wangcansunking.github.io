第4课：第一个工具（Tool Use）

学习目标

  1. 理解 Tool Use（也叫 Function Calling）的完整工作流程
  2. 定义工具 schema（JSON Schema）
  3. 处理 tool_calls 响应
  4. 把工具结果发回给模型

Tool Use 是什么？

Tool Use 让模型能"调用"你定义的函数。流程如下：

你          模型             你的代码
─────────────────────────────────────────
问题 ──────→ 思考需要什么工具
            ← 返回 tool_calls 列表
执行工具 ──→
            工具结果 ───────→
            ← 结合结果给出最终回答

关键理解：模型只是"声明"要调用工具，真正执行的是你的代码。这保证了安全性。

OpenAI 兼容协议下三家（DeepSeek / Kimi / Anthropic）都支持 function calling，字段名一致，本课的代码三家通跑。

第一步：定义工具

工具定义包含三部分：名称、描述、参数 Schema。OpenAI 兼容协议要求外面再包一层 { type: 'function', function: {...} }：

const tools = [
  {
    type: "function" as const,
    function: {
      name: "get_current_time",
      description: "获取当前时间和日期。当用户询问时间相关问题时调用此工具。",
      parameters: {
        type: "object",
        properties: {
          timezone: {
            type: "string",
            description: "时区，例如 'Asia/Shanghai'，默认为 UTC",
          },
        },
        required: [],  // timezone 是可选的
      },
    },
  },
];

描述很重要！ 模型根据描述判断何时使用工具。描述越清晰，模型越准确。

第二步：发送请求

import { client, model } from "../shared/llmClient";

const response = await client.chat.completions.create({
  model,                          // 默认 deepseek-chat
  max_tokens: 1024,
  tools,                          // 把工具列表传给模型
  messages: [
    { role: "user", content: "现在几点了？" },
  ],
});

第三步：检查是否触发工具

const message = response.choices[0].message;

if (response.choices[0].finish_reason === "tool_calls" && message.tool_calls?.length) {
  // 模型要调用工具！遍历 tool_calls
  for (const call of message.tool_calls) {
    console.log("工具名称:", call.function.name);        // "get_current_time"
    console.log("调用参数:", call.function.arguments);   // '{"timezone":"Asia/Shanghai"}' (JSON 字符串)
    console.log("调用 ID:", call.id);                    // "call_xxx"（结果回传时需要）
  }
}

注意：call.function.arguments 是 JSON 字符串（不是对象），需要 JSON.parse() 后再用。

第四步：执行工具，把结果发回

// 实际执行工具
function executeTool(name: string, input: Record<string, unknown>): string {
  if (name === "get_current_time") {
    const tz = (input.timezone as string) ?? "Asia/Shanghai";
    return new Date().toLocaleString("zh-CN", { timeZone: tz });
  }
  return "工具不存在";
}

const call = message.tool_calls![0]!;
const args = JSON.parse(call.function.arguments);
const result = executeTool(call.function.name, args);

// 把工具结果追加到对话，让模型继续
const followUp = await client.chat.completions.create({
  model,
  max_tokens: 1024,
  tools,
  messages: [
    { role: "user", content: "现在几点了？" },
    message,                                              // ← 模型的 tool_calls 请求
    {
      role: "tool",
      tool_call_id: call.id,                              // ← 必须和 tool_call 的 id 匹配
      content: result,                                     // ← 工具执行结果
    },
  ],
});

console.log("最终回答:", followUp.choices[0].message.content);

完整流程代码

import { client, model } from "../shared/llmClient";

const tools = [{
  type: "function" as const,
  function: {
    name: "get_current_time",
    description: "获取当前时间和日期",
    parameters: {
      type: "object",
      properties: {
        timezone: { type: "string", description: "时区，如 Asia/Shanghai" },
      },
      required: [],
    },
  },
}];

// 第一次请求
const response = await client.chat.completions.create({
  model,
  max_tokens: 1024,
  tools,
  messages: [{ role: "user", content: "北京现在几点？" }],
});

const msg = response.choices[0].message;

if (msg.tool_calls?.length) {
  const call = msg.tool_calls[0]!;
  const args = JSON.parse(call.function.arguments) as { timezone?: string };

  // 执行工具
  const timeResult = new Date().toLocaleString("zh-CN", {
    timeZone: args.timezone ?? "Asia/Shanghai",
  });

  // 回传结果，获取最终答案
  const final = await client.chat.completions.create({
    model,
    max_tokens: 1024,
    tools,
    messages: [
      { role: "user", content: "北京现在几点？" },
      msg,
      { role: "tool", tool_call_id: call.id, content: timeResult },
    ],
  });

  console.log(final.choices[0].message.content);
}

工具调用流程总结

用户问题
  ↓
client.chat.completions.create({ tools, messages })
  ↓
finish_reason === "tool_calls"?
  ↓ Yes
遍历 tool_calls，执行本地函数
  ↓
把 tool 结果追加到 messages，再次 create
  ↓
finish_reason === "stop"
  ↓
读取最终 message.content

常见错误

错误 原因
tool_call_id 不匹配 回传时必须等于 tool_calls[].id
忘记 JSON.parse(arguments) arguments 是字符串不是对象
第二次请求漏带 tools 第二次请求也要带 tools 列表
把模型回的 message 转成简单字符串 必须把整个 message 对象（含 tool_calls）作为 assistant 消息原样追加回 messages

关于 Anthropic 原生协议

Anthropic 自家 SDK 也支持 OpenAI 兼容端点（本课代码就能直接用）。如果你确实要用 Anthropic 原生 messages.create 协议，字段映射是：

OpenAI 兼容（本课） Anthropic 原生
tools[].function.parameters tools[].input_schema
message.tool_calls[] content[].type === 'tool_use'
{ role: 'tool', tool_call_id, content } { type: 'tool_result', tool_use_id, content }
finish_reason === 'tool_calls' stop_reason === 'tool_use'
