第1课：Agent Loop — Bash is All You Need

介绍

一个 AI Agent 的本质，是一个带工具调用的循环。本课用 60 行 Python 实现完整的 Agent Loop：模型决策 → 执行工具 → 反馈结果 → 继续推理，无限循环直到任务完成。这是 Claude Code、Cursor、Devin 背后共同的核心模式。

源码参考：github.com/shareAI-lab/learn-claude-code — agents/s01_agent_loop.py

遇到什么问题

LLM 天生是无状态的"问答机"：一次输入，一次输出。但真实任务需要多步操作（读文件、执行命令、修改代码）。如何让模型持续感知环境、采取行动？

flowchart LR
    A[用户输入] --> B[LLM 推理]
    B --> C{stop_reason?}
    C -->|tool_use| D[执行工具]
    D --> E[追加结果到 messages]
    E --> B
    C -->|end_turn| F[返回最终结果]

有哪些方案

方案 A：每次对话独立调用

每轮单独发送一个问题，不携带历史记录。

✅ 最简单 ❌ 模型无法感知多步状态，无法执行需要上下文的任务

方案 B：完整对话历史 + 手动工具分发

维护 messages[] 数组，检测 stop_reason == "tool_use" 后手动执行工具。

✅ 模型可以跨轮次推理 ✅ 工具结果反馈回模型 ❌ 需要手写循环逻辑

方案 C：Agent Loop（推荐）

while response.stop_reason == "tool_use":
    执行所有工具调用
    把 tool_result 追加到 messages
    再次调用 LLM

✅ 自动运行直到任务完成 ✅ 模型自主决定何时停止 ✅ 60 行实现完整闭环

怎么选择

方案 C。Agent Loop 是一切 Agent 的基础骨架。无论多复杂的 Agent（多 Agent 团队、并行任务、记忆系统），最底层都是这个循环。先理解它，再叠加其他机制。

怎么执行

完整实现

# agents/s01_agent_loop.py
import os
import subprocess
from anthropic import Anthropic
from dotenv import load_dotenv

load_dotenv()

client = Anthropic()
MODEL = os.environ["MODEL_ID"]   # e.g. "claude-opus-4-5-20251101"

SYSTEM = f"You are a coding agent at {os.getcwd()}. Use bash to solve tasks. Act, don't explain."

# ── 工具定义：告诉模型它能做什么 ──
TOOLS = [{
    "name": "bash",
    "description": "Run a shell command.",
    "input_schema": {
        "type": "object",
        "properties": {"command": {"type": "string"}},
        "required": ["command"],
    },
}]


def run_bash(command: str) -> str:
    # 安全检查：拦截危险命令
    dangerous = ["rm -rf /", "sudo", "shutdown", "reboot", "> /dev/"]
    if any(d in command for d in dangerous):
        return "Error: Dangerous command blocked"
    try:
        r = subprocess.run(
            command, shell=True, cwd=os.getcwd(),
            capture_output=True, text=True, timeout=120
        )
        out = (r.stdout + r.stderr).strip()
        return out[:50000] if out else "(no output)"
    except subprocess.TimeoutExpired:
        return "Error: Timeout (120s)"


# ── 核心循环：Agent 的心脏 ──
def agent_loop(messages: list):
    while True:
        response = client.messages.create(
            model=MODEL,
            system=SYSTEM,
            messages=messages,
            tools=TOOLS,
            max_tokens=8000,
        )
        # 把模型回复追加到对话历史
        messages.append({"role": "assistant", "content": response.content})

        # 如果模型不再需要工具 → 任务完成
        if response.stop_reason != "tool_use":
            return

        # 执行所有工具调用，收集结果
        results = []
        for block in response.content:
            if block.type == "tool_use":
                print(f"\033[33m$ {block.input['command']}\033[0m")
                output = run_bash(block.input["command"])
                print(output[:200])
                results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": output,
                })

        # 把工具结果作为 user 消息追加 → 触发下一轮推理
        messages.append({"role": "user", "content": results})


# ── 交互入口 ──
if __name__ == "__main__":
    history = []
    while True:
        try:
            query = input("\033[36ms01 >> \033[0m")
        except (EOFError, KeyboardInterrupt):
            break
        if query.strip().lower() in ("q", "exit", ""):
            break
        history.append({"role": "user", "content": query})
        agent_loop(history)
        # 打印最终文本回复
        for block in history[-1]["content"]:
            if hasattr(block, "text"):
                print(block.text)
        print()

关键设计决策

决策 原因
messages 作为参数传入 调用方控制历史，Agent Loop 无状态
stop_reason != "tool_use" 退出 让模型决定何时完成，不硬编码步骤数
工具结果作为 user 消息 Anthropic API 要求的对话格式
一次返回所有工具结果 支持并行工具调用（模型可以同时调用多个工具）
