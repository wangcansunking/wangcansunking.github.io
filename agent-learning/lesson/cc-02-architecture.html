第2课：整体架构 — Agent Loop 是怎么运转的

介绍

上一课我们知道了 Claude Code 用"极简工具"的原因。本课把架构全局拆开看：一次用户请求，经历了什么，才变成最终结果？

看懂这张图，你就能自己从零构建一个 Claude Code。

遇到什么问题

用户说："帮我给这个函数加单元测试。"

AI 需要：

  1. 理解函数在哪、长什么样
  2. 知道项目用什么测试框架
  3. 写出测试代码
  4. 验证测试能跑通
  5. 如果失败，自己修复

这一系列步骤不是一次 API 调用能完成的——AI 需要一个循环，每步读取上一步的结果，决定下一步做什么。

有哪些方案

方案 A：链式调用（Chain）

方案 B：Agent Loop（循环决策）

✅ 能处理任意复杂任务 ✅ 能从错误中恢复 ❌ 需要设计好终止条件，避免死循环

方案 C：多 Agent（分治）

✅ 并行高效 ❌ 协调复杂，适合大任务

怎么选择

核心用方案 B（Agent Loop），复杂任务叠加方案 C（SubAgent）。

Claude Code 的核心就是一个 Loop：

while (true) {
  response = callClaudeAPI(messages, tools)

  if (response.stop_reason === 'end_turn') break  // 任务完成

  // 执行 AI 要求的工具
  toolResults = executeTools(response.tool_use_blocks)

  // 把结果塞回历史，继续循环
  messages.push(assistantMessage, toolResults)
}

这个 Loop 只有 10 行核心逻辑，但能完成几乎所有编程任务。

五大组件详解

1. 工具系统（Tool System）

工具的本质：把 AI 的文字意图转化为实际操作。

interface Tool {
  name: string
  description: string      // AI 根据这个决定何时调用
  input_schema: JSONSchema
}

// 工具执行映射
const toolHandlers = {
  bash:  (input) => exec(input.command),
  read:  (input) => fs.readFile(input.path, 'utf-8'),
  write: (input) => fs.writeFile(input.path, input.content),
  edit:  (input) => applyPatch(input.path, input.old, input.new),
  glob:  (input) => glob(input.pattern),
  grep:  (input) => grep(input.pattern, input.path),
}

3. 权限系统

type PermissionLevel = 'allow' | 'ask' | 'deny'

// 根据工具和参数决定是否需要用户确认
function checkPermission(tool: string, input: any): PermissionLevel {
  if (/rm -rf/.test(input.command)) return 'deny'
  if (tool === 'bash' && input.command.includes('push')) return 'ask'
  return 'allow'
}

4. Skill 系统

Skill = 预设 Prompt + 工具组合，用 / 触发：

const skills = {
  '/commit': { prompt: '分析 diff，生成 commit message 并提交', tools: ['bash'] },
  '/review': { prompt: '审查当前改动，关注安全和性能', tools: ['bash', 'read'] },
  '/test':   { prompt: '运行测试，自动修复失败的用例', tools: ['bash', 'edit'] },
}

5. SubAgent（子 Agent）

主 Agent 可以把子任务委派给独立的子 Agent：

// 主 Agent 调用子 Agent 工具
const result = await spawnSubagent({
  task: "运行测试并修复所有失败用例",
  tools: ['bash', 'read', 'edit'],
  maxTurns: 20,
})

子 Agent 有独立的上下文和权限，互不干扰。
