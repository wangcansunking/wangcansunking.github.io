第8课：Fallback 与可靠性 — 构建不宕机的 AI 服务

介绍

生产环境中 LLM API 会有超时、限流、服务中断。依赖单一 API 意味着一旦出故障整个 AI 服务就崩了。本课构建多层可靠性保障：多模型 Fallback Chain、指数退避重试、熔断器模式（Circuit Breaker）、健康检查与自动恢复，确保服务 99.9% 可用。

可用性目标：

可用性 每年宕机时间 每月宕机时间 需要的措施
99% 87.6 小时 7.3 小时 无重试也能达到
99.9% 8.76 小时 43.8 分钟 需要重试 + Fallback
99.99% 52.6 分钟 4.4 分钟 需要熔断器 + 多区域

实现 99.9% 可用性只需三个组件：指数退避重试 + 熔断器 + 多模型 Fallback。

整体架构：三层可靠性保障

flowchart TD

    B --> C{CircuitBreaker\n检查主模型}
    C -->|OPEN 已熔断| D[跳过，直接尝试备用]
    C -->|CLOSED/HALF_OPEN| E[withRetry\n指数退避重试]

    E -->|第1次调用| F[主模型 Provider 1\ne.g. OpenAI gpt-4o]

    F -->|429 Rate Limit| H[等待 1s → 2s → 4s\n最多重试3次]
    H -->|重试成功| G

    I --> D
    D --> J{CircuitBreaker\n检查备用模型1}

三层保障职责：

  • 第一层（重试）：处理偶发性错误（网络抖动、瞬时限流）
  • 第二层（熔断器）：隔离持续故障的模型，避免无效等待
  • 第三层（Fallback）：当主模型不可用时自动切换备用模型

故障类型与对策

故障类型 HTTP 状态 是否重试 对策
Rate Limit 429 ✅ 退避重试 指数退避，等待后重试；考虑切换备用
临时服务错误 500 / 503 ✅ 退避重试 最多 3 次，间隔递增
超时 timeout ✅ 退避重试 缩短超时后切备用模型
网络断开 ECONNRESET ✅ 退避重试 重连后重试
请求无效 400 ❌ 不重试 参数有问题，重试无意义，直接报错
认证失败 401 ❌ 不重试 API Key 问题，需要人工介入
配额超限 402 / 429（账户级） ❌ 不重试 需要充值或升级计划，切换备用
内容被拒绝 400（safety） ❌ 不重试 内容违规，重试无效

三种模式详解

模式一：指数退避重试

sequenceDiagram
    participant C as 客户端
    participant A as LLM API

    C->>A: 第1次调用
    A-->>C: 429 Rate Limit

    Note over C: 等待 1000ms（+ jitter）

    C->>A: 第2次重试
    A-->>C: 429 Rate Limit

    Note over C: 等待 2000ms（+ jitter）

    C->>A: 第3次重试
    A-->>C: 200 成功

    Note over C,A: 公式：waitTime = min(initialDelay × 2^attempt + jitter, maxDelay)

退避公式说明：

waitTime = min(initialDelay × 2^attempt + jitter, maxDelay)

示例：initialDelay=1000ms, maxDelay=30000ms
  第1次失败后等待：min(1000 × 2^0 + jitter, 30000) ≈ 1000-1300ms
  第2次失败后等待：min(1000 × 2^1 + jitter, 30000) ≈ 2000-2600ms
  第3次失败后等待：min(1000 × 2^2 + jitter, 30000) ≈ 4000-5200ms

为什么需要 jitter（抖动）：如果所有实例在同一时间重试（thundering herd），会再次打爆 API。加入随机抖动（Math.random() × delay × 0.3）能将重试分散在时间轴上。

模式二：熔断器状态机

stateDiagram-v2
    [*] --> CLOSED: 初始状态

    CLOSED --> OPEN: 失败次数 >= 5\n（failureThreshold）
    OPEN --> HALF_OPEN: 等待超时 60s\n（timeout）
    HALF_OPEN --> CLOSED: 连续成功 >= 2次\n（successThreshold）
    HALF_OPEN --> OPEN: 任意1次失败

    CLOSED: CLOSED 关闭\n正常放行所有请求
    OPEN: OPEN 打开\n拒绝所有请求（快速失败）
    HALF_OPEN: HALF_OPEN 半开\n放行少量探测请求

三种状态说明：

状态 含义 行为 转换条件
CLOSED 正常运行 放行所有请求，统计失败次数 失败 >= 5 次 → OPEN
OPEN 已熔断 拒绝所有请求，立即抛出错误（快速失败） 超时 60s → HALF_OPEN
HALF_OPEN 探测恢复 放行少量请求，观察是否恢复 成功 >= 2 次 → CLOSED；失败 → OPEN

熔断器的价值：当一个模型持续故障时，不熔断的话每次请求都要等到超时（可能 30s），有了熔断器可以立即跳过故障节点（1ms），将时间花在可用的备用模型上。

手动实现方案

指数退避重试（retry.ts）

// src/fallback/retry.ts

export interface RetryOptions {
  maxAttempts: number;     // 最大重试次数
  initialDelay: number;    // 初始等待时间（ms）
  maxDelay: number;        // 最大等待时间（ms）
  backoffMultiplier: number; // 退避倍数
  retryableErrors?: (error: Error) => boolean; // 哪些错误需要重试
}

const DEFAULT_RETRY_OPTIONS: RetryOptions = {
  maxAttempts: 3,
  initialDelay: 1000,
  maxDelay: 30000,
  backoffMultiplier: 2,
  retryableErrors: (error) => {
    const message = error.message.toLowerCase();
    return (
      message.includes("rate limit") ||
      message.includes("429") ||
      message.includes("503") ||
      message.includes("timeout") ||
      message.includes("econnreset")
    );
  },
};

export async function withRetry<T>(
  fn: () => Promise<T>,
  options: Partial<RetryOptions> = {}
): Promise<T> {
  const opts = { ...DEFAULT_RETRY_OPTIONS, ...options };
  let lastError: Error;
  let delay = opts.initialDelay;

  for (let attempt = 1; attempt <= opts.maxAttempts; attempt++) {
    try {
      return await fn();
    } catch (error) {
      lastError = error as Error;

      const isRetryable = opts.retryableErrors?.(lastError) ?? true;
      const isLastAttempt = attempt === opts.maxAttempts;

      if (!isRetryable || isLastAttempt) {
        throw lastError;
      }

      // 指数退避 + 随机抖动（避免 thundering herd）
      const jitter = Math.random() * delay * 0.3;
      const waitTime = Math.min(delay + jitter, opts.maxDelay);

      console.log(
        `[Retry] 第 ${attempt} 次失败: ${lastError.message}，等待 ${Math.round(waitTime)}ms 后重试`
      );

      await sleep(waitTime);
      delay = Math.min(delay * opts.backoffMultiplier, opts.maxDelay);
    }
  }

  throw lastError!;
}

function sleep(ms: number): Promise<void> {
  return new Promise((resolve) => setTimeout(resolve, ms));
}

熔断器（circuit_breaker.ts）

// src/fallback/circuit_breaker.ts

type CircuitState = "CLOSED" | "OPEN" | "HALF_OPEN";

interface CircuitBreakerOptions {
  failureThreshold: number;   // 触发熔断的失败次数
  successThreshold: number;   // 半开→关闭需要的成功次数
  timeout: number;            // 熔断持续时间（ms），之后进入 HALF_OPEN
  monitorInterval: number;    // 统计窗口时间（ms）
}

const DEFAULT_CB_OPTIONS: CircuitBreakerOptions = {
  failureThreshold: 5,       // 5次失败后熔断
  successThreshold: 2,       // 2次成功后恢复
  timeout: 60000,            // 熔断 60 秒
  monitorInterval: 120000,   // 2分钟统计窗口
};

export class CircuitBreaker {
  private state: CircuitState = "CLOSED";
  private failureCount = 0;
  private successCount = 0;
  private lastFailureTime = 0;
  private readonly name: string;
  private readonly options: CircuitBreakerOptions;

  constructor(name: string, options: Partial<CircuitBreakerOptions> = {}) {
    this.name = name;
    this.options = { ...DEFAULT_CB_OPTIONS, ...options };
  }

  get currentState(): CircuitState {
    return this.state;
  }

  isOpen(): boolean {
    if (this.state === "OPEN") {
      // 检查是否到了尝试恢复的时间
      if (Date.now() - this.lastFailureTime > this.options.timeout) {
        this.state = "HALF_OPEN";
        this.successCount = 0;
        console.log(`[CircuitBreaker:${this.name}] 进入 HALF_OPEN 状态，尝试恢复`);
        return false;
      }
      return true;
    }
    return false;
  }

  async execute<T>(fn: () => Promise<T>): Promise<T> {
    if (this.isOpen()) {
      throw new Error(
        `[CircuitBreaker:${this.name}] 熔断器已开启，拒绝请求`
      );
    }

    try {
      const result = await fn();
      this.onSuccess();
      return result;
    } catch (error) {
      this.onFailure();
      throw error;
    }
  }

  private onSuccess(): void {
    this.failureCount = 0;

    if (this.state === "HALF_OPEN") {
      this.successCount++;
      if (this.successCount >= this.options.successThreshold) {
        this.state = "CLOSED";
        console.log(`[CircuitBreaker:${this.name}] 恢复正常，进入 CLOSED 状态`);
      }
    }
  }

  private onFailure(): void {
    this.lastFailureTime = Date.now();
    this.failureCount++;

    if (
      this.state === "HALF_OPEN" ||
      this.failureCount >= this.options.failureThreshold
    ) {
      this.state = "OPEN";
      console.log(
        `[CircuitBreaker:${this.name}] 触发熔断，进入 OPEN 状态，${this.options.timeout / 1000}s 后尝试恢复`
      );
    }
  }

  getStats() {
    return {
      name: this.name,
      state: this.state,
      failureCount: this.failureCount,
      successCount: this.successCount,
      lastFailureTime: this.lastFailureTime
        ? new Date(this.lastFailureTime).toISOString()
        : null,
    };
  }
}

多模型 Fallback Chain（fallback_chain.ts）

// src/fallback/fallback_chain.ts
import type { LLMAdapter, LLMMessage, ChatOptions, LLMResponse } from "../unified/types";
import { CircuitBreaker } from "./circuit_breaker";
import { withRetry } from "./retry";

interface FallbackNode {
  adapter: LLMAdapter;
  circuitBreaker: CircuitBreaker;
  priority: number;  // 数字越小优先级越高
}

export class FallbackChain {
  private nodes: FallbackNode[];

  constructor(adapters: LLMAdapter[]) {
    this.nodes = adapters.map((adapter, index) => ({
      adapter,
      circuitBreaker: new CircuitBreaker(adapter.name),
      priority: index,
    }));
  }

  async chat(
    messages: LLMMessage[],
    options?: ChatOptions
  ): Promise<LLMResponse & { provider: string }> {
    const errors: { provider: string; error: string }[] = [];

    // 按优先级尝试每个节点
    for (const node of this.nodes) {
      if (node.circuitBreaker.isOpen()) {
        console.log(`[Fallback] ${node.adapter.name} 熔断中，跳过`);
        continue;
      }

      try {
        const result = await node.circuitBreaker.execute(() =>
          withRetry(
            () => node.adapter.chat(messages, options),
            {
              maxAttempts: 2,
              initialDelay: 1000,
            }
          )
        );

        if (node.priority > 0) {
          console.log(
            `[Fallback] 使用备用模型 ${node.adapter.name}（主模型不可用）`
          );
        }

        return { ...result, provider: node.adapter.name };
      } catch (error) {
        const errorMsg = error instanceof Error ? error.message : String(error);
        errors.push({ provider: node.adapter.name, error: errorMsg });
        console.error(`[Fallback] ${node.adapter.name} 失败: ${errorMsg}`);
      }
    }

    // 所有模型都失败
    throw new Error(
      `所有模型均失败:\n${errors.map((e) => `  ${e.provider}: ${e.error}`).join("\n")}`
    );
  }

  async *stream(
    messages: LLMMessage[],
    options?: ChatOptions
  ): AsyncGenerator<string> {
    for (const node of this.nodes) {
      if (node.circuitBreaker.isOpen()) continue;

      try {
        const generator = node.adapter.stream(messages, options);
        // 先读第一个 chunk 验证连接正常
        const first = await generator.next();
        if (!first.done) {
          yield first.value;
          for await (const chunk of generator) {
            yield chunk;
          }
          return;
        }
      } catch (error) {
        console.error(`[Fallback Stream] ${node.adapter.name} 失败，切换下一个`);
      }
    }

    throw new Error("所有模型流式输出均失败");
  }

  getHealthStatus() {
    return this.nodes.map((node) => ({
      provider: node.adapter.name,
      priority: node.priority,
      circuitBreaker: node.circuitBreaker.getStats(),
    }));
  }
}

健康检查（health_check.ts）

// src/fallback/health_check.ts
import type { LLMAdapter } from "../unified/types";

interface HealthStatus {
  provider: string;
  healthy: boolean;
  latencyMs?: number;
  error?: string;
  lastChecked: string;
}

export class HealthChecker {
  private statuses: Map<string, HealthStatus> = new Map();
  private checkIntervals: Map<string, NodeJS.Timer> = new Map();

  // 单次健康检查
  async checkAdapter(adapter: LLMAdapter): Promise<HealthStatus> {
    const start = Date.now();
    try {
      await adapter.chat(
        [{ role: "user", content: "ping" }],
        { maxTokens: 5 }   // 最小 token 限制，减少成本
      );

      const status: HealthStatus = {
        provider: adapter.name,
        healthy: true,
        latencyMs: Date.now() - start,
        lastChecked: new Date().toISOString(),
      };

      this.statuses.set(adapter.name, status);
      return status;
    } catch (error) {
      const status: HealthStatus = {
        provider: adapter.name,
        healthy: false,
        error: error instanceof Error ? error.message : String(error),
        lastChecked: new Date().toISOString(),
      };

      this.statuses.set(adapter.name, status);
      return status;
    }
  }

  // 启动定期健康检查
  startPeriodicCheck(
    adapters: LLMAdapter[],
    intervalMs: number = 30000
  ): void {
    for (const adapter of adapters) {
      // 立即做一次检查
      this.checkAdapter(adapter).then((status) => {
        console.log(
          `[HealthCheck] ${status.provider}: ${status.healthy ? "✓" : "✗"} ${
            status.latencyMs ? `${status.latencyMs}ms` : status.error
          }`
        );
      });

      // 定期检查
      const timer = setInterval(async () => {
        const status = await this.checkAdapter(adapter);
        console.log(
          `[HealthCheck] ${status.provider}: ${status.healthy ? "健康" : "故障"}`
        );
      }, intervalMs);

      this.checkIntervals.set(adapter.name, timer);
    }
  }

  stopAll(): void {
    for (const timer of this.checkIntervals.values()) {
      clearInterval(timer);
    }
    this.checkIntervals.clear();
  }

  getAllStatuses(): HealthStatus[] {
    return Array.from(this.statuses.values());
  }

  isHealthy(providerName: string): boolean {
    return this.statuses.get(providerName)?.healthy ?? true;
  }
}

生产级使用示例（demo.ts）

// src/fallback/demo.ts
import { OpenAIAdapter, createDeepSeekAdapter } from "../unified/adapters/openai";
import { AnthropicAdapter } from "../unified/adapters/anthropic";
import { FallbackChain } from "./fallback_chain";
import { HealthChecker } from "./health_check";

// 构建 Fallback Chain：OpenAI 主力 → Anthropic 备用 → DeepSeek 兜底
const chain = new FallbackChain([
  new OpenAIAdapter({ defaultModel: "gpt-4o" }),      // 优先级 0（主）
  new AnthropicAdapter(),                              // 优先级 1（备）
  createDeepSeekAdapter(),                             // 优先级 2（兜底）
]);

// 健康检查
const healthChecker = new HealthChecker();
healthChecker.startPeriodicCheck(
  [new OpenAIAdapter(), new AnthropicAdapter(), createDeepSeekAdapter()],
  60000 // 每分钟检查一次
);

async function reliableChat(userInput: string) {
  try {
    const response = await chain.chat(
      [{ role: "user", content: userInput }],
      { temperature: 0.7 }
    );

    console.log(`[使用模型] ${response.provider}`);
    console.log(`[回复] ${response.content}`);
    return response;
  } catch (error) {
    console.error("所有模型均不可用:", error);
    // 降级：返回预设回复，不能直接崩溃
    return {
      content: "抱歉，AI 服务暂时不可用，请稍后重试。",
      provider: "fallback",
    };
  }
}

// 查看系统健康状态
function printHealthStatus() {
  const status = chain.getHealthStatus();
  console.log("\n=== 模型健康状态 ===");
  for (const s of status) {
    console.log(
      `${s.provider}: ${s.circuitBreaker.state} (失败: ${s.circuitBreaker.failureCount})`
    );
  }
}

// 运行演示
await reliableChat("什么是量子计算？");
printHealthStatus();

注意事项

注意点 说明
指数退避必须加随机抖动（jitter） 避免 thundering herd 问题：多个实例同时重试会再次打爆 API，加入 Math.random() × delay × 0.3 能将重试分散到不同时间点
熔断器超时建议 60s 不要太短（< 10s，频繁进入 HALF_OPEN 浪费请求）也不要太长（> 300s，故障恢复后等太久）
流式输出 Fallback 的特殊性 流式请求只能在收到第一个 chunk 后才能确认连接正常；如果在中间失败无法无缝切换，需要重置连接并重新请求备用模型
HealthCheck 用 "ping" + maxTokens:5 减少健康检查成本；每次 ping 消耗约 5-10 tokens，100 个模型 × 每分钟检查 = 每天约 $0.01
所有模型均失败时要有降级文案 不能直接崩溃抛出原始错误给用户；应返回预设的友好提示，并记录完整错误供排查
不重试 400/401/402 错误 这类错误重试无意义且浪费配额；retryableErrors 函数需要明确排除这些状态码
FallbackChain 不应跨任务类型 同一个 FallbackChain 内的模型应该能处理相同类型的任务；不要把纯文本模型和视觉模型混在同一个 chain 里
