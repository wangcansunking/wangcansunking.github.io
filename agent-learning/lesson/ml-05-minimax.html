第5课：MiniMax — 语音+文本多模态模型

介绍

MiniMax 提供文本、语音、视频生成能力，2025 年主力模型为：MiniMax-M1（推理增强旗舰，支持复杂任务）和 MiniMax-Text-01（100 万 token 超长上下文）。API 兼容 OpenAI 格式，同时提供独特的 TTS（文字转语音）和角色扮演能力。本课覆盖基础文本对话、TTS 语音生成、超长上下文使用，以及角色扮演系统 prompt 技巧，帮助构建能"说话"的 AI Agent。

MiniMax 最大的差异化优势是：其他平台基本没有的 TTS 能力，可以直接从文字生成高质量语音，无需第三方 TTS 服务。

整体架构：MiniMax 能力全景

flowchart TD
    MiniMax["MiniMax API<br/>api.minimax.chat"] --> TEXT["文本对话<br/>（OpenAI 兼容格式）"]
    MiniMax --> TTS["TTS 语音生成<br/>（专属 API）"]

    TEXT --> M1["MiniMax-M1<br/>复杂推理 / 旗舰任务"]
    TEXT --> T01["MiniMax-Text-01<br/>超长文档 100万 token"]
    TEXT --> ABAB65S["abab6.5s-chat<br/>快速轻量 日常对话"]
    TEXT --> ABAB65G["abab6.5g-chat<br/>高质量输出"]

    TTS --> SYNC["非流式批量<br/>一次性返回完整音频"]
    TTS --> STREAM["流式实时<br/>逐块返回 边生成边播放"]

    M1 --> PIPELINE["全链路语音对话"]
    ABAB65S --> PIPELINE
    SYNC --> PIPELINE
    STREAM --> PIPELINE

    PIPELINE --> OUTPUT["🎵 文本输入 → LLM生成 → TTS转语音 → 音频输出"]

这个架构的核心：两套端点配合使用。文本走 OpenAI 兼容端点，语音走专属 /v1/t2a_v2 端点，通过一个 MinimaxClient 封装类统一管理。

MiniMax 模型对比

模型 ID 类型 核心特点 适用场景
MiniMax-M1 推理增强旗舰 深度推理，支持复杂多步任务，2025 年最新 代码生成、数学推理、复杂分析
MiniMax-Text-01 超长上下文 100 万 token 上下文窗口，全球最长之一 长文档理解、大型代码库分析
abab6.5s-chat 快速轻量 响应速度快，成本低 日常对话、快速 Q&A、实时交互
abab6.5g-chat 高质量版 输出质量更高，适合精细任务 内容创作、高质量摘要

选型建议：

  • 日常对话、TTS 搭配 → abab6.5s-chat（速度快成本低）
  • 复杂推理、代码任务 → MiniMax-M1
  • 超长文档（>100K token）→ MiniMax-Text-01
  • 高质量内容创作 → abab6.5g-chat

双端点体系设计

MiniMax 有两个独立端点，这是与其他平台最大的架构差异：

flowchart LR
    APP["应用代码"] --> DECISION{请求类型?}

    DECISION -->|文本对话| TEXT_EP["POST /v1/chat/completions<br/>OpenAI 兼容格式"]
    DECISION -->|语音生成| TTS_EP["POST /v1/t2a_v2<br/>MiniMax 专属格式"]

    TEXT_EP --> OPENAI_SDK["可用 OpenAI SDK<br/>openai npm 包"]
    TTS_EP --> FETCH["只能用 fetch<br/>没有对应 SDK"]

    OPENAI_SDK --> RESP1["{ choices[0].message.content }"]
    FETCH --> RESP2["{ data: { audio: 'hex_string' } }"]
功能 端点 SDK 支持 响应格式
文本对话、流式输出 /v1/chat/completions OpenAI SDK 直接用 choices[0].message.content
Function Calling /v1/chat/completions OpenAI SDK 直接用 choices[0].message.tool_calls
TTS 非流式 /v1/t2a_v2 只能用 fetch data.audio（hex 编码）
TTS 流式 /v1/t2a_v2?stream=true 只能用 fetch 逐行 JSON，每行含 data.audio

遇到什么问题

MiniMax 拥有独特的 TTS 和角色扮演能力，但 API 使用方式需要特别注意：

  1. 双端点体系：文本用兼容 OpenAI 的端点，TTS 用专属端点 /v1/t2a_v2，两者不互通，需要封装两套客户端
  2. 音色选择：TTS 有预设音色 ID，需要查文档选择合适的，不同音色有明显风格差异
  3. 角色扮演格式：MiniMax 有专属的 bot_setting 字段用于角色定义（非标准 OpenAI 格式），用 system prompt 也能达到类似效果
  4. 流式音频：TTS 流式输出时每个 chunk 是独立的 JSON 行，包含 hex 编码音频，需要逐行解析后拼接

有哪些方案

方案 A：OpenAI SDK + MiniMax baseURL（基础文本）

import OpenAI from "openai";

const minimax = new OpenAI({
  apiKey: process.env.MINIMAX_API_KEY!,
  baseURL: "https://api.minimax.chat/v1",
});

优点：无需额外依赖，快速接入文本功能，流式输出、Function Calling 全支持。 缺点：无法访问 TTS、视频生成等专属 API，TTS 必须另外用 fetch。

方案 B：MiniMax 官方 API（完整功能）

// 直接调用 MiniMax REST API
const response = await fetch("https://api.minimax.chat/v1/t2a_v2", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.MINIMAX_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ text: "你好，世界", voice_id: "female-shaonv" }),
});

优点：支持 TTS、视频生成等全部独有功能。 缺点：需要手写 HTTP 客户端，没有 SDK 便利性，类型提示较弱。

推荐：两者组合使用

文本对话用方案 A（OpenAI SDK），TTS 语音生成用方案 B（原生 fetch）。封装成统一的 MinimaxClient 类，对外提供一致的调用接口。

核心设计：文字转语音流程

TTS 的数据流有一个容易踩坑的地方：音频以 hex 字符串形式返回，需要手动解码。

sequenceDiagram
    participant App as 应用代码
    participant API as MiniMax API
    participant FS as 文件系统

    App->>API: POST /v1/t2a_v2<br/>{ text, voice_id, speed, format }
    API-->>App: { data: { audio: "hex_string" } }

    Note over App: Buffer.from(hex, 'hex')

    App->>FS: fs.writeFileSync('output.mp3', buffer)
    Note over FS: 可播放的 MP3 文件

流式模式下的数据流略有不同：

sequenceDiagram
    participant App as 应用代码
    participant API as MiniMax API

    App->>API: POST /v1/t2a_v2<br/>{ stream: true, ... }

    loop 每个音频块
        API-->>App: JSON行: {"data":{"audio":"hex_chunk"}}
        Note over App: 解析 JSON → Buffer.from(hex, 'hex')<br/>→ 写入文件流
    end

    API-->>App: 流结束
    Note over App: 关闭文件流，MP3 完整

TTS 语音生成（POST /v1/t2a_v2）

// src/minimax/tts.ts
import * as fs from "fs";
import { MINIMAX_BASE_URL, API_KEY, VOICES } from "./client";

interface TTSOptions {
  text: string;
  voiceId?: string;
  speed?: number;   // 0.5 - 2.0，默认 1.0
  volume?: number;  // 0 - 10，默认 1
  pitch?: number;   // -12 - 12，默认 0
  outputFile?: string;
}

async function textToSpeech(options: TTSOptions): Promise<Buffer> {
  const {
    text,
    voiceId = VOICES.FEMALE_YOUNG,
    speed = 1.0,
    volume = 1,
    pitch = 0,
    outputFile,
  } = options;

  const response = await fetch(`${MINIMAX_BASE_URL}/t2a_v2`, {
    method: "POST",
    headers: {
      Authorization: `Bearer ${API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      model: "speech-01-turbo",
      text,
      stream: false,
      voice_setting: {
        voice_id: voiceId,
        speed,
        vol: volume,
        pitch,
      },
      audio_setting: {
        sample_rate: 32000,
        bitrate: 128000,
        format: "mp3",
        channel: 1,
      },
    }),
  });

  if (!response.ok) {
    const error = await response.json();
    throw new Error(`TTS 失败: ${JSON.stringify(error)}`);
  }

  const data = await response.json();

  // 关键：音频是 hex 编码，必须用 Buffer.from(hex, 'hex') 解码
  const audioBuffer = Buffer.from(data.data.audio, "hex");

  if (outputFile) {
    fs.writeFileSync(outputFile, audioBuffer);
    console.log(`音频已保存: ${outputFile}`);
  }

  return audioBuffer;
}

// 流式 TTS（边生成边播放）
async function streamingTTS(
  text: string,
  voiceId: string = VOICES.MALE_MATURE,
  outputFile: string = "output_stream.mp3"
): Promise<void> {
  const response = await fetch(`${MINIMAX_BASE_URL}/t2a_v2`, {
    method: "POST",
    headers: {
      Authorization: `Bearer ${API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      model: "speech-01-turbo",
      text,
      stream: true,
      voice_setting: {
        voice_id: voiceId,
        speed: 1.0,
        vol: 1,
        pitch: 0,
      },
      audio_setting: {
        format: "mp3",
        sample_rate: 32000,
      },
    }),
  });

  if (!response.ok || !response.body) throw new Error("流式 TTS 失败");

  const writeStream = fs.createWriteStream(outputFile);
  const reader = response.body.getReader();

  console.log("流式音频生成中...");
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    // MiniMax 流式返回：每行是一个 JSON，包含 hex 编码的音频片段
    const lines = new TextDecoder().decode(value).split("\n").filter(Boolean);
    for (const line of lines) {
      try {
        const json = JSON.parse(line);
        if (json.data?.audio) {
          const audioChunk = Buffer.from(json.data.audio, "hex");
          writeStream.write(audioChunk);
        }
      } catch {
        // 跳过非 JSON 行
      }
    }
  }

  writeStream.end();
  console.log(`流式音频已保存: ${outputFile}`);
}

// 批量文本转语音
async function batchTTS(
  texts: string[],
  outputDir: string = "./audio_output"
): Promise<string[]> {
  if (!fs.existsSync(outputDir)) {
    fs.mkdirSync(outputDir, { recursive: true });
  }

  const outputFiles: string[] = [];

  for (let i = 0; i < texts.length; i++) {
    const outputFile = `${outputDir}/audio_${i + 1}.mp3`;
    await textToSpeech({
      text: texts[i],
      voiceId: VOICES.NARRATOR,
      outputFile,
    });
    outputFiles.push(outputFile);
    console.log(`[${i + 1}/${texts.length}] 完成`);

    // 避免触发速率限制
    if (i < texts.length - 1) {
      await new Promise((r) => setTimeout(r, 200));
    }
  }

  return outputFiles;
}

export { textToSpeech, streamingTTS, batchTTS };

TTS 参数说明

参数 类型 取值范围 默认值 说明
text string — 必填 要合成的文字内容
voice_id string 见音色表 female-shaonv 使用的音色 ID
speed number 0.5 ~ 2.0 1.0 语速，1.0 为正常速度
vol number 0 ~ 10 1 音量大小
pitch number -12 ~ 12 0 音调，正值升调，负值降调
format string mp3 / wav / pcm mp3 输出音频格式

bitrate number 32000 ~ 320000 128000 比特率（bps）

角色扮演系统 Prompt 技巧

// src/minimax/roleplay.ts
import { minimax, MODELS } from "./client";
import { textToSpeech } from "./tts";
import { VOICES } from "./client";

interface Character {
  name: string;
  personality: string;
  background: string;
  speakingStyle: string;
  voiceId: string;
}

// 预设角色库
const CHARACTERS: Record<string, Character> = {
  assistant: {
    name: "小助",
    personality: "专业、严谨、友善，总是用清晰的结构回答问题",
    background: "一个拥有广博知识的 AI 助手",
    speakingStyle: "简洁明了，多用要点列表，适时举例",
    voiceId: VOICES.FEMALE_MATURE,
  },
  teacher: {
    name: "文老师",
    personality: "耐心、博学、善于用生活化例子解释复杂概念",
    background: "有 20 年经验的中学物理老师",
    speakingStyle: "循循善诱，先问学生已知什么，再逐步引导",
    voiceId: VOICES.MALE_MATURE,
  },
  narrator: {
    name: "旁白",
    personality: "沉稳、有磁性，擅长讲故事",
    background: "专业播音员",
    speakingStyle: "语速适中，停顿恰当，富有感情",
    voiceId: VOICES.NARRATOR,
  },
};

// 构建角色扮演的 System Prompt
function buildRoleplayPrompt(character: Character): string {
  return `你现在扮演"${character.name}"这个角色。

【角色背景】${character.background}
【性格特征】${character.personality}
【说话风格】${character.speakingStyle}

重要规则：
- 始终保持角色一致性，不要打破角色
- 使用符合角色身份的语气和词汇
- 回复长度控制在 150 字以内，除非用户要求详细解释`;
}

// 角色扮演对话 + 语音输出
async function roleplayWithVoice(
  characterKey: keyof typeof CHARACTERS,
  userMessage: string,
  outputAudio = false
) {
  const character = CHARACTERS[characterKey];

  const response = await minimax.chat.completions.create({
    model: MODELS.ABAB65S,
    messages: [
      { role: "system", content: buildRoleplayPrompt(character) },
      { role: "user", content: userMessage },
    ],
    temperature: 0.8, // 角色扮演用稍高温度，更有创意
    max_tokens: 300,
  });

  const text = response.choices[0].message.content!;
  console.log(`[${character.name}]: ${text}`);

  if (outputAudio) {
    await textToSpeech({
      text,
      voiceId: character.voiceId,
      outputFile: `./roleplay_${Date.now()}.mp3`,
    });
  }

  return text;
}

export { roleplayWithVoice, CHARACTERS, buildRoleplayPrompt };

完整语音对话 Pipeline

文本输入进来，经过 LLM 生成回复，再通过 TTS 转为音频输出——这就是语音对话 Pipeline 的完整链路：

sequenceDiagram
    participant U as 用户
    participant LLM as MiniMax LLM
    participant TTS as MiniMax TTS
    participant AUDIO as 音频文件

    U->>LLM: 发送文字输入
    Note over LLM: model: abab6.5s-chat<br/>system: 语音助手，回复 50 字内
    LLM-->>U: 返回回复文字
    U->>TTS: 发送回复文字 + voice_id
    Note over TTS: POST /v1/t2a_v2<br/>返回 hex 编码音频
    TTS-->>U: { data: { audio: "hex..." } }
    Note over U: Buffer.from(hex, 'hex')
    U->>AUDIO: 写入 reply.mp3
    Note over AUDIO: 可播放的语音回复
// 完整语音对话 Pipeline：文本输入 → LLM → TTS → 音频输出
async function voiceChatPipeline(userInput: string) {
  console.log(`用户: ${userInput}`);

  // 1. LLM 生成回复
  const response = await minimax.chat.completions.create({
    model: MODELS.ABAB65S,
    messages: [
      { role: "system", content: "你是一个简洁的语音助手，回复控制在 50 字以内" },
      { role: "user", content: userInput },
    ],
    temperature: 0.7,
  });

  const replyText = response.choices[0].message.content!;
  console.log(`AI 文本: ${replyText}`);

  // 2. TTS 转语音
  const audioBuffer = await textToSpeech({
    text: replyText,
    voiceId: VOICES.FEMALE_YOUNG,
    speed: 1.1,
    outputFile: "./reply.mp3",
  });

  console.log(`音频大小: ${audioBuffer.length} bytes`);
  return { text: replyText, audio: audioBuffer };
}

export { voiceChatPipeline };

  const paragraphs = [
    "第一章：人工智能的起源。人工智能的概念最早由图灵在1950年提出。",
    "图灵提出了著名的图灵测试：如果一台机器能让人类无法分辨它是机器还是人，那它就具备智能。",
    "此后，AI 经历了多次寒冬与繁荣，直到深度学习的崛起才真正改变了这个领域。",
  ];
  await batchTTS(paragraphs, "./audiobook");
}

注意事项

注意点 说明 解决方法
TTS 音频是 hex 编码 API 返回的是十六进制字符串，不是 base64 用 Buffer.from(hex, 'hex') 解码，不是 Buffer.from(b64, 'base64')
流式 TTS 逐行解析 每个 chunk 是独立 JSON 行，不是连续数据流 按换行符 split，逐行 JSON.parse，提取 data.audio
MiniMax-M1 是推理模型 响应时间比普通模型长，不适合实时对话 复杂任务用 M1，实时对话用 abab6.5s-chat
Text-01 的 1M context 超长文档专用，日常任务用它反而浪费 文档 > 100K token 才需要用 Text-01
TTS 单次文字长度 太长的文字会导致请求超时或质量下降 建议单次 < 500 字，超长文本用 batchTTS 拆分
音色 ID 拼写 音色 ID 是固定字符串，拼错会报错 用 VOICES 常量对象，避免手写字符串
