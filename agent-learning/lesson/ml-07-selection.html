第7课：模型选型 — 什么任务用什么模型

介绍

2025 年可用的 LLM 超过 100 个，盲目使用同一个模型要么浪费成本要么效果不够好。本课建立系统的模型选型框架：能力矩阵、成本对比、场景决策树，并实现 ModelRouter 自动根据任务上下文选择最优模型。

核心问题：一个高质量任务可能需要 $5/1M token 的 Claude Opus，但简单分类任务用 $0.15/1M 的 GPT-4o-mini 就够了。如果全部用同一个模型，成本可能相差 30 倍。

遇到什么问题

所有任务使用同一个模型，导致成本浪费或质量不足：

    T -->|代码生成/调试| O[Claude Opus 4.6 / Sonnet 4.6\n代码能力最强]
    T -->|数学/逻辑推理| R[o3-mini / DeepSeek-R1\n推理专项优化]

    T -->|中文内容处理| D[DeepSeek V3\n中文最强 + 极便宜]
    T -->|多模态图像| V[GPT-4o / Gemini 2.5 Pro\n原生多模态]
    T -->|均衡生产任务| S[Claude Sonnet 4.6\n最佳性价比]

典型痛点：

  • 用 Claude Opus 做简单翻译：单次调用成本是 GPT-4o-mini 的 30 倍
  • 用 GPT-4o-mini 做复杂代码 Review：质量不够，返工成本更高
  • 用 GPT-4o 处理 200K 字长文档：128K 上下文窗口不够用，直接报错
  • 用英文模型处理中文：输出质量明显下降，且 token 消耗更多

有哪些方案

方案 A：手动规则路由

function selectModel(task: { type: string; hasImages?: boolean }): string {
  if (task.hasImages) return "gpt-4o";
  if (task.type === "math") return "o3-mini";
  if (task.type === "simple_qa") return "claude-haiku-4-5";
  return "claude-sonnet-4-6";
}

✅ 可预测，零额外成本 ❌ 需要手动维护，新模型上线要更新规则 ❌ 只考虑了任务类型，忽略语言、输入长度等维度

方案 B：基于成本约束的自动选择

// 自动选择满足质量要求且成本最低的模型
const model = selectCheapestModelMeetingQuality(
  taskType,
  maxCostPer1M: 2.0
);

✅ 自动平衡质量和成本 ❌ 质量要求难以量化为具体数值 ❌ 无法处理能力边界（如超长上下文、多模态）

方案 C：上下文感知路由（推荐）

综合考虑任务类型、输入长度、是否有图像、语言、成本预算，实现 ModelRouter。

✅ 多维度决策，覆盖边界情况 ✅ 可扩展，新增规则无需重构 ✅ 支持成本约束降级，保证服务可用

怎么选择

选择方案 C，实现 ModelRouter 类。生产环境中 90% 的调用可以通过任务类型和几个关键维度做出正确的模型选择，剩余 10% 的特殊场景通过显式参数覆盖。

types.ts

// src/router/types.ts

export type TaskType =
  | "simple_qa"       // 简单问答：今天星期几、翻译单词
  | "code"            // 代码生成、调试、审查
  | "analysis"        // 文档分析、数据分析
  | "creative"        // 创意写作、营销文案
  | "math"            // 数学、逻辑推理
  | "translation"     // 翻译
  | "classification"  // 分类、打标签
  | "summarization";  // 摘要

export interface TaskContext {
  type: TaskType;
  inputLength?: number;           // 输入字符数（估算 Token 数 ≈ 字符数 / 3）
  hasImages?: boolean;
  requiresLongContext?: boolean;  // 是否需要超过 128K Token 的上下文
  language?: "zh" | "en" | "mixed";
  maxCostPer1MTokens?: number;    // 成本约束（美元）
  qualityRequirement?: "high" | "medium" | "low";
}

// 自动检测任务上下文（基于输入文本）
export function detectTaskContext(input: string): Partial<TaskContext> {
  const lower = input.toLowerCase();

  // 语言检测
  const chineseChars = (input.match(/[\u4e00-\u9fff]/g) || []).length;
  const language: "zh" | "en" | "mixed" =
    chineseChars > input.length * 0.3 ? "zh" :
    chineseChars > 0 ? "mixed" : "en";

  // 任务类型检测
  let type: TaskType = "simple_qa";
  if (lower.includes("代码") || lower.includes("函数") || lower.includes("实现") ||
      lower.includes("code") || lower.includes("function") || lower.includes("implement")) {
    type = "code";
  } else if (lower.includes("分析") || lower.includes("summarize") || lower.includes("analyze")) {
    type = "analysis";
  } else if (lower.includes("翻译") || lower.includes("translate")) {
    type = "translation";
  } else if (lower.includes("计算") || lower.includes("数学") || lower.includes("calculate")) {
    type = "math";
  } else if (lower.includes("分类") || lower.includes("classify") || lower.includes("categorize")) {
    type = "classification";
  } else if (lower.includes("摘要") || lower.includes("总结") || lower.includes("summarize")) {
    type = "summarization";
  }

  return {
    type,
    inputLength: input.length,
    language,
    requiresLongContext: input.length > 300_000,
  };
}

model_router.ts

// src/router/model_router.ts
import Anthropic from "@anthropic-ai/sdk";
import OpenAI from "openai";
import { MODEL_CATALOG } from "./model_catalog";
import type { TaskContext } from "./types";

export interface RouteResult {
  content: string;
  model: string;
  inputTokens: number;
  outputTokens: number;
  estimatedCost: number;
}

export class ModelRouter {
  private anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });
  private openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });
  private usageLog: Array<{ model: string; cost: number; timestamp: number }> = [];

  selectModel(ctx: TaskContext): string {
    // 优先规则（强制覆盖）：能力边界优先于成本
    if (ctx.hasImages) return "gpt-4o";
    if (ctx.requiresLongContext) return "gemini-2.0-flash";

    // 中文任务优先 DeepSeek（中文最强且最便宜）
    // 注意：代码和数学任务即使是中文，也优先用专项模型
    if (ctx.language === "zh" && ctx.type !== "code" && ctx.type !== "math") {
      return "deepseek-chat";
    }

    // 按任务类型选择基础模型
    const taskModelMap: Record<string, string> = {
      simple_qa: "gpt-4o-mini",
      classification: ctx.language === "zh" ? "deepseek-chat" : "gpt-4o-mini",
      translation: ctx.language === "zh" ? "deepseek-chat" : "claude-haiku-4-5",
      summarization: "claude-haiku-4-5",
      code: "claude-sonnet-4-6",       // 默认 Sonnet，高质量要求升级到 Opus
      math: "o3-mini",
      analysis: "claude-sonnet-4-6",
      creative: "claude-sonnet-4-6",
    };

    let selected = taskModelMap[ctx.type] || "claude-sonnet-4-6";

    // 质量要求调整
    if (ctx.qualityRequirement === "high") {
      if (ctx.type === "code" || ctx.type === "analysis" || ctx.type === "creative") {
        selected = "claude-opus-4-6";
      }
    } else if (ctx.qualityRequirement === "low") {
      if (["analysis", "code"].includes(ctx.type)) {
        selected = "gpt-4o-mini";
      }
    }

    // 成本约束降级：当所选模型超出预算时，切换到便宜备选
    if (ctx.maxCostPer1MTokens !== undefined) {
      const config = MODEL_CATALOG[selected];
      if (config && config.inputCostPer1M > ctx.maxCostPer1MTokens) {
        // 中英文选择不同的降级模型
        selected = ctx.language === "zh" ? "deepseek-chat" : "gpt-4o-mini";
      }
    }

    return selected;
  }

  async route(
    messages: { role: string; content: string }[],
    ctx: TaskContext
  ): Promise<RouteResult> {
    const modelId = this.selectModel(ctx);
    const config = MODEL_CATALOG[modelId];

    if (!config) throw new Error(`未知模型: ${modelId}`);

    console.log(`[ModelRouter] 模型: ${modelId}（任务: ${ctx.type}，语言: ${ctx.language || "en"}）`);

    const inputTokens = Math.ceil(
      messages.reduce((sum, m) => sum + m.content.length, 0) / 3
    );

    let content = "";

    if (config.provider === "anthropic") {
      const systemMsg = messages.find((m) => m.role === "system");
      const userMessages = messages.filter((m) => m.role !== "system");

      const response = await this.anthropic.messages.create({
        model: modelId,
        max_tokens: 2048,
        system: systemMsg?.content,
        messages: userMessages as Anthropic.MessageParam[],
      });
      content = response.content[0].type === "text" ? response.content[0].text : "";

    } else if (config.provider === "openai" || config.provider === "deepseek") {
      const client = new OpenAI({
        apiKey: config.provider === "deepseek"
          ? process.env.DEEPSEEK_API_KEY
          : process.env.OPENAI_API_KEY,
        baseURL: config.provider === "deepseek"
          ? "https://api.deepseek.com/v1"
          : undefined,
      });

      const response = await client.chat.completions.create({
        model: modelId,
        max_tokens: 2048,
        messages: messages as OpenAI.ChatCompletionMessageParam[],
        temperature: ctx.type === "math" ? 0.1 : 0.7,
      });
      content = response.choices[0].message.content || "";
    }

    const outputTokens = Math.ceil(content.length / 3);
    const estimatedCost =
      (inputTokens / 1_000_000) * config.inputCostPer1M +
      (outputTokens / 1_000_000) * config.outputCostPer1M;

    this.usageLog.push({ model: modelId, cost: estimatedCost, timestamp: Date.now() });

    return { content, model: modelId, inputTokens, outputTokens, estimatedCost };
  }

  // 成本报告
  printCostReport(): void {
    const byModel = new Map<string, { calls: number; cost: number }>();
    for (const entry of this.usageLog) {
      const stats = byModel.get(entry.model) || { calls: 0, cost: 0 };
      stats.calls++;
      stats.cost += entry.cost;
      byModel.set(entry.model, stats);
    }

    console.log("\n=== 成本报告 ===");
    let totalCost = 0;
    for (const [model, stats] of byModel) {
      console.log(`${model}: ${stats.calls} 次调用，$${stats.cost.toFixed(6)}`);
      totalCost += stats.cost;
    }
    console.log(`总成本：$${totalCost.toFixed(6)}`);
  }
}

// src/router/demo.ts
import { ModelRouter } from "./model_router";
import { MODEL_CATALOG } from "./model_catalog";
import { detectTaskContext } from "./types";

function estimateMonthlyCost(
  dailyRequests: number,
  avgInputTokens: number,
  avgOutputTokens: number,
  modelId: string
): void {
  const config = MODEL_CATALOG[modelId];
  if (!config) return;

  const dailyCost =
    (dailyRequests * avgInputTokens / 1_000_000) * config.inputCostPer1M +
    (dailyRequests * avgOutputTokens / 1_000_000) * config.outputCostPer1M;

  console.log(`${modelId}: 日 $${dailyCost.toFixed(2)}，月 $${(dailyCost * 30).toFixed(2)}`);
}

async function runDemo() {
  const router = new ModelRouter();

  // 演示自动路由
  const testCases = [
    {
      messages: [{ role: "user", content: "今天是星期几？" }],
      ctx: { type: "simple_qa" as const },
    },
    {
      messages: [{ role: "user", content: "用 TypeScript 实现一个线程安全的 LRU Cache" }],
      ctx: { type: "code" as const, qualityRequirement: "high" as const },
    },
    {
      messages: [{ role: "user", content: "帮我把这段文字翻译成英文：人工智能正在改变世界" }],
      ctx: { type: "translation" as const, language: "zh" as const },
    },
    {
      messages: [{ role: "user", content: "分析这份 50 万字的合同..." }],
      ctx: { type: "analysis" as const, requiresLongContext: true },
    },
  ];

  for (const tc of testCases) {
    const result = await router.route(tc.messages, tc.ctx);
    console.log(`  模型: ${result.model}，预估成本: $${result.estimatedCost.toFixed(6)}`);
    console.log(`  回答: ${result.content.slice(0, 80)}...\n`);
  }

  router.printCostReport();

  // 成本对比：场景模拟（1万次/天，500 输入 + 200 输出 tokens）

  const models = ["claude-opus-4-6", "claude-sonnet-4-6", "claude-haiku-4-5", "gpt-4o-mini", "deepseek-chat"];
  for (const model of models) {

  // 自动检测演示

    console.log(`输入: "${input.slice(0, 30)}..."`);
    console.log(`  检测: 任务=${detected.type}, 语言=${detected.language}`);
    console.log(`  选择: ${selectedModel}\n`);
  }
}

成本节省计算：智能路由 vs 全用同一模型

假设场景：每天 1 万次请求，平均 500 输入 token + 200 输出 token

策略 月成本估算 节省比例 说明
全用 GPT-4o ~$600 baseline（基准） $2.5×0.5M×10K×30 + $10×0.2M×10K×30
全用 Claude Sonnet 4.6 ~$750 -25%（更贵） 输出价格高于 GPT-4o
全用 Claude Opus 4.6 ~$2,250 -275%（昂贵） 只适合高价值任务
全用 DeepSeek-V3 ~$51 91% 更便宜 但不适合所有场景
全用 GPT-4o-mini ~$27 95% 更便宜 质量最差
智能路由（推荐） ~$120 80% 更便宜 质量保留 + 成本大幅降低

智能路由混合策略：30% 任务用 gpt-4o-mini（$0.15/$0.6），40% 用 deepseek-chat（$0.27/$1.1），20% 用 claude-sonnet-4-6（$3/$15），10% 用 claude-opus-4-6（$5/$25）。按场景权重加权后月均约 $120。

注意事项

注意点 说明
claude-opus-4-6 比旧版 opus-3 便宜很多 claude-3-opus 定价为 $15/$75，claude-opus-4-6 为 $5/$25，降幅超 60%
路由逻辑中中文任务优先 DeepSeek DeepSeek-V3 中文质量接近 GPT-4o，但价格便宜约 90%
超长文档（>128K tokens）只有 Gemini 能处理 GPT-4o 上下文窗口 128K，Claude 200K，Gemini 1M-2M
成本约束降级时注意中英文差异 中文降级选 deepseek-chat，英文降级选 gpt-4o-mini，避免用错语言优化的模型
DeepSeek 上下文仅 64K 如果任务是中文且超长，需要改用 Gemini，不能因为语言优先而忽略容量限制
qualityRequirement 只影响特定任务类型 目前只有 code / analysis / creative 支持 high 升级到 opus；simple_qa 不升级
模型价格会变动 MODEL_CATALOG 中的价格应定期更新，建议从配置文件或环境变量读取
