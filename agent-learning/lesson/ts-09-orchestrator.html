第9课：Orchestrator + Worker 模式

学习目标

  1. 理解多 Agent 协作的核心价值
  2. 实现 Orchestrator 角色：任务分解与协调
  3. 实现 Worker 角色：专注执行单一任务
  4. 掌握 Worker 并行执行的技巧

为什么需要多 Agent？

单个 Agent 面临的瓶颈：

  • Context 限制：复杂任务信息量超出单次上下文
  • 专注度：让一个模型既做研究又写代码又做审核，质量下降
  • 速度：串行执行多个子任务很慢

多 Agent 解决方案：分工 + 并行

架构设计

用户任务
    ↓
Orchestrator（统筹者）
    ├── 任务分解
    ├── 分配给 Workers
    └── 汇总结果
         ↓
┌────────┬────────┬────────┐
Worker1  Worker2  Worker3  Worker4
(研究)   (写作)   (审核)   (润色)
    └────────┴────────┴────────┘
                ↓
           最终结果

Worker 实现

Worker 是专注于单一职责的轻量 Agent：

import { client, model } from "../shared/llmClient";

interface WorkerResult {
  workerId: string;
  task: string;
  result: string;
  tokensUsed: number;
}

async function runWorker(
  workerId: string,
  role: string,        // Worker 的专业角色
  task: string,        // 分配的具体任务
  context: string = "" // Orchestrator 提供的上下文
): Promise<WorkerResult> {
  const response = await client.chat.completions.create({
    model,                        // 默认 deepseek-chat（Worker 用便宜模型节省费用）
    max_tokens: 2048,
    messages: [
      { role: "system", content: `你是一个${role}。请专注完成分配给你的任务，输出结构化的结果。` },
      {
        role: "user",
        content: context
          ? `背景信息：\n${context}\n\n你的任务：\n${task}`
          : `你的任务：\n${task}`,
      },
    ],
  });

  const result = response.choices[0].message.content ?? "";

  console.log(`  ✅ Worker[${workerId}] 完成`);

  return {
    workerId,
    task,
    result,
    tokensUsed: (response.usage?.prompt_tokens ?? 0) + (response.usage?.completion_tokens ?? 0),
  };
}

Orchestrator 实现

async function runOrchestrator(userGoal: string): Promise<string> {
  console.log(`\n🎯 目标: ${userGoal}`);

  // ── 第一步：Orchestrator 分解任务 ────────────────────────────
  // 规划环节可以用更强的模型；想用 R1 的话 LLM_MODEL=deepseek-reasoner
  const planResponse = await client.chat.completions.create({
    model,
    max_tokens: 2048,
    response_format: { type: "json_object" },        // 让模型直接输出 JSON
    messages: [
      { role: "system", content: `你是任务编排专家。将用户目标分解为 3-5 个可并行执行的子任务，
每个子任务需要一个专业 Worker。
输出 JSON 格式：
{
  "tasks": [
    { "id": "1", "role": "角色名称", "task": "具体任务描述" },
    ...
  ]
}` },
      { role: "user", content: `目标：${userGoal}` },
    ],
  });

  // 解析任务计划
  const planText = planResponse.choices[0].message.content ?? "{}";
  const plan = JSON.parse(planText) as {
    tasks: Array<{ id: string; role: string; task: string }>;
  };

  console.log(`\n📋 任务分解（${plan.tasks.length} 个子任务）：`);
  plan.tasks.forEach((t) => console.log(`  [${t.id}] ${t.role}: ${t.task}`));

  // ── 第二步：并行执行所有 Workers ─────────────────────────────
  console.log("\n⚡ 并行执行 Workers...");
  const workerResults = await Promise.all(
    plan.tasks.map((t) => runWorker(t.id, t.role, t.task))
  );

  // ── 第三步：Orchestrator 汇总结果 ────────────────────────────
  const workerSummary = workerResults
    .map((r) => `【${r.workerId} 的结果】\n${r.result}`)
    .join("\n\n---\n\n");

  const totalTokens = workerResults.reduce((sum, r) => sum + r.tokensUsed, 0);
  console.log(`\n📊 Workers 总计消耗 ${totalTokens} tokens`);

  const synthesisResponse = await client.chat.completions.create({
    model,
    max_tokens: 4096,
    messages: [
      { role: "system", content: "你是任务汇总专家。将多个 Worker 的结果整合为完整、连贯的最终输出。" },
      {
        role: "user",
        content: `原始目标：${userGoal}\n\nWorkers 结果：\n${workerSummary}\n\n请汇总为最终答案。`,
      },
    ],
  });

  return synthesisResponse.choices[0].message.content ?? "";
}

使用示例

const result = await runOrchestrator(
  "撰写一篇关于 AI Agent 发展趋势的技术博文，要求包含技术分析、应用案例和未来展望"
);
console.log("\n最终结果：", result);

执行流程：

Orchestrator 规划 → 分解为4个任务：
  [1] 研究员：调研 AI Agent 最新技术趋势
  [2] 案例分析师：收集典型应用案例
  [3] 未来预测师：分析未来发展方向
  [4] 技术编辑：确保技术准确性

并行执行（4个 Worker 同时工作）
  ↓
Orchestrator 汇总 → 完整博文

关键设计决策

决策 建议
Orchestrator 用什么模型 DeepSeek-Reasoner（R1）或 Claude-Opus（规划需要强推理）
Worker 用什么模型 DeepSeek-Chat 或 Kimi-v1-8k（节省费用）
串行还是并行 无依赖的任务用 Promise.all 并行
Worker 之间需要通信吗 通过 Orchestrator 传递上下文，避免直接通信

混搭模型小技巧：让 Orchestrator 和 Workers 用不同的模型。可以创建两个 client 实例，或者直接 inline new OpenAI({...}) 指向不同 baseURL。本课为了简单都用同一个 model。
