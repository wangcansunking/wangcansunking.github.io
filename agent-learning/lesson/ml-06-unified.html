第6课：统一 LLM 接口 — 一套代码接入所有模型

介绍

每个 LLM 提供商都有自己的 SDK 和 API 格式，导致业务代码和具体模型强耦合。想换一个模型，就要修改大量代码。本课介绍三种统一接口方案，核心目标是：业务代码只依赖标准接口，切换模型只需修改一行配置。

三种方案各有适用场景：

  • 方案 A（LiteLLM）：Python 项目首选，39.6k 星，支持 100+ 模型，开箱即用
  • 方案 B（Adapter Pattern）：TypeScript 项目首选，零依赖，类型安全，完全可控
  • 方案 C（Vercel AI SDK）：Next.js / React 前端项目首选

核心问题：四种 SDK，四种写法

以下是目前各主流提供商的 SDK 调用方式对比，差异相当大：

操作 OpenAI Anthropic Gemini DeepSeek
客户端初始化 new OpenAI() new Anthropic() new GoogleGenAI() new OpenAI({baseURL})
发送消息 chat.completions.create messages.create ai.chats.create + chat.sendMessage chat.completions.create
响应内容 choices[0].message.content content[0].text result.text choices[0].message.content
System 消息 放在 messages 数组中 独立的 system 参数 config.systemInstruction 放在 messages 数组中
流式输出 stream: true messages.stream() sendMessageStream() stream: true

这种碎片化导致：多模型 Agent 需要维护多套 SDK 代码，业务逻辑和模型实现混在一起，切换模型成本极高。统一接口层就是解决这个问题的。

整体架构：Adapter 设计模式

flowchart TD
    BIZ["业务代码<br/>runChatAgent / compareProviders"] --> IFACE["LLMAdapter 接口<br/>chat() / stream()"]

    IFACE --> OA["OpenAIAdapter<br/>gpt-4o 等"]
    IFACE --> AA["AnthropicAdapter<br/>claude-sonnet-4-6 等"]
    IFACE --> GA["GeminiAdapter<br/>gemini-2.0-flash 等"]
    IFACE --> DA["DeepSeekAdapter<br/>复用 OpenAIAdapter"]
    IFACE --> KA["KimiAdapter<br/>复用 OpenAIAdapter"]

    OA --> SDK_OAI["openai npm 包"]
    AA --> SDK_ANT["@anthropic-ai/sdk"]
    GA --> SDK_GEM["@google/genai"]
    DA --> SDK_OAI
    KA --> SDK_OAI

Adapter 模式的核心思想：为每个不兼容的接口创建一个包装类，将其转换为统一的接口。业务代码只知道 LLMAdapter 接口，完全不感知底层使用的是哪个 SDK。

有哪些方案

方案 A：LiteLLM（Python，推荐，39.6k 星）

import litellm

# 统一接口：支持 100+ 模型，全部用 OpenAI 格式
response = litellm.completion(
    model="claude-opus-4-6",       # 或 "gpt-4o", "gemini/gemini-2.0-flash"
    messages=[{"role": "user", "content": "你好"}],
)
print(response.choices[0].message.content)

方案 B：自定义 Adapter Pattern（TypeScript）

// 统一接口
interface LLMAdapter {
  chat(messages: LLMMessage[], options?: ChatOptions): Promise<LLMResponse>;
  stream(messages: LLMMessage[], options?: ChatOptions): AsyncGenerator<string>;
}

怎么选择

项目类型 推荐方案 理由
Python 后端服务 方案 A（LiteLLM） 100+ 模型开箱即用，内置可靠性特性
Python 脚本 / 数据处理 方案 A（LiteLLM） 一行切换模型，快速迭代
TypeScript / Node.js 后端 方案 B（Adapter Pattern） 类型安全，零依赖，可完全掌控行为
Next.js / React 应用 方案 C（Vercel AI SDK） 专为前端优化，Streaming 体验好
需要同时支持 Python 和 TS 方案 A + 方案 B 各用各的 不同语言不强求统一

方案A：LiteLLM 完整使用指南

# pip install litellm

import litellm
import asyncio
import os

# 配置 API Keys（LiteLLM 从环境变量自动读取）
# ANTHROPIC_API_KEY, OPENAI_API_KEY, DEEPSEEK_API_KEY 等

# 1. 基础调用（统一 OpenAI 格式）
def chat(model: str, user_message: str, **kwargs) -> str:
    response = litellm.completion(
        model=model,
        messages=[{"role": "user", "content": user_message}],
        **kwargs
    )
    return response.choices[0].message.content

# 切换模型只需改一个字符串
print(chat("claude-opus-4-6", "用 Python 写冒泡排序"))
print(chat("gpt-4o", "用 Python 写冒泡排序"))
print(chat("deepseek/deepseek-chat", "用 Python 写冒泡排序"))  # 中文最强+最便宜
print(chat("gemini/gemini-2.0-flash", "用 Python 写冒泡排序"))

# 2. 流式输出
async def stream_chat(model: str, message: str):
    async for chunk in await litellm.acompletion(
        model=model,
        messages=[{"role": "user", "content": message}],
        stream=True
    ):
        print(chunk.choices[0].delta.content or "", end="", flush=True)
    print()

# 3. Fallback（主模型失败自动切换）
response = litellm.completion(
    model="claude-opus-4-6",
    messages=[{"role": "user", "content": "Hello"}],
    fallbacks=["gpt-4o", "deepseek/deepseek-chat"],
    num_retries=3,
    timeout=30,
    request_timeout=60
)

# 4. 成本追踪
response = litellm.completion(
    model="claude-opus-4-6",
    messages=[{"role": "user", "content": "分析这段代码..."}]
)
cost = litellm.completion_cost(completion_response=response)
print(f"本次调用成本: ${cost:.6f}")

# 5. 批量并行调用（自动处理并发）
async def batch_compare(question: str):
    models = ["claude-opus-4-6", "gpt-4o", "deepseek/deepseek-chat"]
    tasks = [
        litellm.acompletion(
            model=m,
            messages=[{"role": "user", "content": question}],
            max_tokens=200
        )
        for m in models
    ]
    responses = await asyncio.gather(*tasks, return_exceptions=True)

    for model, resp in zip(models, responses):
        if isinstance(resp, Exception):
            print(f"[{model}] 错误: {resp}")
        else:
            text = resp.choices[0].message.content
            cost = litellm.completion_cost(resp)
            print(f"\n[{model}] 成本 ${cost:.6f}:\n{text[:200]}")

asyncio.run(batch_compare("什么是 Agent？用一句话解释"))

方案B：统一接口设计

接口定义原则

设计这套接口时遵循了几个关键原则：

  1. 最小接口面积：只暴露最核心的 chat() 和 stream() 两个方法，不过度设计
  2. 与原始 SDK 相同的心智模型：messages 数组、options 覆盖，和各家 SDK 的思维方式一致
  3. 标准化响应结构：content、usage、model 字段统一，不管底层 SDK 格式如何不同
  4. 可选覆盖：ChatOptions 中的所有字段都是可选的，有合理默认值

接口定义（types.ts）

// src/unified/types.ts

export interface LLMMessage {
  role: "user" | "assistant" | "system";
  content: string;
}

export interface ChatOptions {
  model?: string;           // 覆盖默认模型
  temperature?: number;     // 0-2，默认 0.7
  maxTokens?: number;       // 最大输出 token 数
  systemPrompt?: string;    // 覆盖系统提示（优先级高于 messages 中的 system）
  stream?: boolean;
}

export interface LLMResponse {
  content: string;
  usage?: {
    input: number;    // 输入 token 数
    output: number;   // 输出 token 数
    total: number;
  };
  model?: string;     // 实际使用的模型
  finishReason?: string;
}

export interface LLMAdapter {
  readonly name: string;
  readonly defaultModel: string;
  chat(messages: LLMMessage[], options?: ChatOptions): Promise<LLMResponse>;
  stream(messages: LLMMessage[], options?: ChatOptions): AsyncGenerator<string>;
}

OpenAI Adapter

// src/unified/adapters/openai.ts
import OpenAI from "openai";
import type { LLMAdapter, LLMMessage, ChatOptions, LLMResponse } from "../types";

export class OpenAIAdapter implements LLMAdapter {
  readonly name = "openai";
  readonly defaultModel: string;
  private client: OpenAI;

  constructor(options?: { apiKey?: string; baseURL?: string; defaultModel?: string }) {
    this.client = new OpenAI({
      apiKey: options?.apiKey || process.env.OPENAI_API_KEY,
      baseURL: options?.baseURL,
    });
    this.defaultModel = options?.defaultModel || "gpt-4o";
  }

  async chat(messages: LLMMessage[], options?: ChatOptions): Promise<LLMResponse> {
    const response = await this.client.chat.completions.create({
      model: options?.model || this.defaultModel,
      messages: this.buildMessages(messages, options?.systemPrompt),
      temperature: options?.temperature ?? 0.7,
      max_tokens: options?.maxTokens,
    });

    const choice = response.choices[0];
    return {
      content: choice.message.content || "",
      usage: response.usage
        ? {
            input: response.usage.prompt_tokens,
            output: response.usage.completion_tokens,
            total: response.usage.total_tokens,
          }
        : undefined,
      model: response.model,
      finishReason: choice.finish_reason,
    };
  }

  async *stream(
    messages: LLMMessage[],
    options?: ChatOptions
  ): AsyncGenerator<string> {
    const stream = await this.client.chat.completions.create({
      model: options?.model || this.defaultModel,
      messages: this.buildMessages(messages, options?.systemPrompt),
      temperature: options?.temperature ?? 0.7,
      max_tokens: options?.maxTokens,
      stream: true,
    });

    for await (const chunk of stream) {
      const delta = chunk.choices[0]?.delta?.content;
      if (delta) yield delta;
    }
  }

  private buildMessages(
    messages: LLMMessage[],
    systemPrompt?: string
  ): OpenAI.ChatCompletionMessageParam[] {
    const result: OpenAI.ChatCompletionMessageParam[] = [];

    if (systemPrompt) {
      result.push({ role: "system", content: systemPrompt });
    }

    for (const msg of messages) {
      if (msg.role === "system" && systemPrompt) continue; // 已有 systemPrompt，跳过
      result.push({ role: msg.role, content: msg.content });
    }

    return result;
  }
}

// DeepSeek 复用 OpenAI Adapter（只改 baseURL 和 defaultModel）
export function createDeepSeekAdapter(): OpenAIAdapter {
  return new OpenAIAdapter({
    apiKey: process.env.DEEPSEEK_API_KEY,
    baseURL: "https://api.deepseek.com/v1",
    defaultModel: "deepseek-chat",
  });
}

// Kimi 同理
export function createKimiAdapter(): OpenAIAdapter {
  return new OpenAIAdapter({
    apiKey: process.env.MOONSHOT_API_KEY,
    baseURL: "https://api.moonshot.cn/v1",
    defaultModel: "moonshot-v1-8k",
  });
}

Anthropic Adapter

// src/unified/adapters/anthropic.ts
import Anthropic from "@anthropic-ai/sdk";
import type { LLMAdapter, LLMMessage, ChatOptions, LLMResponse } from "../types";

export class AnthropicAdapter implements LLMAdapter {
  readonly name = "anthropic";
  readonly defaultModel: string;
  private client: Anthropic;

  constructor(options?: { apiKey?: string; defaultModel?: string }) {
    this.client = new Anthropic({
      apiKey: options?.apiKey || process.env.ANTHROPIC_API_KEY,
    });
    this.defaultModel = options?.defaultModel || "claude-sonnet-4-6";
  }

  async chat(messages: LLMMessage[], options?: ChatOptions): Promise<LLMResponse> {
    // Anthropic 的 system 消息是独立参数，不在 messages 数组中
    const { systemPrompt, anthropicMessages } = this.separateSystemMessage(
      messages,
      options?.systemPrompt
    );

    const response = await this.client.messages.create({
      model: options?.model || this.defaultModel,
      max_tokens: options?.maxTokens || 8192,
      system: systemPrompt,
      messages: anthropicMessages,
      temperature: options?.temperature ?? 0.7,
    });

    const contentBlock = response.content[0];
    const content = contentBlock.type === "text" ? contentBlock.text : "";

    return {
      content,
      usage: {
        input: response.usage.input_tokens,
        output: response.usage.output_tokens,
        total: response.usage.input_tokens + response.usage.output_tokens,
      },
      model: response.model,
      finishReason: response.stop_reason || undefined,
    };
  }

  async *stream(
    messages: LLMMessage[],
    options?: ChatOptions
  ): AsyncGenerator<string> {
    const { systemPrompt, anthropicMessages } = this.separateSystemMessage(
      messages,
      options?.systemPrompt
    );

    const stream = await this.client.messages.stream({
      model: options?.model || this.defaultModel,
      max_tokens: options?.maxTokens || 8192,
      system: systemPrompt,
      messages: anthropicMessages,
      temperature: options?.temperature ?? 0.7,
    });

    for await (const event of stream) {
      if (
        event.type === "content_block_delta" &&
        event.delta.type === "text_delta"
      ) {
        yield event.delta.text;
      }
    }
  }

  private separateSystemMessage(
    messages: LLMMessage[],
    overrideSystem?: string
  ): {
    systemPrompt: string | undefined;
    anthropicMessages: Anthropic.MessageParam[];
  } {
    const systemMsg = messages.find((m) => m.role === "system");
    const systemPrompt = overrideSystem || systemMsg?.content;

    const anthropicMessages: Anthropic.MessageParam[] = messages
      .filter((m) => m.role !== "system")
      .map((m) => ({
        role: m.role as "user" | "assistant",
        content: m.content,
      }));

    return { systemPrompt, anthropicMessages };
  }
}

Gemini Adapter（使用新版 @google/genai SDK）

// src/unified/adapters/gemini.ts
// 使用 2025 年新版 @google/genai SDK（替代旧版 @google/generative-ai）
import { GoogleGenAI } from "@google/genai";
import type { LLMAdapter, LLMMessage, ChatOptions, LLMResponse } from "../types";

export class GeminiAdapter implements LLMAdapter {
  readonly name = "gemini";
  readonly defaultModel: string;
  private ai: GoogleGenAI;

  constructor(options?: { apiKey?: string; defaultModel?: string }) {
    this.ai = new GoogleGenAI({
      apiKey: options?.apiKey || process.env.GEMINI_API_KEY!,
    });
    this.defaultModel = options?.defaultModel || "gemini-2.0-flash";
  }

  async chat(messages: LLMMessage[], options?: ChatOptions): Promise<LLMResponse> {
    const modelId = options?.model || this.defaultModel;
    const { history, lastUserMessage, systemInstruction } =
      this.convertMessages(messages, options?.systemPrompt);

    const chat = this.ai.chats.create({
      model: modelId,
      history,
      config: {
        systemInstruction,
        temperature: options?.temperature ?? 0.7,
        maxOutputTokens: options?.maxTokens,
      },
    });

    const result = await chat.sendMessage({ message: lastUserMessage });

    return {
      content: result.text ?? "",
      usage: result.usageMetadata
        ? {
            input: result.usageMetadata.promptTokenCount || 0,
            output: result.usageMetadata.candidatesTokenCount || 0,
            total: result.usageMetadata.totalTokenCount || 0,
          }
        : undefined,
      model: modelId,
    };
  }

  async *stream(
    messages: LLMMessage[],
    options?: ChatOptions
  ): AsyncGenerator<string> {
    const modelId = options?.model || this.defaultModel;
    const { history, lastUserMessage, systemInstruction } =
      this.convertMessages(messages, options?.systemPrompt);

    const chat = this.ai.chats.create({
      model: modelId,
      history,
      config: {
        systemInstruction,
        temperature: options?.temperature ?? 0.7,
      },
    });

    const response = await chat.sendMessageStream({ message: lastUserMessage });

    for await (const chunk of response) {
      if (chunk.text) yield chunk.text;
    }
  }

  // 把统一格式转换为 Gemini 的 history + 最后一条消息格式
  private convertMessages(
    messages: LLMMessage[],
    overrideSystem?: string
  ): {
    history: { role: string; parts: { text: string }[] }[];
    lastUserMessage: string;
    systemInstruction?: string;
  } {
    const nonSystemMessages = messages.filter((m) => m.role !== "system");
    const systemMsg = messages.find((m) => m.role === "system");
    const systemInstruction = overrideSystem || systemMsg?.content;

    if (nonSystemMessages.length === 0) {
      throw new Error("至少需要一条用户消息");
    }

    const lastMessage = nonSystemMessages[nonSystemMessages.length - 1];
    if (lastMessage.role !== "user") {
      throw new Error("最后一条消息必须是用户消息");
    }

    // Gemini history 不包含最后一条消息
    const history = nonSystemMessages.slice(0, -1).map((m) => ({
      role: m.role === "assistant" ? "model" : "user", // assistant → model
      parts: [{ text: m.content }],
    }));

    return {
      history,
      lastUserMessage: lastMessage.content,
      systemInstruction,
    };
  }
}

Factory 工厂函数

// src/unified/factory.ts
import type { LLMAdapter } from "./types";
import { OpenAIAdapter, createDeepSeekAdapter, createKimiAdapter } from "./adapters/openai";
import { AnthropicAdapter } from "./adapters/anthropic";
import { GeminiAdapter } from "./adapters/gemini";

export type ProviderName =
  | "openai"
  | "anthropic"
  | "gemini"
  | "deepseek"
  | "kimi";

export function createLLM(provider: ProviderName): LLMAdapter {
  switch (provider) {
    case "openai":
      return new OpenAIAdapter();
    case "anthropic":
      return new AnthropicAdapter();
    case "gemini":
      return new GeminiAdapter();
    case "deepseek":
      return createDeepSeekAdapter();
    case "kimi":
      return createKimiAdapter();
    default:
      throw new Error(`未知的 provider: ${provider}`);
  }
}

// 从环境变量读取默认 provider
export function createDefaultLLM(): LLMAdapter {
  const provider = (process.env.LLM_PROVIDER as ProviderName) || "openai";
  return createLLM(provider);
}

使用示例：业务代码完全解耦

// src/unified/demo.ts
import { createLLM, createDefaultLLM } from "./factory";
import type { LLMMessage } from "./types";

// 业务代码：完全不依赖具体的模型 SDK
async function runChatAgent(provider: "openai" | "anthropic" | "gemini") {
  const llm = createLLM(provider);

  const messages: LLMMessage[] = [
    { role: "user", content: "用 TypeScript 写一个冒泡排序函数" },
  ];

  console.log(`=== 使用 ${llm.name}（${llm.defaultModel}）===`);

  // 同样的代码，切换 provider 只需改一个参数
  const response = await llm.chat(messages, {
    temperature: 0.3,
    systemPrompt: "你是一个专业的 TypeScript 开发者，代码简洁高效。",
  });

  console.log("回复:", response.content);
  console.log("Token 用量:", response.usage);
}

// 流式输出：所有 adapter 同样的调用方式
async function runStreamingChat() {
  const llm = createDefaultLLM(); // 从环境变量读取

  const messages: LLMMessage[] = [
    { role: "user", content: "解释一下 React Hooks 的工作原理" },
  ];

  process.stdout.write(`[${llm.name}] `);
  for await (const chunk of llm.stream(messages)) {
    process.stdout.write(chunk);
  }
  console.log("\n");
}

// 对比多个模型的输出
async function compareProviders(question: string) {
  const providers = ["openai", "anthropic", "gemini"] as const;

  const results = await Promise.all(
    providers.map(async (provider) => {
      const llm = createLLM(provider);
      const response = await llm.chat(
        [{ role: "user", content: question }],
        { temperature: 0.5, maxTokens: 200 }
      );
      return { provider, content: response.content, usage: response.usage };
    })
  );

  for (const result of results) {
    console.log(`\n=== ${result.provider} ===`);
    console.log(result.content);
    console.log(`Token: 输入 ${result.usage?.input}, 输出 ${result.usage?.output}`);
  }
}

// 测试三个 provider
Promise.all([
  runChatAgent("openai"),
  runChatAgent("anthropic"),
  runChatAgent("gemini"),
]).then(() => runStreamingChat());

Adapter 实现要点

AnthropicAdapter 中最关键的处理是 system 消息的分离。Anthropic API 不接受 messages 数组中的 system 角色消息，必须作为独立参数传入：

flowchart TD
    INPUT["输入 messages 数组<br/>[{role:system,...}, {role:user,...}]"]
    INPUT --> FIND["查找 system 消息"]
    FIND --> SYS["提取 system 内容<br/>作为独立 system 参数"]
    FIND --> FILTER["过滤掉 system 消息<br/>剩余作为 messages 参数"]
    SYS --> API["Anthropic API 调用<br/>{ system: '...', messages: [...] }"]
    FILTER --> API

GeminiAdapter 的消息转换也有特殊处理：Gemini 使用 history + 最后一条消息的结构，而不是完整的 messages 数组。

各 Adapter 差异处理说明

Adapter 特殊处理 原因
OpenAIAdapter system 消息放在 messages 数组第一位 OpenAI API 规范，支持 messages 中的 system 角色
AnthropicAdapter system 作为独立参数，messages 中过滤掉 system Anthropic API 要求，messages 只能包含 user/assistant
GeminiAdapter assistant 角色替换为 model，history 不含最后一条消息 Gemini API 特性，角色名不同且使用 history + 当前消息结构
DeepSeekAdapter 复用 OpenAIAdapter，仅修改 baseURL 和 apiKey DeepSeek API 完全兼容 OpenAI 格式
KimiAdapter 复用 OpenAIAdapter，仅修改 baseURL 和 apiKey Kimi API 完全兼容 OpenAI 格式

注意事项

注意点 说明
Gemini 使用新版 SDK 必须用 @google/genai，不是旧版的 @google/generative-ai，初始化方式不同
Anthropic 默认模型 默认模型为 claude-sonnet-4-6，不是 claude-3-5-sonnet 等旧版模型名
Gemini 默认模型 默认模型为 gemini-2.0-flash，不是 gemini-pro
DeepSeek 复用 OpenAI Adapter 直接传 baseURL 和 apiKey 参数，无需单独实现 DeepSeekAdapter 类
systemPrompt 优先级 ChatOptions.systemPrompt 优先级高于 messages 数组中的 system 消息，会覆盖后者
Gemini 最后一条消息 convertMessages 方法会把最后一条 user 消息从 history 中分离出来单独发送，是 Gemini API 的要求
