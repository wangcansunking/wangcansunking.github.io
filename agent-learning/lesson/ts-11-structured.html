第11课：结构化输出

学习目标

  1. 理解为什么需要结构化输出
  2. 用 Zod 定义输出 Schema
  3. 使用 OpenAI response_format: { type: 'json_schema' } 获取类型安全的结果
  4. 处理嵌套结构和枚举类型

为什么需要结构化输出？

让模型返回自然语言很容易，但如果你需要把结果存入数据库或传给下一个系统，就需要可靠的 JSON 格式：

❌ 自然语言（难以解析）：
"该职位是一个高级工程师职位，薪资在 20-30K 之间，要求 5 年经验..."

✅ 结构化 JSON（直接使用）：
{
  "title": "高级工程师",
  "salary": { "min": 20000, "max": 30000 },
  "requiredYears": 5
}

Provider 支持情况

Provider 强约束 JSON Schema 仅 JSON 模式
OpenAI / GPT ✅ response_format: { type: 'json_schema', strict: true } ✅ { type: 'json_object' }
DeepSeek ⚠️ 暂未支持 strict schema ✅ { type: 'json_object' }
Kimi ⚠️ 暂未支持 strict schema ✅ { type: 'json_object' }
Anthropic ✅ messages.parse() + Zod（原生 SDK） ✅

本课提供两种实现：

  1. 通用版（DeepSeek 默认）：用 response_format: { type: 'json_object' } + Zod 在客户端验证（兼容三家）
  2. Anthropic 版：用原生 messages.parse() 享受更强约束

安装依赖

npm install zod zod-to-json-schema

通用版：JSON 模式 + Zod 验证（DeepSeek / Kimi / 任意 OpenAI 兼容）

import { client, model } from "../shared/llmClient";
import { z } from "zod";
import { zodToJsonSchema } from "zod-to-json-schema";

// 1. 用 Zod 定义期望的输出结构
const JobPostingSchema = z.object({
  title: z.string().describe("职位名称"),
  company: z.string().describe("公司名称"),
  location: z.string().describe("工作地点"),
  salary: z.object({
    min: z.number().describe("最低薪资（元/月）"),
    max: z.number().describe("最高薪资（元/月）"),
  }).describe("薪资范围"),
  requiredYears: z.number().describe("要求工作年限"),
  skills: z.array(z.string()).describe("必须技能列表"),
  isRemote: z.boolean().describe("是否支持远程办公"),
});

type JobPosting = z.infer<typeof JobPostingSchema>;

// 2. 把 Zod schema 转成 JSON Schema，喂给系统提示，让模型严格按它输出
async function parseJob(text: string): Promise<JobPosting> {
  const jsonSchema = zodToJsonSchema(JobPostingSchema);

  const response = await client.chat.completions.create({
    model,
    max_tokens: 2048,
    response_format: { type: "json_object" },          // 强制 JSON 模式
    messages: [
      {
        role: "system",
        content: `你是数据提取助手。严格按以下 JSON Schema 输出，不要任何额外文字：
${JSON.stringify(jsonSchema, null, 2)}`,
      },
      { role: "user", content: `从以下招聘信息中提取结构化数据：\n\n${text}` },
    ],
  });

  // 3. 客户端用 Zod 验证（兜底校验）
  const raw = JSON.parse(response.choices[0].message.content ?? "{}");
  return JobPostingSchema.parse(raw);                   // 不通过会抛 ZodError
}

// 4. 使用
const job = await parseJob(`招聘：高级 TypeScript 工程师 | 字节跳动 | 北京/上海
薪资：25K-40K，5年以上经验
技能要求：TypeScript、React、Node.js、微服务架构
可远程：支持每周2天居家办公`);

console.log(`职位：${job.title} @ ${job.company}`);
console.log(`薪资：${job.salary.min / 1000}K - ${job.salary.max / 1000}K`);
console.log(`技能：${job.skills.join(", ")}`);
console.log(`远程：${job.isRemote ? "支持" : "不支持"}`);

关键：response_format: { type: 'json_object' } 强制模型只输出合法 JSON，不会有"```json"包裹或前后多余文字；Zod.parse() 在客户端做最终校验。

枚举和联合类型

const SentimentSchema = z.object({
  sentiment: z.enum(["positive", "negative", "neutral"]).describe("情感倾向"),
  confidence: z.number().min(0).max(1).describe("置信度 0-1"),
  keywords: z.array(z.string()).describe("关键词列表"),
  summary: z.string().describe("一句话总结"),
});

type SentimentResult = z.infer<typeof SentimentSchema>;

async function analyzeSentiment(text: string): Promise<SentimentResult> {
  const jsonSchema = zodToJsonSchema(SentimentSchema);
  const response = await client.chat.completions.create({
    model,
    max_tokens: 512,
    response_format: { type: "json_object" },
    messages: [
      { role: "system", content: `按此 Schema 严格输出 JSON：${JSON.stringify(jsonSchema)}` },
      { role: "user",   content: `分析以下文本的情感倾向：\n\n${text}` },
    ],
  });

  return SentimentSchema.parse(JSON.parse(response.choices[0].message.content ?? "{}"));
}

const analysis = await analyzeSentiment(
  "这个产品真的超出预期！界面设计精美，性能流畅，客服也很及时。强烈推荐！"
);
console.log(analysis.sentiment);    // "positive"
console.log(analysis.keywords);     // ["超出预期", "精美", "流畅", "推荐"]

复杂嵌套结构

const BookingIntentSchema = z.object({
  intent: z.enum([
    "create_booking", "modify_booking", "cancel_booking", "query_booking",
  ]).describe("用户意图"),

  booking: z.object({
    type: z.enum(["flight", "hotel", "car"]).describe("预订类型"),
    destination: z.string().optional().describe("目的地"),
    checkIn: z.string().optional().describe("入住日期 YYYY-MM-DD"),
    checkOut: z.string().optional().describe("离开日期 YYYY-MM-DD"),
    passengers: z.number().optional().describe("人数"),
  }).describe("预订详情"),

  urgency: z.enum(["low", "medium", "high"]).describe("紧急程度"),
  missingInfo: z.array(z.string()).describe("还需要用户提供的信息"),
});

调用方式与上面 parseJob 完全一样——通用模板：

async function parseStructured<T>(schema: z.ZodSchema<T>, userInput: string): Promise<T> {
  const jsonSchema = zodToJsonSchema(schema);
  const response = await client.chat.completions.create({
    model,
    max_tokens: 2048,
    response_format: { type: "json_object" },
    messages: [
      { role: "system", content: `严格按此 JSON Schema 输出：${JSON.stringify(jsonSchema)}` },
      { role: "user",   content: userInput },
    ],
  });
  return schema.parse(JSON.parse(response.choices[0].message.content ?? "{}"));
}

// 用法
const booking = await parseStructured(BookingIntentSchema, "我想订明天从北京到上海的机票，两个人，越快越好");
console.log(booking.intent);            // "create_booking"
console.log(booking.booking.type);      // "flight"
console.log(booking.missingInfo);       // ["出发时间", "舱位要求"]

错误处理

try {
  const data = await parseJob(text);
  // 安全使用
} catch (e) {
  if (e instanceof z.ZodError) {
    console.error("模型输出不符合 Schema：", e.errors);
    // 可以重试，让模型修正
  } else if ((e as any).status === 400) {
    console.error("API 拒绝请求（可能触发内容审查）");
  } else {
    throw e;
  }
}

Anthropic 原生 messages.parse()（更强约束）

如果你切到 Anthropic 原生 SDK，可以享受 SDK 内置的 schema 验证：

import Anthropic from "@anthropic-ai/sdk";
import { zodOutputFormat } from "@anthropic-ai/sdk/helpers/zod";

const ant = new Anthropic();
const result = await ant.messages.parse({
  model: "claude-opus-4-7",
  max_tokens: 2048,
  messages: [{ role: "user", content: "..." }],
  output_config: { format: zodOutputFormat(JobPostingSchema) },
});

const job = result.parsed_output!;     // 类型化

差异点：

通用版（DeepSeek/Kimi） Anthropic 原生
response_format: { type: 'json_object' } output_config: { format: zodOutputFormat(...) }
客户端 Zod.parse() 校验 SDK 自动验证
失败抛 ZodError parsed_output === null，需检查 stop_reason

与 Tool Use 的对比

方法 适用场景
response_format: 'json_object' + Zod 提取信息、分类、数据转换
Tool Use（function calling） 触发函数调用、外部操作
纯提示词要求 JSON 不推荐（格式不稳定，无 strict 保证）

小结

// 三步走：定义 → 解析 → 使用
const Schema = z.object({ ... });                              // 1. 定义结构
const response = await client.chat.completions.create({       // 2. JSON 模式
  response_format: { type: "json_object" },
  messages: [...],
});
const data = Schema.parse(JSON.parse(response.choices[0].message.content)); // 3. 验证类型化使用
