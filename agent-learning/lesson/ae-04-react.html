第4课：ReAct 模式 — 推理与行动交替

介绍

ReAct（Reasoning + Acting）是最经典的 Agent 范式：让模型在每次行动前先"推理"（Thought），再"行动"（Action），再"观察"（Observation），循环直到答案。ReAct 的 Thought 让 Agent 的决策过程可观察，大幅提升可调试性。

遇到什么问题

纯工具调用 Agent 没有明显的推理步骤，难以调试和干预：

flowchart TD
    A[用户问题] --> B{ReAct 循环}
    B --> C[Thought\n推理当前状态和下一步]
    C --> D[Action\n选择并执行工具]
    D --> E[Observation\n获取工具执行结果]
    E --> F{是否有最终答案?}
    F -->|否| C
    F -->|是| G[Final Answer]

    style D fill:#d4edda
    style E fill:#d1ecf1

有哪些方案

方案 A：普通工具调用 Agent（无推理步骤）

// 直接调用工具，没有中间思考
const result = await openai.chat.completions.create({
  tools: [...], messages: [...]
});

✅ 简单，延迟低 ❌ 无推理过程，出错难以调试 ❌ 复杂任务容易跑偏

方案 B：ReAct（Thought → Action → Observation 循环）

// 强制模型输出 Thought 再 Action
const systemPrompt = `
你必须按以下格式输出：
Thought: [你的推理过程]
Action: [工具名称]
Action Input: [工具参数]
`;

✅ 推理过程可观察，方便调试 ✅ 复杂任务表现更稳定 ❌ 比直接调用多 50-200ms 延迟

方案 C：ReAct + Structured Output（推荐）

使用 JSON Schema 强制结构化输出，而不是依赖文本解析。

✅ 更可靠，不会因格式错误失败 ✅ TypeScript 类型安全

怎么选择

选择方案 C（ReAct + Structured Output）。生产环境中文本解析容易失败，用 JSON Schema 强制输出结构保证可靠性。同时展示推理链，方便监控和调试。

怎么执行

ReAct 数据结构

// src/react/types.ts

export interface Thought {
  reasoning: string;       // 当前状态分析
  plan: string;            // 下一步计划
}

export interface Action {
  tool: string;            // 工具名称
  input: Record<string, any>; // 工具参数
}

export interface Observation {
  result: string;          // 工具执行结果
  success: boolean;
  error?: string;
}

export interface ReActStep {
  thought: Thought;
  action?: Action;
  observation?: Observation;
  isFinal: boolean;
  finalAnswer?: string;
}

export interface Tool {
  name: string;
  description: string;
  parameters: Record<string, any>; // JSON Schema
  execute: (input: Record<string, any>) => Promise<string>;
}

ReAct Agent 实现

// src/react/react_agent.ts
import OpenAI from "openai";
import type { Tool, ReActStep } from "./types";

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

const REACT_SYSTEM_PROMPT = `你是一个使用 ReAct 框架的 AI Agent。

对于每一步，你必须输出以下 JSON 格式：
{
  "thought": {
    "reasoning": "分析当前情况和已有信息",
    "plan": "下一步要做什么以及为什么"
  },
  "action": {
    "tool": "工具名称（或 final_answer）",
    "input": {}
  },
  "is_final": false,
  "final_answer": null
}

如果已经有足够信息回答，设置 is_final: true 并填写 final_answer。
可用工具：{TOOLS}`;

export class ReActAgent {
  private tools: Map<string, Tool> = new Map();
  private maxSteps: number;

  constructor(tools: Tool[], maxSteps: number = 10) {
    for (const tool of tools) {
      this.tools.set(tool.name, tool);
    }
    this.maxSteps = maxSteps;
  }

  async run(userQuestion: string): Promise<{
    answer: string;
    steps: ReActStep[];
    totalSteps: number;
  }> {
    const steps: ReActStep[] = [];
    const messages: OpenAI.ChatCompletionMessageParam[] = [
      {
        role: "system",
        content: REACT_SYSTEM_PROMPT.replace(
          "{TOOLS}",
          this.getToolDescriptions()
        ),
      },
      { role: "user", content: userQuestion },
    ];

    for (let step = 0; step < this.maxSteps; step++) {
      console.log(`\n[ReAct] 步骤 ${step + 1}/${this.maxSteps}`);

      // 1. 让 LLM 推理
      const response = await openai.chat.completions.create({
        model: "gpt-4o",
        messages,
        response_format: { type: "json_object" },
        temperature: 0.3,
      });

      const rawOutput = response.choices[0].message.content!;
      let parsed: any;

      try {
        parsed = JSON.parse(rawOutput);
      } catch {
        console.error("[ReAct] JSON 解析失败:", rawOutput);
        break;
      }

      const thought = parsed.thought || { reasoning: "", plan: "" };
      const action = parsed.action;
      const isFinal = parsed.is_final;
      const finalAnswer = parsed.final_answer;

      console.log(`  Thought: ${thought.reasoning}`);
      console.log(`  Plan: ${thought.plan}`);

      if (isFinal && finalAnswer) {
        console.log(`  Final Answer: ${finalAnswer}`);
        steps.push({ thought, isFinal: true, finalAnswer });
        return { answer: finalAnswer, steps, totalSteps: step + 1 };
      }

      if (!action?.tool) break;

      // 2. 执行工具
      console.log(`  Action: ${action.tool}(${JSON.stringify(action.input)})`);
      const tool = this.tools.get(action.tool);

      let observation: string;
      let success = true;

      if (!tool) {
        observation = `错误：工具 "${action.tool}" 不存在`;
        success = false;
      } else {
        try {
          observation = await tool.execute(action.input);
        } catch (error) {
          observation = `执行错误: ${error instanceof Error ? error.message : String(error)}`;
          success = false;
        }
      }

      console.log(`  Observation: ${observation.substring(0, 100)}...`);

      steps.push({
        thought,
        action: { tool: action.tool, input: action.input },
        observation: { result: observation, success },
        isFinal: false,
      });

      // 3. 把结果加入对话历史
      messages.push(
        { role: "assistant", content: rawOutput },
        {
          role: "user",
          content: `Observation: ${observation}\n\n继续推理，输出下一步的 JSON。`,
        }
      );
    }

    return {
      answer: "达到最大步骤数，未找到答案",
      steps,
      totalSteps: steps.length,
    };
  }

  private getToolDescriptions(): string {
    return Array.from(this.tools.values())
      .map((t) => `- ${t.name}: ${t.description}`)
      .join("\n");
  }
}

工具定义示例

// src/react/tools.ts
import type { Tool } from "./types";

export const searchTool: Tool = {
  name: "search_web",
  description: "搜索互联网获取最新信息",
  parameters: {
    type: "object",
    properties: {
      query: { type: "string", description: "搜索关键词" },
    },
    required: ["query"],
  },
  execute: async (input) => {
    // 实际使用时调用搜索 API（如 Tavily、SerpAPI）
    console.log(`[搜索] ${input.query}`);
    return `搜索 "${input.query}" 的结果：找到相关信息 3 条...`;
  },
};

export const calculatorTool: Tool = {
  name: "calculator",
  description: "执行数学计算",
  parameters: {
    type: "object",
    properties: {
      expression: { type: "string", description: "数学表达式，如 '2 + 3 * 4'" },
    },
    required: ["expression"],
  },
  execute: async (input) => {
    try {
      const result = eval(input.expression); // 实际生产请用安全的表达式解析器
      return `计算结果: ${input.expression} = ${result}`;
    } catch {
      return `计算失败: 无效的表达式`;
    }
  },
};

export const readFileTool: Tool = {
  name: "read_file",
  description: "读取本地文件内容",
  parameters: {
    type: "object",
    properties: {
      path: { type: "string", description: "文件路径" },
    },
    required: ["path"],
  },
  execute: async (input) => {
    const fs = await import("fs");
    try {
      const content = fs.readFileSync(input.path, "utf-8");
      return content.slice(0, 2000); // 限制返回长度
    } catch {
      return `读取失败: 文件不存在或无权限`;
    }
  },
};

完整演示

// src/react/demo.ts
import { ReActAgent } from "./react_agent";
import { searchTool, calculatorTool } from "./tools";

async function runDemo() {
  const agent = new ReActAgent([searchTool, calculatorTool], 8);

  const questions = [
    "如果每月节省 15% 的工资，年薪 20 万的人一年能存多少钱？",
    "查一下最新的 iPhone 价格，然后计算如果我买 3 台需要多少钱",
  ];

  for (const q of questions) {
    console.log(`\n${"=".repeat(60)}`);
    console.log(`问题: ${q}`);
    const result = await agent.run(q);
    console.log(`\n最终答案: ${result.answer}`);
    console.log(`总步骤: ${result.totalSteps}`);
  }
}

runDemo().catch(console.error);

下一步

下一课学习第5课：Plan-and-Execute — 先规划后执行。ReAct 是"走一步看一步"，适合简单任务。但长任务（如"帮我写一份完整的市场分析报告"）容易偏离目标。Plan-and-Execute 先用强模型做全局规划，再逐步执行，更适合复杂长任务。
