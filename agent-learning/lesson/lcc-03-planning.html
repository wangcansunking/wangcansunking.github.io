第3课：Planning — TodoWrite 强制规划

介绍

没有规划的 Agent 会"漂移"：随机尝试工具、重复相同操作、在中途忘记目标。本课引入 todo_write 工具，强制 Agent 在开始任务前写出步骤列表，任务完成时更新状态。这是 Claude Code 中 TodoWrite 机制的精简复现。

源码参考：github.com/shareAI-lab/learn-claude-code — agents/s03_todo_write.py

遇到什么问题

对比同一个任务（"重构这个模块并补充测试"）的两种 Agent 行为：

无规划 有规划
随机修改文件 先列出 5 个步骤
完成一半忘记测试 每步完成后更新 ✓
10 轮后还没完成 5 轮内完成
用户不知道进度 实时可见进度 
flowchart LR
    A[接收任务] --> B[todo_write 写计划]
    B --> C[执行 Step 1]
    C --> D[todo_write 标记完成]
    D --> E[执行 Step 2]
    E --> F[...]
    F --> G[所有步骤 ✓ → 完成]

有哪些方案

方案 A：系统 prompt 要求规划

SYSTEM = "Before starting any task, write a plan as a numbered list..."

✅ 无需新工具 ❌ 模型可能忽略指令，没有强制约束力

方案 B：单独的 plan.txt 文件

模型用 write_file 写计划，用 read_file 读状态。

✅ 计划持久化，Agent 重启后可恢复 ❌ 格式不统一，难以程序化读取

方案 C：专用 todo_write 工具（推荐）

def todo_write(todos: list[dict]) -> str:
    """
    todos: [{"task": "...", "status": "pending|in_progress|done", "priority": "high|medium|low"}]
    """
    state["todos"] = todos
    return f"Plan updated: {len(todos)} tasks"

✅ 结构化状态，可程序化读取和展示 ✅ 模型每轮都能看到当前计划状态 ✅ 可扩展为持久化存储

怎么选择

方案 C。专用工具给模型明确的"规划入口"，比 prompt 指令更可靠。系统 prompt 注入当前 Todo 状态，让模型始终知道还剩哪些步骤。

怎么执行

Todo 状态管理

# agents/s03_todo_write.py
import json
from pathlib import Path

# 内存状态（可换成文件持久化）
state = {"todos": []}


def todo_write(todos: list) -> str:
    """更新整个 todo 列表"""
    state["todos"] = todos
    lines = []
    for t in todos:
        icon = {"pending": "○", "in_progress": "◐", "done": "✓"}.get(t["status"], "?")
        lines.append(f"  {icon} [{t.get('priority','medium').upper()}] {t['task']}")
    return "Plan:\n" + "\n".join(lines)


def todo_read() -> str:
    """供 system prompt 读取当前状态"""
    if not state["todos"]:
        return "No active plan."
    return todo_write(state["todos"])

动态系统 Prompt

def build_system_prompt() -> str:
    """每轮重建，注入最新 todo 状态"""
    todos = todo_read()
    return f"""You are a coding agent at {WORKDIR}.

CURRENT PLAN:
{todos}

Rules:
- Start every new task with todo_write to create a plan
- Mark tasks in_progress when starting them
- Mark tasks done immediately after completing them
- Use tools to act, don't just explain"""

注册 Todo 工具

TOOL_HANDLERS = {
    "bash":       lambda **kw: run_bash(kw["command"]),
    "read_file":  lambda **kw: run_read(kw["path"], kw.get("limit")),
    "write_file": lambda **kw: run_write(kw["path"], kw["content"]),
    "edit_file":  lambda **kw: run_edit(kw["path"], kw["old_text"], kw["new_text"]),
    "todo_write": lambda **kw: todo_write(kw["todos"]),  # ← 新增
}

TOOLS.append({
    "name": "todo_write",
    "description": "Update the task plan. Call this at task start and after each step.",
    "input_schema": {
        "type": "object",
        "properties": {
            "todos": {
                "type": "array",
                "items": {
                    "type": "object",
                    "properties": {
                        "task":     {"type": "string"},
                        "status":   {"type": "string", "enum": ["pending", "in_progress", "done"]},
                        "priority": {"type": "string", "enum": ["high", "medium", "low"]},
                    },
                    "required": ["task", "status"],
                },
            }
        },
        "required": ["todos"],
    },
})

更新后的 Agent Loop

def agent_loop(messages: list):
    while True:
        # 每轮重建 system prompt，注入最新 todo 状态
        system = build_system_prompt()

        response = client.messages.create(
            model=MODEL, system=system,
            messages=messages, tools=TOOLS, max_tokens=8000,
        )
        messages.append({"role": "assistant", "content": response.content})

        if response.stop_reason != "tool_use":
            return

        results = []
        for block in response.content:
            if block.type == "tool_use":
                handler = TOOL_HANDLERS.get(block.name)
                output = handler(**block.input) if handler else f"Unknown: {block.name}"
                # 高亮 todo 更新
                if block.name == "todo_write":
                    print(f"\033[32m{output}\033[0m")
                else:
                    print(f"> {block.name}: {output[:150]}")
                results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": output,
                })
        messages.append({"role": "user", "content": results})
