第9课：Agent Memory 分层 — 文件化记忆系统

介绍

很多 Agent 项目把"memory"做成单一数据库或大文件。Claude Code 不是。

它把记忆做成分 4 层、文件化、用户可见可治理的系统：Auto / Session / Agent / Team Memory，每层独立目录、独立 prompt、独立更新策略，再加上一个轻量召回器选出本轮真正需要的那几个文件。

源码定位

文件 角色
src/memdir/memdir.ts 底层文件存储 + buildMemoryPrompt
src/memdir/paths.ts 路径解析与 isAutoMemoryEnabled
src/memdir/findRelevantMemories.ts 相关记忆召回
src/services/SessionMemory/sessionMemory.ts 会话摘要
src/tools/AgentTool/agentMemory.ts Agent 专属记忆
src/tools/AgentTool/agentMemorySnapshot.ts Memory 资产化

4 层架构

+---------------------------+
| Auto Memory               |  → 用户/项目长期协作记忆
| <memoryBase>/projects/    |     • durable
| <git-root>/memory/        |     • 跨会话持续
+---------------------------+
+---------------------------+
| Session Memory            |  → 当前会话摘要
| ~/.claude/sessions/.../   |     • 0o600 权限
| session-mem-*.md          |     • token 阈值触发
+---------------------------+
+---------------------------+
| Agent Memory              |  → 某个 agent 类型的专属记忆
|   user / project / local  |     • 三种 scope
|   /agent-memory/<type>/   |     • 自动绑定 read/write/edit 工具
+---------------------------+
+---------------------------+
| Team Memory               |  → 团队共享 repo 级知识
| 带 pull/push/checksum     |     • 可同步
|   /optimistic-locking     |     • 受控
+---------------------------+

每层都靠：
  - 入口文件 MEMORY.md（≤200 行 / 25KB 截断保护）
  - topic memories/*.md（独立文件存每条记忆）
  - findRelevantMemories（轻量召回器选 ≤5 个文件）

底层存储：MEMORY.md 索引 + topic 文件

src/memdir/memdir.ts 里几个关键常量：

export const ENTRYPOINT_NAME    = 'MEMORY.md';
export const MAX_ENTRYPOINT_LINES = 200;
export const MAX_ENTRYPOINT_BYTES = 25_000;

设计意图：

  • MEMORY.md 是入口索引，不是正文
  • 每条 memory 单独一个 markdown 文件
  • agent / model 默认只看到 MEMORY.md 索引；需要细节再读具体文件

为什么不把所有记忆堆一个大文件？因为单文件会有四个问题：

  1. prompt 爆炸
  2. 更新冲突
  3. 历史垃圾难治理
  4. 单条错误污染整份上下文

索引 + 分文件是对这四个问题的工程答案。

buildMemoryPrompt：装配上下文的硬截断保护

真实源码（src/memdir/memdir.ts:272）：

export function buildMemoryPrompt(params: {
  displayName: string;
  memoryDir:   string;
  extraGuidelines?: string[];
}): string {
  const entrypoint = params.memoryDir + ENTRYPOINT_NAME;

  // ⚠️ 同步读取（getSystemPrompt 在 React render 路径上，不能 await）
  let entrypointContent = '';
  try {
    entrypointContent = fs.readFileSync(entrypoint, { encoding: 'utf-8' });
  } catch { /* 文件不存在静默忽略 */ }

  const lines = buildMemoryLines(...);

  if (entrypointContent.trim()) {
    const t = truncateEntrypointContent(entrypointContent);  // ⚠️ 硬截断
    lines.push(`## ${ENTRYPOINT_NAME}`, '', t.content);
  }
  return lines.join('\n');
}

两个工程细节：

  1. 同步读取：getSystemPrompt() 在 React render 路径上，不能 await
  2. 硬截断保护：超过 200 行 / 25KB 自动切掉，防 prompt 爆炸

findRelevantMemories：不是全塞 prompt，而是做选择

很多 Agent 一旦有 memory，就把所有历史塞 prompt。Claude Code 不是。

findRelevantMemories() 做的事：

memoryDir
  -> scanMemoryFiles()              扫文件头
  -> 过滤 alreadySurfaced 已展示过的
  -> formatMemoryManifest()         "文件名 + 描述"清单
  -> sideQuery(...)                 调一个轻量模型选
  -> 最多选 5 个 memory 文件
  -> 返回绝对路径 + mtimeMs

几个关键细节：

  • 选的是文件名，不是整段正文
  • 选择器只看 header / manifest，不灌全文
  • MEMORY.md 本身不在这里选（已单独注入 system prompt）
  • recentTools 影响选择（避免重复召回当前活跃工具的文档）
  • alreadySurfaced 避免每轮都选同一批

所以这层是轻量检索器，不是全文向量库。

Session Memory：智能摘要的触发逻辑

阈值（src/services/SessionMemory/sessionMemoryUtils.ts）：

minimumMessageTokensToInit = 10000;     // 短会话不开
minimumTokensBetweenUpdate = 5000;      // 每次更新最小间隔
toolCallsBetweenUpdates    = 3;         // 工具调用次数阈值

shouldExtractMemory 真实源码（src/services/SessionMemory/sessionMemory.ts:134）：

export function shouldExtractMemory(messages: Message[]): boolean {
  const currentTokenCount = tokenCountWithEstimation(messages);

  if (!isSessionMemoryInitialized()) {
    if (!hasMetInitializationThreshold(currentTokenCount)) return false;
    markSessionMemoryInitialized();
  }

  const hasMetTokenThreshold    = hasMetUpdateThreshold(currentTokenCount);
  const hasMetToolCallThreshold = countToolCallsSince(messages, lastMemoryMessageUuid) >= 3;
  const hasToolCallsInLastTurn  = hasToolCallsInLastAssistantTurn(messages);

  // token 阈值始终必要；
  // 在自然断点（无 tool_use）或双阈值都满足时才触发
  const shouldExtract =
    (hasMetTokenThreshold && hasMetToolCallThreshold) ||
    (hasMetTokenThreshold && !hasToolCallsInLastTurn);

  if (shouldExtract) {
    lastMemoryMessageUuid = messages[messages.length - 1]?.uuid;
    return true;
  }
  return false;
}

!hasToolCallsInLastTurn 是寻找自然断点——防止在 tool_use 链中间截断、生成孤立摘要。

Session Memory 文件权限：0o600

async function setupSessionMemoryFile(ctx) {
  const sessionMemoryDir = getSessionMemoryDir();
  await fs.mkdir(sessionMemoryDir, { mode: 0o700 });   // 目录：只有属主可读写执行

  const memoryPath = getSessionMemoryPath();
  await writeFile(memoryPath, '', {
    mode: 0o600,    // 文件：只有属主可读写
    flag: 'wx',     // O_CREAT|O_EXCL：文件不存在才创建，防覆盖
  });
}

0o700 + 0o600 说明：Session Memory 是敏感本地状态，不是普通缓存。

Memory 提取子 agent 的最小权限

第 6 课已经看过这个例子，这里再点一下它的精妙。

// src/services/SessionMemory/sessionMemory.ts:460
export function createMemoryFileCanUseTool(memoryPath: string): CanUseToolFn {
  return async (tool, input) => {
    if (
      tool.name === FILE_EDIT_TOOL_NAME &&
      input.file_path === memoryPath
    ) {
      return { behavior: 'allow', updatedInput: input };
    }
    return { behavior: 'deny', message: `only ${FILE_EDIT_TOOL_NAME} on ${memoryPath} is allowed` };
  };
}

Session Memory 提取子 agent 是严密沙箱化的摘要代理：

  • 只有 FileEditTool
  • 只能编辑那个精确路径
  • 连 FileReadTool / Bash / Grep 都被 deny

这是把"最小权限原则"用到极致的范例。

Agent Memory：让 agent 真正"成长"

Auto Memory 是用户 / 项目维度的；Agent Memory 是某个 agent 类型维度的。它有三种 scope：

Scope 目录 用途
user <memoryBase>/agent-memory/<type>/ 跨项目复用的 agent 长期记忆
project <cwd>/.claude/agent-memory/<type>/ 当前项目共享
local <cwd>/.claude/agent-memory-local/<type>/ 当前机器/工作区

所以 agent 不再是静态 prompt 模板，而是：

agent =
  静态角色 prompt
  + memory scope
  + memory 目录
  + 可写 memory tools (Write/Edit/Read 自动注入)
  + snapshot 初始化能力

这意味着同一个 agent 被多次召唤，它的记忆是累积的，不只是"重新读一遍 prompt"。

Agent Memory Snapshot：把记忆做成可分发资产

src/tools/AgentTool/agentMemorySnapshot.ts 引入了一个有意思的概念——snapshot 让 agent memory 可以随项目分发。

<cwd>/.claude/agent-memory-snapshots/<type>/
  ├── snapshot.json          ← updatedAt 时间戳
  └── <topic-1>.md
  └── <topic-2>.md

checkAgentMemorySnapshot() 返回三种动作：

动作 触发
none 没 snapshot 或已是最新
initialize 有 snapshot 但本地 memory 空
prompt-update 有 snapshot，本地非空，但 snapshot 更新了

这把 agent memory 从"运行时副产物"提升成了可分发的角色资产——一个项目可以发布带初始记忆的 agent，让团队成员一起从同一起点开始。

Pattern 提炼

Pattern 1：索引 + 分文件，而不是单大文件

MEMORY.md 索引 + topic-*.md 主文件。让记忆可治理、可单独删除、可单独编辑。

Pattern 2：硬截断保护必不可少

const MAX_ENTRYPOINT_LINES = 200;
const MAX_ENTRYPOINT_BYTES = 25_000;

任何动态注入 prompt 的字段都需要硬上限。

Pattern 3：自然断点触发，不破坏工具链

shouldExtract = ... && !hasToolCallsInLastAssistantTurn(messages);

后台任务在自然语义断点触发，不打断进行中的工具链。

Pattern 4：子 agent 最小权限

createMemoryFileCanUseTool(memoryPath)

每个子任务用专属 CanUseToolFn 闭包，把允许的工具+路径硬编码进去。

Pattern 5：Memory 也是产品资产

Snapshot 让 agent memory 可分发、可版本化、可初始化——它不是日志，是资产。

自己实现：60 行的 MEMORY.md 索引系统

// my-memory.ts
import * as fs from 'fs/promises';
import * as path from 'path';

const MAX_INDEX_LINES = 200;
const MAX_INDEX_BYTES = 25_000;

export class MarkdownMemory {
  constructor(private memoryDir: string) {}

  async ensureDir() {
    await fs.mkdir(this.memoryDir, { recursive: true, mode: 0o700 });
  }

  async getIndex(): Promise<string> {
    const indexPath = path.join(this.memoryDir, 'MEMORY.md');
    let raw = '';
    try { raw = await fs.readFile(indexPath, 'utf-8'); } catch {}

    return this.truncate(raw);
  }

  private truncate(raw: string): string {
    let s = raw.trim();
    const lines = s.split('\n');
    if (lines.length > MAX_INDEX_LINES) {
      s = lines.slice(0, MAX_INDEX_LINES).join('\n') + '\n\n> WARNING: truncated';
    }
    if (Buffer.byteLength(s, 'utf-8') > MAX_INDEX_BYTES) {
      s = s.slice(0, MAX_INDEX_BYTES) + '\n\n> WARNING: byte-truncated';
    }
    return s;
  }

  async appendTopic(slug: string, content: string, indexLine: string) {
    await this.ensureDir();
    const topicPath = path.join(this.memoryDir, `${slug}.md`);
    await fs.writeFile(topicPath, content, { mode: 0o600 });

    const indexPath = path.join(this.memoryDir, 'MEMORY.md');
    let index = '';
    try { index = await fs.readFile(indexPath, 'utf-8'); } catch {}
    if (!index.includes(slug)) {
      index += `\n- [${slug}](./${slug}.md) — ${indexLine}\n`;
      await fs.writeFile(indexPath, index, { mode: 0o600 });
    }
  }

  /** 用关键词从索引里轻量召回（不读 topic 全文） */
  async findRelevant(keyword: string, max = 5): Promise<string[]> {
    const index = await this.getIndex();
    const lines = index.split('\n').filter(l => l.includes(keyword));
    return lines.slice(0, max);
  }
}

// ===== 用法 =====
const mem = new MarkdownMemory('/Users/xx/.my-agent/memory');
await mem.appendTopic('user-prefers-typescript', '# User prefers TypeScript\n\n用 TS 不要 JS', '用户明确要求 TypeScript');

// 后续会话装配 prompt 时
const indexContent = await mem.getIndex();
const promptWithMemory = `# Persistent Memory\n\n${indexContent}\n\n# Current Task\n${userInput}`;

60 行就实现了 Claude Code memory 的核心 pattern：索引文件 + topic 文件 + 硬截断 + 0o600 权限。

本课小结

  • 4 层 memory：Auto / Session / Agent / Team，每层独立目录与策略
  • MEMORY.md 是索引，不是正文——单条记忆单独成文件
  • 硬截断保护（200 行 / 25KB）和 0o600 权限是基本动作
  • findRelevantMemories 是轻量检索器，不是全文向量库
  • Session Memory 提取子 agent 用最严格的精确路径权限
  • Agent Memory Snapshot 让记忆成为可分发的角色资产
