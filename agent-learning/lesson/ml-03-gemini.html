第3课：Gemini — Google AI + 多模态原生支持

介绍

Gemini 是 Google 的旗舰大语言模型系列，具备以下核心优势：

  • 原生多模态：天生支持文本、图像、音频、视频混合输入，不需要额外插件
  • 超长上下文：Gemini 2.5 Pro 支持高达 200万 token 的上下文窗口，业界领先
  • 原生 Function Calling：内置工具调用支持，格式与 OpenAI 略有不同
  • 文件 API：支持上传大型文件（PDF、图片、视频等）并在对话中引用

2025 年 SDK 重要变更：Google 推出了全新的 @google/genai SDK，取代了旧版 @google/generative-ai。新版 SDK 的 API 设计更简洁，功能更完整。本课所有代码均使用新版 SDK。

SDK 迁移指南：旧版 vs 新版

操作 旧版 @google/generative-ai 新版 @google/genai

导入 import { GoogleGenerativeAI } from "@google/generative-ai" import { GoogleGenAI } from "@google/genai"

注意：旧版 response.text() 是函数，新版 response.text 是属性。这是最常见的迁移错误！

遇到什么问题

从 OpenAI 切换到 Gemini 时，有 4 个核心差异需要注意：

1. role 名称不同

OpenAI 用 "assistant"，Gemini 用 "model"。如果写错会直接报错。

2. content 格式不同

OpenAI 的消息内容是字符串，Gemini 必须是 parts 数组格式：

flowchart LR
    subgraph OpenAI格式
        A["{ role: 'user', content: '你好' }"]
    end
    subgraph Gemini格式
        B["{ role: 'user', parts: [{ text: '你好' }] }"]
    end
    A -->|迁移时需要转换| B

3. 工具（Tool）格式不同

OpenAI 使用 JSON Schema 字符串，Gemini 使用自己的 FunctionDeclaration 格式，需要用 Type 枚举指定类型。

4. history 管理方式不同

Gemini 的 chat history 不包含当前正在发送的消息。历史记录是过去的对话轮次，当前消息通过 sendMessage 发送。这与 OpenAI 不同（OpenAI 把所有消息包括当前的都放在 messages 数组里）。

有哪些方案

方案 A：使用 @google/genai SDK（推荐）

直接使用 Google 官方最新 SDK，功能完整，支持所有 Gemini 特性（文件上传、多模态、Function Calling 等）。

优点：功能最完整、文档最新、支持 File API 和流式输出

缺点：与 OpenAI 代码不兼容，需要单独维护

方案 B：通过 OpenAI 兼容模式

Gemini 提供了 OpenAI 兼容的 API 端点，可以直接用 OpenAI SDK 调用。

优点：复用现有 OpenAI 代码

缺点：部分 Gemini 特有功能（如 File API、原生多模态）无法使用

怎么选择

推荐方案 A（@google/genai SDK）。

原因：

  1. Gemini 的核心优势（长上下文、多模态、File API）只有原生 SDK 才能完整使用
  2. 2025 年新 SDK 设计简洁，学习成本低
  3. Google 官方维护，安全可靠

只有当你已有大量 OpenAI 代码需要快速切换时，才考虑方案 B。

核心设计：消息格式转换

Gemini 的多轮对话设计与 OpenAI 有本质区别：

sequenceDiagram
    participant Dev as 开发者代码
    participant SDK as @google/genai SDK
    participant API as Gemini API

    Dev->>SDK: ai.chats.create({ model, history: [过去的对话] })
    Note over SDK: history 只包含历史轮次
    Note over SDK: 格式: {role: "user"/"model", parts: [{text}]}

    Dev->>SDK: chat.sendMessage({ message: "新问题" })
    SDK->>API: 发送 history + 新消息
    API-->>SDK: 返回响应
    SDK-->>Dev: response.text (属性，非方法)

    Note over Dev,API: 每次 sendMessage 后，SDK 自动更新内部 history

关键点：

  • history 数组存放过去已完成的对话轮次（user + model 成对出现）
  • sendMessage 发送当前新消息，SDK 自动将其加入历史
  • 不需要手动维护历史数组

多轮对话（注意 history 格式）

// multi_turn.ts
import { ai, GEMINI_MODELS } from "./client";

async function multiTurnChat() {
  // 重要：history 中 role 必须是 "user" 或 "model"
  // 不能用 "assistant"！这是最常见的错误
  const chat = ai.chats.create({
    model: GEMINI_MODELS.FLASH,
    history: [
      {
        role: "user",
        parts: [{ text: "你好，我叫小明，是一名前端工程师。" }],
      },
      {
        role: "model", // 注意：是 "model" 不是 "assistant"
        parts: [{ text: "你好，小明！很高兴认识你。作为前端工程师，你主要使用哪些技术栈呢？" }],
      },
    ],
    config: {
      systemInstruction: "你是一个友好的技术顾问，善于根据用户的背景给出针对性建议。",
    },
  });

  // 第一轮：发送新消息
  const response1 = await chat.sendMessage({
    message: "我最近想学习 AI 开发，应该从哪里入手？",
  });
  console.log("回答 1：", response1.text);

  // 第二轮：继续对话，SDK 自动维护历史
  const response2 = await chat.sendMessage({
    message: "我的 Python 基础不太好，这样可以吗？",
  });
  console.log("回答 2：", response2.text);

  // 第三轮
  const response3 = await chat.sendMessage({
    message: "能推荐一些具体的学习资源吗？",
  });
  console.log("回答 3：", response3.text);
}

流式输出

// streaming.ts
import { ai, GEMINI_MODELS } from "./client";

async function streamingOutput() {
  console.log("开始流式输出...\n");

  const stream = await ai.models.generateContentStream({
    model: GEMINI_MODELS.FLASH,
    contents: "请详细介绍一下 Transformer 架构的工作原理，包括注意力机制的核心思想。",
    config: {
      systemInstruction: "你是一位擅长用通俗易懂语言解释技术概念的讲师。",
    },
  });

  // 逐 chunk 打印，实现打字机效果
  for await (const chunk of stream) {
    if (chunk.text) {
      process.stdout.write(chunk.text);
    }
  }

  console.log("\n\n流式输出完成。");
}

async function streamingChat() {
  console.log("\n=== 流式多轮对话 ===\n");

  const chat = ai.chats.create({
    model: GEMINI_MODELS.FLASH,
    history: [],
  });

  const questions = [
    "什么是向量数据库？",
    "它和传统数据库有什么区别？",
  ];

  for (const question of questions) {
    console.log(`\n问：${question}`);
    console.log("答：");

    const stream = await chat.sendMessageStream({
      message: question,
    });

    for await (const chunk of stream) {
      if (chunk.text) {
        process.stdout.write(chunk.text);
      }
    }
    console.log("\n");
  }
}

async function main() {
  await streamingOutput();
  await streamingChat();
}

main().catch(console.error);

Function Calling（工具格式与 OpenAI 不同）

// function_calling.ts
import { Type } from "@google/genai";
import { ai, GEMINI_MODELS } from "./client";

// Gemini 工具定义：使用 Type 枚举，不用 JSON Schema 字符串
const weatherTool = {
  name: "get_weather",
  description: "获取指定城市的当前天气信息",
  parameters: {
    type: Type.OBJECT,
    properties: {
      city: {
        type: Type.STRING,
        description: "城市名称，例如：北京、上海",
      },
      unit: {
        type: Type.STRING,
        enum: ["celsius", "fahrenheit"],
        description: "温度单位",
      },
    },
    required: ["city"],
  },
};

const searchTool = {
  name: "search_web",
  description: "搜索网络获取最新信息",
  parameters: {
    type: Type.OBJECT,
    properties: {
      query: {
        type: Type.STRING,
        description: "搜索关键词",
      },
    },
    required: ["query"],
  },
};

// 模拟工具执行
function executeWeather(city: string, unit: string = "celsius") {
  const temps: Record<string, number> = { 北京: 22, 上海: 28, 广州: 32 };
  const temp = temps[city] ?? 20;
  return {
    city,
    temperature: unit === "fahrenheit" ? temp * 1.8 + 32 : temp,
    unit,
    condition: "晴天",
    humidity: "45%",
  };
}

function executeSearch(query: string) {
  return {
    results: [
      { title: `关于"${query}"的最新资讯`, url: "https://example.com/1" },
      { title: `"${query}"深度分析`, url: "https://example.com/2" },
    ],
  };
}

async function functionCallingDemo() {
  const response = await ai.models.generateContent({
    model: GEMINI_MODELS.FLASH,
    contents: "北京今天天气怎么样？用摄氏度显示。",
    config: {
      tools: [{ functionDeclarations: [weatherTool, searchTool] }],
    },
  });

  // 检查是否有工具调用
  const candidate = response.candidates?.[0];
  const content = candidate?.content;

  if (!content?.parts) {
    console.log("直接回答：", response.text);
    return;
  }

  for (const part of content.parts) {
    if (part.functionCall) {
      const { name, args } = part.functionCall;
      console.log(`模型调用工具：${name}`);
      console.log("参数：", args);

      // 执行工具
      let toolResult;
      if (name === "get_weather") {
        toolResult = executeWeather(args!.city as string, args!.unit as string);
      } else if (name === "search_web") {
        toolResult = executeSearch(args!.query as string);
      }

      console.log("工具结果：", toolResult);

      // 将工具结果返回给模型
      // 注意：Gemini 的工具结果格式使用 functionResponse 包裹
      const finalResponse = await ai.models.generateContent({
        model: GEMINI_MODELS.FLASH,
        contents: [
          { role: "user", parts: [{ text: "北京今天天气怎么样？用摄氏度显示。" }] },
          { role: "model", parts: [{ functionCall: { name, args } }] },
          {
            role: "user",
            parts: [
              {
                functionResponse: {  // Gemini 特有格式，不同于 OpenAI
                  name,
                  response: toolResult,
                },
              },
            ],
          },
        ],
        config: {
          tools: [{ functionDeclarations: [weatherTool, searchTool] }],
        },
      });

      console.log("\n最终回答：", finalResponse.text);
    }
  }
}

functionCallingDemo().catch(console.error);

OpenAI 工具格式 vs Gemini 工具格式对比：

对比项 OpenAI 格式 Gemini 格式
工具定义结构 { type: "function", function: { name, description, parameters } } { name, description, parameters }
参数类型写法 JSON Schema 字符串 "type": "object" Type 枚举 type: Type.OBJECT
工具放置位置 顶层 tools 数组 config.tools[0].functionDeclarations
工具调用结果角色 role: "tool" role: "user" + functionResponse
调用结果格式 { role: "tool", tool_call_id: "...", content: "结果字符串" } { functionResponse: { name, response: {...} } }

图像输入（Gemini 原生多模态）

// vision.ts
import * as fs from "fs";
import * as path from "path";
import { ai, GEMINI_MODELS } from "./client";

// 将图片转换为 base64
function imageToBase64(imagePath: string): { data: string; mimeType: string } {
  const imageBuffer = fs.readFileSync(imagePath);
  const base64Data = imageBuffer.toString("base64");
  const ext = path.extname(imagePath).toLowerCase();
  const mimeTypes: Record<string, string> = {
    ".jpg": "image/jpeg",
    ".jpeg": "image/jpeg",
    ".png": "image/png",
    ".gif": "image/gif",
    ".webp": "image/webp",
  };
  return {
    data: base64Data,
    mimeType: mimeTypes[ext] ?? "image/jpeg",
  };
}

async function analyzeImage(imagePath: string) {
  const { data, mimeType } = imageToBase64(imagePath);

  // 多模态输入：文本 + 图片
  const response = await ai.models.generateContent({
    model: GEMINI_MODELS.FLASH,
    contents: [
      {
        role: "user",
        parts: [
          { text: "请详细描述这张图片的内容，包括主要元素、颜色、构图和可能传达的含义。" },
          {
            inlineData: {   // Gemini 图片内联格式
              mimeType,
              data,
            },
          },
        ],
      },
    ],
  });

  console.log("图像分析结果：", response.text);
}

async function compareImages(imagePath1: string, imagePath2: string) {
  const img1 = imageToBase64(imagePath1);
  const img2 = imageToBase64(imagePath2);

  // 多图对比：Gemini 原生支持在一次请求中传入多张图片
  const response = await ai.models.generateContent({
    model: GEMINI_MODELS.PRO,
    contents: [
      {
        role: "user",
        parts: [
          { text: "请对比这两张图片的异同，从内容、风格、色调等角度进行分析。" },
          { inlineData: { mimeType: img1.mimeType, data: img1.data } },
          { text: "第二张图片：" },
          { inlineData: { mimeType: img2.mimeType, data: img2.data } },
        ],
      },
    ],
  });

  console.log("对比分析：", response.text);
}

async function extractChartData(chartImagePath: string) {
  const { data, mimeType } = imageToBase64(chartImagePath);

  // 从图表中提取结构化数据
  const response = await ai.models.generateContent({

        parts: [
          {

1. 图表类型（柱状图/折线图/饼图等）
2. 标题和坐标轴标签
3. 所有数据点或数值

长文档处理（200万 token 上下文）

// long_context.ts
import * as fs from "fs";
import { ai, GEMINI_MODELS } from "./client";

// 方式一：直接在 contents 中放入大量文本（适合 < 100 万 token）
async function analyzeLongText(longText: string) {
  const response = await ai.models.generateContent({
    model: GEMINI_MODELS.PRO, // 使用 Pro 版本的 200 万 token 上下文
    contents: [
      {
        role: "user",
        parts: [
          {
            text: `以下是一份很长的文档，请进行全面分析：\n\n${longText}\n\n
请提供：
1. 核心主题摘要（500字以内）
2. 关键论点列表
3. 重要数据或事实
4. 潜在风险或问题`,
          },
        ],
      },
    ],
  });

  return response.text;
}

// 方式二：通过 File API 上传文件（适合大型文件，如 PDF、视频）
async function uploadAndAnalyzeFile(filePath: string) {
  console.log("正在上传文件...");

  // 上传文件到 Gemini File API
  const uploadedFile = await ai.files.upload({
    file: filePath,
    config: {
      displayName: "待分析文档",
    },
  });

  console.log(`文件上传成功，ID: ${uploadedFile.name}`);
  console.log(`文件状态: ${uploadedFile.state}`);

  // 等待文件处理完成（变为 ACTIVE 状态）
  // 重要：上传后不能立即使用，必须等待处理完成
  let file = uploadedFile;
  while (file.state === "PROCESSING") {
    console.log("文件处理中，等待 2 秒...");
    await new Promise((resolve) => setTimeout(resolve, 2000));
    file = await ai.files.get({ name: uploadedFile.name! });
  }

  if (file.state === "FAILED") {
    throw new Error("文件处理失败");
  }

  console.log("文件已就绪，开始分析...");

  // 使用 fileData 引用已上传的文件（不需要重新传输内容）
  const response = await ai.models.generateContent({
    model: GEMINI_MODELS.PRO,
    contents: [
      {
        role: "user",
        parts: [
          { text: "请对这份文档进行全面的摘要和分析，提取关键信息。" },
          {
            fileData: {           // 引用已上传文件，用 fileUri 而不是 inlineData
              mimeType: file.mimeType!,
              fileUri: file.uri!,
            },
          },
        ],
      },
    ],
  });

  // 分析完成后清理文件释放存储
  await ai.files.delete({ name: file.name! });
  console.log("文件已删除");

  return response.text;
}

async function main() {
  // 测试长文本处理
  const sampleLongText = "这是一份示例长文档内容。".repeat(1000);
  console.log("=== 长文本分析 ===");
  const result = await analyzeLongText(sampleLongText);
  console.log(result);

  // 测试文件上传（需要提供实际 PDF 文件路径）
  const pdfPath = "./sample.pdf";
  if (fs.existsSync(pdfPath)) {
    console.log("\n=== 文件上传分析 ===");
    const fileResult = await uploadAndAnalyzeFile(pdfPath);
    console.log(fileResult);
  }
}

关键注意事项

注意点 说明 错误示例 正确做法
role 名称 Gemini 用 "model" 表示 AI，不是 "assistant" role: "assistant" role: "model"
parts 数组格式 消息内容必须是 parts 数组，不能直接用字符串 content: "你好" parts: [{ text: "你好" }]
response.text 是属性 新版 SDK 中 text 是属性，不是方法 response.text() response.text
chat history 不含当前消息 history 只放过去的对话，当前消息通过 sendMessage 发送 把新消息也加入 history 用 chat.sendMessage({ message: "..." })
File API 需等待 ACTIVE 上传文件后必须轮询等待状态变为 ACTIVE 上传完立即使用 循环检查 file.state === "PROCESSING"
Function calling 结果格式 工具结果用 functionResponse 包裹，role 为 "user" 用 OpenAI 的 role: "tool" 格式 { role: "user", parts: [{ functionResponse: {...} }] }
Type 枚举 Gemini 工具参数类型用 Type.STRING 等枚举，不用字符串 "type": "string" type: Type.STRING
模型选择成本 Pro 版本成本高出 Flash 约 25 倍，日常任务用 Flash 即可 所有场景都用 Pro 根据任务复杂度选择合适模型
