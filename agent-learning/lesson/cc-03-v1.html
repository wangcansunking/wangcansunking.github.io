第3课：MVP v1 — 60行实现最小可用 Agent

介绍

理论讲完了，动手写。

本课目标：用最少的代码，实现一个真正能用的 AI 编程助手——能执行 Bash 命令，能自己决策，能完成多步任务。

代码量：约 60 行。功能：真实可用。

遇到什么问题

我们想让 AI 能执行这样的对话：

用户：帮我查看当前目录有哪些文件，然后统计一下 .ts 文件的数量

AI：好的，先看一下目录...
[调用 bash: ls -la]
[调用 bash: find . -name "*.ts" | wc -l]
当前目录共有 12 个 .ts 文件。

实现这个，需要解决三个问题：

  1. 如何让 AI 调用工具？ → Claude API 的 Tool Use 机制
  2. 如何循环直到完成？ → Agentic Loop
  3. 如何让 AI 保持上下文？ → Messages 历史数组

有哪些方案

方案 A：每次调用都传完整历史

messages = [user, assistant, tool_result, assistant, tool_result, ...]
// 每轮都把完整历史传给 API

✅ 实现最简单 ❌ token 消耗线性增长（长任务会很贵）

方案 B：摘要压缩

// 超过阈值后，把早期历史压缩成摘要
if (messages.length > MAX) compress(messages)

✅ 控制 token 消耗 ❌ 实现复杂

方案 C：只保留最近 N 轮

messages = messages.slice(-N)

✅ 简单粗暴 ❌ 会丢失重要历史

怎么选择

MVP 阶段用方案 A（最简单），等 v2/v3 再加上下文管理优化。

原则：先让它跑起来，再让它跑得好。

怎么执行

完整代码

import Anthropic from "@anthropic-ai/sdk";
import { exec } from "child_process";
import { promisify } from "util";

const execAsync = promisify(exec);
const client = new Anthropic();

// ── 工具定义 ──────────────────────────────────────────
const tools: Anthropic.Tool[] = [
  {
    name: "bash",
    description: "执行 Bash 命令。用于读文件、运行测试、Git 操作等一切需要终端的任务。",
    input_schema: {
      type: "object",
      properties: {
        command: { type: "string", description: "要执行的 Bash 命令" },
      },
      required: ["command"],
    },
  },
];

// ── 工具执行 ──────────────────────────────────────────
async function executeTool(name: string, input: Record<string, string>): Promise<string> {
  if (name === "bash") {
    try {
      const { stdout, stderr } = await execAsync(input.command, {
        cwd: process.cwd(),
        timeout: 30_000,      // 30秒超时
        maxBuffer: 1024 * 512 // 512KB 输出上限
      });
      return stdout || stderr || "(命令执行完毕，无输出)";
    } catch (err: any) {
      return `错误: ${err.message}`;
    }
  }
  return `未知工具: ${name}`;
}

// ── Agent Loop ────────────────────────────────────────
async function runAgent(userMessage: string): Promise<void> {
  const messages: Anthropic.MessageParam[] = [
    { role: "user", content: userMessage }
  ];

  console.log(`\n👤 用户: ${userMessage}\n`);

  while (true) {
    const response = await client.messages.create({
      model: "claude-opus-4-6",
      max_tokens: 4096,
      tools,
      messages,
      system: "你是一个 AI 编程助手。通过 Bash 工具完成用户的任务。每次执行前简短说明你在做什么。",
    });

    // 把 AI 回复加入历史
    messages.push({ role: "assistant", content: response.content });

    // 打印文本输出
    for (const block of response.content) {
      if (block.type === "text" && block.text) {
        console.log(`🤖 ${block.text}`);
      }
    }

    // 任务完成，退出循环
    if (response.stop_reason === "end_turn") break;

    // 执行所有工具调用
    const toolResults: Anthropic.ToolResultBlockParam[] = [];

    for (const block of response.content) {
      if (block.type === "tool_use") {
        console.log(`\n🔧 执行: ${block.name}(${JSON.stringify(block.input)})`);
        const result = await executeTool(block.name, block.input as Record<string, string>);
        console.log(`📤 结果: ${result.slice(0, 200)}${result.length > 200 ? "..." : ""}`);

        toolResults.push({
          type: "tool_result",
          tool_use_id: block.id,
          content: result,
        });
      }
    }

    // 把工具结果加入历史，继续循环
    messages.push({ role: "user", content: toolResults });
  }
}

// ── 入口 ──────────────────────────────────────────────
const task = process.argv[2] ?? "查看当前目录结构，告诉我有哪些文件类型";
runAgent(task).catch(console.error);

代码解析

部分 行数 作用
工具定义 ~12行 告诉 AI 有什么工具可用
工具执行 ~12行 真正执行 Bash 命令
Agent Loop ~35行 核心循环逻辑

关键循环逻辑：

1. 调用 API → 得到 response
2. 如果 stop_reason = end_turn → 完成，break
3. 如果 stop_reason = tool_use → 执行工具，结果加入 messages，继续循环

运行方式

# 安装依赖
npm install @anthropic-ai/sdk

# 运行（需要设置 ANTHROPIC_API_KEY）
export ANTHROPIC_API_KEY="your-key"
npx ts-node mini_claude_code_v1.ts "帮我统计当前目录的 TypeScript 文件数量"

实际效果

🤖 我来帮你统计一下 TypeScript 文件的数量。

🔧 执行: bash({"command":"find . -name '*.ts' -not -path '*/node_modules/*' | wc -l"})
📤 结果: 23

下一步

v1 能用，但有明显的局限：

  • 只有 Bash：读/写文件通过 Bash 做，效率低、容易出错
  • 无权限检查：危险命令会直接执行
  • 无 CLAUDE.md：没有项目上下文
  • 无 token 管理：长任务会超出上下文

下一课我们解决这些问题，加入 Read/Write/Edit/Glob 工具，加入权限系统，加入 CLAUDE.md 支持。
