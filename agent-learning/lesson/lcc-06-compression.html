第6课：Context Compression — 无限会话

介绍

Agent 持续运行时，对话历史 tokens 不断增长，最终触碰模型上下文上限（200K tokens）。本课实现自动检测 + 智能压缩策略：当历史接近阈值时，调用 LLM 将早期消息压缩为摘要，保留最近 N 条原始消息，实现无限运行的 Agent 会话。

源码参考：github.com/shareAI-lab/learn-claude-code — agents/s06_context_compact.py

遇到什么问题

长时间运行的 Agent（多小时任务、多轮交互）：

会话开始: 500 tokens
30 分钟后: 15,000 tokens
2 小时后:  80,000 tokens
4 小时后:  190,000 tokens → 接近 200K 上限
4.1 小时: 💥 context_length_exceeded 错误

简单粗暴地截断历史会导致模型"失忆"——不知道它之前做了什么决策。

flowchart LR
    A[正常对话循环] --> B{tokens > 阈值?}
    B -->|否| A
    B -->|是| C[用 LLM 压缩早期历史]
    C --> D[保留最近 K 条原始消息]
    D --> E[摘要 + 近期历史 = 新 messages]
    E --> A

有哪些方案

方案 A：固定窗口截断

只保留最近 N 条消息，丢弃早期历史。

✅ 实现简单 ❌ 模型失去早期上下文，决策连贯性断裂

方案 B：Anthropic 原生 Compaction（方案 B）

使用 Anthropic beta API 的 compact_20260112 功能，服务端自动处理压缩。

✅ 零实现成本 ✅ 服务端优化的压缩质量 ❌ 需要 beta header，对压缩逻辑无控制权

方案 C：自主 LLM 摘要压缩（推荐）

if count_tokens(messages) > COMPRESS_THRESHOLD:
    summary = llm_summarize(messages[:-KEEP_RECENT])
    messages = [{"role": "user", "content": summary}] + messages[-KEEP_RECENT:]

✅ 完全掌控压缩策略 ✅ 可根据任务特性定制摘要 prompt ✅ 与任何 Anthropic 模型兼容

怎么选择

实际项目优先用方案 B（Anthropic 原生 Compaction，见第7课 TypeScript 系列），因为质量更高且零成本。本课实现方案 C，理解原理，并应用于需要自定义压缩逻辑的场景。

怎么执行

Token 计数

# agents/s06_context_compact.py

COMPRESS_THRESHOLD = 100_000   # 100K tokens 触发压缩
KEEP_RECENT        = 10        # 保留最近 10 条消息原始内容


def count_tokens(messages: list) -> int:
    """估算 messages 的 token 数（精确计数需调用 API）"""
    response = client.messages.count_tokens(
        model=MODEL,
        messages=messages,
        tools=TOOLS,
    )
    return response.input_tokens

LLM 摘要函数

def compress_history(messages: list, keep_recent: int = KEEP_RECENT) -> list:
    """
    将早期历史压缩为摘要，保留最近 keep_recent 条消息。
    返回新的 messages 列表。
    """
    if len(messages) <= keep_recent:
        return messages

    old_messages = messages[:-keep_recent]
    recent_messages = messages[-keep_recent:]

    print(f"\033[33m[Compressing {len(old_messages)} messages → summary]\033[0m")

    # 调用 LLM 生成摘要
    summary_response = client.messages.create(
        model=MODEL,
        max_tokens=2000,
        system="You are a conversation summarizer. Create a concise but complete summary "
               "that preserves all key decisions, actions taken, files modified, "
               "and important findings. Future actions depend on this context.",
        messages=[{
            "role": "user",
            "content": f"Summarize this conversation history:\n\n"
                       + format_messages_for_summary(old_messages),
        }],
    )

    summary_text = summary_response.content[0].text
    print(f"[Summary]: {summary_text[:200]}...")

    # 构建压缩后的 messages：摘要 + 近期原始消息
    compressed = [
        {
            "role": "user",
            "content": f"[CONVERSATION HISTORY SUMMARY]\n{summary_text}\n[END SUMMARY]\n\n"
                       "Above is a summary of earlier conversation. Continue from here.",
        },
        {
            "role": "assistant",
            "content": "Understood. I'll continue based on the summary and recent context.",
        },
    ] + recent_messages

    return compressed


def format_messages_for_summary(messages: list) -> str:
    """将 messages 格式化为可读文本供 LLM 摘要"""
    parts = []
    for msg in messages:
        role = msg["role"].upper()
        content = msg["content"]
        if isinstance(content, str):
            parts.append(f"[{role}]: {content[:500]}")
        elif isinstance(content, list):
            for block in content:
                if isinstance(block, dict):
                    if block.get("type") == "text":
                        parts.append(f"[{role}]: {block['text'][:500]}")
                    elif block.get("type") == "tool_use":
                        parts.append(f"[{role} TOOL]: {block['name']}({str(block['input'])[:200]})")
                    elif block.get("type") == "tool_result":
                        parts.append(f"[TOOL RESULT]: {str(block['content'])[:300]}")
    return "\n".join(parts)

集成到 Agent Loop

def agent_loop(messages: list):
    while True:
        # ── 压缩检查 ──
        token_count = count_tokens(messages)
        if token_count > COMPRESS_THRESHOLD:
            messages = compress_history(messages)
            print(f"\033[33m[Compressed to {count_tokens(messages)} tokens]\033[0m")

        response = client.messages.create(
            model=MODEL, system=SYSTEM,
            messages=messages, tools=TOOLS, max_tokens=8000,
        )
        messages.append({"role": "assistant", "content": response.content})
        if response.stop_reason != "tool_use":
            return
        results = []
        for block in response.content:
            if block.type == "tool_use":
                handler = TOOL_HANDLERS.get(block.name)
                output = handler(**block.input) if handler else f"Unknown: {block.name}"
                results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": output,
                })
        messages.append({"role": "user", "content": results})

压缩效果

压缩前：
  messages: 156 条，98,500 tokens
  早期消息：146 条 → 压缩为 800 token 摘要
  近期消息：10 条保留原文

压缩后：
  messages: 12 条，6,300 tokens
  模型继续运行，无缝衔接

会话可以无限期运行，token 成本保持稳定。
