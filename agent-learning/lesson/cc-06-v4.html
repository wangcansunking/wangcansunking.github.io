第6课：MVP v4 — Skill 系统

介绍

前几课我们把 Agent 的"能力"建好了。本课解决使用效率问题：

每次要做代码审查，你都要输入："帮我审查这次改动，关注安全漏洞、性能问题、代码风格..."

用 Skill 系统，你只需要输入：/review

遇到什么问题

随着 Agent 变强，用户的高频需求逐渐固定：

/commit   → 生成规范的 commit message 并提交
/review   → 审查当前 git diff
/test     → 运行测试，自动修复失败用例
/explain  → 解释指定文件或函数的作用

每次都要重新描述这些任务，既麻烦又容易描述不准确。

有哪些方案

方案 A：硬编码命令处理

if (input === '/commit') {
  runSpecificCommitLogic()
} else if (input === '/review') {
  runSpecificReviewLogic()
}

✅ 简单 ❌ 每增加一个 Skill 都要改代码 ❌ 不灵活，无法组合

方案 B：Skill 注册表（数据驱动）

const skills = {
  '/commit': { prompt: '...', tools: ['bash'] },
  '/review': { prompt: '...', tools: ['bash', 'read'] },
}
// 触发时注入 system prompt + 工具集，走普通 Agent Loop

✅ 新增 Skill 只需加一条数据 ✅ 可复用完整的 Agent Loop ✅ 支持参数（/explain src/auth.ts）

方案 C：动态加载外部 Skill 文件

// 从 ~/.claude/skills/ 目录加载
const skills = loadSkillsFromDirectory('~/.claude/skills/')

✅ 用户可自定义 ❌ 需要文件系统和 schema 验证

怎么选择

用方案 B，内置常用 Skill，同时保留方案 C 的扩展能力。

关键思路：Skill 本质上是"带特定 System Prompt 和工具集的 Agent 调用"，不需要特殊处理，只需在启动前注入正确的配置。

怎么执行

Skill 注册表

interface Skill {
  name: string;
  description: string;
  systemPromptAddition: string;  // 追加到主 system prompt
  tools: string[];               // 允许使用的工具
  requireArgs?: boolean;         // 是否需要参数
}

const SKILLS: Record<string, Skill> = {
  "/commit": {
    name: "commit",
    description: "分析 git diff，生成规范的 commit message 并提交",
    systemPromptAddition: `
## Commit 规范
- 格式：<type>(<scope>): <subject>
- type: feat/fix/refactor/docs/test/chore
- subject: 用中文描述，不超过 50 字
- 先用 git diff 查看改动，再生成 message`,
    tools: ["bash"],
  },

  "/review": {
    name: "review",
    description: "审查当前 git diff，找出潜在问题",
    systemPromptAddition: `
## 代码审查重点
1. 安全漏洞（SQL 注入、XSS、敏感信息泄露）
2. 性能问题（N+1 查询、内存泄漏、不必要的循环）
3. 错误处理（未捕获的异常、缺少边界检查）
4. 代码风格（命名规范、重复代码）
先运行 git diff 获取改动内容。`,
    tools: ["bash", "read"],
  },

  "/test": {
    name: "test",
    description: "运行测试，自动修复失败的用例",
    systemPromptAddition: `
## 测试修复流程
1. 先运行测试，收集失败信息
2. 阅读失败的测试用例
3. 找到对应的实现代码
4. 修复 Bug（不是修改测试！）
5. 再次运行测试验证
循环直到所有测试通过。`,
    tools: ["bash", "read", "edit"],
  },

  "/explain": {
    name: "explain",
    description: "解释指定文件或函数的作用和实现逻辑",
    systemPromptAddition: `
## 解释规范
- 先概述整体作用（1-2句）
- 再逐步解释关键逻辑
- 指出依赖关系和副作用
- 用通俗语言，避免照抄代码`,
    tools: ["read", "glob", "bash"],
    requireArgs: true,
  },
};

Skill 解析与执行

function parseInput(raw: string): { skill?: Skill; args: string } {
  const trimmed = raw.trim();

  for (const [command, skill] of Object.entries(SKILLS)) {
    if (trimmed === command || trimmed.startsWith(command + " ")) {
      const args = trimmed.slice(command.length).trim();
      return { skill, args };
    }
  }

  return { args: trimmed }; // 普通消息，无 Skill
}

async function runAgent(userInput: string): Promise<void> {
  const { skill, args } = parseInput(userInput);

  // 构建 System Prompt
  let systemPrompt = BASE_SYSTEM_PROMPT;
  let userMessage = args || userInput;

  if (skill) {
    systemPrompt += `\n\n## 当前任务模式：${skill.name}\n${skill.systemPromptAddition}`;

    if (skill.requireArgs && !args) {
      console.log(`ℹ️  用法：/${skill.name} <目标文件或函数>`);
      return;
    }

    // Skill 消息：如果有参数则包含，否则用 Skill 名称作为指令
    userMessage = args
      ? `${skill.description}：${args}`
      : skill.description;

    console.log(`\n🎯 执行 Skill: ${skill.name}${args ? ` (${args})` : ""}\n`);
  }

  // 过滤工具（Skill 可以限制工具集）
  const allowedTools = skill
    ? tools.filter(t => skill.tools.includes(t.name))
    : tools;

  // 走普通 Agent Loop
  await runAgentLoop(userMessage, systemPrompt, allowedTools);
}

帮助命令

function showHelp(): void {
  console.log("\n📖 可用命令：\n");
  for (const [command, skill] of Object.entries(SKILLS)) {
    console.log(`  ${command.padEnd(12)} ${skill.description}`);
  }
  console.log("\n  或直接输入任意任务描述\n");
}
