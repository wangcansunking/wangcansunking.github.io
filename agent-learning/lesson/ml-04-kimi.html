第4课：Kimi — 长文本专家（月之暗面）

介绍

Kimi 是月之暗面（Moonshot AI）推出的大语言模型，在中文长文本处理领域表现突出：

  • 完全兼容 OpenAI API：只需修改 apiKey 和 baseURL，无需改动其他代码
  • 128K 超长上下文：一次性处理整本书、完整合同、超长研究报告
  • 中文优化：对中文分词和语义理解有专项优化，特别适合中文文档处理
  • 文件 API：支持直接上传 PDF、Word、TXT 等文件并提取内容分析
  • 性价比高：相比 GPT-4 同等能力，Kimi 在中文长文本场景价格更具优势

适用场景：法律合同审查、学术论文分析、长篇报告摘要、整本书内容问答。

Kimi 模型一览

模型 ID 上下文 适用场景 价格特点
kimi-latest 128K 推荐默认，自动匹配最新最好版本 自动优化，综合性价比最佳
moonshot-v1-8k 8K 短文本对话、简单问答 最便宜，适合高频简单任务
moonshot-v1-32k 32K 中等长度文档、多轮对话 中等价格，平衡性能与成本
moonshot-v1-128k 128K 超长文档、完整书籍分析 按实际使用 token 计费

推荐：日常开发直接用 kimi-latest，它会自动匹配当前最佳版本，无需手动追踪模型版本更新。

当你需要根据文档长度动态选择模型时，可以参考以下决策流程：

中文估算参考：中文约 1.5 字/token，英文约 4 字/token。一份 10 万字的中文报告约 67,000 tokens，需要用 128k 模型。

遇到什么问题

从 OpenAI 直接迁移到 Kimi 时，有 4 个关键差异需要了解：

1. baseURL 不同

必须将请求指向 Kimi 的端点：https://api.moonshot.cn/v1

2. 模型名称不同

不能用 gpt-4 等 OpenAI 模型名，必须用 Kimi 的模型名（moonshot-v1-8k 等）。

3. 文件 API 独立

Kimi 有自己的文件上传 API，用于上传 PDF、Word 等文档并提取文本内容，然后将内容插入对话中使用。

4. 中文语义优化

Kimi 对中文的理解和生成质量比 OpenAI 在某些中文场景下更优，特别是涉及中国法律、政策、文化背景的内容。

有哪些方案

方案 A：OpenAI SDK + 修改 baseURL（推荐）

使用 openai 包，只修改 apiKey 和 baseURL，其余代码保持不变。

优点：

  • 代码改动最小，只需两行配置
  • TypeScript 类型完整
  • 与 OpenAI 代码可无缝切换

缺点：

  • 文件 API 需要用原生 fetch 或 Kimi 专用方法调用

方案 B：原生 fetch 调用

直接用 fetch 调用 Kimi REST API，无需任何 SDK。

优点：无依赖，轻量

缺点：需要手写请求格式，无类型提示，维护成本高

怎么选择

推荐方案 A（OpenAI SDK + baseURL）。

理由：

  1. 最小改动，降低迁移风险
  2. 如果以后需要切回 OpenAI 或切换其他 OpenAI 兼容服务，只改两行配置
  3. TypeScript 类型支持完整，开发体验好

核心设计：OpenAI 兼容层工作原理

sequenceDiagram
    participant Code as 你的代码
    participant SDK as OpenAI SDK
    participant Kimi as Kimi API
    participant Resp as 响应处理

    Code->>SDK: new OpenAI({ apiKey: KIMI_KEY, baseURL: KIMI_URL })
    Code->>SDK: client.chat.completions.create({ model: "moonshot-v1-8k", ... })
    SDK->>Kimi: POST https://api.moonshot.cn/v1/chat/completions
    Note over Kimi: 接收 OpenAI 格式请求
    Note over Kimi: 用 Kimi 模型处理
    Kimi-->>SDK: 返回 OpenAI 格式响应
    SDK-->>Code: ChatCompletion 对象（与 OpenAI 完全一致）
    Code->>Resp: response.choices[0].message.content

你需要改的 vs 不需要改的：

你需要改的 你不需要改的
apiKey：换成 Kimi API Key 请求体结构（messages、temperature 等）
baseURL：换成 Kimi 端点 响应体解析方式
model：换成 Kimi 模型名 流式输出处理逻辑
文件上传（用 Kimi File API） 错误处理逻辑

安装与配置

npm install openai
# .env

// client.ts
import OpenAI from "openai";

// 只需修改 apiKey 和 baseURL，其余与 OpenAI 完全一致
export const client = new OpenAI({
  apiKey: process.env.KIMI_API_KEY!,
  baseURL: "https://api.moonshot.cn/v1",
});

export const KIMI_MODELS = {
  LATEST: "kimi-latest",           // 推荐默认：自动匹配最新最好版本
  SHORT: "moonshot-v1-8k",         // 短文本：最便宜
  MEDIUM: "moonshot-v1-32k",       // 中文档：平衡性价比
  LONG: "moonshot-v1-128k",        // 超长文档：128K 上下文
} as const;

export type KimiModel = (typeof KIMI_MODELS)[keyof typeof KIMI_MODELS];

// 根据估计 token 数自动选择最合适的模型
export function selectModel(estimatedTokens: number): KimiModel {
  if (estimatedTokens < 6000) {
    return KIMI_MODELS.SHORT;    // 最便宜
  } else if (estimatedTokens < 28000) {
    return KIMI_MODELS.MEDIUM;   // 平衡
  } else {
    return KIMI_MODELS.LONG;     // 超长文档
  }
}

// 估算中文文本的 token 数（中文约 1.5 字/token）
export function estimateChineseTokens(text: string): number {
  return Math.ceil(text.length / 1.5);
}

// basic.ts
import { client, KIMI_MODELS } from "./client";

async function basicChat() {
  const response = await client.chat.completions.create({
    model: KIMI_MODELS.LATEST,
    messages: [
      {
        role: "system",
        content: "你是一个专业的文档分析助手，善于提炼关键信息和结构化总结。",
      },
      {
        role: "user",
        content: "请简单介绍一下月之暗面（Moonshot AI）公司。",
      },
    ],
    temperature: 0.3,
  });

  console.log("回答：", response.choices[0].message.content);
  console.log("使用 tokens：", response.usage?.total_tokens);
}

async function multiTurnChat() {
  const messages: OpenAI.ChatCompletionMessageParam[] = [
    {
      role: "system",
      content: "你是一个法律助手，熟悉中国合同法。",
    },
  ];

  const questions = [
    "什么是合同的必备条款？",
    "违约金条款有哪些注意事项？",
    "能给我一个简单的服务合同模板吗？",
  ];

  for (const question of questions) {
    messages.push({ role: "user", content: question });

    const response = await client.chat.completions.create({
      model: KIMI_MODELS.LATEST,
      messages,
    });

    const answer = response.choices[0].message.content!;
    messages.push({ role: "assistant", content: answer });

    console.log(`\n问：${question}`);
    console.log(`答：${answer.substring(0, 200)}...`);
  }
}

import OpenAI from "openai";

async function main() {
  console.log("=== 基础对话 ===");
  await basicChat();

  console.log("\n=== 多轮对话 ===");
  await multiTurnChat();
}

流式输出

// streaming.ts
import { client, KIMI_MODELS } from "./client";

async function streamingChat(prompt: string) {
  console.log(`\n问：${prompt}`);
  console.log("答：");

  const stream = await client.chat.completions.create({
    model: KIMI_MODELS.LATEST,
    messages: [
      { role: "system", content: "你是一个知识渊博的助手，回答简洁清晰。" },
      { role: "user", content: prompt },
    ],
    stream: true, // 开启流式输出
  });

  let fullResponse = "";

  for await (const chunk of stream) {
    const delta = chunk.choices[0]?.delta?.content ?? "";
    if (delta) {
      process.stdout.write(delta);
      fullResponse += delta;
    }
  }

  console.log("\n");
  console.log(`（共 ${fullResponse.length} 字）`);

  return fullResponse;
}

async function streamingDocumentAnalysis(documentText: string) {
  console.log("开始流式分析文档...\n");

  const stream = await client.chat.completions.create({
    model: KIMI_MODELS.LONG,
    messages: [
      {
        role: "system",
        content: "你是专业的文档分析师。",
      },
      {
        role: "user",
        content: `请分析以下文档并生成摘要：\n\n${documentText}`,
      },
    ],
    stream: true,
    max_tokens: 1500,
  });

  for await (const chunk of stream) {
    const delta = chunk.choices[0]?.delta?.content ?? "";
    if (delta) {
      process.stdout.write(delta);
    }

    // 检查是否完成
    if (chunk.choices[0]?.finish_reason === "stop") {
      console.log("\n\n分析完成。");
    }
  }
}

async function main() {
  await streamingChat("请解释什么是向量数据库，以及它在 AI 应用中的作用。");

  const sampleDoc = "人工智能正在快速改变各行各业。".repeat(100);
  await streamingDocumentAnalysis(sampleDoc);
}

main().catch(console.error);

文件读取与分析（Kimi 文件 API）

// file_api.ts
import * as fs from "fs";
import * as path from "path";
import { client, KIMI_MODELS } from "./client";

// 上传文件到 Kimi（返回 file_id）
async function uploadFile(filePath: string): Promise<string> {
  console.log(`正在上传文件：${path.basename(filePath)}`);

  const fileStream = fs.createReadStream(filePath);
  const fileName = path.basename(filePath);

  // 使用 OpenAI SDK 的 files.create（兼容 Kimi File API）
  const uploadedFile = await client.files.create({
    file: fileStream as any,
    purpose: "assistants",
  });

  console.log(`上传成功，file_id: ${uploadedFile.id}`);
  return uploadedFile.id;
}

// 获取文件内容（提取文本）
async function getFileContent(fileId: string): Promise<string> {
  // Kimi 文件内容提取端点
  const response = await fetch(
    `https://api.moonshot.cn/v1/files/${fileId}/content`,
    {
      headers: {
        Authorization: `Bearer ${process.env.KIMI_API_KEY}`,
      },
    }
  );

  if (!response.ok) {
    throw new Error(`获取文件内容失败: ${response.statusText}`);
  }

  const data = await response.text();
  return data;
}

// 删除文件（释放存储）
async function deleteFile(fileId: string): Promise<void> {
  await client.files.del(fileId);
  console.log(`文件 ${fileId} 已删除`);
}

// 完整流程：上传文件 → 提取内容 → 分析 → 删除文件
async function analyzeFileWithKimi(
  filePath: string,
  analysisPrompt: string
): Promise<string> {
  let fileId: string | null = null;

  try {
    // 步骤 1：上传文件
    fileId = await uploadFile(filePath);

    // 步骤 2：提取文件内容
    console.log("正在提取文件内容...");
    const fileContent = await getFileContent(fileId);
    console.log(`提取完成，内容长度：${fileContent.length} 字符`);

    // 步骤 3：将文件内容加入对话并分析
    const response = await client.chat.completions.create({
      model: KIMI_MODELS.LONG,
      messages: [
        {
          role: "system",
          content: "你是专业的文档分析助手，能够准确理解和分析各类文档内容。",
        },
        {
          role: "system",
          content: `以下是文件内容：\n\n${fileContent}`,
        },
        {
          role: "user",
          content: analysisPrompt,
        },
      ],
    });

    return response.choices[0].message.content!;
  } finally {
    // 步骤 4：无论成功失败，都删除文件释放存储
    if (fileId) {
      await deleteFile(fileId);
    }
  }
}

async function main() {
  const testFilePath = "./sample.pdf";

  if (fs.existsSync(testFilePath)) {
    console.log("=== 文件分析 ===");
    const result = await analyzeFileWithKimi(
      testFilePath,
      "请总结这份文档的主要内容，并列出3个最重要的观点。"
    );
    console.log("\n分析结果：");
    console.log(result);
  } else {
    console.log("提示：请提供 sample.pdf 文件以测试文件 API 功能");
    console.log("支持格式：PDF、Word(.docx)、TXT、Markdown 等");
  }
}

main().catch(console.error);

Kimi 文件 API 流程

sequenceDiagram
    participant Dev as 开发者代码
    participant KimiFiles as Kimi File API
    participant KimiChat as Kimi Chat API

    Dev->>KimiFiles: POST /v1/files (上传文件)
    KimiFiles-->>Dev: { id: "file_xxx", status: "ok" }

    Dev->>KimiFiles: GET /v1/files/{file_id}/content
    KimiFiles-->>Dev: 文件文本内容（已提取）

    Note over Dev: 将文件内容插入消息

    Dev->>KimiChat: POST /v1/chat/completions
    Note over Dev: messages 包含文件内容 + 分析问题
    KimiChat-->>Dev: 分析结果

    Dev->>KimiFiles: DELETE /v1/files/{file_id}
    KimiFiles-->>Dev: 删除成功（释放存储）

文件类型 格式 注意事项
PDF 文档 .pdf 支持文本型 PDF，扫描版 PDF（图片）可能提取效果差
Word 文档 .docx, .doc 支持文字内容，复杂排版可能丢失格式
纯文本 .txt 完美支持，无损提取

适用场景 vs 不适用场景

最适合用 Kimi 不推荐用 Kimi
整本书分析（10-30万字） 图像识别、图片描述
长篇法律合同审查 代码执行与调试
学术论文全文理解 需要实时联网搜索的任务
超长研究报告摘要 复杂数学计算

完整项目文档问答 超低延迟实时对话（用 Flash 版本更快）
多版本合同对比 需要代码解释器执行的任务

常见注意事项

注意点 说明
优先用 kimi-latest kimi-latest 会自动选择当前最新最好的 Kimi 版本，无需手动追踪版本更新。适合大多数场景。
128K 模型按实际 token 计费 发送一份 10 万字的文档，即使问题很简单，也会消耗大量 input tokens。批量处理长文档成本需要评估。
文件上传后记得删除 Kimi 文件 API 有存储配额限制，每次分析完成后调用 deleteFile 释放存储，避免积压。
中文分词估算 中文约 1.5 字/token，英文约 4 字/token。一份 10 万字中文文档 ≈ 67,000 tokens，接近 8K 模型上限的 8 倍。
system 消息放文件内容 将提取的文件内容放在第二个 system 消息里，而不是 user 消息，有助于模型更好地理解文档角色。
温度设置要低 文档分析、合同审查等场景建议 temperature: 0.1-0.3，减少幻觉，提高准确性。
baseURL 末尾不加斜杠 baseURL 应为 "https://api.moonshot.cn/v1"，末尾不加 /，否则可能导致路径拼接错误。
