第3课：流式输出

学习目标

  1. 理解为什么需要流式输出
  2. 用 client.chat.completions.create({ stream: true }) 实现打字机效果
  3. 处理 content 文本流和 reasoning_content 思考流（DeepSeek-R1）
  4. 用累加器获取完整响应

为什么需要流式？

非流式请求：模型生成完所有内容后，一次性返回。等待时间可能长达数秒，用户体验差。

流式请求：模型生成一个 token 就立刻发送一个，用户看到文字像"打字"一样逐字出现。

非流式：[等待 3 秒...] → "TypeScript 是一种...完整回答"
流  式：T → y → p → e → S → c → r → i → p → t → ...（实时出现）

额外好处：流式请求不受 HTTP 超时限制。对于 max_tokens 很大的请求，务必用流式。

基础流式输出

import { client, model } from "../shared/llmClient";

// 加 stream: true，返回一个异步可迭代的 chunk 流
const stream = await client.chat.completions.create({
  model,                              // 默认 deepseek-chat
  max_tokens: 4096,
  stream: true,
  messages: [{ role: "user", content: "解释一下什么是 TypeScript 泛型？" }],
});

let fullText = "";

// for await 遍历每个 chunk
for await (const chunk of stream) {
  const delta = chunk.choices[0]?.delta?.content ?? "";
  if (delta) {
    process.stdout.write(delta);     // 不换行，形成打字机效果
    fullText += delta;
  }
}

console.log(`\n\n总计 ${fullText.length} 字符`);

Chunk 结构

每个 chunk 形如：

{
  id: "chatcmpl-xxx",
  choices: [
    {
      index: 0,
      delta: {
        role: "assistant",          // 仅第一个 chunk 含
        content: "TypeScript",      // 本次新增的文本片段
        reasoning_content: "...",   // R1 模型才有：思考过程
        tool_calls: [...],          // tool 流也走 delta
      },
      finish_reason: null,           // 流结束才会变成 "stop" / "tool_calls"
    },
  ],
}

最后一个 chunk 通常含 usage 字段（提供商各异，DeepSeek/Kimi 会在最后一个 chunk 给出 prompt_tokens / completion_tokens）。

处理 Thinking（推理模型 R1）

DeepSeek-R1 在回答前先推理，思考过程通过 reasoning_content 字段流式传输：

const stream = await client.chat.completions.create({
  model: "deepseek-reasoner",       // R1 模型
  max_tokens: 16000,
  stream: true,
  messages: [{ role: "user", content: "请分析这段代码的复杂度..." }],
});

let inThinking = false;
let inAnswer = false;

for await (const chunk of stream) {
  const delta = chunk.choices[0]?.delta as { content?: string; reasoning_content?: string };

  if (delta?.reasoning_content) {
    if (!inThinking) { process.stdout.write("\n[💭 思考中...]\n"); inThinking = true; }
    process.stdout.write(delta.reasoning_content);  // 思考过程
  }

  if (delta?.content) {
    if (!inAnswer) { process.stdout.write("\n\n[回答]\n"); inAnswer = true; }
    process.stdout.write(delta.content);             // 最终回答
  }
}

何时用 R1？ 适合复杂推理任务：数学题、代码分析、逻辑推断。简单问答不需要开启，会增加 token 消耗。 Anthropic Claude 也有思考能力，但走的是 thinking block + 流事件 thinking_delta，详见末尾差异表。

流式 vs 非流式对比

// ─── 非流式 ───────────────────────────────────
const response = await client.chat.completions.create({
  model,
  max_tokens: 1024,
  messages: [{ role: "user", content: "Hello" }],
});
const text = response.choices[0].message.content;

// ─── 流式（推荐用于 max_tokens > 4096 的场景）────
const stream = await client.chat.completions.create({
  model,
  max_tokens: 16000,
  stream: true,
  messages: [{ role: "user", content: "Hello" }],
});

let fullText = "";
for await (const chunk of stream) {
  const delta = chunk.choices[0]?.delta?.content;
  if (delta) { process.stdout.write(delta); fullText += delta; }
}

完整示例：带进度的流式对话

import { client, model } from "../shared/llmClient";

async function streamChat(userMessage: string) {
  console.log(`用户: ${userMessage}\n`);
  console.log("助手: ");

  const stream = await client.chat.completions.create({
    model,
    max_tokens: 4096,
    stream: true,
    messages: [{ role: "user", content: userMessage }],
  });

  let charCount = 0;
  let outputTokens = 0;

  for await (const chunk of stream) {
    const delta = chunk.choices[0]?.delta?.content;
    if (delta) {
      process.stdout.write(delta);
      charCount += delta.length;
    }
    // DeepSeek/Kimi 在最后一个 chunk 提供 usage
    if (chunk.usage) outputTokens = chunk.usage.completion_tokens ?? 0;
  }

  console.log(`\n\n📊 统计：${charCount} 字符 | ${outputTokens} tokens`);
}

await streamChat("用 100 字解释什么是闭包？");

小结

方法 场景
chat.completions.create({ stream: false }) 简单请求，等待完整响应（默认）
chat.completions.create({ stream: true }) 实时显示、长文本、大 max_tokens
chunk.choices[0].delta.content 最简单的逐 token 处理
chunk.choices[0].delta.reasoning_content DeepSeek-R1 思考链
累加 delta.content 自己拼装最终完整文本

关于 Anthropic 原生协议

如果你切到 Anthropic 原生 SDK，流式 API 是 client.messages.stream()：

OpenAI 兼容（本课） Anthropic 原生
client.chat.completions.create({ stream: true }) client.messages.stream({ ... })
for await (const chunk of stream) stream.on("text", delta => ...) 或 for await (const event of stream)
chunk.choices[0].delta.content delta.text 事件
chunk.choices[0].delta.reasoning_content thinking content block + thinking_delta 事件
累加文本 await stream.finalMessage() 直接拿完整对象
