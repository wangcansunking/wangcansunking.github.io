谷歌、OpenAI都在用：MinerU 文档解析项目如何支撑 AI 军备竞赛？

一个 2000 星的开源项目，在 GitHub Trending 上登顶，不稀奇。但一个专注"文档解析"这种听起来像上个时代遗留问题的项目，一年拿下 6 万星标、调用量超 10 亿次，还被 Google Gemini 3 和 OpenAI GPT-5 的官方评测体系直接采纳——这就不是运气能解释的了。

MinerU，这个上海人工智能实验室 OpenDataLab 团队做的东西，正在被华为、阿里、Google 等上百家企业塞进自己的数据管线里。而它背后的核心推手何聪辉，一个拿过 ACM 戈登·贝尔奖的超算博士，下周要在 AICon 上海站把 MinerU 从 v1 到 v2.5-Pro 的技术路线图完整摊开。

高质量文档数据，已经从"锦上添花"变成了 AI 军备竞赛的入场券。

一个"无聊"赛道，为什么突然变成了兵家必争之地

文档解析这件事，往前倒五年，基本是 OCR 厂商和内网扫描件归档系统才会聊的话题。但 LLM 预训练和 RAG 规模化应用把局面彻底掀翻了。

先说预训练侧。GPT-4 级别的模型已经把互联网上能爬的公开文本吃得差不多了，接下来想继续提升能力，必须去啃那些"锁在 PDF 里"的结构化知识——学术论文、财报年报、技术手册、专利文献。这些文档不是纯文本，里面有公式、表格、多栏布局、阅读顺序跳转，传统爬虫 + 正则表达式的路子根本处理不了。何聪辉在演讲提纲里把这一点讲得很直白："LLM 预训练对海量高质量知识的渴求"和"RAG 系统对精准检索的刚需"，构成了双轮驱动。缺了文档解析这环，模型就相当于被断了高质量知识供给的粮道。

再讲 RAG 侧。2024 年以来，企业级 RAG 系统从 Demo 走向生产环境，最头疼的问题不是向量检索的 recall，而是文档切块（chunking）阶段的信息损失。一个包含三栏表格的 PDF 页面，如果解析引擎把表格拆成碎片、公式识别成乱码、阅读顺序搞反，后面再怎么优化 embedding 和检索策略都是白费功夫。MinerU 在这个场景下卡住了位置——它解决的恰好是"从原始文档到干净结构化数据"这最脏最累的一步。

这个赛道的竞争格局也在快速升温。何聪辉提到的 OmniDocBench，是文档理解领域一个综合评测基准，目前处于"百家争鸣"阶段。微软的 LayoutLM 系列、Adobe 的 PDF Extract API、以及一批基于 GPT-4V 和 Gemini 的多模态方案都在抢这个市场。MinerU 的差异化在于全链路开源 + 垂直领域深度优化——它不是通用多模态模型的轻量封装，而是从公式识别（UniMERNet）、布局检测（DocLayout-YOLO）到表格解析（OTSL）全自研了一条技术栈。

从拼凑工具库到"数据为王"，MinerU 的技术进化论藏着一条行业暗线

MinerU 的三个版本迭代，本质上映射了 AI 基础设施领域过去两年最大的认知迁移：架构创新正在让位于数据工程。

v1 阶段是典型的 Pipeline 思路：预处理 → 解析 → 后处理 → 格式转换，四阶段流水线。核心组件包括 UniMERNet（公式识别专用模型）和 DocLayout-YOLO（布局检测模型），本质上是一个"拼凑工具库"——把不同任务拆开，各用专用模型解决，最后拼接结果。这条路能跑通，但工程维护成本高，组件之间的误差传导很难根治。

v2.5 做了架构层面的关键创新：解耦式多模态大模型（Decoupled VLM）。核心思路是把文档解析拆成两阶段——先用低分辨率做全局布局理解（Stage I），再切到原分辨率做局部精细识别（Stage II）。这一步直接解决了端到端模型 $O(N^2)$ 的 token 复杂度问题，在效率上打出了一波优势。何聪辉这次会详细拆解这个架构设计，对做多模态模型的团队来说，参考价值不小。

但真正让我觉得有意思的是 v2.5-Pro。这个版本的核心洞察可以用一句话概括：当模型架构趋同之后，性能瓶颈已经不在模型本身，而在训练数据的质量和分布。 v2.5-Pro 和上一版用的是同一参数规模，但 Overall 指标从 92.98 跃升到 95.69——这 2.71 个百分点的提升，几乎全部来自数据工程的优化。

何聪辉把这套方法论称为"数据炼金术"，包含三个核心组件：DDAS（多样性感知采样，用视觉特征聚类 + 难度感知来筛选高价值训练样本）、CMCV（异构模型交叉验证，用不同模型互相校验识别结果，精准定位单模型短板）、以及 Judge-and-Refine 自动校正 + 定向专家标注的闭环。这套组合拳的价值在于，它不只是在 MinerU 这个单一项目上有效——任何做垂直领域模型训练的团队，面临高质量标注数据稀缺的困境时，这套数据引擎方法论都可以迁移复用。

再往深一层看，MinerU 背后其实藏着上海人工智能实验室 OpenDataLab 团队更大的野心：做 AI 数据基础设施的标准制定者。他们的评测基准被 Google Gemini 3 和 OpenAI GPT-5 官方采纳，而且是其中唯一来自中国团队的成果，这个信号比 GitHub 6 万星标更值得关注。因为这意味着，在"什么样的文档解析算好"这个定义权上，中国团队已经开始参与全球标准的制定。

何聪辉这次还会首次公开两个前沿探索方向：AgenticOCR（让文档解析从"全量扫读"转向按需动态解析，由 Agent 决定"我现在需要读文档的哪一块"）和 MinerU-Diffusion（基于扩散模型的非自回归解码，做到 3.26 倍速度提升）。这两个方向如果跑通，文档解析的定义会从"静态提取"变成"动态智能体感知"——不再是工具被动地吐结构化数据，而是 Agent 主动去"读"文档、理解上下文、只提取当前任务需要的信息。

这个思路跟当下 Agent 从 Demo 向工程化落地的大趋势高度吻合。6 月 26 日 AICon 上海站，何聪辉这场演讲大概率会成为文档解析方向今年最重要的技术分享之一。到时候 MinerU 团队会不会顺手放出 v3.0 的路线图——这可能是现场最大的悬念。
