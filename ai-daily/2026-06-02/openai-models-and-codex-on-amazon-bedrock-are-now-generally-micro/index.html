亚马逊Bedrock正式上线GPT-5.5和Codex，定价与OpenAI持平

亚马逊和 OpenAI 这对曾经在 AI 军备竞赛中各自为营的巨头，把关系推到了一个新的刻度上。一份 AWS 机器学习博客的更新，悄然宣告：GPT-5.5、GPT-5.4 以及 Codex，在 Amazon Bedrock 上正式全面可用（GA）。这不是一个“即将推出”的预告，而是“今天就可以部署到生产环境”的通告。而且，定价直接对标 OpenAI 官方一手价，没有额外加价。

这距离两家公司宣布“扩大合作伙伴关系”仅仅过去了一个月。一个月，从握手到产品落地，速度之快，让整个开发者社区的讨论从“他们真的会合作吗”直接跳到了“要不要把工作流迁到 Bedrock 上”。

这大概是云计算史上最务实的一次“打不过就加入”——只不过，加入的是对方的客户群。

从“你死我活”到“你的模型，我的底座”

要理解这次 GA 的冲击力，得往回看。亚马逊和 OpenAI 的关系从来不是温情脉脉的。AWS 自己押注了 Titan 模型，重金投入 Anthropic（前后砸了 80 亿美元），把 Claude 系列当作 Bedrock 的旗舰筹码。而 OpenAI 则把 Microsoft Azure 作为独家云服务商，Sam Altman 和 Satya Nadella 肩并肩站台的画面过去几年里出现了无数次。两边是泾渭分明的阵营：一边是 AWS + Anthropic，另一边是 Azure + OpenAI。

但现在，GPT-5.5 出现在了 Bedrock 的模型目录里。而且不是阉割版——官方博客明确写着“the most advanced frontier model from OpenAI”，定价“matches OpenAI first-party rates”。这意味着，一个已经在 AWS 上跑了五年工作负载的金融公司，现在可以不用把数据搬出 VPC，不用重建 IAM 权限体系，不用重新做合规审计，就能直接调用 GPT-5.5 来生成招股书分析段落。一个用惯了 S3 和 Lambda 的创业团队，可以用 Codex 来重构整个代码库，而推理流量全部留在自己选的 Region 里。

这种“留在原地接入最前沿模型”的便利性，比任何技术白皮书都有说服力。我看到官方博客里那段关于 Bedrock 推理引擎的描述时，第一反应是：这是冲着企业客户最痛的三个点去的——性能波动、硬件故障后的请求丢失、以及合规。Bedrock 给每个客户独立的隔离队列，做自动容量管理，保证高负载下性能不抖；每个请求的完整状态被持续持久化，节点重启了也能从断点继续跑，而不是重头再来；所有调用自动继承 AWS 那套 IAM、VPC、PrivateLink、KMS 加密和 CloudTrail 审计日志。而且，prompts 和 responses 不会被拿去训练模型，也不会分享给模型提供商。

这些能力，在 OpenAI 自己的平台上并非全都默认提供，或者说需要额外配置。但在 Bedrock 上，它们是开箱即用的默认项。对于 Amgen 这样的大型药企，CTO Sean Bruich 在公告里的原话很直白：

“在像我们这样问题复杂、对科学准确性和决策质量要求极高的领域，GPT-5.5 和前沿模型在能力、质量和一致性上带来了引人注目的进步。让这些模型在 AWS 上可用，为我们提供了一条重要的新路径，可以在负责任 AI 框架内探索和扩展这些能力。”

这不是客套话。制药公司要把 AI 用到药物研发流程里，数据不出合规边界、每一步可审计、模型不被第三方用于训练，是底线而不是加分项。AWS 正好把这个底线变成了标准配置，顺便把 OpenAI 的模型放上去。

Codex 上架：500 万开发者的“就地武装”

更让我觉得有意思的，是 Codex 的 GA。OpenAI 自己的数据是“每周有超过 500 万人使用 Codex”。这个数字背后，是大量在 VS Code、JetBrains、Xcode 里写代码的开发者，以及用 CLI 和 Codex App 接入的团队。现在，所有这些入口的推理流量，都可以选择路由到 Amazon Bedrock，推理留在选定 Region 内，满足数据驻留要求。

计费方式是纯 per-token，没有 seat license，没有 per-developer 承诺。这对比 GitHub Copilot 的月付席位模式，是一种完全不同的弹性——你可以在一个 sprint 里让 50 个人重度使用 Codex，下一个 sprint 只用 5 个人，成本跟着 token 量走，不养闲账户。而且，Codex 现在跑的是 GPT-5.5 做推理，官方称“相比之前版本，完成相同工作的效率更高、质量更好”。它能在整个仓库范围内保持上下文，推理模糊的失败原因，用工具检查假设，并在修改代码时感知系统间的连接关系。

Autodesk 的 VP Ritesh Bansal 在公告里提到，团队正在评估如何用这些工具“加速开发工作流，支持更明智的决策”。对于 Autodesk 这种做建筑设计软件的公司，代码库庞大、跨团队协作复杂，Codex 能理解整个仓库上下文的能力，可能比单纯生成一段函数更有价值。

但这里有一个容易忽略的细节：Codex on Bedrock 的推理，用的是 Bedrock 的推理引擎，而不是 OpenAI 自己的基础设施。这意味着一件事——AWS 把 OpenAI 的模型和工具，变成了自己云上的一项原生能力。开发者不需要去 OpenAI 注册账号，不需要管理另一套 API key，一切都在 AWS 控制台里完成，账单也合并到 AWS 的月度结算里。对于那些已经在 AWS 上有大量消费承诺（commitments）的企业，Codex 的 token 消耗直接计入现有承诺额度，这在财务上是一个相当顺滑的体验。

下一步：Agent 和 Daybreak 的伏笔

公告里还埋了两个钩子。一个是“Amazon Bedrock Managed Agents, powered by OpenAI”——即将推出，让用户部署基于 OpenAI agent harness 的生产级智能体，每个 agent 有自己的身份、记录每一步操作以供审计，所有模型推理跑在 Bedrock 上。

另一个是 Daybreak，OpenAI 的“改变软件构建和防御方式”的愿景产品，包含 cyber models 和 Codex Security，专门帮安全团队识别漏洞、审查代码风险、指导整个开发生命周期的修复。当 Daybreak 上线 Bedrock 时，安全团队可以直接沿用 AWS 上已有的治理和操作框架。

这两个预告说明，这次 GA 不是终点，而是 AWS 把 OpenAI 全家桶逐步搬上 Bedrock 的第一步。从模型到编码代理，再到托管智能体和安全模型，AWS 显然想把 OpenAI 的能力变成 Bedrock 上的一个“订阅套餐”，而不是一次性的模型上架。

这让我想起 Anthropic 最近签下的那笔 22 万张 H100 的算力大单——算力供应商之一，正是把数据中心租给 Anthropic 的 SpaceX。而 OpenAI 这边，则在用 AWS 的渠道触达那些永远不会离开 AWS 的企业客户。三家公司，两两之间既有竞争又有交易，关系网复杂得像一部科技版的《三国演义》。

但有一点是清晰的：企业 AI 的战场，正在从“谁的模型最强”转向“谁能让最强模型在企业的私有环境里安全、合规、可审计地跑起来”。AWS 在后者上积累了近二十年的信任，现在它把门打开，让 OpenAI 走进来——而 OpenAI 似乎也很乐意走进来，毕竟，没有哪家公司会因为坚持“只用自家云”而拒绝几万个 AWS 上的企业客户。

接下来就看微软怎么接招了。Azure OpenAI Service 一直是 GPT 系列的独家云入口，现在这个“独家”被 AWS 撕开了一道口子。而且价格还一模一样。这或许不是一场谁赢谁输的战争，而是云厂商之间围绕“模型分发权”的一场重新洗牌。开发者们大概会很高兴——毕竟，多一个选择，从来不是坏事。
