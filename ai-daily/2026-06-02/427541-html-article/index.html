神秘公司月烧34亿Claude账单，亚马逊微软取消AI排行榜

一个工程师的年薪 50 万美元。他一年烧掉的 Token 账单，25 万美元。黄仁勋说，如果这个数字不到 25 万，他会“深感担忧”。

现在，有一家企业一个月就烧掉了 5 亿美元的 Claude 账单。按这个逻辑推演，黄仁勋应该感到极度舒适——但这家企业的 CFO 大概率已经瘫在办公椅上。

2026 年 5 月，Axios 爆出了一条让整个硅谷沉默的消息：一家至今没有公开身份的公司，因为老板给员工开通了 Anthropic 企业授权后忘了设置用量上限，30 天内烧掉了 5 亿美元。折合人民币 34 亿元。平均每天 1.13 亿。每小时 470 万。每分钟 7.8 万。

如果按 Anthropic 目前约 470 亿美元的年化收入算，这一家公司一个月的消耗，就占到了 Anthropic 月收入的近八分之一。换句话说，Anthropic 每赚 8 块钱，就有 1 块来自这家“忘了设上限”的神秘企业。

Token 经济学的第一定律：烧得越多，赚得越狠——前提是你不是那个买单的人。

亚马逊的排行榜，被自己人玩崩了

就在这家神秘公司天价账单曝光的几乎同一时间，《金融时报》挖出了另一条消息：亚马逊已经悄悄取消了内部 AI 使用排行榜。

取消的原因，说起来颇具黑色幽默。亚马逊员工为了在排行榜上冲名次，开始疯狂刷 Token，执行大量没有实际价值的任务。排行榜设立的初衷是鼓励大家多用 AI，结果变成了一个“谁更会烧钱”的锦标赛。

这让我想起了一个经典的管理学悖论：当一个指标变成目标，它就不再是一个好指标。

亚马逊的遭遇不是孤例。Meta 也经历过几乎一模一样的剧本。内部被曝出建立了 Claude 使用排行榜，单月消耗数十亿 Token。从最初的鼓励“多用、多刷”，到现在逐渐收紧激励，转向更关注实际产出。这个转向本身，就是一个信号。

多邻国的故事更值得玩味。这家语言学习巨头曾计划将 AI 使用情况纳入员工绩效考核。但员工直接反问了一个让管理层无法回答的问题：“我们是不是必须为了使用 AI 而使用 AI？”最终，多邻国撤回了这个决定。

今年 4 月的一档播客中，多邻国 CEO 路易斯·冯·安总结得相当坦诚：

感觉我们不是在为实际结果负责，而是在试图强推一些并不总是适合的东西。

这句话，几乎可以成为过去 18 个月整个“Token 狂热”时代的墓志铭。

微软翻脸：六个月前迎进门，六个月后扫地出门

如果说亚马逊取消排行榜还算是“内部调整”，那微软的动作就更加直接和不留情面了。

2026 年 6 月 30 日之前，微软将取消 Experiences + Devices 部门大部分 Claude Code 授权。这个部门负责 Windows、Microsoft 365、Outlook、Teams、Surface 等核心产品。工程师们被要求迁移到自家的 GitHub Copilot CLI。

而 Claude Code 进入微软内部，距离这个截止日期不过 6 个月。

微软给出的理由很符合硅谷传统——Claude Code 已经完成了帮助团队学习和探索的阶段，是时候回到“Eat Your Own Dog Food”，使用自己的产品了。这个说法在文化上无懈可击。但你很难不注意到，Token 账单一定在某个邮件链里被反复转发过。

（值得一提的是，“Eat Your Own Dog Food”这句硅谷黑话，意思是用自己的产品。微软这次是真的在吃，只是换了个口味。）

微软的转身速度，折射出一个更大的趋势：企业开始从“先用再说”的疯狂实验阶段，进入“精细运营”的冷静期。这个切换，可能比很多人预想的来得更快。

Uber 的坦白：4 个月烧完全年预算，Token 和产出没有线性关系

Uber 的故事可能是所有案例中最直白、也最有杀伤力的。

Uber CTO Praveen Neppalli Naga 在今年早些时候透露，公司工程师仅用 4 个月就烧完了全年的 Claude Code 预算。4 个月，全年预算归零。这意味着接下来 8 个月要么追加预算，要么砍用量。

但真正关键的，是 Uber 首席运营官 Andrew Macdonald 随后做出的公开表态：

AI Token 消耗与最终发布的有价值产品之间，似乎并不存在明显的线性关系。

翻译成大白话：烧掉更多 Token，并不一定意味着创造出更多价值。

这句话如果放在 2025 年，可能会被当作异端。但在 2026 年 5 月，它听起来更像是一个迟到的共识。

Uber 不是一家小公司。它不是那种会被几百万美元账单吓到的初创企业。但即便是 Uber，也开始认真审视每一个 Token 背后的 ROI。这个信号，比任何一份研究报告都更有说服力。

Token 不是价值。完成任务、交付产品、创造收入，才是价值。这个道理本该是常识，但在过去一年半里，整个行业似乎集体忘记了。

DeepSeek 的沉默转向，和米哈游的 200 万学费

大洋彼岸的故事，同样耐人寻味。

今年 4 月发布 V4 时，DeepSeek 在技术报告里顺带提了一句：V4 已经成为公司内部员工日常使用的 Agentic Coding 模型，体验优于 Claude Sonnet 4.5，交付质量接近 Opus 4.6。

官方没有明确提及成本因素。但如果你是一家拥有自研顶尖模型的公司，用自己的模型完成同样的工作，显然是一个更经济的选择。DeepSeek 没有高调宣布“弃用 Claude”，它只是默默地把自己的模型用起来了。这种沉默本身，比任何宣言都更有分量。

国内的另一个故事来自米哈游。米哈游的郑银河曾分享过一次经历：某个 Agent 项目上线后，一晚上烧掉了 200 万元 Token 费用。

200 万人民币，一个晚上。

这笔学费不算便宜。但它让米哈游内部清醒地意识到一个问题：Agent 的自主循环一旦失控，烧钱的速度是按秒计算的。一个没有收敛条件的 Agent 工作流，本质上是一个开着水龙头的水槽。你以为是让它在思考，实际上它只是在空转。

“Tokenmaxxing”文化：一个被发明出来又被扔进垃圾堆的词

在 Hacker News 上，有人给过去半年流行的那股风气起了个名字：Tokenmaxxing。

这个词是大语言模型术语“Token”和后缀“-maxxing”的组合。后者源自互联网俚语，指的是在某件事上做到极致、疯狂堆资源。Tokenmaxxing 翻译过来就是：把“烧了多少 Token”当成 AI 用率和生产力的衡量指标，然后疯狂往上堆。

这股文化的起点，可以追溯到 2026 年 3 月黄仁勋在《All-In Podcast》上的那段公开背书。当时他说，如果一个年薪 50 万美元的工程师每年消耗的 Token 不到 25 万美元，他会深感担忧。

这句话在当时被很多企业奉为圭臬。逻辑链条看起来也很清晰：如果 AI 真的能提升工程师效率，那么多烧一些 Token，本质上是在购买生产力。用的越多，省得越多。

但问题在于，这个逻辑链条有一个隐蔽的断裂点：它假设 Token 消耗量和生产力提升之间存在线性关系。而 Uber 的 Andrew Macdonald 用真实账单证明了，这个假设并不成立。

Hacker News 上的讨论中，有人把矛头直接指向了这种文化本身。在他们看来，问题并不在于 AI，而在于企业把“烧更多 Token”当成了目标本身。如果使用更高效的模型、合理控制 Agent 工作流，或者让人类参与关键决策，成本本可以低得多。

也有人开了个玩笑：这场运动最大的赢家，从头到尾可能只有黄仁勋。

这句话是玩笑，但不完全是玩笑。英伟达的 GPU 是 Token 生产的基础设施。每多烧一个 Token，背后都需要算力来支撑。而算力，就是英伟达的营收。

不是退潮，是退烧

不过，如果把这一系列事件简单理解为“AI 热潮退去”，可能就错过了真正重要的东西。

不少开发者在 Hacker News 上的观点值得认真对待。他们认为，这不是 AI 热潮退去的信号。恰恰相反。这意味着企业终于开始从“先用再说”的狂热阶段，进入“精细运营”的成熟阶段。

过去两年，企业最担心的是员工不用 AI。而现在，越来越多企业开始担心另一件事：AI 是不是用得太多了。

这个转变本身，恰恰证明了 AI 的渗透率已经达到了一个相当高的水平。如果不重要，谁会去操心用量？

未来大家关注的重点，可能不再是谁烧掉更多 Token，而是谁能用更少的 Token 完成更多任务。高效 Agent 工作流、廉价模型处理简单任务、昂贵模型只负责关键决策、更严格的预算和权限管理——这些听起来像常识的东西，正在成为下一阶段的核心课题。

我自己的感受是，这场“退烧”来得比预期更快，但方向比预期更健康。一个行业如果连“烧钱买幻觉”和“烧钱买生产力”都分不清，那才是真正的问题。现在，至少 CFO 们开始拿起计算器了。

黄仁勋的 Token 经济学或许没有翻车，但买单的人终于学会看账单了。
