王晓野披露：仅10%企业从AI获价值，记忆管理成关键破局点

2026 年 6 月，北京。一场千人规模的 AIGC 峰会上，亚马逊云科技的技术总监王晓野走上台，没有画大饼，没有炫 Demo，而是丢出了一组让人沉默的数字：87% 的企业说自己在“大规模部署 AI”，但真正拿到价值的，只有 10%。

这个落差，比任何技术路线之争都更刺眼。过去两年，全行业都在“养龙虾”——个人开发者、创业团队、大厂实验室，都在 Mac mini 上跑出了令人惊艳的 Agent。可一旦把这些东西搬进企业机房，搬进几千个并发用户、几百套遗留系统、几十条合规红线交织的生产环境，问题就变了味。

Demo 从来不缺掌声，缺的是让它活过第一个季度。

Token 贵，不是单价贵，是你喂给模型的垃圾太多了。

个人养龙虾和企业养龙虾，隔着一条你没见过的河

王晓野的演讲里，有一个比喻反复出现：个人养龙虾 vs 企业养龙虾。个人用户配好环境、跑起来、截个图发朋友圈，完事。但企业场景里，你要让几千个 Agent 同时在线，不能串台、不能失忆、不能越权，还要能自动重启、自动扩容、自动在权限边界内执行任务——这已经是另一套工程体系了。

他给了一个很具体的切面：记忆管理。企业里不同员工、不同部门的 Agent 需要共享一部分知识，又必须严格隔离另一部分。如果记忆生命周期没人管，长期记忆里堆满过期信息、矛盾知识，Agent 的判断就会退化。更致命的是，很多团队把几千条 skills 一股脑塞进 prompt，让模型自己去挑——“这就相当于你让一个人背完整本百科全书再回答一个简单问题，Token 全浪费在无效上下文上了。”

亚马逊云科技给出的解法是 Bedrock AgentCore，一个被他们称为“Harness”的东西。这个词选得很准——Harness 本意是马具、挽具，用来驾驭力量。模型是引擎，但引擎需要方向盘、刹车、仪表盘。AgentCore 的九大模块里，Runtime 管横向扩展，Memory 管记忆生命周期，Code Interpreter 和 Browser 给 Agent 装上手脚，Identity 和 Gateway 负责把企业现有的 CRM、ERP 接进来，Policy、Evaluation、Observability 则负责让整个流程可视、可控、可审计。

这套东西听起来不性感，但恰恰是那 77% 的企业（87% 宣称部署 vs 10% 真正拿到价值）掉进去的鸿沟。

有意思的是，亚马逊在同一个平台上还塞进了一个“Managed Agent”——由 OpenAI 提供前沿模型和他们在 ChatGPT 里积累的 Harness 最佳实践，再套上亚马逊的云安全基础设施。这意味着企业可以选：要么用 Bedrock AgentCore 自己搭，模型随便换；要么直接用 OpenAI 打包好的那一套，开箱即用。王晓野说这话时语气很平淡，但背后的信号不小：企业级 Agent 的竞争，已经从“谁模型更强”转向了“谁能把模型管住”。

15% 的决策将由 AI 自主完成——不是辅助，是拍板

王晓野引用了 Gartner 的一组预测：2028 到 2030 年，超过 15% 的企业日常工作决策将由 Agent 或 AI 完全自主完成。关键词是“自主”——不是帮人整理数据、生成草稿，而是直接做出决策。

这组数字放在两年前，会被当成科幻。但今天，配合麦肯锡的另一组数据看：82% 的企业领导者表示将在 2026-2028 年增加“数字员工”比例，Agent 带来的增量商业市场规模可能从 2.6 万亿跳到 4.4 万亿美元。这些数字堆在一起，说明一件事：Agent 不再是“试试看”的实验项目，它正在变成企业预算表里的正式科目。

但王晓野反复强调的另一句话更值得琢磨：“过去 30 年里，个人生产力其实从来没有被真正大规模颠覆过。”Office 换了十几个版本，邮件客户端从 Outlook 到 Slack 到 Teams，工作方式本质上没变。Agent 的出现，第一次让人感觉到“我不用亲自做这件事了”——这种体感，用过“龙虾”的人都懂。

亚马逊自己的产品 Quick 就是一个信号。王晓野说他每天早上要花 20 分钟到 1 小时清理各平台的任务，Quick 能主动提醒、主动建议“你该给某人约个会了”，甚至在长期使用后形成个人知识图谱，决策越来越像用户本人。这种“深度个性化”的 Working Agent，正在把 Agent 从开发者玩具变成白领员工的日常工具。

回到那组 87% vs 10% 的数据。王晓野的整场演讲，其实就在回答一个问题：那 77% 的企业到底卡在哪了？他的答案分五层——算力性价比、模型选择权、AI-ready 数据平台、Harness 管控体系、Agent 应用落地。每一层都有坑，每一层都有人摔过。

亚马逊云科技 CEO Matt Garman 说“每一个应用都将会被重构”。这句话 2023 年就有人喊过，但 2026 年再说，味道已经不一样——不是畅想，是已经看到重构正在发生。只是重构的前提，不是有一个更强的模型，而是有一套能让模型安全、可信、不中断跑起来的工程底座。

那些还在往 prompt 里塞几千条 skills 的团队，可能还没意识到：Token 贵的真正原因，不是算力成本下不来，是你喂给模型的东西里，有一半是它根本不该看到的垃圾。
