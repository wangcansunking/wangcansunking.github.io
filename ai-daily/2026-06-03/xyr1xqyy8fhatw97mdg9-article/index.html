亚马逊撤榜背后：Claude月度账单5亿美元，Agentic协作如何掏空预算？

财务主管盯着屏幕上的数字，以为是系统故障。刷新，再刷新，还是那个数：5 亿美元。一个月。

不是被黑客攻击，不是云配置错误，不是某个失控的自动化脚本。这笔钱是公司给全体员工开通 Claude 授权后，大家“正常使用”烧出来的。折合人民币超过 30 亿，够在北京国贸买一栋楼，够养一支千人研发团队一整年，够做很多事情——偏偏不够让业务真正变好。

扒开账单细节，最荒诞的部分浮出水面：相当大一块消耗，来自员工跑任务时反复撞上报错，一遍遍手动点“重试”点出来的。AI 在报错，人在点重试，账单在疯涨。三个角色各忙各的，谁都没让谁更聪明。

把 Token 当 KPI，本质上是在用真金白银给形式主义买加速器。

一场全员 AI 的狂欢，烧出了 60 万亿 Token 的烟火

这家公司当然不是唯一踩坑的。Meta 内部有人做了一个叫 Claudeonomics 的榜单——名字取得戏谑，专门统计谁用 AI 用得最猛。30 天，全公司烧掉超过 60 万亿个 Token。60 万亿，这个数字大到已经失去了直觉感知。换算一下：如果每个 Token 是一秒，60 万亿秒大约是 190 万年，够人类从直立行走进化到发朋友圈。

“榜一大哥”一人就占了 2810 亿 Token，折合将近 50 万美金一个月。一个人，一个月，50 万美金。这还不是在做什么惊天动地的 AI 研究，大概率就是日常跑任务、调 prompt、让 Agent 反复试错——和那位点重试点出 5 亿的哥们是同类。

Uber 的故事更扎心。公司最早给 5000 名工程师配齐 Claude Code，几个月内使用率猛冲，表面上一片繁荣。结果一季度刚过，CTO 就叫苦不迭：2026 全年的 AI 编程预算已提前烧光。全年预算，三个月，没了。

国内这边同样热闹。在阿里云峰会上，米哈游技术负责人分享了一个案例：有员工为了做一个项目，搭了几十个 Agent 协作，一晚上烧掉 200 万人民币的 Token。一晚上，200 万。第二天早上醒来，项目进度可能还卡在原地，但账单已经到了。

（米哈游的人说这话时不知道是什么表情——也许是一种“我们有钱但也经不起这么造”的复杂。）

我翻这些案例的时候，脑子里一直有个画面：一群聪明人，拿到了一个更聪明的工具，然后集体用它做了最笨的事情——比谁用得更多。

1000 倍消耗背后，是一场心照不宣的合谋

为什么 Token 账单会如此失控？高盛 2025 年 5 月的报告《Decoding the Agentic Economy》给出了一个精准的数字：

在 Agentic 模式下，模型由于需要不断“思考 - 检索 - 调用工具 - 重新读取完整上下文”的循环，其 Token 消耗量达到了普通问答模式的 1000 倍。

1000 倍。这个数字值得停下来想一想。普通问答模式，你问一句，模型答一句，一个来回几百到几千 Token。Agentic 模式呢？模型要思考——生成内部推理链；要检索——调用外部知识库；要调用工具——执行代码、查询数据库、发 API 请求；然后每次调用完，还要把完整的上下文重新读一遍，确保不出幻觉。一轮下来，几万 Token 是常态，几十万 Token 不稀奇。

你以为它在思考，其实它也在烧钱。而且烧的节奏是 1000 倍加速。

更有意思的是上游的态度。模型公司、云厂商、芯片公司，心照不宣地把“多用 AI、多烧 Token”包装成了先进生产力的进步。你用得越多，说明你转型越积极；你烧得越多，说明你 AI 化程度越高。这套叙事太顺滑了，顺滑到没人好意思质疑——直到账单来了。

同一笔钱，在两家公司的账本上写着完全不同的名字：一个叫增长，一个叫事故。

现在再看 Anthropic 一季度收入 48 亿美元、预计二季度翻倍至 109 亿美元这条曲线，是不是就不难理解了？企业 API、Claude Code、无数公司内部那套“全员 AI”“Agent 先跑起来再说”的组织冲动——这些才是真正的燃料。下游“事故性”烧掉的钱，转身就成了模型公司财报上实打实的进账。

再看英伟达，最新季度收入 816 亿美元。黄仁勋昨天在 GTC Taipei 上说，从产业角度看，Token 已经成为资产和营收单位。这句话聪明之处在于，它悄悄偷换了主语。

对英伟达来说，Token 确实是资产。Token 越多，推理越多；推理越多，GPU、网络、液冷都得跟着卖。对 Anthropic 来说也是资产，企业每多跑一轮 Agent，最后都会变成它的营收和利润率。但站在下游企业的账本上，Token 首先是成本。成本不是不能花，但只有换回更短流程、更少返工、更强交付，才有资格叫资产。如果只是为了排行榜和先进性证明来烧，那不过是更贵的形式主义。

亚马逊动手了：当排行榜被污染，指标就死了

亚马逊并不是一开始就清醒的。它内部曾有一个叫 KiroRank 的榜单，挂在自家的 Kiro 开发者平台上，按工程师消耗的 Token 量排名。加上公司此前下了硬指标，要求八成以上员工每周必须用 AI，榜单一出，员工纷纷开始 Tokenmaxxing——这个词值得记住，它是 Token 和 maximizing 的合成，意思是“刷 Token”。

派 Agent 去干根本没必要的活，把 Token 烧成排名，再把排名内化成职场安全感。这逻辑链条太熟悉了：以前卷工时，现在卷 Token；以前 KPI 注水，现在云账单注水。技术进步了，职场的底色一直没变。

直到亚马逊高级副总裁 Dave Treadwell 终于坐不住了，在内部大会上呼吁：

不要为了用 AI 而用 AI。

榜单随即被下线。新指标换成了“normalised deployments”，即标准化部署量——衡量工程师到底有没有用 AI 交付出真正有用的代码，而不是单纯数你烧了多少 Token。

作为云计算霸主，亚马逊当然信 AI。但它也亲自证明了一件事：AI 使用量这个指标，太容易被污染。你统计 Token，员工就会刷 Token；你统计 Prompt 数，员工就会编 Prompt；你暗示“不用 AI 就落后”，员工就会想办法证明自己没落后。

经济学里有个古德哈特定律：当一个指标变成目标，它就不再是一个好指标。到了 AI 时代，这句话可以换一个说法：当 Token 成了 KPI，它就不再是生产力指标，而是一张会自动膨胀的云账单。

亚马逊并不是唯一踩刹车的人。Shopify 已经把 Token 排行榜改成更中性的使用仪表盘，加上熔断机制——烧到一定额度自动切断，不给失控留空间。Duolingo 曾经想把 AI 使用情况纳入绩效考核，后来撤回。微软也开始收缩部分外部 AI 编程工具授权。这些公司的共同点是什么？它们都不是不用 AI，而是发现“用量”这个指标根本不可靠，用它来考核人，等于花钱鼓励造假。

（我看到亚马逊撤榜这个细节时第一反应是：Dave Treadwell 可能是在某次月度复盘会上看到了 KiroRank 的数据，然后问了一个很朴素的问题——“排第一的这位，他交付了什么？”答案大概率不太好看。）

你的成本是老黄的资产，但成本不是不能花

那些已经烧掉的钱，并不会凭空消失。一家公司的失控，往往是另一家公司的报表。

Anthropic 单季度收入破百亿、估值破万亿美元；英伟达季度收入 816 亿、黄仁勋在 GTC 上笑容满面。这些数字背后，每一分钱都来自下游企业的 API 调用、Agent 循环、以及无数个“先跑起来再说”的决策。

现在再看黄仁勋那句话，味道就变了。他说 Token 已经成为资产和营收单位——这话从他嘴里说出来完全正确。对英伟达，Token 是资产；对 Anthropic，Token 是营收。但对你呢？对你的公司呢？如果 Token 消耗没有换回更短的交付周期、更少的线上事故、更低的返工率，那它就不是资产，是纯纯的支出。财务追着你问“这钱花得值不值”时，你不能回答“但我们 AI 用得很多”。

Uber COO 在经历了激进推行后的醒悟，值得每个正在制定 AI 策略的人读三遍：

我们的代码确实多交付了一些，但很难把它和“给用户做出了更有用的功能”划上等号。

这句话的杀伤力在于，它承认了一个尴尬的事实：AI 用得多，不等于用得好。代码多交付了一些，但那些代码有没有让用户觉得这个 App 更好用了？有没有让司机匹配更快了？有没有让账单更清晰了？说不清楚。

代码分析公司 GitClear 扒了 2.2 亿行代码，数据更触目惊心：用 AI 辅助后，写完两周内就得返工的代码翻了 9 倍；复制粘贴的重复代码翻了 8 倍。9 倍返工率，8 倍冗余代码。这意味着什么？意味着很多企业只是把人类低效的地方，换成了模型以更贵的方式低效一遍。

以前是实习生写出需要重构的代码，现在是 AI 以 1000 倍的成本写出需要重构的代码。

这才是最荒诞的部分：你花了更多钱，得到了更快的交付速度，但返工率也同步飙升。速度快了，质量没变好，甚至可能更差——因为 AI 生成的代码往往缺少统一的设计思路，像是一群不同的人各自写了一段然后硬拼在一起。

真正的战场不在 Token 榜单上，而在业务深处

当然，这并不意味着要回到保守主义，否则投资 AI 的价值。问题是，很多公司把 AI 落地理解得太浅了——以为给员工开账号就是 AI 转型，以为使用率上去了就是组织进步，以为 Token 烧得多就是 AI 用得深。

真正的 AI 组织提效，并不在 Token 榜单上，而发生在业务深处。企业 AI 落地最难的地方，不是给员工开账号，而是让模型穿透工作流。什么叫穿透工作流？是 AI 不只是帮你写一段代码，而是理解这段代码在整个系统里扮演什么角色，知道依赖的上下游是谁，能在写完之后自动跑测试、过 CI/CD、在 staging 环境验证，然后生成 changelog 通知相关团队。这一套下来，Token 消耗可能比单纯写代码高 10 倍，但它换回的是端到端的效率提升，而不是一堆等着返工的代码。

这也是为什么现在 OpenAI、Anthropic 都在重金招募 Forward-Deployed Engineer——前向部署工程师。这个岗位的职责不是写 API 文档，不是做 demo，而是深入客户内部去拆流程、理权限、做集成。模型公司自己都已经意识到，光是 API 层面的交付，根本落不了地。你得钻到客户的业务里，理解他们的数据长什么样、审批流程怎么走、哪些环节是瓶颈、哪些人掌握关键决策权——然后才能让 AI 真正嵌入进去，而不是浮在表面烧 Token。

（这个岗位的出现本身就说明一个问题：模型公司卖的不再只是模型能力，而是“落地能力”。卖 API 是卖汽油，派 FDE 是帮你把车造好、路修通、教会你司机怎么开。后者的客单价和续费率，远不是前者能比的。）

上游公司当然会继续讲 Token 的故事，但那毕竟是别人的增长故事。普通企业如果不先把自己的业务问题、流程结构想清楚，就急着把别人的故事搬进自己的 KPI，最后只会沦为别人财报里的配角。你这一头的事故，就是那一头的营收。

这轮 AI 淘金热里，卖铲子的已经赢麻了。至于那些买了铲子就开始挖、但连矿在哪都没搞清楚的人，账单会替他们记住一切。会烧 Token 并不算本事，能说清楚“这笔钱到底让组织哪里变好了”才算。
