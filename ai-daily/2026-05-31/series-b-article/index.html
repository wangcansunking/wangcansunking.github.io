2026-05-31AI Daily

OpenRouter获1.13亿美元B轮融资：Alphabet旗下基金领投，NVIDIA等巨头跟投

2024 年秋天，一个叫 Alex Atallah 的人做了一个决定：把 OpenRouter 从项目变成公司。那时候，每周用这个平台流转的 token 量是 5 万亿。

18 个月后，这个数字翻了 5 倍。

2026 年 5 月 28 日，OpenRouter 宣布完成 1.13 亿美元 B 轮融资。领投方是 CapitalG——Alphabet 旗下的独立成长基金。跟投名单拉出来，几乎是一张企业软件领域的全明星阵容：NVIDIA 的风投部门 NVentures、ServiceNow Ventures、MongoDB Ventures、Snowflake Ventures、Databricks Ventures，以及 AMP PBC 和 Pace Capital。老股东 a16z 和 Menlo Ventures 继续加码。

一家做 API 网关的公司，把整个企业软件栈的头部玩家都变成了自己的股东——这不是融资，这是生态绑定的艺术。

我盯着这份投资者名单看了很久，最让我愣神的不是金额——1.13 亿美元在 2026 年的 AI 赛道算不上惊天动地。真正有意思的是结构：这轮投资方里没有一家是纯粹的财务投资者。CapitalG 背靠 Google Cloud 的渠道网络，NVIDIA 手里攥着全球最紧缺的 GPU 供给，ServiceNow、MongoDB、Snowflake、Databricks 分别是各自赛道的平台级公司——ITSM、文档数据库、数据仓库、数据湖仓。它们不是来赌 OpenRouter 独立 IPO 的，它们是来确保自己的企业客户在拥抱多模型 AI 时，流量会经过一个跟自家产品深度集成的路由层。

这不是一个 1.13 亿美金的融资故事。这是一个关于“AI 时代中间件如何吃掉价值链”的样本案例。

25 万亿 token 在跑，8 百万开发者在上船

先看一组硬数字。

OpenRouter 在公告里写了三句话：过去六个月，平台周 token 处理量从 5 万亿增长到 25 万亿；今年预计处理超过一千万亿（1 quadrillion）token；服务超过 800 万开发者，接入 400+ 模型。

5 万亿到 25 万亿，半年 5 倍——这不是增速曲线，这是垂直起跳。

让我做一道简单的算术。25 万亿 token 一周，摊到每天是约 3.6 万亿 token。如果平均每个 token 的推理成本在 2026 年已经压到了极低的水平——假设混合模型均价在每百万 token 几美分到十几美分之间——那么 OpenRouter 每周经手的流水规模已经是一个相当可观的数字。

（当然，OpenRouter 自己不是模型提供商，它赚的是路由层和网关层的差价/抽成。具体抽佣比例官方从未披露过，但行业惯例在几个点到十几个点之间浮动。）

但比流水更重要的，是那 800 万开发者。800 万是什么概念？GitHub 在 2023 年全球开发者数量刚破 1 亿，其中真正活跃在 AI 应用层的开发者可能在千万级别。OpenRouter 说自己有 800 万注册开发者，意味着它几乎覆盖了全球 AI 应用开发者的大半壁江山。

这引出一个关键问题：为什么开发者不直接去用 Anthropic、OpenAI、Google、Meta 的官方 API，而要绕一道 OpenRouter？

答案藏在另一个数字里：400+ 模型。

任何一个开发者，如果同时要用 Claude 4 做长文本推理、用 Gemini 3 做多模态理解、用 Llama 4 做本地微调、用 Stable Diffusion 出图、用 Whisper 做语音转文字——他需要对接至少 5 套不同的 API 规范、5 套计费系统、5 套 rate limit 策略、5 套内容安全审核标准。而且一旦某个供应商宕机（在 2025-2026 年这几乎每个月都在发生），他的整个应用就挂了。

OpenRouter 做的事情，本质上是把这些复杂度吞掉，吐出来一个统一的接口。开发者只需要调一个 API，剩下的路由、failover、成本优化、合规检查，全部由中间层搞定。

这听起来像二十年前 Akamai 做的事情——只是当时 CDN 分发的是网页和视频，现在分发的是智能。

推理负载从“实验”变成“生产”，中间件必须成人

OpenRouter 的公告里有一句很关键的话，我把它原样摘出来：

AI is rapidly shifting from experimentation into critical production apps and agents, and that transition requires infrastructure that works reliably at scale, across providers, across modalities, and across use cases.

这句话不是 PR 话术。它精确地描述了 2025-2026 年 AI 产业正在经历的一次底层迁移：推理负载的性质变了。

2023-2024 年，大部分 token 消耗来自“实验”——开发者在 Playground 里试 prompt，创业团队做 PoC，企业创新部门跑概念验证。那时候 token 量虽然增长快，但对可靠性的要求并不高。一次请求失败，重试就行了。

但到了 2026 年，情况完全不同。AI 应用开始进入企业的核心生产系统。客服机器人替代了人工坐席，代码助手嵌入了 CI/CD 流水线，AI agent 直接操作数据库和 ERP 系统。这些场景里，一次推理失败不只是“重试一下”的问题——可能是订单丢失、流水线停摆、合规红线被触发。

这就是为什么 OpenRouter 的 B 轮投资方全是企业级平台公司。ServiceNow 的客户在把 AI agent 部署到 ITSM 流程里，Snowflake 的客户在用 AI 分析数据仓库里的 PB 级数据，Databricks 的客户在湖仓上跑 AI 训练和推理流水线。这些场景的共性需求是：推理必须可靠、可审计、可治理。

OpenRouter 在过去一年里堆的功能，几乎全部指向这个“成人礼”。多模态推理支持——不只是文本，还包括图像、音频、语音合成、转录、embedding 和视频模型。企业控制面板——Workspaces、消费管理、guardrails、零数据留存策略。智能路由——不只是简单的负载均衡，而是基于质量、成本和延迟的感知路由，在模型提供商之间自动做 failover。

（这里有一个容易被忽略的细节：zero-data-retention 策略。对于受 SOC 2、HIPAA、GDPR 约束的企业客户来说，API 网关是否留存请求数据是一个一票否决项。OpenRouter 把这个做成了可配置策略，说明它已经摸到了合规敏感型客户的真实痛点。）

800 万开发者不是因为 OpenRouter 便宜才来的——他们是因为自己搞不定多模型运维才来的。

投资方名单里藏着的是“分销协议”，不是“估值倍数”

回到这轮融资的投资者构成。我列一个对照表：

  • CapitalG → Alphabet / Google Cloud
  • NVentures → NVIDIA
  • ServiceNow Ventures → ServiceNow
  • MongoDB Ventures → MongoDB
  • Snowflake Ventures → Snowflake
  • Databricks Ventures → Databricks

这个名单让我想起 2018-2020 年 Snowflake 和 Databricks 各自的融资历程。那时候两家公司都在疯狂引入战略投资方——不是因为缺钱，而是因为需要 Salesforce、AWS、Azure 的渠道网络来触达企业客户。Snowflake 在 IPO 前拿了 Salesforce Ventures 的钱，换来的是 Salesforce 的客户经理在全球帮它卖数据仓库。

OpenRouter 现在在做一模一样的事情。

CapitalG 的进入意味着 OpenRouter 会深度接入 Google Cloud 的 Marketplace 和销售渠道。一个 GCP 的企业客户，未来可能在控制台里直接开通 OpenRouter，费用走 GCP 账单，合规审核由 Google 背书。

NVentures 的进入则更微妙。NVIDIA 不只是一家 GPU 厂商，它在 2025-2026 年已经变成了 AI 推理基础设施的全栈玩家——从硬件（H200/B200）到推理软件栈（TensorRT-LLM）到云服务（DGX Cloud）。OpenRouter 作为一个聚合了 400+ 模型的推理网关，天然需要跟 GPU 供给和推理优化深度耦合。NVIDIA 投它，意味着未来 OpenRouter 的路由决策可以直接感知底层 GPU 的可用性和成本，做更精细的调度。

至于 ServiceNow、MongoDB、Snowflake、Databricks——这四家分别覆盖了企业 IT 的四个关键领域：工作流自动化、操作型数据库、分析型数据库、数据工程。它们共同的需求是：让客户在自己的平台上直接调用 AI 能力，而不需要跳出去对接十几个模型 API。

这轮融资的本质不是“OpenRouter 值多少钱”，而是“谁需要确保 OpenRouter 的路由层跟自己的平台绑定”。

我甚至怀疑，这轮融资的条款里可能包含了一些跟估值无关的商业协议——比如 GCP Marketplace 的分成比例、NVIDIA GPU 的优先供给权、各平台对 OpenRouter 的默认集成深度。这些细节不会写在公告里，但它们才是决定 OpenRouter 下一步能跑多快的真正变量。

多模型世界的“路由器”，会是下一个千亿美金赛道吗？

OpenRouter 的故事让人忍不住对标一个历史类比：CDN 行业的 Akamai，或者 API 网关领域的 Kong 和 Apigee。但仔细想想，这个类比其实低估了 OpenRouter。

CDN 解决的是“内容离用户更近”的问题，价值在于降低延迟和带宽成本。API 网关解决的是“微服务之间如何安全通信”的问题，价值在于安全治理和流量管理。

而 OpenRouter 解决的问题比这两者都复杂：它需要在 400+ 模型、数十家提供商之间，实时做出“这个请求应该发给谁”的决策。这个决策涉及多个变量——模型能力是否匹配任务？提供商的 API 当前是否健康？价格是否在预算范围内？延迟是否满足 SLA？输出内容是否合规？数据是否需要留存？

更关键的是，这些变量是动态变化的。一个模型可能在上午 10 点还很稳定，到中午就因为流量洪峰开始排队。一家提供商可能在周二调低了价格，到周五又因为 GPU 紧缺提价。OpenRouter 的智能路由层需要实时感知这些变化，并且在毫秒级做出调度决策。

这实际上是一个“AI 推理市场的做市商”角色。

做市商在金融市场上赚的是 bid-ask spread。OpenRouter 在 AI 推理市场上赚的是模型提供商和开发者之间的信息差和复杂度差。

而且这个市场还在急剧膨胀。OpenRouter 今年预计处理超过 1 quadrillion token——这个数字在 2025 年听起来像天文数字，但到了 2027 年可能只是月活量。因为 AI agent 的 token 消耗模式跟人类完全不一样：一个人类用户一天可能发几十条消息，消耗几万 token；但一个 AI agent 在后台持续运行，一天可能消耗几千万甚至几亿 token。

如果 agent 真的成为下一个计算范式，那么 agent 之间的通信、agent 调用工具时的推理、agent 自我反思时的多轮推理——所有这些 token 流量都需要经过某个路由层。OpenRouter 卡住的，就是这个位置。

当然，OpenRouter 不是没有风险。最大的风险来自模型提供商自身的战略选择。目前 OpenAI、Anthropic、Google 都允许第三方网关聚合自己的 API——因为这能帮它们触达更多开发者。但如果有一天，某家巨头决定关闭第三方聚合的通道，强迫开发者直连自己的平台，OpenRouter 的模型供给就会出现缺口。

（不过从目前来看，这个风险不大。因为开发者对多模型的需求太强了，任何一家敢封杀聚合网关的模型提供商，都会面临开发者大规模流失。这跟当年 Netflix 封杀 CDN 聚合商不一样——Netflix 有独家内容，AI 模型没有。）

另一个风险是开源模型的崛起。如果 Llama 4、Mistral、DeepSeek 这些开源模型的性能追平甚至超越闭源模型，企业可能会选择自建推理基础设施，绕过所有网关。但即使在这种场景下，OpenRouter 仍然有价值——因为自建推理也需要路由层来管理不同开源模型的版本、量化策略、硬件适配。只是价值捕获的链条会从“API 差价”转向“基础设施管理”。

一家没有公开创始人的公司，正在悄悄吃掉 AI 推理的中间层

最后说一个让我觉得特别有意思的细节：OpenRouter 的这轮融资公告，从头到尾没有提到任何一位创始人的名字。

这太反常了。在 2026 年的 AI 行业，融资公告几乎是创始人个人品牌的秀场。Sam Altman、Dario Amodei、Demis Hassabis——这些名字本身就是估值的一部分。但 OpenRouter 的公告里，主语永远是“we”，不是“I”也不是某位 CEO。

（我查了一下，OpenRouter 的创始人是 Alex Atallah 和 Alan Zhang。Alex Atallah 之前是 OpenSea 的联合创始人兼 CTO——OpenSea 在 NFT 时代最高估值 130 亿美元。他 2022 年离开 OpenSea，2023 年开始做 OpenRouter。这段背景在本次公告里完全没提。）

这种低调可能是刻意的。OpenRouter 做的是基础设施层的生意，它的客户是开发者，不是消费者。基础设施公司的核心信任资产不是创始人魅力，而是可靠性、中立性和持久性。一个过于鲜明的创始人 IP，反而可能让客户担心“如果这个人离开怎么办”。

OpenRouter 想卖给企业客户的不是“某个天才的故事”，而是“一个你不需要关心的中间层”。

这恰恰是基础设施生意最难做、也最有价值的地方。最好的基础设施是你每天都在用，但从不主动想起的东西——就像 DNS、CDN、TCP/IP。OpenRouter 的目标，显然是成为 AI 时代的 TCP/IP——一个开发者默认接入、企业默认信任、模型提供商默认兼容的通用路由层。

1.13 亿美元的 B 轮融资，买的不是增长，买的是成为默认选项的权利。

而那张投资者名单，就是默认选项的背书人。

接下来的看点很简单：这 1.13 亿美元会烧在什么地方？是继续砸开发者增长，把 800 万变成 2000 万？还是砸企业销售团队，去跟 ServiceNow、Snowflake 的客户经理一起敲开全球 500 强的门？还是砸技术团队，把智能路由从“好用”做到“不可替代”？

三种选择对应三种结局。开发者增长路线会让 OpenRouter 变成 Stripe——量大、利薄、依赖生态。企业销售路线会让它变成 Palantir——量不大但单价极高、定制化重。技术壁垒路线会让它变成 Cloudflare——靠技术代差吃下整个市场。

不管是哪种结局，有一件事已经确定了：在 AI 从单模型走向多模型的路上，有人提前在收费站的位置盖好了房子。而且盖房子用的砖，是 NVIDIA、Google、ServiceNow、MongoDB、Snowflake 和 Databricks 亲手递过来的。

这笔账，怎么看都划算。
