从零手写到多智能体：这份435节AI工程课程如何打造你的兵工厂？

ai-daily · 2026 年 5 月 26 日 21:41 · 来源：GitHub Trending python

凌晨三点，旧金山某间共享公寓的厨房灯还亮着。笔记本电脑风扇嗡嗡作响，屏幕上是一个 GitHub 仓库——rohitg00/ai-engineering-from-scratch。光标停在 README 最上方的两行数字上：84% 的学生已经在用 AI 工具，但只有 18% 觉得自己有能力在职业场景里驾驭它们。

这两个数字之间的巨大裂缝，就是这份课程表想要填平的东西。

这不是教程，这他妈是一套兵工厂。

从线性代数到自主集群：一份把骨架全拆给你看的课程表

我打开课程大纲的时候，第一反应是这人在开玩笑。435 节课，20 个阶段，大约 320 小时的纯学习时间，横跨 Python、TypeScript、Rust 和 Julia 四种语言。从线性代数的直觉开始，一路杀到多智能体集群和自主系统。中间没有任何跳过按钮。

但这还不是最狠的。

最狠的是每一节课的交付物。大多数 AI 课程结束时会跟你说“恭喜你学会了 X”。这个仓库的每节课结束时，你手里多了一个可重用的工具——一条 prompt、一个 skill、一个 agent，或者一个 MCP server。不是作业，是你可以直接粘贴进日常工作流的武器。435 节课就是 435 个这样的构件。

课程分层的逻辑非常清晰。Phase 1 是数学基础——线性代数、微积分、概率论、信息论、傅里叶变换，22 节课，用 Python 和 Julia 两种语言做到。Phase 2 是经典机器学习，18 节课，线性回归、决策树、SVM、集成方法，全是从零手写。Phase 3 进入深度学习核心，13 节课，感知机、多层网络、反向传播、激活函数、优化器、正则化——最后让你自己搭一个迷你框架，然后才允许你碰 PyTorch 和 JAX。

这种“先用裸写理解原理，再给你框架”的做法，在 Phase 14 的智能体工程里达到了极致。第 1 课是智能体循环：大约 120 行纯 Python，零依赖，从头做到一个 ReAct 风格的 loop。然后同一课的 outputs/ 文件夹里产出两样东西：一个叫 skill-agent-loop.md 的 skill 文件，可以直接丢给 Claude 或 Cursor 使用；一个叫 prompt-debug-agent.md 的调试 prompt，用来诊断智能体跑偏的步骤。

我看到这里的时候愣了一下——这是在逼你把肌肉记忆刻进骨头里。

先把自己弄疼，再给你止痛药，这样你永远不会忘记疼的感觉。

84% 在用，18% 觉得自己行——这道裂缝究竟是怎么形成的

回来看开头那组数字。84% 的学生已经在日常中使用 AI 工具，这个比例放在 2025 年的语境里并不令人惊讶。ChatGPT 的周活用户早就过了 2 亿，GitHub Copilot 的付费用户超过 180 万，Cursor 的估值在 2025 年初冲到了 26 亿美元。AI 工具的渗透率在学生群体里只会更高——作业、论文、代码、简历，能用的地方全用上了。

但 18% 觉得自己能在职业场景里用好它——这个数字才是真正值得琢磨的。它意味着 66% 的学生处于一个尴尬的中间地带：会用，但不知道为什么能用；能跑出结果，但解释不了结果是怎么来的。

这正是当前 AI 教育最大的断层。网上铺天盖地的教程教你“5 分钟搭一个 chatbot”、“一行代码部署你的第一个 agent”。你 hook 了一个 function 到 agent 上，但你讲不清楚 attention 在模型内部到底做了什么。你 fine-tune 了一个 LLaMA，但你看不懂 loss curve 为什么在那个点突然抖了一下。

这份课程表的设计者显然被这种断层激怒了。他在 README 里写了一句话，语气几乎带着火气：“大多数 AI 教材教的都是碎片。这里一篇论文，那里一篇 fine-tune 帖，别的地方又一个花哨的 agent demo。这些碎片几乎从来对不齐。”

于是他把整个知识体系做成了脊椎。20 个阶段从数学开始，层层堆叠。数学是地基，agent 和生产是屋顶。你可以跳过底层直接上屋顶，但他在文档里冷冷地补了一句：“跳了就别问为什么屋顶塌了。”

课程里内置的智能体技能系统也很有意思。你可以在 Claude、Cursor、Codex、OpenClaw、Hermes 里直接调用 /find-your-level 命令，回答 10 道题，系统会把你映射到合适的起始阶段，并生成一条个性化的学习路径，附带每阶段的预估小时数。每完成一个阶段，跑一次 /check-understanding <phase>，8 道题加反馈，告诉你哪些课需要回头补。

这是个聪明的设计。320 小时的课程量摆在那里，没人能一口吞下。但如果你已经会写反向传播，不需要再从 Phase 3 的第一课开始——跳过，节省时间，把精力花在你真正缺的东西上。

Phase 14 的智能体工程部分尤其值得单拎出来说。42 节课，从最基础的 agent loop 开始，一直延伸到 Anthropic 的 workflow patterns、LangGraph 的状态图、AutoGen 的 actor model、CrewAI 的角色编排，然后是计算机使用能力（Claude、OpenAI CUA、Gemini）、语音 agent（Pipecat、LiveKit）、可观测性（Langfuse、Phoenix、Opik），以及整整 12 课专门讲 agent workbench——为什么能力够强的模型在实际任务中仍然会失败，以及如何用 scope contracts、verification gates、reviewer agent 这些东西把失败率压下去。

最后 12 课每课都附带一个 mission.md 文件，在 agent 打开完整课程文档之前，先给它一份任务简报。这个细节透露出设计者脑子里想的不只是“教会你”，而是“让你交付出来的东西能用在真实项目里”。

Phase 17 的基础设施与生产部分则直接面向已经在干活的人——vLLM 的服务内部机制、SGLang 的 RadixAttention、TensorRT-LLM 在 Blackwell 上的 FP8 和 NVFP4、EAGLE-3 的推测性解码、多区域 LLM 服务和 KV cache 局部性、冷启动缓解、模型路由作为降成本原语、AI 网关（LiteLLM、Portkey、Kong、Bifrost）、FinOps 的单位经济学……28 节课，几乎把 2025 年 LLM 生产部署的整个技术栈从头到尾犁了一遍。

Phase 18 的安全与对齐部分同样不手软。30 节课，从指令遵循作为对齐信号开始，一路穿过奖励黑客、谄媚放大、宪法 AI、mesa-optimization、睡眠特工、上下文欺骗、对齐伪装、AI 控制、可扩展监督、红队自动化、多轮越狱、ASCII 艺术攻击、间接 prompt 注入、差分隐私、SynthID 水印、欧盟 AI 法案合规……这不是“AI 伦理入门”那种水课，这是给真的要在生产环境里承担安全责任的人准备的。

我数了一下，整个课程表里出现的具体产品名和框架名超过 120 个。DeepSeek-V3 的架构走读、Jamba 的混合 SSM-Transformer、SAM 3 的开放词汇分割、Diffusion Transformers 和 Rectified Flow、ColPali 的视觉原生文档 RAG、Moshi 和 Hibiki 的流式语音到语音——2024 年末到 2025 年初的前沿进展几乎全被收进去了。

这让我想起另一组数字。2025 年 4 月，Stack Overflow 的开发者调查显示，76% 的受访者已经在使用 AI 工具开发，但同时有 45% 的人表示对 AI 工具的输出质量存在信任问题。信任问题的根源在哪？不是工具不够好，而是用工具的人看不懂工具的内部逻辑。

当你的技能栈在 API 调用层断了，信任就会在断口处漏掉。

这份课程表在 GitHub 上的协议是 MIT——完全开源，免费，可以在你自己的笔记本上跑。没有 5 分钟视频，没有复制粘贴部署，没有保姆式引导。设计者在 README 里甚至没有放一个“捐赠”按钮。

这种姿态在 2025 年的 AI 教育市场里几乎算得上反常。市面上主流的 AI 课程定价从 49 美元的 Udemy 到 4000 美元的 bootcamp 不等，企业级培训报价轻松过万。而这个人把 435 节课、320 小时的内容全部开源，附带四语言做到，附带 435 个可重用交付物，附带智能体技能系统，附带个性化路径生成器——然后标了个 MIT。

我不知道 rohitg00 是谁。GitHub 主页上信息不多。但我猜他大概也受够了那些“5 分钟学会 AI”的东西，受够了那些碎片知识拼不成一张完整地图的挫败感，受够了明明每天都在用 AI 却解释不了它为什么在某个 prompt 下突然胡言乱语的无力感。

于是他决定从零开始，把整条路重新铺一遍。从线性代数铺到多智能体集群。从裸写反向传播铺到生产环境的 KV cache 优化。从手写 tokenizer 铺到 MCP server 的安全认证。

而且他把路开放给所有人走。
