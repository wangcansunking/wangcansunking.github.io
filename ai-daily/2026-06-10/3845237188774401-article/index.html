微信AI Agent新玩家：得物首批接入，如何应对订单量翻倍挑战？

如果你打开得物 App，选一双倒钩 AJ1，点「鉴别」，系统会告诉你得寄到上海仓库让 8 个鉴定师轮流过眼。你再打开微信，跟一个对话框说「帮我找双 42 码的熊猫 Dunk，要过鉴定的」，20 秒后它把商品链接、鉴别报告、下单按钮全塞进聊天窗口里。同一个需求，两条路径，体验差了一个代际。

这就是 2026 年 5 月 7 日发生的事情——不是某个创业公司的概念 demo，不是 Demo Day 的 PPT，而是 36 氪快讯里一条不到 200 字的新闻：得物 App 正式接入微信 AI Agent 生态，首批上线场景导购、商品鉴别、购买下单、品质履约四件事。

电商的入口，正在从 App 图标退化成聊天框里的一行字。

得物为什么必须抢这张船票

得物这家公司有个奇特的生存状态：它的复购率在潮流电商里排第一，但它的 App 打开频次在综合电商里排不进前二十。用户只在「想买东西」或「想鉴定」的时候打开它，其他时间它躺在手机第三屏吃灰。这在移动互联网时代不是问题，但在 AI Agent 时代会变成致命伤。

因为 Agent 的逻辑是反 App 的。用户不再主动打开某个应用，而是对着一个统一入口说需求，Agent 在后台调用一堆服务商的 API，拼出一个完整流程。谁在 Agent 的「服务菜单」里，谁就能被调用；谁不在菜单里，就等于不存在。得物显然意识到了这一点——它必须赶在所有潮流电商之前，把自己的鉴别能力、导购能力、履约能力全部拆成 API，喂进微信 Agent 的嘴里。

这个判断不是空穴来风。腾讯在 2026 年 Q1 的财报电话会上明确提过，微信 Agent 生态的「服务接入伙伴」已经超过 300 家，涵盖出行、餐饮、电商、本地生活四大类。据 36 氪此前的报道，美团、滴滴、京东健康都在首批名单里，得物是潮流电商赛道第一个吃螃蟹的。

第一波接入的玩家分两种：高频刚需，或者高壁垒垂类。得物属于后者，但它的壁垒正在被时间融化。

鉴别能力是得物的护城河——但护城河里要有水

得物的鉴别体系是它跟淘宝、拼多多、抖音电商拉开差距的核心武器。据公开资料，得物在全国有 7 个鉴别实验室，签约鉴定师超过 300 人，部分热门鞋款的鉴别报告能做到「8 人交叉过检」。这个能力在传统电商里是增值服务，在 Agent 生态里会变成基础设施。

什么意思？当用户用微信 Agent 说「我要一双正品倒钩 AJ1」，Agent 不会只给你一个淘宝链接——它会优先调用得物的 API，因为只有得物能在返回商品的同时带上「8 人鉴别用」的标签。这个标签在聊天界面里可能就是一行绿色小字，但它背后的信用权重，是 300 个鉴定师五年时间堆出来的。

但问题也在这里。得物的鉴别能力依赖人工，300 个鉴定师一年能处理的订单量是有上限的。据《晚点 LatePost》2025 年的一篇报道，得物在 2025 年 Q4 的日均鉴别订单量已经突破 80 万单，高峰期鉴定师排班排到凌晨两点。接入微信 Agent 之后，订单量如果翻倍，人力能不能跟上？得物官方目前没有公布过自动化鉴别的技术进展，但行业里已经有竞品在跑了——字节跳动的「识货」App 在 2026 年 3 月上线了 AI 鉴别功能，据称准确率做到 92%，虽然跟人工鉴别的 99.5% 还有差距，但成本只有人工的 1/20。

得物的护城河是人堆出来的，但 AI 正在把这条河填平。

（值得一提的是，得物的鉴别体系还有个隐藏优势：它不光鉴别真伪，还鉴别「品相」——鞋盒有没有破损、配件齐不齐全、氧化程度如何。这个维度 AI 目前还做不到，因为它需要主观判断。但话说回来，大部分用户其实不 care 鞋盒有没有一个角皱了，他们只 care 是不是正品。）

微信 Agent 的野心：吃掉所有低频 App 的入口

微信做 Agent 的逻辑，跟当年做小程序一脉相承。2017 年小程序上线的时候，张小龙说「用完即走」，意思是小程序不需要下载、不需要注册、用完就关掉。八年过去了，小程序的日活超过 6 亿，但「用完即走」这件事并没有真正做到——用户还是得在微信里翻半天才能找到那个三个月前用过一次的小程序。

Agent 解决的就是这个「翻半天」的问题。用户不用找了，直接说需求，Agent 自己去找对应的小程序或 API。这意味着微信从一个「应用容器」变成了一个「意图分发引擎」。得物接入 Agent 生态之后，它的服务不再依赖用户主动打开 App 或小程序，而是被 Agent 在合适的时机「唤醒」。

这个模式对得物这种低频刚需 App 来说是降维打击。一个用户可能一年只买四次球鞋，但他每次买的时候都需要鉴别。在传统模式下，得物得花广告费把他从抖音、小红书、B 站「捞」回来；在 Agent 模式下，用户对着微信说一句话，Agent 就把得物的服务推到他面前，获客成本趋近于零。

得物接入微信 Agent，本质上是用服务调用替代广告投放。

据 36 氪快讯原文，接入后用户可以用微信 Agent「直接调用得物的场景导购、商品鉴别、购买下单、品质履约等服务」。这四件事拆开看，每一件都对应一个独立的 API 调用：

  • 场景导购：Agent 理解用户需求（「我想买一双适合夏天穿的球鞋，预算 1500 以内」），调用得物的商品库和导购算法，返回推荐结果。
  • 商品鉴别：Agent 在返回商品信息的同时，调取该商品的鉴别记录（如果有的话），或者触发鉴别流程（如果用户要求）。
  • 购买下单：Agent 直接调用得物的订单系统，在聊天界面内完成支付，不需要跳转。
  • 品质履约：Agent 调用物流和售后系统，实时反馈订单状态。

这四个环节全在微信里闭环，用户从头到尾不用打开得物 App。这意味着得物的 App 本身会退化成什么？退化成一个小程序和 Agent 背后那个「跑 API 的服务器」。用户感知不到它，但它无处不在。

但得物不是唯一的选择——Agent 生态的「服务竞价」逻辑

这里有一个得物可能不愿意面对的问题：Agent 会不会只把得物当成一个「鉴别工具」，而不是一个「购物入口」？

当用户说「帮我找双 AJ1」，Agent 可以同时调用得物、淘宝、京东、拼多多的商品库，然后按价格、鉴别标签、物流速度排序。如果淘宝上的同款比得物便宜 200 块，但没有鉴别标签，Agent 会不会同时返回两个结果，让用户自己选？大概率会。那用户会选哪个？价格敏感型用户选淘宝，正品焦虑型用户选得物。

问题在于，得物的价格通常不是最低的。得物的模式是「卖家寄货→平台鉴别→买家收货」，中间多了一道鉴别环节，成本必然体现在价格上。如果 Agent 把价格和鉴别标签并列展示，得物的转化率会不会被低价竞品稀释？得物官方没有公开过接入 Agent 后的具体数据，但据 36 氪报道，得物内部对 Agent 渠道的定位是「增量市场」，而不是「存量迁移」——他们赌的是那些原本不在得物消费的人群，会用 Agent 的鉴别标签第一次接触得物。

得物在赌一个假设：当用户同时看到「便宜 200 块但没鉴别」和「贵 200 块但 8 人过检」两个选项时，会有一部分人选择后者。

这个假设能不能成立，取决于两个变量：一是用户对正品的焦虑程度，二是 Agent 的推荐算法会不会给「鉴别标签」更高的权重。第二个变量尤其关键——如果微信 Agent 的排序逻辑是「价格优先」，得物会被压在最下面；如果 Agent 的排序逻辑是「信任优先」，得物就是第一名。据腾讯官方此前的表态，微信 Agent 的推荐算法会综合考虑「服务质量、用户评价、履约能力」三个维度，但没有明确说会不会考虑「鉴别能力」这个维度。

（括号副注：腾讯自己不做电商，但微信支付和微信小店的 GMV 在 2025 年突破了 2 万亿。Agent 生态的推荐算法如果偏向自有体系内的服务商，得物这种外部接入的玩家会不会被边缘化？这是所有接入微信 Agent 的公司都在担心的问题，但没人公开说出来。）

潮流电商的终局：不是 App 的战争，是 API 的战争

得物这次接入微信 Agent，让我想起 2024 年 Karpathy 说过的一句话——他说未来的软件公司都会变成「API 公司」，用户不直接跟你的 UI 交互，而是用 Agent 调用你的能力。当时很多人觉得这是五年后的事，但 2026 年 5 月，得物已经把这套逻辑跑通了。

这件事的延伸影响可能比表面看起来大得多。得物是潮流电商里第一个接入微信 Agent 的，但不是唯一一个想做这件事的。据 Tech 星球 2026 年 4 月的报道，毒 App（得物的老对手）也在跟微信 Agent 团队接触，预计 Q3 会有进展。如果毒也接入，两个平台的鉴别能力会在 Agent 里直接 PK——谁的鉴别报告更详细、谁的鉴别速度更快、谁的鉴别成本更低，这些原本藏在 App 后台的指标，会全部暴露在用户面前。

到那时候，潮流电商的竞争就完全变味了。过去是比谁的 App 体验好、谁的市场投放狠、谁的社区氛围浓。以后是比谁的 API 响应速度快、谁的鉴别数据更结构化、谁的履约系统更稳定。得物做了五年多的「鉴别壁垒」，会被 Agent 生态拉平到一个统一的接口标准上，用户根本感知不到背后的服务商是谁——他们只会看到「鉴别用」四个字。

当用户不再关心服务商是谁，品牌本身就会消失。得物在 Agent 生态里最大的风险不是被竞品打败，而是变成一个没有名字的「鉴别 API」。

得物显然也意识到了这一点。据 36 氪快讯，得物在接入 Agent 的同时，保留了所有服务环节的「品牌露出」——Agent 返回的商品卡片上会明确标注「得物鉴别」「得物发货」，而不是一个匿名的「第三方鉴别」。这是在跟 Agent 生态谈条件：我可以把能力拆成 API 给你用，但我的名字必须留在每一个调用结果里。

这个条件能不能一直守住，取决于得物在潮流电商赛道的话语权。如果有一天微信 Agent 的推荐算法觉得「鉴别」这个环节不需要品牌露出，只需要一个统一的「正品保证」标签，得物跟毒、识货、nice 的鉴别能力会被混在一起，用户分不清谁是谁。到那一天，「得物」这个名字在 Agent 生态里的价值，就只剩下老用户的认知惯性了。

戏谑收尾：一场没有硝烟的入口迁移

得物接入微信 Agent 这件事，本质上是一个信号——超级 App 正在用 Agent 吃掉所有垂类 App 的入口，而垂类 App 只能选择「打不过就加入」。得物是第一批上船的，但不是最后一个。接下来半年，你会看到更多垂类玩家排着队把自己的核心能力拆成 API，喂进微信、支付宝、抖音的 Agent 生态里。

但问题在于，上了船之后，船往哪开就不是你能决定的了。微信 Agent 的推荐算法调一次权重，得物的订单量可能就波动 30%。腾讯如果有一天决定自己做潮流电商的「统一鉴别标准」，得物和毒都得变成这个标准下面的「认证服务商」——从平台降级为供应商。

这让我想起 2018 年支付宝小程序刚上线时，一堆第三方支付公司抢着接入，觉得抱上了大腿。五年后回看，那些公司大部分都消失了，因为支付宝自己把它们的核心能力全做了。得物今天接入微信 Agent，跟当年那些支付公司接入支付宝小程序，剧本会不会重演？我不知道，但得物的 300 个鉴定师应该比我先知道答案——毕竟，他们每天经手的 80 万双鞋里，已经开始混进 Agent 渠道的订单了。
