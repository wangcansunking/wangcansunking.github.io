谷歌砍掉学生 API，AI 写作者如何高效去除“AI 味”？

一个 200M 参数的时间序列模型，预测窗口从 2048 拉到 16000。Google 悄悄放出的 TimesFM 2.5，连一篇正经论文都没发，只在 GitHub 上更新了 README。

与此同时，linux.do 论坛上一个帖子正在发酵——「请问 codex 写的文章如何高效地去 AI 味？」帖主试过 op7418 做的 Humanizer-zh skill，试过论坛佬友自制的去味工具，试过 d4 pro、Claude，结论是 Gemini 3.1 Pro 效果最好。但 GitHub 学生权益砍了，API 额度没了，路又堵死了。

去 AI 味这件事，已经从「锦上添花」变成了「求生刚需」——而工具链的脆弱性，正在把写作者逼进死角。

句式惯性：不是 prompt 能解决的问题

帖主描述了一个非常具体的痛点——无论怎么约束 prompt，模型输出始终逃不开「先 XX 后 XX」「不要只看 XX 要看 XX」这类句式模板。这个问题的根源不在 prompt engineering 层面，而在训练数据的统计分布上。

大语言模型本质上是一个条件概率机器。它预测下一个 token 时，依赖的是训练语料中高频出现的序列模式。中文互联网上「先…后…」「不要…要看…」这类转折结构，在科技类、评论类、教程类文本中的出现频率极高——因为它们确实好用，逻辑清晰，上手门槛低。问题在于，当模型被要求生成 3000 字以内的短篇电子类文章时，这类句式就成了「最安全的选择」：不会出错，不会跑偏，但读起来像流水线产品。

op7418 的 Humanizer-zh 尝试用规则 + 替换策略来解决这个问题。它的思路是识别 AI 生成文本中的高频句式，然后用更自然的表达替换。但这种方案有一个天然天花板——规则能覆盖的句式是有限的，而模型的输出变化是无限的。你修掉「先 XX 后 XX」，它下次可能换成「一方面 XX 另一方面 XX」；你再修掉这个，它又学会了「从 A 到 B 的转变」。这是一场永远打不完的 whack-a-mole。

帖主提到 Gemini 3.1 Pro 效果最好，这跟 Google 在 Gemini 系列上对长文本生成质量的优化方向有关。Gemini 的训练数据中包含了大量非结构化、口语化、带有个人风格的中文语料（YouTube 字幕、播客转录、论坛帖子），这让它在模拟「人味」上天然比 Claude 那种偏学术、偏正式的写作风格更灵活。但问题在于——Google 把 GitHub 学生包的 API 额度砍了，这条路现在走不通了。

真正的问题：你要去的是「味」还是「痕迹」？

帖主的诉求是「去 AI 味」，但仔细看帖子里的描述——「看到就已经恼火了」「写了提示词约束还是这样」——会发现真正的痛点其实有两个层面。

第一层是句式重复。这是最容易感知的「AI 味」来源，也是 Humanizer-zh 这类工具试图解决的问题。但句式重复本质上是一个统计问题，不是风格问题。你让真人写 100 篇 3000 字的电子类短评，他也会不自觉地重复某些句式——只是重复的模式更随机、更不可预测。AI 的重复之所以刺眼，是因为它的分布太均匀了，均匀到像节拍器。

第二层是判断缺失。这是更深的层面，也是当前所有「去味工具」都没碰到的核心问题。帖主说不要「不要只看 XX 要看 XX」，这种句式之所以讨厌，不是因为它语法有问题，而是因为它假装在提供洞察，实际上什么都没说。真正的「人味」来自于写作者基于自身经验做出的判断——哪怕是错的、偏激的、不全面的，它至少是一个立场。

Claude 的写作风格偏学术，GPT 偏教程，Gemini 偏口语，但它们在「提供判断」这件事上表现都不好。因为模型被 RLHF 训练成了「安全优先」的机器——它倾向于给出平衡的、不惹争议的、两头都不得罪的表达。而「不要只看 XX 要看 XX」恰恰是这种安全策略的产物：它既没有否定你，也没有肯定你，它只是把话题转移到了一个更安全的讨论维度。

去 AI 味的终极解法，不是换模型也不是写更长的 prompt——是往输出里注入模型天然缺失的东西：偏见、立场、以及敢于说「我认为」的勇气。

这就解释了为什么帖主试了一圈工具都不满意。Humanizer-zh 改的是皮，改不了骨。Claude 和 GPT 的底层安全对齐决定了它们写不出真正带刺的东西。Gemini 3.1 Pro 效果好，可能是因为它在安全对齐和表达自由度之间找到了一个更宽松的平衡点——但这条路现在被砍了。

说到底，linux.do 这个帖子暴露的不仅是一个工具问题，而是一个正在蔓延的焦虑：当 AI 写作工具越来越普及，写出来的东西越来越像 AI，那些真正需要「人味」的场景——个人博客、独立评论、有态度的产品评测——正在被同质化的输出淹没。而现有的去味方案，要么靠规则打补丁（Humanizer-zh），要么靠更贵的模型（Claude），要么靠随时可能被砍的福利（GitHub 学生包 + Gemini）。

帖主最后问「求大佬支招」，但这个问题可能没有银弹。或许最务实的解法反而是最笨的那个——写完初稿，自己大声读一遍，把所有让你「恼火」的句式手动改掉。AI 能帮你生成 80% 的内容，但最后那 20% 的「人味」，目前还是得靠人自己往上撒盐。
