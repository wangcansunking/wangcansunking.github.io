Anthropic与OpenAI定价转向：企业客户成本为何飙升2倍？

ai-daily · 2026 年 5 月 28 日 02:50 · 来源：Hacker News Frontpage

2026 年 5 月 27 日，Simon Willison 在他的博客上敲下了一行标题：“I think Anthropic and OpenAI have found product-market fit”。这位以冷静著称的开发者工具作者，这次语气里带着一种“终于等到这一天”的确定感。他摊开自己的账单：过去 30 天，如果按 API token 计价，他消耗了 2,180.16 美元——但他只付了 200 美元月费。一边是 Anthropic 的 100 美元 Max 计划，一边是 OpenAI 的 100 美元 Pro 计划，两个订阅加起来，把他这种重度编程用户的成本压到了不到实际消耗的十分之一。

这个数字让他以为自己发现了某种行业潜规则：重度用户靠订阅薅羊毛，企业客户应该也能拿到类似的折扣。然后他发现自己错了——大错特错。

企业客户正在被反向薅羊毛，而他们还没完全反应过来。

四月定价闪电战：从订阅补贴到 API 计价

故事要从 Anthropic 的一次悄然转向说起。Simon 追查了半天也没找到确切日期，但他锁定了一个时间窗口：2025 年 8 月，Anthropic 还在宣传企业计划“Claude seats 包含足够典型工作日的用量”。到了 2026 年 4 月 14 日，The Information 报道了定价变更的消息，但 Anthropic 发言人声称这个改动实际上在 2025 年 11 月就已经生效。也就是说，过去半年里，续约的企业客户才陆续发现自己签的新合同变成了 20 美元/seat/月的基础费，外加 API token 按量计价。

OpenAI 的节奏更紧凑。2026 年 4 月 2 日，Codex 定价从按消息计费切换为按 API token 使用量计费，覆盖新的和现有的 Plus、Pro、ChatGPT Business 以及新的 Enterprise 计划。21 天后，也就是 4 月 23 日，这个改动扩展到所有现有 Enterprise 计划——包括 Edu、Health、Gov 和 ChatGPT for Teachers。Simon 仔细对比了“credits”和 API token 的换算关系，结论是：credits 成本就是 API token 成本的精确映射。

这意味着什么？到 2026 年 4 月，OpenAI Codex 和 Anthropic Claude Code/Cowork 的企业级成本，等于它们的公开 API 价格。过去那种“签一年合同拿巨额折扣”的时代结束了。

更狠的还在后面。GPT-5.5（4 月 23 日发布）的 API 价格是 GPT-5.4 的 2 倍。Opus 4.7（4 月 16 日发布）在计入新 tokenizer 的影响后，价格比 Opus 4.6 高出约 1.4 倍。两家公司在同一个月释放了新一代前沿模型，同时把企业客户锁定在了更高的 API 定价上。

这个时间点的巧合让人很难不产生联想。Anthropic 和 OpenAI 都在筹备 IPO，但这不足以解释如此激进的定价动作。Simon 的判断更直接：他们终于找到了产品-市场匹配，而那个匹配的载体是编程代理——Claude Code、Cowork 和 Codex。

编程代理：烧 token 的印钞机

ChatGPT 在 2023 年 2 月成为史上增长最快的消费级应用时，没人怀疑它找到了 PMF。但那个 PMF 有一个致命缺陷：赚不到钱。2026 年 2 月，OpenAI 吹嘘 ChatGPT 有超过 9 亿周活用户，但只有 5,000 万是付费订阅者——占比 5.6%。每个月收 10-20 美元的生意不算差，但要覆盖万亿级别的基础设施投入，你需要 10-20 亿订阅用户持续付费四年。这个算术题无解。

编程代理改写了这个方程式。它们烧 token 的量级完全不同。Simon 自己的数据就是一个微型样本：每月 1,000 美元的 API 成本/供应商，而他的使用强度还远没到“日夜不停跑 agent”的程度。一个年薪 30 万美元的软件工程师，如果每天用 Claude Code 节省 2 小时，企业为他支付每月 2,000 美元的 token 费用是完全合理的——这笔账算得过来。

而且编程代理的边界远不止写代码。“一个编程代理是可以自动化任何你能用打字输入计算机来完成的事情的工具”，Simon 写道。这意味着它的适用人群从软件工程师扩展到所有高薪知识工作者。2025 年 11 月发布的模型让代理变得“真正有用”，六个月后，企业开始为此付出真金白银。

招聘数据从另一个角度佐证了这个判断。Simon 用 Claude Code 爬了 OpenAI 和 Anthropic 的招聘网站，又用 Datasette 做了分析——他自己称之为“吃狗粮”。结果显示：OpenAI 的 703 个开放职位中，229 个（32.6%）与销售和支持相关——客户经理、“Go To Market”、“前向部署工程师”这类角色。Anthropic 的 390 个开放职位里，105 个（26.9%）看起来是企业向的。两家 AI 实验室的商业模式居然建立在对人力的重度依赖上——企业销售合同不会自己签自己，这个讽刺意味拉满了。

与此同时，企业端开始传出“AI 账单失控”的惊呼。Uber CTO Praveen Neppalli Naga 表示公司“2026 年才过了几个月就用完了全年的 AI 预算”，主要归因于 Claude Code。Uber COO Andrew Macdonald 在 Rapid Response 播客上说了句“上季度 25% 的代码提交来自 Claude Code，但很难说这就等于多产出了 25% 的消费者功能”。这番话被媒体加工成“Uber COO 说越来越难 justify AI 支出”。

微软那边也在取消 Claude Code 许可证，官方说法是鼓励工程师用自家的 Copilot CLI，但 The Verge 记者 Tom Warren 的消息源说“这也是财务决定”，触发点是微软 6 月 30 日的财年截止日。Simon 对这些“AI 失败故事”的解读完全不同：定价最好的信号就是客户倒吸一口凉气，然后还是掏了钱。Uber 预算超支、微软砍许可证——这恰恰说明产品强到让企业欲罢不能。

客户骂骂咧咧地续费，才是真正的 PMF。

Anthropic 的算力胃口从另一个维度揭示了推理成本的规模。SpaceX 的 S-1 文件披露了一则奇异的数据：2026 年 5 月，Anthropic 与 SpaceX 签订了云服务协议，租用 COLOSSUS 和 COLOSSUS II 的算力，每月支付 12.5 亿美元，合同持续到 2029 年 5 月。Anthropic 自己的公告说，这笔交易让他们“提高 Claude Code 和 Claude API 的使用上限”，强烈暗示 COLOSSUS 被用于推理而非训练。Anthropic 已经从其他供应商那里拿了海量算力，现在又愿意为单一供应商的额外容量每月烧 12.5 亿——推理预算膨胀到什么程度，这个数字给了你一个参照系。

Anthropic 的收入结构也在这条轨道上剧烈重构。2025 年 8 月，VentureBeat 引述知情人士称，仅 Cursor 和 GitHub Copilot 就贡献了 Anthropic 当时 40 亿美元收入中的 12 亿。如今 Anthropic 被传将在 2026 年 Q2 达到 109 亿美元收入，甚至可能首次做到盈利。从依赖少数大 API 客户到直接面向企业卖 Claude Code——这是在切掉中间商的蛋糕。Cursor 开始投资自有模型也就不奇怪了。

2025 年 11 月是“代理变得好用”的转折点，2026 年 4 月是“代理开始赚钱”的转折点。Simon 说，等 Anthropic 和 OpenAI 的 IPO S-1 文件公开时，我们就能看到经过审计的真实数字。那一天的到来，可能比任何人预想的都快。

毕竟，谁能拒绝一个让客户一边骂贵一边刷卡的产品呢？
