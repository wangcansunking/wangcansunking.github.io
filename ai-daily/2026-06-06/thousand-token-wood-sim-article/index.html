黑客松项目用 Llama 3.2 3B 模型，跑出10个NPC的微型市场经济体

你有没有见过一个游戏，NPC 会自己砍树、自己盖房子、自己搞贸易，而且这一切不是靠后台脚本硬编码，而是靠一个 30 亿参数的小模型实时推理出来的？

2025 年 7 月，Hugging Face 办了一场叫“Build Small”的黑客松，主题很明确：用小模型做大事。参赛者被要求用参数量不超过 30 亿的模型，跑出能让人眼前一亮的东西。最终冲上热门榜的，是一个叫“Thousand Token Wood”的项目。

它没有炫酷的 3D 渲染，没有复杂的战斗系统。它只是一个网页沙盒，里面住着 10 个 NPC，每个 NPC 的大脑，是一台本地跑着的 Llama 3.2 3B。

这场黑客松最大的讽刺在于：当巨头们在用万亿参数模型争“世界模型”的王座时，一个 3B 模型已经在沙盒里跑出了一个完整的小型经济体。

10 个 NPC 的微型文明，没有一行 if/else

Thousand Token Wood 的设计哲学，用一句话概括就是：让 NPC 自己决定该干什么。

在传统的游戏 AI 里，NPC 行为无非是状态机+行为树。农民看到田空了就去播种，商人看到库存低了就去进货，这些都是程序员事先写死的逻辑链条。但 Thousand Token Wood 把这一层全剥掉了。每个 NPC 每 5 秒接收一次环境信息——自己背包里有什么、周围有什么资源、附近有谁、当前的市场价格是多少——然后把这一切压缩成一段文本 prompt，喂给 Llama 3.2 3B，模型输出一个动作。

我最初看到这个架构的时候，第一反应是：这不就是 AI agent 圈里天天喊的“感知-决策-执行”循环吗？只不过这一次，循环里跑了 10 个 agent，而且它们共享同一个物理空间和经济系统。

项目团队在 Hugging Face 上公开了完整的运行 trace，40 条记录，每一条都记录了某个 NPC 在某个时刻看到了什么、想了什么、做了什么。翻这些 trace 的感觉，像在读一本微型人类学田野笔记。有一个叫“Willow”的 NPC，角色设定是伐木工，某天早上她走到市场，发现木材价格跌了，于是决定“今天先不砍树了，去帮铁匠搬矿石，赚点差价”。这个决策完全不是程序员预设的，是模型根据价格信号自己推出来的。

3B 参数，塞进了一个亚当·斯密。

价格信号、套利行为和自发分工

Thousand Token Wood 最让我愣神的地方，不是 NPC 会动，而是它们之间出现了经济学家才关心的东西：价格发现和自发分工。

项目里内置了一个简单的市场系统。每个 NPC 可以把自己采集或生产的物品挂到市场上卖，也可以从市场上买别人挂的东西。价格不是固定的——当某种物品供过于求，价格下跌；供不应求，价格上涨。这个机制本身不稀奇，稀奇的是 3B 模型居然能理解并利用它。

在公开的 trace 里，我看到一个叫“Finn”的 NPC，角色是商人。他的行为模式跟其他 NPC 完全不同——他不生产任何东西，只做一件事：低价买入，高价卖出。某条 trace 显示，Finn 发现铁矿石的市场价是 8 金币/单位，但铁匠“Bram”因为急需铁矿石，愿意出 12 金币收购。Finn 立刻从市场上扫了一批铁矿石，转手卖给 Bram，净赚 4 金币差价。

（值得一提的是，这种套利行为在真实世界里是高频交易公司的核心业务，而这里只是一个 3B 模型在沙盒里自己琢磨出来的。）

更复杂的模式出现在分工层面。最初 10 个 NPC 的角色是预设的——3 个伐木工、2 个矿工、1 个铁匠、1 个农夫、1 个商人、1 个面包师、1 个建筑工。但跑了几百轮之后，角色边界开始模糊。伐木工 Willow 偶尔去挖矿，矿工偶尔去帮铁匠打下手，商人 Finn 甚至一度因为木材利润高，自己跑去砍了两天树。

这不是混乱，这是劳动力市场的流动性。

经济学里有个概念叫“比较优势”——即使一个人样样都比别人强，他依然应该专注做自己最擅长的事，把其他事交给别人。Thousand Token Wood 里的 NPC 当然不懂李嘉图的理论，但价格信号在替它们做这个计算。当木材价格高时，更多人涌入伐木；当面包供不应求时，连铁匠都会考虑要不要去帮面包师搬面粉。

“Build Small”背后的算力焦虑

Thousand Token Wood 之所以值得认真聊，不只是因为项目本身有趣，更因为它踩中了一个正在加速的趋势：小模型的逆袭。

Hugging Face 办“Build Small”黑客松，本身就带着一种态度。2025 年上半年，整个 AI 圈的叙事主轴是“scaling law 还有没有效”。GPT-5 迟迟未出，Claude 4 虽然强但贵得离谱，Gemini Ultra 在 benchmark 上刷榜但开发者用不起。与此同时，Llama 3.2 3B、Qwen 2.5 3B、Phi-3-mini 这批小模型，在特定任务上的表现已经逼近甚至超过两年前的大模型。

Hugging Face 的官方博客在介绍这次黑客松时说得很直白——他们鼓励参赛者用“能在边缘设备上跑起来的模型”做“真正有用的东西”。Thousand Token Wood 的整个推理管线跑在一台 MacBook 上，3B 模型用 llama.cpp 量化后，内存占用不到 2GB。10 个 NPC，每 5 秒一次推理，CPU 风扇甚至没怎么转。

当 OpenAI 在为一轮 GPT-5 训练烧掉几亿美元时，一台 MacBook 正在安静地运行一个微型文明。

这种对比本身就在拷问一个根本问题：我们到底需要多大的模型？如果一个 3B 模型能在沙盒里涌现出套利、分工、价格发现这些复杂经济行为，那那些万亿参数模型多出来的 9997 亿参数，到底在干什么？是在背维基百科的每一个脚注，还是在真正提升推理能力？

从沙盒到现实：多 agent 经济的想象空间

Thousand Token Wood 目前只是一个黑客松项目，代码量不大，场景也极其简化——只有木材、石头、铁矿石、面包、工具这几种物品。但它打开的想象空间，远比项目本身要大。

第一个方向是游戏。过去二十年，开放世界游戏一直在追求“更真实的 NPC”。从《上古卷轴》的 Radiant AI 到《荒野大镖客 2》里每个 NPC 都有自己的日程表，游戏行业在 NPC 行为模拟上投入了海量资源。但这些系统本质上是编剧和程序员手写的规则，NPC 的行为丰富度受限于设计文档的厚度。

Thousand Token Wood 展示的路径完全不同：不给 NPC 写规则，只给 NPC 一个目标（比如“活下去，赚钱”），然后让模型自己驱动行为。这意味着，一个 50 人的小团队，可能做出 NPC 行为丰富度超越 3A 大作的游戏——因为 NPC 的“剧本”不是写出来的，是实时生成的。

第二个方向是经济模拟。经济学家一直想要一个“社会实验室”——可以调参数、跑实验、观察宏观现象如何从微观行为中涌现。目前的主流工具是 agent-based modeling（ABM），但 ABM 里的 agent 通常是极其简化的规则引擎，跟真实人类的决策复杂度差了几个数量级。

如果把 Thousand Token Wood 的思路放大——用更大一点的模型，更丰富的环境，更多的 agent——是不是就能跑出一个更有参考价值的模拟经济体？中央银行调利率、政府发补贴、贸易战加关税，这些政策在真实世界里试错成本极高，但如果能先在模拟器里跑一万轮呢？

（当然，3B 模型的决策质量离“可信任的政策模拟”还差得远。但方向是对的。）

第三个方向，也是最让我兴奋的，是 AI agent 之间的自主协作。2025 年上半年的 AI 叙事，已经从“单个 agent 能做什么”转向了“多个 agent 怎么配合”。Anthropic 在推 MCP 协议，OpenAI 在搞 Assistants API 的群组调用，LangChain 和 CrewAI 在抽象 multi-agent 编排层。但 Thousand Token Wood 走了一条更激进的路线：它不给 agent 编排，agent 之间唯一的协调机制是市场价格。

市场本身成了那个“看不见的手”，而模型只是这只手上的神经元。

3B 模型的极限与陷阱

说了这么多好话，该泼冷水了。

Thousand Token Wood 的 trace 里，暴露了 3B 模型在长期运行中的几个硬伤。第一个是“记忆坍缩”。Llama 3.2 3B 的上下文窗口是 128K token，项目团队用了滑动窗口机制来管理每个 NPC 的记忆。但跑了几百轮之后，NPC 开始出现“失忆”症状——忘记自己昨天在干什么，忘记跟谁做过交易，甚至忘记自己背包里有什么。

在一条 trace 里，伐木工 Willow 连续三次去市场卖同一批木材，因为模型“不记得”自己已经卖过了。这种重复行为在单个 NPC 身上只是搞笑，但如果 10 个 NPC 都开始犯糊涂，整个经济系统的效率会迅速崩坏。

第二个问题是“目标漂移”。NPC 的角色设定（“你是一个伐木工，你的目标是……”）是写在 system prompt 里的，但 3B 模型对长 prompt 的遵循能力有限。跑久了之后，NPC 的行为会逐渐偏离初始设定。铁匠 Bram 在某个阶段突然决定“我要去探索世界”，丢下铁匠铺走了三天，回来发现市场已经被另一个 NPC 占领了。

第三个，也是最根本的问题：3B 模型没有真正的“世界模型”。它只是在做 next token prediction，它对“砍树”的理解不是基于物理规则，而是基于训练数据里关于砍树的文本描述。这意味着当环境出现训练数据里没有的组合时，模型会胡编。比如有一次，一个 NPC 试图“用面包跟树换木材”——因为在模型的训练数据里，“交换”是一个高频模式，但“面包”和“树”之间的可交换性约束，模型并没有真正学到。

3B 模型能涌现经济行为，但涌不出常识。

黑客松背后的草根 AI 运动

聊完技术，我想聊聊这场黑客松本身。

Hugging Face 的“Build Small”不是一场孤立的活动。2025 年上半年，整个开发者社区正在形成一股“去中心化 AI”的暗流。起因是多方面的：大模型 API 涨价（OpenAI 在 4 月调高了 GPT-4o 的价格，Anthropic 的 Claude 4 每百万 token 成本是 3B 模型的几百倍）、开源模型性能跃升、以及越来越多人意识到“不是所有任务都需要一个博士级别的模型”。

在 Hugging Face 上，Llama 3.2 3B 的下载量在过去三个月涨了四倍。Qwen 2.5 3B 的微调变体在 Hugging Face 模型库里已经超过了 200 个。Phi-3-mini 被塞进了树莓派、无人机、甚至一个智能门锁里。

大模型是军备竞赛，小模型是人民战争。

Thousand Token Wood 在这个语境下的意义，不只是“一个有趣的黑客松项目”。它是一个信号：当算力民主化到一台 MacBook 就能跑一个微型文明时，创新会从哪里长出来？不会是 OpenAI 的办公室里，不会是 Google 的 TPU 集群上，而会是某个开发者的周末 side project 里。

Hugging Face 的 CEO Clem Delangue 在 6 月的一条推文里写道：“The future of AI is not one model to rule them all, but millions of small models, each specialized, each running locally, each owned by its user.”（AI 的未来不是一个模型统领一切，而是数百万个小模型，每个专精一项，每个本地运行，每个属于它的用户。）

Thousand Token Wood 就是这条推文的一个活的注脚。

一个沙盒，和它撬动的万亿市场

最后，让我们回到那个最俗但也最真实的问题：这东西能赚钱吗？

Thousand Token Wood 本身大概不会直接商业化。但它的技术路径——用小模型驱动多 agent 系统，让 agent 用市场机制自主协调——正在被多个行业盯上。

供应链优化是一个显而易见的场景。一个全球物流网络里，成千上万个节点（仓库、卡车、港口、零售商）需要实时决策。传统做法是用运筹学模型做全局优化，计算量大、响应慢、而且一旦出现计划外事件（比如港口罢工），整个模型要重新跑。如果把每个节点变成一个轻量级 agent，用本地小模型做局部决策，用价格/优先级信号做全局协调，系统的鲁棒性和响应速度可能会上一个台阶。

另一个场景是 IoT 设备集群。一个智能工厂里有几百台设备，每台设备有自己的状态、维护周期、能耗曲线。让一个中心化 AI 去管几百台设备，通信延迟和单点故障都是大问题。但如果每台设备跑一个几百 MB 的模型，自己管自己，设备之间用拍卖/竞价来协调产能——这不就是 Thousand Token Wood 的工业版吗？

一个 3B 模型在沙盒里跑出来的经济逻辑，可能正在撬动万亿级的工业市场。

当然，从黑客松项目到工业级产品，中间隔着一万个坑。模型幻觉怎么解决？agent 间的恶意行为（比如商人 Finn 如果学会欺诈）怎么防范？系统出现级联故障时谁来兜底？这些问题 Thousand Token Wood 一个都答不了——但它至少提出了一个问题：如果最简单的 3B 模型就能涌现出市场行为，那当我们把模型再放大一点、环境再复杂一点、agent 再多一点时，会发生什么？

我不知道答案，但我知道有人在 MacBook 上跑这个实验的时候，OpenAI 正在为 GPT-5 的千亿美元融资路演，Anthropic 刚签下了 22 万张 H100 的算力合同，马斯克还在 X 上跟人吵架。

世界在分叉。一条路通往万亿参数的超级大脑，另一条路通往数百万个小模型组成的分布式智能。Thousand Token Wood 不过是第二条路上的一棵小树苗——但树苗底下，可能藏着一整片森林。
