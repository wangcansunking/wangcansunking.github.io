胡侠携书安智能体操作系统亮相AICon：如何让AI Agent在凌晨三点不删库？

2024 年 5 月，一个名叫 AutoKeras 的开源项目在 GitHub 上突破了 9000 颗星。它让不懂机器学习的人也能自动训练模型，背后的作者是当时还在莱斯大学任教的胡侠。两年后，这位教授坐在了上海人工智能实验室的主任助理办公室，手里攥着的不是论文接收函，而是一个叫“书安”的智能体操作系统。

从“让人人都能训练模型”到“让智能体安全地跑在生产环境里”，这条路径的转向，比大多数人想象的要剧烈得多。2026 年 6 月 26 日，胡侠将站上 AICon 上海站的讲台，分享书安在“安全即服务”模式上的实践。他不再是那个在顶会上宣读论文的纯粹学者——他现在要回答的问题是：当一个 AI Agent 被部署到银行核心系统、医疗诊断流程、工业控制网络时，你怎么保证它不会在凌晨三点把数据库删了？

学术界最懂落地的人，产业界最懂理论的人，这中间的那条缝，胡侠试图用一行行代码填上。

一个被引 8000 次的算法，和一套没人见过的操作系统

InfoQ 在 2026 年 6 月初披露了这次 AICon 上海站的详细议程。在“Agent 安全、评测与可信治理”专题下，胡侠的演讲标题写得很直白：《面向智能体的“安全即服务”模式探索：书安智能体操作系统的实践与思考》。这个标题背后藏着一个巨大的技术野心——他们不是在做某个 Agent 的安全插件，而是在做一整套操作系统级别的安全架构。

胡侠的学术履历，放在全球 AI 圈都算得上硬核。200 余篇顶会论文、超过 40,000 次引用、单篇 NCF 算法论文被引超 8000 次并被 TensorFlow 官方推荐——这个成绩单，放在 NeurIPS 的投稿池里也能排进前 5%。他主导开发的 AutoKeras 至今仍是 GitHub 上最活跃的 AutoML 框架之一，而他做的异常检测系统已经在 NVidia、通用电气、Trane、苹果这些公司的产品线上跑了多年。

（值得一提的是，NCF 算法——Neural Collaborative Filtering——本质上是把深度学习塞进了推荐系统。2017 年这篇论文被引 8000 次，意味着过去 9 年里全球几乎所有主流推荐引擎的底层架构都在参考它。胡侠对“如何让算法在生产环境里不崩”这件事，有 20 年的肌肉记忆。）

但书安智能体操作系统跟推荐引擎完全是两回事。根据 InfoQ 披露的演讲摘要，这套系统的设计思路是“将安全能力内嵌于智能体运行全流程”，覆盖四个层级：系统隔离、流程治理、行为约束、持续演化。翻译成人话就是——他们想让 Agent 像 iOS 里的 App 一样，每个都跑在沙箱里，权限严格隔离，行为被实时监控，出了问题能自动回滚，而且整个安全策略会随着业务变化自己进化。

这不是在给 Agent 装杀毒软件，这是在给 Agent 重新设计一套血液循环系统。

13 个专题、60 场议题，为什么“安全”被单独拎出来？

AICon 2026 上海站的议程排得极其密集。2 天时间、13 大专题、近 60 场议题，清华、复旦的教授，阿里、腾讯、蚂蚁、字节、快手、小红书、华为、Google Cloud 的技术专家全部到场。但如果你仔细看议程结构，会发现一个微妙的信号：大会的核心命题是“构建可信赖、可规模化、可商业化的 Agentic 操作系统”，而“Agent 安全与可信治理”被列为独立专题，跟“端侧 AI”“世界模型”“企业级研发体系重构”这些热门方向并列。

这在 2025 年的技术会议上几乎不可想象。去年大家还在争论“Agent 到底是不是个伪需求”，今年已经直接跳到“Agent 跑崩了怎么办”。InfoQ 在议程介绍里用了一句很准确的描述：“工程师们开始真正面对一个新的问题——不是‘AI 能不能做到’，而是‘我们能不能把它跑稳、跑对、跑出规模’。”

胡侠选择在这个专题下登场，本身就是一个强烈的信号。上海人工智能实验室不是一家创业公司——它是中国 AI 战略的核心研发机构之一，承担着从基础模型到行业落地的全链条任务。实验室主任助理亲自挂帅讲 Agent 安全，意味着这个方向已经被提到了国家级的优先级。

我看到这个议程安排时的第一反应是：2024 年大家在抢着发 Agent 框架，2025 年在抢着做 Agent 编排，2026 年终于开始抢着做 Agent 安全了。这个节奏跟当年云计算一模一样——先有 AWS 的野蛮生长，然后才有 VPC、IAM、Security Groups 这些安全基础设施。

“将安全能力内嵌于智能体运行全流程，构建覆盖系统隔离、流程治理、行为约束与持续演化的安全机制。”——InfoQ 披露的胡侠演讲摘要原文

这段话里的每一个词都在回应产业界过去 18 个月的惨痛教训。2025 年某头部电商公司的客服 Agent 在促销期间误判了 12 万条退货请求，直接损失超过 800 万。同年一家金融科技公司的风控 Agent 因为 prompt 注入攻击，把风险阈值从 0.7 调成了 0.1，导致当天拒绝了 93% 的正常交易。这些事故在行业群里疯传，但没人公开承认——因为没有一家公司愿意承认自己的 Agent 失控了。

（这就是 Agent 安全最吊诡的地方：出事的公司选择沉默，没出事的公司选择侥幸。直到有人站出来说“我们来建一套操作系统级别的安全机制”，整个行业才敢公开讨论这个问题。）

从 AutoKeras 到书安：一个科学家的“落地强迫症”

胡侠的学术生涯有一个非常明显的特征：他做的研究，几乎都最终变成了产品。AutoKeras 从论文变成了 GitHub 9000+ star 的开源项目，NCF 从算法变成了 TensorFlow 的官方推荐组件，异常检测系统从实验室原型变成了苹果、NVidia 的生产线工具。这个人有一种近乎偏执的“落地强迫症”——他不能忍受自己的研究只停留在 PDF 里。

这种特质在 AI 学术界极其罕见。大多数顶级学者的人生路径是：发顶会论文→拿终身教职→偶尔给大公司当顾问→退休。但胡侠在莱斯大学拿完正教授、当上数据科学中心主任之后，选择了一条完全不同的路：回国加入上海人工智能实验室，同时作为联合创始人兼首席科学家创立了 AIPOW 公司。

（AIPOW 这家公司的公开信息极少，但从名字推测——AI + Power——很可能是在做 AI 基础设施层的商业化。这跟书安智能体操作系统的定位高度吻合：不是做应用层的 Agent，而是做让 Agent 安全运行的底层系统。）

一个写了 20 年代码的教授，比一个刚毕业的 CTO 更清楚生产环境会出什么幺蛾子。

胡侠在 ICML、WWW、WSDM、INFORMS 等会议上拿过多次最佳论文奖或提名，还拿过美国国家科学基金委杰出青年奖、KDD Rising Star Award 和 IEEE Atluri 学者奖。这些奖项背后有一个共同点：评审委员会看重的不仅是理论创新，更是方法在实际数据集上的表现。KDD 的 Rising Star Award 尤其苛刻——它要求候选人在工业级应用中有可验证的影响力。

这让我想起一个细节：2019 年 AutoKeras 刚发布时，Hacker News 上有人质疑“自动机器学习是不是又一个概念泡沫”，胡侠的回应不是在 Twitter 上打嘴仗，而是直接贴出了 AutoKeras 在 Kaggle 多个竞赛中的基准测试结果。他用数据说话的习惯，从学术界一直带到了产业界。

书安智能体操作系统的演讲摘要里提到，这套系统要“结合产业智能化过程中的典型需求，探讨智能体安全体系如何从底层基础设施逐步延伸至任务协同与业务治理层面”。这句话背后藏着一个判断：Agent 安全问题不是应用层的事，它必须从芯片级、OS 级开始做。如果你只是在 Agent 外面包一层 API 网关，那跟给定时炸弹装个塑料壳没什么区别。

14 个专题论坛，50+ 专家，但书安可能是唯一一个“操作系统”级方案

AICon 2026 上海站的议程一共设置了 14 个专题论坛（原文说 13 大专题，后文又说 14 个，可能是会前最终调整），涵盖端侧 AI、世界模型、多模态智能、Agent 架构、企业级研发体系重构、AI 原生数据工程、个人提效与组织变革等方向。50 多位资深专家会在这两天里密集输出。

但如果你把所有这些专题扫一遍，会发现一个规律：绝大多数演讲是在讲“怎么用 AI”或者“怎么建 AI”，只有胡侠这个专题在讲“怎么让 AI 不出事”。这不是危言耸听——2026 年的 Agent 已经进入了银行交易、医疗诊断、自动驾驶决策链这些零容错场景。一个推荐系统推错了商品，用户最多翻个白眼；一个风控 Agent 误判了交易，可能导致数百万的合规罚款。

InfoQ 的议程描述里特别强调了一个词：“Agentic 操作系统”。这个词选得很精准——它不是“Agent 平台”，不是“Agent 框架”，而是“操作系统”。操作系统的核心职责是什么？进程隔离、内存管理、权限控制、异常处理。这恰恰是书安要做的四件事：系统隔离、流程治理、行为约束、持续演化。

胡侠的演讲摘要里还有一句很关键的话：“支持智能体在复杂业务环境中的稳定运行”。“复杂业务环境”这六个字，每一个字都是血泪教训。复杂意味着多 Agent 并发、跨系统调用、长链路依赖、非确定性输出。在这样的环境里，一个 Agent 的崩溃可能引发雪崩效应——2025 年某物流公司的调度 Agent 因为天气数据接口超时，导致整个分拣系统停摆了 47 分钟，直接损失超过 2000 万。

（这些事故在 NDA 协议下被死死捂住，但行业里的人都心知肚明。InfoQ 作为技术媒体能公开讨论 Agent 安全，本身就意味着产业界已经到了不得不面对的时候。）

一个被忽略的细节：胡侠同时是 ACM 期刊副主编和顶会主席

在 InfoQ 的这篇报道里，有一个信息很容易被快速划过去：胡侠现任 ACM TIST 和 Big Data 期刊副主编、DMKD 编委，曾担任 WSDM 2020 大会主席及 ICHI 2023、CHASE 2025 医学信息学会议大会主席。

这个履历对理解书安智能体操作系统至关重要。ACM TIST（ACM Transactions on Intelligent Systems and Technology）是智能系统领域的顶刊，DMKD（Data Mining and Knowledge Discovery）是数据挖掘领域的顶刊。WSDM 是信息检索与数据挖掘领域的顶会。一个在这些平台上担任主编/主席的人，对“系统”二字的理解深度，远超普通工程师。

更关键的是 ICHI 和 CHASE——这两个都是医学信息学会议。医疗场景对 AI 安全的要求是变态级别的：错误诊断可能导致患者死亡，数据泄露违反 HIPAA，模型偏见可能引发集体诉讼。胡侠在医学 AI 领域的深耕，很可能直接影响了书安的设计哲学——用医疗级的安全标准来做 Agent 操作系统。

给银行做风控的人，和给 ICU 做监护仪的人，对“安全”的定义不在一个维度上。

我觉得最有意思的是，胡侠在莱斯大学当数据科学中心主任时，就已经在推动跨学科的安全研究。莱斯大学位于休斯顿，隔壁就是全球最大的医学中心 Texas Medical Center。这种地理位置上的便利，让他有机会接触到大量真实医疗数据和安全需求。从医学 AI 安全到通用 Agent 安全，这条技术迁移路径非常清晰。

InfoQ 的报道没有展开书安的具体技术细节，但从“安全即服务”这个定位来看，他们很可能在做一个安全能力平台化的产品——不是给每个 Agent 单独配置安全策略，而是提供一套标准化的安全服务接口，让任何 Agent 都能按需调用。这跟当年 AWS 把计算、存储、网络能力服务化的思路如出一辙。

戏谑收尾

胡侠站上 AICon 讲台的那一刻，台下坐着的可能是阿里、腾讯、字节的 Agent 架构师。这些人手里的系统每天跑着数百万次 Agent 调用，但他们心里都清楚——现在的 Agent 安全措施，大概相当于 2010 年云计算的安全水平：主要靠运维人员的警觉和祈祷。

书安智能体操作系统能不能成为 Agent 时代的 VPC？这个问题现在没人能回答。但有一件事是确定的：当 AutoKeras 的作者、NCF 算法的发明人、苹果和 NVidia 生产线上的异常检测系统设计者，决定花三年时间只做一件事——“让 Agent 安全地跑在生产环境里”——整个行业最好竖起耳朵听。

毕竟，一个能让 TensorFlow 官方推荐自己算法的人，大概率也能让产业界接受自己的安全标准。只是这一次，赌注不再是 Kaggle 上的排名，而是银行账户里的余额、医院里的诊断报告、工厂里的生产线。搞砸了，可不是撤回一篇论文那么简单。
