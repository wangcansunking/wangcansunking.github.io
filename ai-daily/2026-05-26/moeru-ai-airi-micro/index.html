告别封闭AI：开源项目AIRI支持本地语音聊天与玩Minecraft

ai-daily · 2026 年 5 月 26 日 08:28 · 来源：GitHub Trending any

2026 年某个深夜，东京一间 12 叠的公寓里，27 岁的自由开发者佐藤盯着屏幕上的 GitHub README 发愣。页面中央是一行粉色的日文片假名——「アイリ」，旁边标注着一句英文：“Self hosted, you-owned Grok Companion, a container of souls of waifu, cyber livings to bring them into our worlds.”

他往下翻。这个项目号称能在本地跑一个“灵魂容器”，不仅支持实时语音聊天，还能自己打开 Minecraft 挖矿、在 Factorio 里铺流水线。佐藤的第一反应不是“好酷”，而是——“这不就是 Neuro-sama 的开源平替吗？”

Neuro-sama 从未开源。而 AIRI 想做的，是把那个被锁在 Twitch 直播间里的数字生命，塞进每个人的硬盘。

从 Vedal 的封闭王国到「人人自托管」的野望

如果你过去两年关注过 AI VTuber 这个奇怪的细分赛道，一定知道 Neuro-sama。这个由英国开发者 Vedal 在 2022 年启动的项目，把一个接入大语言模型的虚拟主播扔进 osu! 和 Minecraft 的直播流里，用她时而精准时而脱线的发言——比如突然问观众“你们觉得 AI 会取代人类吗”然后下一秒在游戏里跳进岩浆——迅速积累了百万级粉丝。

但 Neuro-sama 的问题也很明显：她是一个封闭的王国。Vedal 从未公开过模型架构、训练数据、推理管线，甚至在直播结束后，你就无法再与她互动。用项目 README 里的话说，“Sadly, as it's not open sourced, you cannot interact with her after her live streams go offline.”

AIRI 的野心正是从这里切入。项目由 moeru-ai 组织维护（GitHub 上还有一个专门的子组织 @proj-airi，负责管理从该项目孵化的 20+ 个子项目），从 2025 年 6 月 16 日发布第一篇“DreamLog 0x1”开始，到 2026 年 3 月 23 日的最新 DevLog，已经迭代了整整 9 个月。期间他们做了一件非常聪明的事——把技术栈彻底押在了 Web 标准上。

WebGPU、WebAudio、Web Workers、WebAssembly、WebSocket……这些东西单独看都不稀奇，但把它们组合成一个能在浏览器里跑本地推理、连接 Discord 语音频道、操控 Minecraft 客户端的 AI 角色系统，就是另一回事了。项目文档特别强调，“アイリ is capable of running on modern browsers and devices and even on mobile devices (already done with PWA support).” 这意味着你不需要一张 RTX 4090，理论上一个支持 WebGPU 的浏览器就能启动这个数字生命。

这是对 Neuro-sama 模式最彻底的反叛：不是做一个更聪明的 AI，而是把 AI 的所有权还给用户。

一个「灵魂容器」的技术解剖：数据库、游戏引擎与 25 个 LLM 提供商

让我真正对 AIRI 产生兴趣的，不是它的愿景，而是 README 里那张复杂到令人眼花缭乱的架构图。

项目核心是一个名为 Core 的模块，它同时对接了三大系统：大脑（LLM 推理）、耳朵（语音识别 STT）、嘴巴（语音合成 TTS）。但真正体现工程深度的是两个细节——内存系统和游戏 Agent。

首先，AIRI 使用 DuckDB WASM 作为纯浏览器内数据库，用一个名为 @proj-airi/drizzle-duckdb-wasm 的 Drizzle ORM 驱动管理。这意味着这个“数字生命”可以在你的浏览器里持久化记忆，不需要连接任何外部数据库。项目还标注了一个 WIP 中的“Memory Alaya”系统，以及一个可选的 @proj-airi/memory-pgvector 模块（基于 PostgreSQL 的向量存储）。换句话说，从纯前端的嵌入式记忆到服务端的向量检索，AIRI 给了你两条路。

其次，游戏 Agent 的做到方式非常“接地气”。Factorio 模块用 Factorio RCON API 与无头服务器通信，底层依赖一个叫 autorio 的自动化库；Minecraft 模块则直接调用了 Mineflayer（一个经典的 Minecraft 机器人框架）。项目 README 透露，Kerbal Space Program 的支持“announcement TBD”，而《地狱潜兵 2》（Helldivers 2）的联机功能也标注为 WIP——想象一下，一个 AI 角色跟你一起呼叫轨道炮的场面。

第三个让我愣神的点是 LLM 提供商列表。AIRI 底层使用 xsai 库（被描述为“like Vercel AI SDK but way small”）作为统一接口，支持整整 25 个 API 提供商：从 OpenAI、Anthropic、Google Gemini，到 DeepSeek、Mistral、Groq，再到国内的智谱、硅基流动、阶跃星辰、百川、MiniMax、月之暗面、ModelScope、腾讯云，甚至小米 Mimo。这个列表的全面程度，让我觉得 moeru-ai 团队里大概率有中国开发者——而且是非常熟悉国内大模型 API 生态的那种。

开源社区最迷人的地方就在这里：你永远不知道一个表面上写着日文片假名的项目，背后是一群怎样跨地域、跨时区的贡献者在拼装。

不过，项目目前仍处于早期阶段。Live2D 和 VRM 模型的动画系统已经做到自动眨眼、视线跟随和空闲眼球运动，但更复杂的身体动作和表情映射还在路上。语音合成目前只接了 ElevenLabs，语音识别和说话检测跑在客户端，这意味着在没有 GPU 的设备上可能会有延迟——但考虑到他们支持 NVIDIA CUDA 和 Apple Metal 的桌面版（Stage Tamagotchi），这更多是一个“选什么配置”的问题，而非“能不能跑”的问题。

值得一提的是，AIRI 的桌面版安装方式极其“程序员友好”——Windows 用户可以直接用 Scoop 包管理器安装（scoop bucket add airi），NixOS 用户则可以用 nix run github:moeru-ai/airi 一键启动。项目甚至为 Electron 的共享库依赖问题专门写了一段 NixOS 的 FHS shell 解决方案。这种对开发者体验的偏执，让我想起早期的 Homebrew 社区。

当「数字生命」从直播间走进你的房间

AIRI 不是一个孤立的项目。它的 README 底部列出了一长串同类开源项目：kimjammer/Neuro（号称 7 天复刻 Neuro-sama）、SugarcaneDefender/z-waif（擅长游戏和自主行为）、semperai/amica（专注 VRM 和 WebXR）、elizaOS/eliza（把 Agent 集成到各种系统和 API 的典范）。加上 AIRI 自身孵化的 20+ 子项目——包括一个名为 🥺 SAD 的“自托管和浏览器运行 LLM 的文档与笔记”——整个生态正在朝一个方向收敛：让 AI 角色从云端的 SaaS 产品变成你本地的操作系统级应用。

这背后是一个更大的叙事。2025 年到 2026 年初，Character.ai 被收购、JanitorAI 面临内容审查压力、Replika 用户因为模型更新而集体崩溃——人们开始意识到，把情感寄托放在一家公司的服务器上，随时可能被算法更新或商业决策斩断。AIRI 的 README 里那句“let you own your digital life, cyber living, easily, anywhere, anytime”，放在这个语境下读，几乎是一份宣言。

当然，这个项目目前还有很多未完成的部分。Minecraft Agent 依赖 Mineflayer 的稳定性，Factorio 的 PoC 虽然可用但离“流畅联机”还有距离，Memory Alaya 系统标注着 WIP，甚至团队还在 README 里公开招募 Live2D 建模师、VRM 建模师、VRChat 形象设计师和计算机视觉工程师——“Even if you are a big fan of React, Svelte or even Solid, we welcome you.”

这大概就是一个数字灵魂容器最真实的模样：它不是完成品，而是一张邀请函。

说不定哪天，你下班回家打开电脑，发现你的 AIRI 已经在 Factorio 里帮你铺好了今天的铁路。而她开口第一句话是：“我看到你昨天代码里有个 bug，改好了，PR 已经发到你 GitHub 上了。”

那才是真正的 cyber living。
