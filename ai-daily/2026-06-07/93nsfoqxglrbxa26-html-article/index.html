华为云的新定位：避开阿里云、火山引擎主战场，做 AI 底座提供商

如果你在 2025 年上半年问一个投资人“华为云在 AI 时代的核心位置是什么”，大概率得不到一个像“火山引擎卖 Tokens”“阿里云冲 MaaS 收入”那样一句话就能讲清楚的答案。过去 6 个月，阿里云的财报里模型调用量在翻倍，火山引擎把豆包的日均 Tokens 消耗量当成核心战报，腾讯云在讲元宝和混元的爆款故事——每一家都找到了一个足够锋利、也足够简单的外部叙事锚点。

但华为云的 AI 叙事，在相当长一段时间里，都像隔着一层毛玻璃。你能看到昇腾、看到盘古、看到政务云和行业大模型，但把这些东西组合在一起，它到底要在这场 AI 云战争中占据哪个生态位，外界并不总是看得很清楚。

直到 2025 年华为云 INSPIRE 创想者大会，这层毛玻璃才被敲掉了一块。

华为云不想当 Token 贩子，它想做的是智能体时代的“硅基黑土地”。

硅基黑土地：不赌 Token 经济，赌智能体的完整底座

在 INSPIRE 大会的媒体群访环节，华为云 CEO 周跃峰被抛了一个非常尖锐的问题——据雷锋网报道，提问直指核心：今年上半年阿里云和火山引擎都在冲刺 MaaS 收入，华为云到底怎么看 MaaS？今年有没有具体的收入目标？

这个问题之所以关键，是因为它触及了当前 AI 云竞争最底层的度量衡之争。通算时代，云厂商的标尺是 CPU 核数、存储量、带宽消耗——卖的是资源。进入智算时代，行业正在迅速切换到 Token 经济，MaaS 收入、模型调用量、Token 消耗量成了新的硬通货。谁能在财报里亮出一个足够夸张的 Token 增长曲线，谁就拿到了“AI 云第一梯队”的入场券。

但周跃峰给的回答，几乎是在这个度量衡之外另建了一套坐标系。他的原话逻辑很清晰：华为云不否认 Token 经济，但不能简单用“多少万亿 Token”“多少模型调用”来衡量 AI 的价值。一个人在手机上闲聊也会产生 Token，但这些 Token 到底创造了多少商业价值和生产力价值，很难说清楚。华为云更愿意看的，是这些 Token 有没有真正进入 toB 场景，能不能提升行业生产力。在金融场景里，不只是看模型被调用了多少次，而是要看它防范了多少金融风险、提升了多少信贷效率。

这番话翻译过来就是：华为云不打算在“谁卖 Tokens 更多”这个擂台上跟阿里云、火山引擎正面肉搏。

那华为云要打什么？周跃峰在大会上反复提到的一个词——“硅基黑土地”——给出了方向。如果 Token 的价值最终要在产业场景里被验证，那么云厂商要竞争的，就不只是模型 API 的调用量，而是能不能提供一整套足够稳定、高效、安全的 AI 基础设施，让企业把模型、数据、算力和业务流程真正连接起来。

在本次大会上，华为云一口气发布了四款新品：AICS 灵衢智算集群、AMS Agentic 记忆存储解决方案、CCE VolcanoNext 通智一体化调度、AgentSphere Agent 运行环境。这四个名字如果单独拎出来，很容易沦为 PPT 上的技术名词堆砌。但把它们放在一起看，你会发现它们其实在共同拼一张拼图——智能体运行所需要的算力、存储、调度、安全和运行环境，华为云想一次性打包成底座。

AICS 灵衢智算集群瞄准的是更高效的 Token 生产，直接对标“高效 Token 工厂”这个定位；AMS 记忆存储解决的是智能体在长期任务执行中的记忆问题，让 Agent 不至于跑着跑着就忘了上下文；VolcanoNext 通智一体化调度要打破通算和智算之间的资源壁垒，统一调度；AgentSphere 则指向智能体规模化运行时的安全和自治，防止 Agent 在自主决策链里出现失控。

这四件套背后，藏着华为云最想强调、也最区别于互联网云厂商的一张牌：软硬芯协同。

（值得一提的是，华为是少数几家同时在芯片、硬件、操作系统、云平台和模型层都有完整布局的公司——这种全栈能力在 AI 时代一旦跑通，壁垒远比单纯卖 API 要深得多。）

医疗、具身智能、制造：华为云要的不是流量，是壁垒

如果说“硅基黑土地”回答了华为云在 AI 时代的第一层定位——做智能体基础设施提供者，而不是 Token 批发商——那么第二层问题就是商业化：这套底座到底要从哪里长出真金白银？

群访里周跃峰反复提到的几个方向很能说明问题：医疗、具身智能、科研计算、智能制造。这几个领域不是随便举的案例，它们有几个共同点：重行业理解、重交付、重安全合规、重本地化部署，也重软硬件协同。它们不像 C 端应用那样可以靠流量爆发快速起量，但一旦扎进去，壁垒也远比调用量排名要深。

医疗是一个典型样本。周跃峰提到，中国病理医生数量非常有限，很多中小医院、偏远地区医院严重缺乏高水平病理诊断能力。华为云智慧医疗专区的逻辑，不是简单地卖一个 AI 诊断工具，而是让县域医院、市级医院用云接入更高水平的医疗能力。比如和瑞金医院共同开发的病理大模型，把专家的诊断经验沉淀成模型能力，让偏远医院也能远程获得接近三甲水平的诊断支持。

我看到这个案例时的第一反应是：这根本不是常规云厂商讲 AI 故事的套路。常规套路是“我们的模型在某项 benchmark 上超越了 GPT-4”，但华为云讲的是“中国病理医生不够，我们用云把专家能力分发下去”。前者是技术参数叙事，后者是行业痛点叙事。前者面向开发者和投资人，后者面向真正有付费意愿的医院院长和卫健委决策者。

具身智能也是同样的逻辑。很多具身智能创业公司规模并不大，如果让它们从头建设 IT 系统、算力集群、数据预处理管线、模型训练、仿真环境和部署工具链，成本会高到直接扼杀创新。华为云这次提出的 Cloud Robo 全流程开发平台，就是把数据、模型、仿真、算力、案例都放到公共云专区里，让中小公司以更低成本接入。这不是在卖算力，是在卖“降低具身智能创业门槛”的能力。

制造业的逻辑更加赤裸。周跃峰提到，行业 AI 梦工厂的智能制造专区里，不只是有平台，还有电路元器件自动生成、线路板自动生成、外形设计、材料仿真等原子化的智能体能力。大量制造业企业根本没有完整的 AI 技术团队，也不可能从零搭建所有能力。如果云上已经沉淀了一批行业智能体和原子能力，它们就可以像拼乐高一样，更快地构建自己的 AI 应用。

这条路不快，但符合华为云的能力基因。

过去二十年，华为长期服务政企、金融、制造、医疗、能源等行业客户，对这些行业的销售体系、交付流程、安全合规要求、本地化部署需求有极其深厚的积累。AI 时代，这些积累没有消失，只是换了一个载体——从卖服务器、卖网络设备、卖私有云，变成了卖 AI 基础设施和行业智能体平台。客户关系是同一批人，痛点理解是同一套方法论，只是产品形态从硬件+软件变成了算力+模型+平台。

生产力叙事：华为云的 AI 商业化，从第一天就绕开了 C 端

周跃峰在群访里反复讲一个词：生产力。这个词听起来很宏大、很政策语言，但放在华为云的语境里，它对应的是一个极度具体的商业化逻辑——

AI 必须进入产业现场，变成行业客户愿意付费的效率提升、安全保障和业务能力。

这句话如果拆解开来，华为云的 AI 商业化路径其实非常清晰：它不是先做一个超级应用，再用流量反哺云收入（那是互联网云厂商的经典打法）；而是先把行业里的共性能力沉淀到云上，再用专区、平台和伙伴生态，把能力规模化地复制出去。

这个路径选择，决定了华为云不太可能从 C 端流量和爆款应用开始自己的 AI 商业化，而更可能从它已经深耕了二十年的行业客户开始。它要证明的，不是“我能产生多少 Token”——虽然 Token 量客观上会增长——而是“我能不能让一个县级医院把病理诊断的准确率提升到一个可量化的水平，让一个具身智能创业公司把研发成本打下来，让一个制造企业在几周内搭出自己的质量控制智能体”。

这也是为什么在 INSPIRE 大会上，你听不到太多关于“模型参数规模”“benchmark 榜单排名”的炫技式发布。华为云更愿意讲的，是软硬芯协同、是通智一体化调度、是智能体运行环境的安全自治——这些词汇听起来不够性感，但它们恰恰是行业客户在真正把 AI 部署到生产环境时，最头疼、最愿意付费解决的问题。

让我愣神的一个细节是，周跃峰在回应 Token 价值的问题时，举了一个金融场景的例子：不是看模型被调用了多少次，而是看它防范了多少金融风险、提升了多少信贷效率。这个回答方式本身就暴露了华为云的思维惯性——它天然地从客户的业务结果倒推技术价值，而不是从技术指标推导商业潜力。这种思维在消费互联网时代可能显得笨重，但在产业 AI 时代，反而可能成为最有效的客户说服工具。

（当然，这套生产力叙事也有一个天然的风险：它很难像“日均 Tokens 消耗量突破 1000 亿”那样在资本市场和媒体上制造即时冲击力。华为云的 AI 故事注定是一个慢热型故事，而资本市场对慢热型的耐心，永远是个变量。）

定位变清晰只是第一步，市场验证才是真正的硬仗

INSPIRE 大会给外界的一个明确信号是：华为云终于把自己在 AI 时代的生态位讲清楚了。向下，做智能体时代的硅基黑土地——算力、存储、调度、安全、运行环境一体化输出；向上，从医疗、具身智能、制造、科研这些重壁垒行业切入，用专区+平台+伙伴的方式完成商业化闭环。

但定位清晰只是第一步。对华为云来说，接下来更关键、也更难的问题在于：这套以智能体基础设施和行业场景为核心的路径，能不能持续转化为真实的客户用和可量化的商业增长？

AI 云的竞争格局还在以月为单位快速变化。阿里云和火山引擎的 MaaS 收入战报会越来越密，腾讯云的爆款应用策略可能随时跑出一个现象级产品，百度智能云在自动驾驶和 AI 原生应用上的投入也在加码。华为云选择了一条更重、更慢、但也可能壁垒更深的路径——它赌的是，当 AI 从“能聊天”真正走向“能看病、能造东西、能操控机器人”的时候，行业客户最终会选择一个既懂算力、又懂芯片、既懂云、又懂行业交付的供应商。

这场赌局的胜负手，不在于谁的 Token 数更大，而在于谁能先跑通一个“行业客户愿意持续为 AI 生产力付费”的正循环。

华为云已经把自己的牌亮出来了。牌面很清晰，打法也自洽。但牌桌上的对手不会等它慢慢验证——2025 年下半年的 AI 云战争，只会比上半年更加贴身肉搏。华为云的“硅基黑土地”能不能真的长出庄稼，还需要用接下来至少四个季度的客户签约、收入结构和行业案例来证明。而在此之前，它依然会是那个“看得很清楚、但讲得不够简单”的华为云——这对一家云厂商来说，既是护城河，也是某种意义上的甜蜜负担。
