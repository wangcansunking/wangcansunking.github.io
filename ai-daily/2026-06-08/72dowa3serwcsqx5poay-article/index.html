GitLab裁员350人重写Git：AI Agent让代码提交暴增100倍

GitLab 2027 财年第一季度的财报电话会，本来是一场标准的胜利巡游。营收 2.642 亿美元，同比暴增 23%，毛利率稳在 88%，净亏损从 3590 万美元收窄到区区 500 万美元。CEO Bill Staples 和 CFO Jessica Ross 轮番上台，讲“结构性顺风”、讲“股票回购 240 万股”、讲“全年利润指引上调”。如果只看数字，这几乎是一家 SaaS 公司在宏观经济逆风中能交出的最漂亮答卷。

但电话会结束后不到一个月，2026 年 6 月初，GitLab 宣布裁员 14%，约 350 人离开这家全远程办公的明星公司。

一边是营收超预期、亏损收窄、客户大额订单增长 18%，另一边是裁掉七分之一的员工、退出 22 个国家、压平三层管理层级。这两件事同框的违和感，让整个科技圈都愣了一下。

这不是一次“困境式裁员”——它是智能体时代的第一场基础设施起义。

被 AI 智能体“逼到极限”的，不是人力，是 Git 本身

GitLab 的裁员叙事里藏着一个极其反常的信号。通常企业裁员的说辞无外乎“降本增效”“聚焦核心业务”“优化运营效率”，但 Bill Staples 在财报电话会上给出的理由，跟利润率没有半毛钱关系。他说的是：

智能体以机器级规模运行，正在把一众竞争对手逼到极限。本季度，我们启动了对 Git 的一次代际重构，以支持做到 100 倍增长所需的规模与功能。

注意这个措辞——“代际重构”“100 倍增长”。Git 这个版本控制系统从 2005 年 Linus Torvalds 写下第一行代码开始，经历过无数次优化和扩展，但从来没有人说过它需要一次“代际重构”。GitHub 托管着超过 4 亿个仓库，GitLab 自己服务着数十万企业客户，这套基础设施在过去二十年里被证明是足够强韧的。

直到 AI 智能体开始以“机器级规模”提交代码。

GitLab CEO 描述了一个让所有开发者基础设施团队后背发凉的现象：智能体工作负载正在对系统施加超出原本设计承载能力的压力。一位未具名的大型科技公司工程负责人把智能体形容为“精神失常的实习生”——它们能写代码，但缺乏判断力，产出的 PR 数量是人类开发者的数十倍甚至上百倍，其中夹杂着大量需要人工审核的低质量提交、重复变更和安全漏洞。

这里有一个被大多数行业观察者忽略的细节：GitLab 的竞争对手 GitHub 已经先踩过这个坑。TechCrunch 在 6 月 3 日的报道中明确指出，GitHub 曾因大量 AI 驱动的提交涌入而面临稳定性挑战，影响了系统正常运行时间。这说明 GitLab 面对的不是一个假设性的威胁，而是一个已经被验证过的生产环境事故。

Staples 透露的信息量远比裁员本身大得多。GitLab 已经与一家未披露名称的 AI 实验室合作，重新设计并重建面向 AI 工作负载的基础设施。同时，他们正在构建“对智能体优化”的 API，用于存储和检索上下文——包括代码。这还不是全部：公司还在投入开发编排工具，以协调 AI 智能体与开发者之间的软件开发流程，构建上下文层，并将治理工具直接内嵌进平台。

（值得一提的是，“对智能体优化的上下文 API”这个方向，跟 Anthropic 的 Claude 集成路径高度吻合，而 GitLab 的 Duo Agent Platform 今年早些时候已经深度集成了 Claude 模型。）

当你的版本控制系统需要为机器人重构底层架构，裁掉 350 个人类员工只是最不起眼的副作用。

退出 22 个国家，同时悄悄在印度招人

裁员消息出来后，X（原 Twitter）上的开发者社区反应迅速且尖刻。有人截图发了一条推文：“LinkedIn 上的招聘信息才是真正的路线图。”截图里，GitLab 在宣布裁员的同一周，LinkedIn 上挂着多个面向印度的远程职位。

这条推文被疯狂转发后，所有面向印度的 GitLab 远程岗位从 LinkedIn 上“悄然”消失。但印度最大的招聘网站 NAUKRI 上，新的 GitLab 职位仍在不断出现。

GitLab 官方给出的重组方案里写了四项运营调整：退出 22 个国家、压平组织结构、在部分职能中最多削减三层管理层级、加大基础设施投入。这四条里，“退出 22 个国家”是执行难度最高的一条。GitLab 作为一家从成立第一天就全远程办公的公司，2500 名员工分布在数十个国家，这意味着要在不同司法辖区处理员工解聘，而各地在通知期和遣散规则上各不相同。财报里明确写着，3000 万至 3500 万美元的税前重组费用将分摊在四个季度确认，其中约 1900 万美元在当前季度计入——这正是跨国裁员的典型财务特征。

但真正让观察者感到不安的，是印度招聘与全球裁员之间的时间线重叠。根据人力资源公司 Xpheno Pvt 的数据，过去一年中，美国科技大厂 Meta、苹果、微软、奈飞和谷歌在印度的员工总数增长了 16%。开源社区 GitHub 去年发布的报告预测，到 2030 年，印度将新增超过 3560 万名开发者，以总数 5750 万超越美国，成为全球开发者数量最多的国家。

而最关键的，可能是那个所有科技公司都不愿意公开讨论的数字：印度工程师的平均薪资仅为美国同岗位的四分之一。

我看到这个消息的第一反应是，GitLab 的“重构”逻辑其实非常自洽——如果智能体正在以机器规模生成代码，那么人类开发者的角色将从“写代码”转向“审核、治理和编排”。在这种范式转移下，用更低成本的人力覆盖更多时区的审核需求，从财务和运营角度都是合理的选择。但这件事的观感实在太差了：一边在财报电话会上大谈“结构性顺风”和股票回购，一边让 350 个人离开，同时把岗位转移到薪资只有四分之一的地区。

裁员的公司都在赚钱，这不是巧合

把目光从 GitLab 身上移开，会发现一个令人不安的宏观图景。根据 Statista 的数据，2026 年科技行业的裁员人数已经突破 10 万人，如果趋势持续，这个数字有可能超过 2024 年和 2025 年的水平。但跟 2022 年末那一波“寒冬裁员潮”完全不同——这次裁员的公司，绝大多数都在赚钱，而且赚得比以往更多。

GitLab 营收增长 23%，毛利率 88%，订阅收入从 1.945 亿美元跃升至 2.393 亿美元，年经常性收入超过 10 万美元的客户数量增长了 18%。非 GAAP 运营利润率从 12% 提升到 14%，GAAP 净亏损几乎抹平。

Intuit 在宣布裁员 1800 人时，季度营收同比增长 13%。Block（前 Square）裁掉 931 人，全年毛利润增长 22%。Cisco 裁了 4000 多人，同期发布了超出预期的财报。Cloudflare 裁掉约 300 人，营收增长 32%。Meta 在 2023 年裁掉 2 万多人后，股价翻了三倍。

GitLab 的 CFO Jessica Ross 在财报电话会上完全没有提“裁员”两个字。她的发言聚焦在“稳健的财务基础”和“股票回购计划”——本季度 GitLab 回购了约 240 万股股票。CEO Bill Staples 的表态则更加高调：

智能体时代正在为 GitLab 带来结构性利好，从第一季度平台活动的加速增长以及 GitLab Duo Agent Platform 的初步成果中可以清晰看到。GitLab 是唯一一个用单一控制平面、单一数据模型，同时保持云和 AI 模型中立性的、覆盖完整软件生命周期的平台。

这段话的信息密度极高。“单一控制平面”“单一数据模型”“云和 AI 模型中立性”——Staples 在试图将 GitLab 定位成智能体时代的核心基础设施，一个既不被 AWS 绑定、也不被 Anthropic 或 Google 的单一模型锁死的“可信企业级平台”。而做到这个定位的代价，包括裁掉 350 人、退出 22 个国家、将研发体系重组成约 60 个规模更小但权限更高的端到端团队。

过去裁员是业绩差的信号。2026 年，裁员是 AI 转型的入场券。

下一代 Git 重构：一次被迫的自我革命

TechCrunch 的报道里有一个被大多数人略过的细节：GitLab 的裁员公告里提到，公司已经开始用 AI 智能体重塑内部流程，将评审、审批和交接等环节自动化，并计划据此对公司各岗位规模相应调整。换句话说，GitLab 正在拿自己当试验品，测试智能体究竟能替代多少人类工作流。

而更大的赌注在 Git 本身。Staples 说的“代际重构”到底意味着什么，官方没有给出技术细节，但可以从已知信息中拼出一个轮廓：与未披露名称的 AI 实验室合作重建基础设施 + 对智能体优化的上下文 API + 支持 100 倍增长所需的规模与功能 + 编排工具协调 AI 与人类开发者。结合起来看，这很像是在构建一套原生支持 AI 智能体作为一等公民的版本控制系统。

传统的 Git 工作流是线性的：开发者写代码 → 提交 commit → 推送到分支 → 创建 PR → 人类审核 → 合并。但智能体的工作流是爆炸式的：一个 Agent 可能在几分钟内创建几十个分支、提交数百个 commit，其中大部分需要被丢弃或合并，审核压力从“每人每天几个 PR”变成“每人每天几十上百个 PR”。现有的 Git 架构在元数据管理、并发处理、上下文存储方面都没有为这种负载做过设计。

（GitHub 的系统稳定性问题，大概率就是被这种“提交风暴”打崩的。）

GitLab 的应对策略是把治理和编排直接做进平台层——不是事后审核，而是实时治理；不是人工判断每个 commit，而是用智能体审核智能体的输出。这听起来像是用魔法打败魔法，但实际上可能是唯一可行的路径。当提交量达到“机器级规模”时，任何依赖人类判断的质量控制机制都会瞬间崩溃。

Staples 在电话会上特别强调了一个点：“GitLab 所解决的问题包括安全、合规以及大规模治理，在智能体时代只会变得更加复杂。”这句话的潜台词是——如果你觉得现在的 DevSecOps 已经够复杂了，等智能体开始在你的代码库里自由活动时，你会怀念今天的一切。

裁掉 350 人只是开始，每个开发者都将面对同样的拷问

GitLab 的故事之所以值得被认真对待，不是因为它裁了 350 个人——科技行业每天都有公司在裁员。而是因为它的叙事逻辑自洽到让人无法反驳：智能体在逼着基础设施升级 → 升级需要资源 → 资源从人力成本中释放 → 释放出来的人力正好是那些可以被智能体替代的岗位。

GitLab 把节省下来的大部分成本重新投入研发和 AI 产品，而不是简单地转化为利润率。这意味着裁员不是为了短期美化财报，而是为了在一个正在被智能体重塑的市场里抢到基础设施层的船票。60 个更小、权限更高、端到端负责的研发团队，几乎将独立团队数量翻倍——这是典型的“用更少的人做更多的事”的敏捷化改造，而 AI 智能体被寄望于填补人力缩减后的执行缺口。

但 GitLab 的印度招聘事件提醒所有人，技术叙事的背后永远是成本结构在驱动决策。当一家公司一边说“智能体时代的结构性利好”，一边把岗位从薪资高的地区迁移到薪资低的地区，它的行为比它的措辞更诚实。印度工程师平均薪资是美国同岗位的四分之一，而 AI 智能体的“机器级规模”意味着时区覆盖和审核带宽比单个工程师的深度技能更重要——这个算账逻辑一旦成立，全球范围内的开发者岗位地理分布将在未来五年内经历一次剧烈重构。

GitHub 去年那份预测——印度到 2030 年新增 3560 万开发者——在 AI 智能体大规模介入软件开发的背景下，读起来有了完全不同的意味。OpenAI 和 Anthropic 相继在新德里和班加罗尔开设首个印度办公室，分别挖角 WhatsApp 和微软前高管担任印度“1 号员工”，两家公司都公开表示印度是继美国之外的第二大市场。这不是巧合，这是整个行业在用脚投票。

GitLab 预计 2027 财年第二季度营收在 2.72 亿至 2.74 亿美元之间，届时首笔约 1900 万美元的重组费用将正式计入财报。一个刚刚裁掉约七分之一员工的公司，其新的运营形态也将第一次完整地呈现在市场面前。投资者在盘后已经把股价推高了——市场用钱投票的速度，永远比任何技术博客的评论都快。

那些在 X 上截图质疑印度招聘的开发者们，其实问出了更深层的问题：当 Git——那个每个开发者职业生涯第一天就要学会的工具——正在为机器人重构底层架构，写代码的人类在十年后还剩下什么角色？

GitLab 的 CEO 给出的答案是“治理、编排和上下文管理”。听起来体面，但一个需要 350 人离开、60 个更小团队顶上、印度招聘悄悄的转型，显然不会给所有人预留座位。
