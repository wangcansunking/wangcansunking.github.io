胡侠携书安智能体系统亮相AICon：AI Agent安全难题何解？

一个 AutoML 框架被 TensorFlow 官方推荐，一篇推荐算法论文被引超 8000 次，一个异常检测系统跑进了苹果和英伟达的生产线——这些成果背后是同一个名字：胡侠。而现在，这位上海人工智能实验室的领军科学家，把注意力转向了一个更棘手的问题：当 AI Agent 开始接管业务流程，谁来保证它不会在关键时刻“发疯”？

安全不是 Agent 的外挂插件，而是它能否走出 demo 的生死线。

从 AutoKeras 到书安：一个科学家的安全转向

胡侠的履历在机器学习圈子里属于那种“不需要介绍”的级别。美国莱斯大学正教授、数据科学中心主任，联合创立 AIPOW 公司并担任首席科学家，在 ICLR、NeurIPS、KDD、WWW、SIGIR 这些顶会顶刊上发表超过 200 篇论文，总被引次数突破 40,000 次。他主导开发的 AutoKeras 至今仍是 GitHub 上最常用的 AutoML 框架之一，而那篇提出 NCF（Neural Collaborative Filtering）算法的论文，单篇被引超 8000 次，直接被 TensorFlow 收编为官方推荐做到。

但这些数字背后有一个容易被忽略的细节：胡侠的异常检测系统已经被部署到了 NVidia、通用电气、Trane、苹果等企业的产品里。这意味着他不是第一次处理“从论文到产线”的工程化难题——只不过这一次，他要面对的不再是静态的数据异常，而是动态的、多步决策的、可能产生连锁反应的 Agent 行为。

2026 年 6 月 26 日至 27 日，在上海举办的 AICon 大会上，胡侠将在“Agent 安全、评测与可信治理”专题发表主题演讲，题目是《面向智能体的“安全即服务”模式探索：书安智能体操作系统的实践与思考》。据 InfoQ 中国 AI 报道，这场大会集结了清华、复旦等高校教授，以及阿里、腾讯、蚂蚁、字节、快手、小红书、华为、Google Cloud 等数十家头部公司的技术专家，2 天时间塞进了 13 大专题、近 60 场议题和一个动手实验室——而胡侠的分享，被放在了“安全”这个最不性感但最要命的环节。

书安智能体操作系统的核心思路，是把安全能力内嵌到 Agent 运行的每一个环节里——不是事后审计，不是外挂防火墙，而是从系统隔离、流程治理、行为约束到持续演化，全部做成原生能力。换句话说，书安试图回答的问题是：如果 Agent 要在金融、医疗、工业这些“出事就是大事”的领域跑起来，安全体系能不能像操作系统一样，成为它默认的底层基础设施？

这个思路之所以值得关注，是因为过去一年行业踩过的坑太多了。Agent 架构怎么设计？记忆怎么管理？多智能体之间如何协调？这些问题在 demo 阶段都可以用“先跑起来再说”糊弄过去，但一旦进入生产环境，一个幻觉引发的错误决策、一次任务链断裂导致的业务中断、一个权限溢出造成的数据泄露，都足以让 CIO 连夜叫停整个 AI 项目。InfoQ 在大会介绍里用了一句话总结当下的真实困境：“不是‘AI 能不能做到’，而是‘我们能不能把它跑稳、跑对、跑出规模’。”

安全即服务：从“能不能”到“敢不敢”的最后一公里

让我愣神的是“安全即服务”这个提法。它暗示了一个行业共识正在形成：Agent 的安全能力，最终会像云计算里的 IAM（身份与访问管理）或数据库里的加密模块一样，变成一种可以标准化、可以接入、可以按需调用的基础设施层。书安试图做的，就是把系统隔离、行为约束、流程治理这些安全机制，从“每个团队自己造轮子”变成“操作系统默认提供”。

这个逻辑在产业场景里尤其关键。胡侠的分享会结合“产业智能化过程中的典型需求”，讨论安全体系如何从底层基础设施逐步延伸到任务协同与业务治理层面。这句话翻译过来就是：Agent 安全不是一个技术点的单点突破，而是一条从芯片到应用的全链路工程。系统隔离解决的是“Agent 能不能访问不该访问的数据”，流程治理解决的是“多 Agent 协作时谁来拍板、谁来回滚”，行为约束解决的是“Agent 的每一步操作能不能被预测和审计”，持续演化解决的是“安全策略能不能跟上 Agent 能力的升级速度”。

值得一提的是，胡侠的学术背景里有大量“系统落地”的基因。他开发的异常检测系统之所以能进入苹果和英伟达的产品线，靠的不是论文里的理论最优解，而是工程上能扛住真实负载的鲁棒性。这种从顶会论文到产线代码的跨越能力，恰恰是当前 Agent 安全领域最稀缺的——懂安全的往往不懂 Agent 架构，懂 Agent 的往往没做过企业级安全合规，而胡侠的履历刚好踩在了两个圈的交集上。

AICon 这次大会的其他专题也侧面印证了 Agent 工程化的热度。端侧 AI、世界模型与多模态智能突破、Agent 架构与工程化实践、企业级研发体系重构、AI 原生数据工程——14 个专题论坛、50 多位资深专家，几乎把 Agent 从模型到应用的全链路拆了一遍。但安全专题被单独拎出来，而且放在了和“架构与工程化实践”并列的位置，这本身就说明了一件事：2026 年的 Agent 行业，已经从“能不能造出来”的阶段，正式进入了“敢不敢用上去”的阶段。

胡侠这次要讲的“书安”，目前公开信息还很有限，但“智能体操作系统”这个定位已经透露了不少信息。操作系统级别的安全不是给应用加个权限弹窗，而是从进程隔离、内存管理、系统调用这一层就开始做约束。如果把这个思路搬到 Agent 领域，意味着每一个 Agent 的执行环境、每一次工具调用、每一段记忆读写，都要经过安全层的仲裁——这在性能开销和工程复杂度上都是巨大的挑战，但也是唯一能让 Agent 在金融交易、医疗诊断、工业控制这些“零容忍”场景里跑起来的方式。

大会的票务信息已经挂出来了，联系电话 13269078023，扫码可咨询详情。2 天、近 60 场分享、1 个动手实验室——规模不小，但真正能让人记住的，往往是那些回答了“出事怎么办”的议题。毕竟，Agent 行业最不缺的是酷炫的 demo，最缺的是能扛住凌晨三点生产事故的工程底气。

胡侠从 AutoKeras 走到书安，从“让模型自动训练”走到“让 Agent 安全运行”——这条路线拐了个弯，但方向没变：都在做“让 AI 真正能用起来”这件事。只不过这一次，他要解决的不是算法精度问题，而是信任问题。而信任，从来都是技术落地前最后一道、也最难跨越的门槛。
