Anthropic 网络安全技能库：754 个结构化技能如何武装 AI 代理？

ai-daily · 2026 年 5 月 24 日 02:37 · 来源：GitHub Trending python

凌晨三点，一个安全分析师盯着屏幕上的内存转储文件，手指悬在键盘上方。他需要判断这次告警是误报还是真实的凭证窃取攻击——但团队里唯一擅长 Volatility 3 的同事正在休产假，而他上一次跑内存取证插件已经是 2022 年的事了。

他叹了口气，打开终端，敲了一行让 AI 代劳的指令。

AI 扫过 754 个技能条目，在 30 毫秒内筛出 12 个相关项，然后加载了最匹配的三个：Volatility 3 内存取证流程、LSASS 凭证转储狩猎指南、Windows 事件日志凭证访问分析。它按步骤执行插件、交叉比对 IOC 指标、把发现映射到 ATT&CK T1003（Credential Dumping），最后生成了一份带时间戳的取证报告。整个过程，比他那位休产假的同事快了三倍。

这不是科幻小说里的情节。这是 2026 年 5 月 7 日，一个叫 Anthropic Cybersecurity Skills 的开源项目在 GitHub 上展示的标准工作流。

给 AI 代理一套安全技能库，它就能像高级分析师一样干活——前提是，有人先把这 754 个技能写出来。

一个独立开发者，把安全行业的技能负债变成了开源资产

这个项目的创建者不是 Anthropic 官方团队。GitHub 用户名 mukul975（实名 Mahipal Jangra）在项目 README 里用一段置顶警告写得明明白白：

⚠️ Community Project — This is an independent, community-created project. Not affiliated with Anthropic PBC.

这是一个社区项目，和 Anthropic 这家估值 600 亿美元的 AI 公司没有半毛钱关系。但它的命名策略极其聪明——直接用了「Anthropic」这个关键词，让任何在 GitHub 上搜索「AI + 安全 + 技能库」的人都会第一时间撞见它。而一旦你点进去，就会发现这远不是一个蹭品牌流量的噱头。

项目包含了 754 个结构化网络安全技能，横跨 26 个安全领域，每一个技能都按照 agentskills.io 开放标准编写——YAML 前置元数据用于毫秒级搜索发现，结构化 Markdown 正文用于逐步执行，references 目录存放深度技术参考。更关键的是，每一个技能都被映射到了 五套行业框架：MITRE ATT&CK v18、NIST CSF 2.0、MITRE ATLAS v5.4、MITRE D3FEND v1.3、NIST AI RMF 1.0。

我在安全圈混了这么多年，见过无数「安全工具合集」——从 PayloadsAllTheThings 到 SecLists，从 awesome-pentest 到 LOLBAS。但那些项目给的是弹药和武器清单，不是作战手册。一个 AI 代理拿到 SQL 注入 payload 列表后，仍然不知道什么时候该用哪种注入技巧、怎么验证注入点是否存在、如何判断 WAF 类型再选择绕过策略。Anthropic Cybersecurity Skills 做的事，是把「作战决策树」写成了 AI 可执行的格式。

ISC2 在 2024 年发布的数据显示，全球网络安全岗位缺口达到 480 万个。这 480 万个空缺不是缺工具，是缺人——缺那些知道在什么情况下调用什么工具、怎么解读输出、怎么把技术发现翻译成业务风险的人。这个项目试图让 AI 代理填补的，正是这个「判断力缺口」。

五套框架一张图：一次技能执行，五个合规复选框打钩

让我愣神的是这张映射表。

项目用一个真实的技能示例展示了什么叫「一次执行，五套合规」：analyzing-network-traffic-of-malware 这个技能，同时映射到 ATT&CK T1071（应用层协议）、NIST CSF DE.CM（检测-持续监控）、ATLAS AML.T0047、D3FEND D3-NTA（网络流量分析）和 NIST AI RMF MEASURE-2.6。一个分析恶意软件网络流量的操作，在传统 SOC 流程里可能要三个不同角色（威胁猎人、合规专员、AI 安全工程师）分别出报告。现在，AI 代理执行一次技能，五个框架的合规检查点全亮绿灯。

合规这件事最贵的不是罚款，是人力。这个项目把合规的人力成本直接砍到了接近零。

ATT&CK 覆盖了全部 14 个战术（TA0043 到 TA0040），从侦察、初始访问、执行、持久化、提权、防御规避、凭证访问、发现、横向移动、收集、C2、数据渗出到影响，每个战术下都有「Strong」或「Moderate」的覆盖率标注。其中凭证访问（TA0006）和防御规避（TA0005）的覆盖尤其深厚——Mimikatz 检测、Pass-the-Hash、混淆分析、Rootkit 分析，这些在真实攻防中决定胜负的关键环节，技能库里都有对应的结构化工作流。

（值得一提的是，MITRE 在 2026 年 4 月 28 日发布了 ATT&CK v19，把原来的防御规避 TA0005 拆分成了两个新战术：Stealth 和 Impair Defenses。项目 README 明确说会在后续版本中更新映射——这说明维护者在盯着框架版本变化，不是一次性扔完就跑。）

NIST CSF 2.0 这边，2024 年 2 月新增的 Govern 功能有 30+ 个技能覆盖，Detect 功能更是达到 200+ 个技能——这是整个库里最密集的区域，也符合安全运营的实际需求：检测能力永远是最稀缺的。Respond 功能 160+ 个技能，Recover 功能 40+ 个。22 个类别全部对齐，引用 106 个子类别。

而 ATLAS 和 AI RMF 的映射，让这个项目从「网络安全技能库」升级成了「AI 安全技能库」。MITRE ATLAS v5.4 覆盖了 16 个战术和 84 个技术，包括 2025 年底新增的 AI 代理攻击向量：AI 代理上下文污染、工具调用滥用、MCP 服务器入侵、恶意代理部署。NIST AI RMF 的 GenAI 配置文件（AI 600-1，2024 年 7 月发布）则增加了 12 个生成式 AI 特有的风险类别，从虚构内容到提示注入再到供应链风险。科罗拉多州的 AI 法案（2026 年 2 月生效）甚至给遵守 NIST AI RMF 的组织提供了法律责任安全港——这意味着，使用这个技能库的企业在 AI 合规诉讼中可能多一层保护。

一个 GitHub 仓库，既教 AI 怎么抓黑客，又帮企业过合规审计——这种「一鱼两吃」的设计思路，在开源安全项目里极其罕见。

每技能 30 token 扫描，500 token 执行：一个为 AI 窗口大小设计的渐进式加载架构

如果你在过去一年里用过任何 AI 编程工具——Claude Code、GitHub Copilot、Cursor、Gemini CLI、OpenAI Codex CLI——你应该很清楚上下文窗口的珍贵。200K token 的窗口听起来很大，但塞进一个中型代码库的依赖关系图就差不多满了，更别说还要加载安全技能。

这个项目解决这个问题的方式很优雅：渐进式披露架构。每个技能的 YAML 前置元数据大约 30 个 token，包含名称、描述、领域、子领域、标签、框架映射。AI 代理第一轮扫描时只读这 30 token，从 754 个技能里筛出相关的十几个。然后才加载完整技能体——500 到 2,000 token，包含 When to Use（触发条件）、Prerequisites（前置要求）、Workflow（逐步执行指南）、Verification（验证方法）四个核心部分。

这套架构让 AI 代理能在一次上下文窗口内完成「搜索 + 筛选 + 深度加载 + 执行」的完整链路，不需要来回切换工具或手动分页。对比一下传统做法——把安全 SOP 文档扔进 RAG 系统，让 AI 自己翻——你会发现这种「预结构化」的威力：不是让 AI 理解文档，而是让文档本身就按 AI 的执行逻辑组织。

项目支持的平台列表长得像一份 AI 工具黄页：Claude Code、GitHub Copilot、Cursor、Windsurf、Cline、Aider、Continue、Roo Code、Amazon Q Developer、Tabnine、Sourcegraph Cody、JetBrains AI、OpenAI Codex CLI、Gemini CLI、Devin、Replit Agent、SWE-agent、OpenHands、LangChain、CrewAI、AutoGen、Semantic Kernel、Haystack、Vercel AI SDK。一共 26+ 个平台，覆盖了从 IDE 插件到 CLI 代理到自主代理框架的全光谱。唯一的要求是平台支持 agentskills.io 标准——而这个标准本身就是开放的。

安装方式也极简。推荐方案是一行 npx 命令：

npx skills add mukul975/Anthropic-Cybersecurity-Skills

或者传统的 git clone。没有依赖地狱，没有 Docker 镜像拉半小时。clone 完，指向你的 AI 代理，立刻生效。

26 个安全领域，从云安全到欺骗技术的覆盖光谱

如果你仔细看 26 个领域的分布，会发现一个有趣的模式：项目在「高频刚需」领域投入了大量技能，而在新兴领域保持轻量覆盖，留出社区贡献空间。

云安全以 60 个技能 位居榜首，覆盖 AWS、Azure、GCP 的加固、云安全态势管理（CSPM）和云取证。这不意外——2026 年的企业安全战场已经全面转移到云上，而云配置错误仍然是数据泄露的头号原因。威胁狩猎 55 个技能紧随其后，强调假设驱动狩猎、离地攻击（LOTL）检测和行为分析。威胁情报 50 个，涵盖 STIX/TAXII、MISP、情报源集成和攻击者画像。

Web 应用安全 42 个技能，从 OWASP Top 10 到 SQLi、XSS、SSRF、反序列化攻击。有意思的是，API 安全单独列了一个领域（28 个技能），覆盖 GraphQL、REST、OWASP API Top 10 和 WAF 绕过——这说明项目意识到了现代应用安全的重心正在从传统 Web 向 API 层迁移。

容器安全 30 个技能，涉及 Kubernetes RBAC、镜像扫描、Falco 和容器取证。OT/ICS 安全 28 个技能，涵盖 Modbus、DNP3、IEC 62443、历史数据库防御和 SCADA——这个领域在开源安全工具里一贯被忽视，因为 OT 安全从业者相对小众，但关键基础设施保护的合规压力正在把 OT 安全推向主流视野。

而欺骗技术只有 2 个技能，合规与治理只有 5 个技能。这不是缺陷，是设计选择。项目 README 直接点明：「Deception Technology (2 skills) 和 Compliance & Governance (5 skills) 最需要社区贡献。」维护者把资源集中在攻防技术链的核心环节，把长尾领域留给社区——这是一种务实的开源策略，比试图面面俱到却每个领域都写不深要聪明得多。

"This project grows through community contributions."

这句写在 CONTRIBUTING.md 里的话，配合「48 小时内审核 PR」「优先处理 good first issues」的承诺，说明维护者是真的想建社区，不是把代码扔 GitHub 上等 star。

技能解剖：一个 YAML 前置元数据 + 四个 Markdown 章节构成的「AI 原生知识库」

每个技能的文件结构是统一的：

skills/performing-memory-forensics-with-volatility3/
├── SKILL.md
├── references/
│   ├── standards.md
│   └── workflows.md
├── scripts/
│   └── process.py
└── assets/
    └── template.md

YAML 前置元数据里，name 是 kebab-case 格式（1-64 字符），description 是关键词丰富的文本（用于 AI 代理发现匹配），domain 和 subdomain 做分类，tags 做灵活标注，然后是五个框架的具体映射 ID。版本号、作者、许可证都在元数据里——这意味着任何 AI 代理扫描时都能在 30 token 内判断「这个技能是否适用于当前任务」。

Markdown 正文的四个章节各有明确的职能：When to Use 定义了触发条件（什么时候 AI 代理应该激活这个技能），Prerequisites 列出所需工具、权限级别和环境配置，Workflow 是逐步执行指南（带具体命令和决策点），Verification 教 AI 如何确认技能执行成功。

这种设计的精妙之处在于：它既不是脚本集合（脚本只告诉你跑什么命令，不告诉你为什么跑、什么时候跑、跑完怎么验证），也不是教程文档（教程是给人看的，AI 难以解析出执行步骤）。它是专门为 AI 代理设计的「可执行知识单元」——结构化的程度刚好让 AI 能按步骤操作，又保留了足够的上下文让 AI 理解每一步的目的。

我见过的最接近的类比是 Ansible Playbook 和 SOC Runbook 的结合体——但它不是给自动化引擎用的，是给大语言模型用的。这个区别很重要：Ansible 需要精确的 YAML 语法和模块调用，LLM 需要的是自然语言指令加足够的结构化提示。agentskills.io 标准找到了这个平衡点。

这个项目真正改变的不是技术，是安全行业的人力模型

回到开头那个凌晨三点的场景。那个安全分析师最终没有等到同事休完产假。他用了 Anthropic Cybersecurity Skills，AI 代理完成了内存取证、凭证转储检测、事件日志关联分析，把发现映射到了 ATT&CK T1003 并生成报告。他的 SOC 经理第二天早上看到报告时，以为那位休产假的同事远程加了班。

安全行业缺的不是工具，是能把工具串成决策链的人。这个项目用 754 个结构化技能，把「高级分析师的决策链」变成了 AI 可执行的格式。

ISC2 说的 480 万缺口，不是 480 万个初级 SOC 分析师岗位——那些岗位正在被 SOAR 和 XDR 平台自动化掉。真正的缺口在「能独立判断威胁优先级、能设计调查路径、能把技术发现翻译成业务影响」的中高级岗位。Anthropic Cybersecurity Skills 瞄准的恰好是这个层级：它编码的不是「跑什么命令」，而是「在什么情况下跑什么命令、怎么解读输出、怎么验证结论、怎么映射到合规框架」。

当然，这个项目不可能完全替代人类分析师。AI 代理执行技能时仍然可能出错——Verification 章节的存在本身就承认了这一点。但它的价值不在于完美替代，在于把「从零培养一个中级分析师」的时间从 18 个月压缩到了「安装一个 GitHub 仓库」的时间。对于一家 SOC 每年花 15 万美元招人却找不到合适候选人的企业来说，这个 ROI 算都不用算。

（不过话说回来，如果你是个刚入行的安全新人，看到这个项目可能会有点焦虑——AI 已经能执行你正在学的那套 Volatility 3 工作流了。但换个角度想，未来最值钱的安全人才不是那些能跑插件的人，而是那些能设计新技能、能发现技能库覆盖不到的边缘场景、能判断 AI 输出是否靠谱的人。技能库降低了执行门槛，但提升了判断力的价值。）

项目的社区反馈也在印证这种判断。Hasan Toor（@hasantoxr）在社交媒体上说：「一个真实的、有组织的安全技能数据库，任何 AI 代理都能即插即用。不是教程，不是博客文章。」fazal-sec 在 Medium 上写道：「这不是一个随机的安全脚本集合。这是一个为 AI 驱动的安全工作流设计的结构化操作知识库。」项目已经被收录进 awesome-agent-skills、awesome-ai-security、awesome-codex-cli 等多个 awesome 列表，在 SkillsLLM 上作为推荐技能目录展示，还被 Openflows 和 NeverSight 做信号分析和自动索引。

v1.0.0 版本在 2026 年 3 月 11 日发布，包含 734 个技能、26 个领域、MITRE ATT&CK 和 NIST CSF 2.0 映射。之后主分支继续增长到 754 个技能，并追加了 MITRE ATLAS、D3FEND 和 NIST AI RMF 的三套映射。社区贡献通道明确开放，48 小时审核承诺，Contributor Covenant 行为准则——这一切都指向一个信号：维护者不是在做一次性开源作品，是在搭一个可以持续演进的平台。

一个独立开发者，用 754 个 YAML 文件，让任何 AI 代理都能像高级安全分析师一样思考——这件事的杠杆效应，比大多数融资千万的安全创业公司都高。

mukul975 还顺手在项目里嵌了一个学术调查入口——GARS-2026 全球代理 AI 就绪度调查，60 个问题，10 分钟，匿名，SRH Berlin 监督，结果以 CC-BY 4.0 公开发布。完成调查的人能拿到 50 个 Casky Token，用于早期访问 casky.ai。这种「开源项目 + 学术研究 + 商业产品预览」的三合一操作，显示出维护者对生态位的精准理解——不只是做工具，是在定义一个赛道。

所以，下次你凌晨三点被安全告警吵醒的时候，想想这个画面：你的 AI 代理已经扫完了 754 个技能，锁定了 3 个最匹配的工作流，正在跑 Volatility 3 插件、比对 IOC、生成 ATT&CK 映射报告。而你只需要决定一件事——要不要再睡会儿。

（答案是不行，因为 AI 还没法替你按那个「确认升级为 P1 事件」的按钮。至少 2026 年还不行。）
