字节扣子3.0发布：4个AI Agent组团打工，手机就能远程操控

你打开扣子，给它看了一眼《我的世界》的视频，说了句“做一个类似的游戏项目”。几分钟后，一个可交互的方块世界就在浏览器里跑起来了。

这不是某个游戏引擎的宣传片，这是字节跳动旗下 AI 开发平台扣子（Coze）在 2026 年 6 月发布的 3.0 版本实测。但视频生成游戏只是这场发布的开胃菜——真正的重头戏藏在后面：扣子 3.0 把桌面端、电脑端、手机端全打通了，你甚至可以用手机远程遥控电脑上的 Agent 干活。

过去一年，AI 产品的竞争被简化成了模型能力的军备竞赛：谁更聪明，谁推理更强，谁上下文更长。但扣子 3.0 这次没跟风，它选择了一条更“接地气”的路——让一群术业有专攻的 Agent 组团给你打工。

AI 产品的下一站，不是更强的模型，而是更像真实团队的工作系统。

一个项目、四个 Agent、零行代码

先看一个让科技编辑血压升高的需求：做一个 AI 热点追踪仪表盘。

这个需求听起来简单，但真正上手时，编辑要先判断今天哪些新闻值得写，还要补背景、查事实、找角度，接着把需求拆成页面模块，最后才是代码做到。过去，这类任务通常会得到一个“大而全”的回答：前面讲选题，中间讲页面，后面突然开始写代码——看起来什么都有，实际什么都落不了地。

扣子 3.0 换了一种做法。在同一个项目里安排了 4 个 Agent 协作：小曜（扣子自带的通用 Agent）、选题写作 Agent、产品设计 Agent 和前端开发 Agent。用户只需要新建一个项目，@这几个 Agent，附上一段 Prompt，任务下达瞬间，几个专家 Agent 就各自开忙了。

从实测效果来看，Agent 们有来有往地交流过后，一个 AI 热点追踪仪表盘就诞生了。整个过程的价值不在于某个 Agent 多聪明，而在于它把一个复杂任务拆成了不同角色的连续劳动——选题 Agent 负责判断，产品 Agent 负责拆结构，代码 Agent 负责做到。角色清楚之后，任务自然就更容易推进。

这让我想起一个细节：扣子 3.0 这次强调的“@一下，全队开工”，本质上是在解决真实工作里一个长期被忽略的问题——真实工作很少是一问一答解决的。写一篇稿子、做一个 Demo、剪一条视频、开发一个小应用，都需要多轮推进、多角色协作、不断沉淀中间产物。过去很多 AI 对话更像临时问答，今天问一个问题，明天再问一个问题，中间的资料、角色、产物经常散落在不同会话里。

扣子 3.0 把任务从对话推进到了项目。资料、角色、任务、产物都放进一个项目空间里，围绕同一个目标持续推进。这一点，可能是这次升级里最被低估的地方。

本地 Agent 一键入队：从自己组团队到把老伙计拉进来

更有意思的还在后面。

如果你是一个已经在本地用熟了 Claude Code、Codex CLI 或 OpenClaw 的开发者，扣子 3.0 现在支持把这些本地 Agent 一键导进来。这意味着，这件事从“自己组一个 AI 团队”，变成了“把你原来用顺手的 AI 工具也拉进团队”。

我们实测了一个更工程化的任务：让扣子 3.0 做一个 AI 编辑部桌宠。虽然这类应用很迷你，但它足以考验扣子 3.0 几件事——能不能把一个模糊创意变成清晰产品结构，能不能把好玩和有用结合起来，能不能直接生成可运行代码并说明后续如何扩展。

从效果来看，扣子 3.0 已经做到了从产品设定、交互逻辑、前端页面，一直走到后续工程扩展。Prompt 是这样的（原文未提供完整 Prompt，实测中给出了任务描述），Agent 们自己就把活分了。

真正好用的 Agent，不应该只比单点能力，而要看它能不能进入真实工作流。

这里有一个容易被忽略的细节：扣子 3.0 的编程项目里，支持导入 GitHub 项目或本地项目继续开发，也支持一键部署到线上。这意味着它不是从头造轮子，而是可以接着你已有的代码库继续干活。对于开发者来说，这比“AI 帮你写一个新项目”要实用得多——毕竟真实工作里，大部分时间是在改代码，而不是从零写代码。

视频项目：把资产沉淀下来，而不是每次重新定义

短视频创作的麻烦，在于它从来不是单点任务。脚本、分镜、角色、道具、镜头、字幕、配乐、剪辑节奏，每一个环节都能单独写一大段，但真正难的是它们要互相对齐。

扣子 3.0 这次新增了视频项目。实测中，我们先在编辑部群里 @多个 Agent 生成了一篇文章（让 Agent 自己卷起来，用户只需 pick 最好的结果），然后新建一个视频项目，把文章丢进去，附上一句简单的 Prompt。视频生成项目默认的模型是 Seedance 2.0，也有很多视频风格可选。不到一会，适配编辑部文章的短视频就出来了。

但比“能生成视频”更重要的，是扣子 3.0 的项目化能力在这里的发挥：角色、道具、文档等资产可以在项目里沉淀下来，后续不必每次重新定义。这意味着，如果你是一个持续做视频内容的创作者，你定义的“编辑部吉祥物”角色、“科技感片头”道具、“标准字幕样式”文档，都可以在项目里复用，而不是每次生成视频都从头调教一遍。

这让我想起一个行业痛点：过去一年，AI 视频生成工具层出不穷，Runway、Pika、Sora 都在卷生成质量。但创作者真正的痛点往往不是“生成一段视频”，而是“持续稳定地生成一系列风格统一的视频”。扣子 3.0 的项目化设计，至少在资产沉淀这个环节，给出了一个更务实的解法。

（值得一提的是，Seedance 2.0 是字节跳动自研的视频生成模型，扣子 3.0 把它作为默认模型集成进视频项目，这个“全家桶”式的整合策略，跟 OpenAI 把 DALL-E 集成进 ChatGPT 的逻辑如出一辙。）

手机遥控电脑：Agent 从工具走向协作对象的关键一步

最后一个实测，可能是这次扣子 3.0 升级里最有想象空间的部分。

我们在本地电脑桌面上放了一个英伟达财报的 PDF，然后在扣子 3.0 的手机端，直接让它从电脑桌面读取文件并解读。过去很多 AI 工具最大的断点就在这里：它能给建议，但拿不到你的本地材料；它能写提纲，但不知道你电脑里那份速记到底讲了什么；它能分析趋势，但你还得先把文件上传、整理、复制、粘贴。

桌面端和移动端打通后，Agent 可以在获得授权的前提下处理本地文件，任务也能在手机上继续推进。这意味着，你在地铁上想到一个点子，可以掏出手机让扣子去读你办公室电脑里的一份数据报告，然后直接生成分析结论。任务不再被某一台电脑绑住。

Agent 产品从“能说”走向“能做”，跨端打通是那条看不见的分界线。

扣子 3.0 这次覆盖了手机端（iOS / Android）、电脑端（Mac OS / Windows）、网页端（coze.cn），三端同步更新。这个动作本身比很多人意识到的要重要得多——因为 AI 产品的竞争正在从“谁的模型更强”转向“谁的产品更能融入用户的工作流”，而工作流天然是多端切换的。

我在看到手机遥控电脑这个功能的时候，第一反应是：这不就是 AI 版的远程桌面吗？但仔细一想，区别很大。远程桌面是你手动操作另一台电脑，而扣子 3.0 是 Agent 在获得授权后自动处理文件、推进任务。前者是把屏幕搬过来，后者是把能力搬过去。

行业技能包：不是加几个提示词，而是封装专业方法

扣子 3.0 这次还上线了行业技能包，覆盖自媒体、法律、金融、互联网、医疗健康等场景。

这些技能包不是简单给 Agent 加几个提示词，而是把行业数据库、专业方法和高频工作流封装成可调用能力。比如金融场景里，可以围绕 A 股、基金等任务调用实时数据引擎和专业分析能力；法律场景里，可以检索法律法规和专业内容，辅助合同审阅、诉讼策略材料整理；医疗健康场景里，可以识别体检报告、结构化提取关键信息、整理健康档案。

这意味着，Agent 不只是更会回答，而是能带着更贴近行业经验的能力进入具体任务。对新人来说，它降低了从零摸索的门槛；对专业人士来说，它可以先接住检索、整理、初稿、结构化分析这些高频环节，把人的时间留给真正需要判断和经验的部分。

（我在想，这个技能包的设计思路其实跟企业软件里的“行业解决方案”很像，只是扣子把它做成了可插拔的 Agent 能力模块。字节跳动在 toB 领域的积累，在这里终于找到了一个跟 AI 产品结合的点。）

三层升级：项目化、Agent 编排、工具链打通

把这几组实测放在一起看，扣子 3.0 最重要的升级可以拆成三层。

第一层是项目化。过去 AI 对话像临时问答，资料、角色、产物散落在不同会话里。扣子 3.0 把它们放进一个项目空间，围绕同一个目标持续推进。第二层是 Agent 编排。不是把几个模型摆在一起让用户自己切，而是让不同 Agent 按角色参与任务——选题、资料、产品、代码、本地工程检查，各司其职。第三层是工具链打通。编程项目、视频项目、本地 Agent 接入、行业技能包、多端同步、桌面端本地文件处理，一整套能力覆盖下来，Agent 才能真正干活，而不是只停留在对话框里。

AI 产品的竞争，正在从“谁更聪明”变成“谁更能融入工作流”。

过去一年，行业里有一个很明显的趋势：模型能力在快速趋同。GPT-5、Claude 4、Gemini 3，各家在 benchmark 上的差距越来越小。但到了 Agent 产品阶段，另一个问题开始变得更关键——一个 AI 再聪明，能不能和其他 AI、工具、人类一起工作？

扣子 3.0 给出的答案，是把 Agent 产品做成一个协作系统。它面向的不只是开发者，也包括创作者、复杂业务工作者和新手用户。开发者可以接入 Claude Code、Codex CLI、OpenClaw 等本地 Agent，也可以在编程项目里并行开发；创作者可以把脚本、分镜、视频、音乐和续集放进同一个项目；复杂业务工作者可以调用行业技能包；新手用户则不需要理解 API、MCP 这些概念，也能先从一个项目开始。

这让我想起量子位在报道里引用的那句话：

“87% 用户不懂代码，却用 AI 跑通了一人公司。”

这句话出现在扣子 3.0 的发布会上（据量子位报道），它指向的是一个更底层的趋势：AI 工具的使用门槛正在从“会不会写代码”变成“会不会描述需求”。扣子 3.0 的项目化、Agent 编排、技能包，本质上都是在降低“描述需求”到“拿到结果”之间的摩擦。

如果说早期 AI 产品像一个随问随答的聪明人，那扣子 3.0 想做的，更像一个能开工的小团队。它不一定每个环节都替你做到极致，但它开始让不同 Agent 围绕同一个项目协作起来。这就是这次升级最值得关注的地方。

以后你打开一个项目，@一下，AI 全队开工——这件事，可能比任何单点 benchmark 的提升都更值得期待。
