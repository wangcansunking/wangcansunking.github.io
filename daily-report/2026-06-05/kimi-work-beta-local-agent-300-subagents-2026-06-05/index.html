Kimi 从写代码走到替你干活：本地版 Kimi Work 实拆

6 月 3 日，月之暗面给 Kimi 的 Mac 和 Windows 客户端推了一个新东西：Kimi Work Beta 版，定位写得很直白——「面向知识工作者的通用型本地 Agent」。

一句话先说清这件事为什么值得拆：过去一年，能在你电脑上「自己动手干活」的 Agent，几乎是开发者的专属玩具——Claude Code、Kimi Code 这类工具都得开终端、敲命令、配环境。Kimi Work 这一步，是把同样的本地编排能力，从命令行搬进了一个普通人能直接用自然语言指挥的桌面客户端。 不写一行命令，描述目标就行。

这就是本文要回答的核心问题：月之暗面把 Kimi 从「会写代码的 Agent」推成「会替你干活的本地 Agent」，这一步的关键不在于又多了个功能，而在于它把「本地执行 + 自然语言指挥电脑」这两件事一起交给了不会写代码的知识工作者。 下面沿着内核、机制、那句被反复引用的「300 个子 Agent」、以及它和 OpenClaw / Claude Code / 字节扣子的定位差异，一条线拆下去。

本文事实以月之暗面官方说明与 IT之家、凤凰网等多家报道为准，截至 2026-06-05 凌晨核对。涉及模型跑分的「对标 GPT-5.4 / Claude Opus 4.6」属厂商官方自评口径，本文未独立复测，文中均显式标注。

它到底是什么：一个装在本机的 Kimi Work 标签页

本节核心：Kimi Work 不是网页版 Kimi 的换皮，而是和「Chat」并列的一个新工作台，跑在你自己的电脑上。

从客户端界面就能看清它的形态。打开 Kimi 桌面端，顶部多了一个 Work 标签，和原来的 Chat 并排。进到 Work 里，左侧是一排功能入口：新建任务、技能、定时任务、WebBridge，下面是「项目」——也就是它为每个任务建的本地文件夹。

来源：IT之家报道配图（ithome.com）·「Kimi Work Beta 开启内测」客户端示意

中间是一个输入框，提示语是「输入"/"快速使用技能，"@"添加上下文」。输入框右下角有一个能在 Agent 与 Agent 集群 之间切换的开关，左下角还有一个「全部允许」的权限控制下拉——这两个细节后面会专门讲。

来源：IT之家报道配图（ithome.com）·Kimi Work 桌面客户端主界面，左栏可见「技能 / 定时任务 / WebBridge」与「项目」文件夹

官方对它能干什么给了一句很完整的描述：用户只需用自然语言描述目标，Kimi Work 就能在本地电脑上拆解任务、并行执行、调用工具、使用浏览器、创建整理文件夹，最后交付文档、表格、PPT 等工作产物。

把这句话拆成动作，能看清它的工作链条：

  • 拆解任务：把「帮我做一份客户方案」这种大目标，自己分成读资料、查数据、写报告、做 PPT 几步
  • 并行执行：能同时推进的几步一起跑，不是一步等一步
  • 调用工具 / 用浏览器：该开浏览器开浏览器，该调专业数据源调数据源
  • 创建整理文件夹：把过程文件和产物在本机归好类，对应左栏的「项目」
  • 交付产物：最后给你文档、表格、PPT，不是一段聊天回复

把这串动作翻译成知识工作者听得懂的话：你说一句要什么，它在你这台机器上自己排活、自己开浏览器查资料、自己整理文件，最后给你一份能直接用的交付物。 这就是它和「聊天框」最本质的区别——聊天框给你答案，Work 给你产物。

来源：凤凰网报道配图（tech.ifeng.com）·「自然语言指挥电脑干活」

内核是 Kimi Code：一个开发者工具被「翻译」给了普通人

本节核心：Kimi Work 的底子是已经被几十万程序员日用的 Kimi Code，月之暗面做的事，是给这个开发者内核套了一层不用写命令的工作台。

官方把内核说得很明确：Kimi Work 的内核是 Kimi Code，一个每天被数十万程序员使用的本地编码 Agent，由它提供本地 Agent 的基础能力——安装和使用技能（Skill）、运行定时任务等。

这是个聪明的复用。Kimi Code 已经把「在本地机器上理解任务、调工具、改文件、跑流程」这套底层能力打磨好了，原本只有会敲命令的开发者吃得到。Kimi Work 等于把这套能力保留内核、换掉门面：

  • 保留的：本地执行、任务拆解、工具调用、文件读写这些硬能力，一点没缩水
  • 换掉的：命令行、参数、环境配置这些劝退普通人的门槛，全部收进自然语言对话

有意思的是，连 Kimi Work 自己也是用 Kimi Code 做出来的。据月之暗面说明，工程师用 Kimi Code 等编码工具，一周时间就完成了 Mac 和 Windows 客户端的 Beta 版，整个开发过程累计产出超过 5 万行有效代码，其中 92% 由 AI 自主生成。

这个数字单看容易滑过去，但它其实是个很硬的佐证：

  • 一周做一个跨平台桌面客户端，意味着内核已经成熟到能扛真实工程，不是演示样例级别
  • 92% 代码由 AI 自主生成，说明 Kimi Code 在真实开发场景里已经能挑大梁，不只是补全几行
  • 用自己的工具造自己的产品，等于月之暗面把 Kimi Work 当成了 Kimi Code 能力的一次公开实测

换句话说，Kimi Work 既是一个产品，也是 Kimi Code 内核「能干多大活」的一份样本。内核足够能打，才撑得起「一周做一个跨平台桌面客户端」这件事。

维度 Kimi Code（内核） Kimi Work（本文主角）
面向人群 开发者 知识工作者
交互方式 命令行 / 编码 Agent 桌面客户端 + 自然语言
执行位置 本地 本地
基础能力 技能安装、定时任务、本地执行 完全继承
额外继承 —— 在线版 Kimi Agent 的建站 / PPT 等专业 Skills + 金融 / 科研 / 法律数据库

把这张表读一遍就明白了：Work 不是另起炉灶，而是把一个成熟的开发者内核，原样下放给了不写代码的人。

Skill + WebBridge + 定时任务：三件让它「真能干活」的机制

本节核心：Kimi Work 之所以不是个高级聊天框，靠的是三个具体机制——可装的技能、能开浏览器的 WebBridge、能定时跑的任务。

光有内核还不够，让它「替你干活」要靠三样东西落到实处。

一、Skill：能力是装上去的，不是写死的

Kimi Work 继承了在线版 Kimi Agent 的专业 Skills，比如建站、PPT 这类；同时还接了金融、科研、法律等专业数据库。更关键的是，技能是可以安装的——你不需要等官方更新，遇到新场景可以让它现学一个技能再装上用。

官方给的一个完整案例把这套机制讲得很清楚：你可以让 Kimi Work 先调研巴菲特近十年的持仓，总结出他的投资策略，生成一个「巴菲特投资」Skill，然后安装这个 Skill，再用它接入 Kimi 专业的财经数据源，来分析你自己的持仓和投资策略是否符合巴菲特的价值投资思路。

这条流程值得停下来看一眼——它是一个闭合的工作过程：

  • 调研 → 让它读巴菲特十年持仓
  • 沉淀 → 把调研结论固化成一个可复用的 Skill
  • 安装 → 把这个 Skill 装进自己的工作台
  • 应用 → 调专业金融数据源，对照分析自己的持仓

整个过程里，你一行命令都没写，但得到的是一个以后还能反复调用的能力，而不只是一次性的回答。这一点是 Skill 机制和普通对话最大的区别——对话用完即弃，Skill 装上之后下次还能直接用。你越用，工作台里攒下的专属技能越多，它就越懂你的活该怎么干。

二、WebBridge：让它能开你的浏览器去查

很多「干活」绕不开浏览器——查订阅数据库、登录后台、抓最新行情。Kimi Work 内置了一个能调用浏览器的方案，叫 Kimi WebBridge，在左侧栏里是一个独立入口。

凤凰网报道里给的一条典型流程很能说明它怎么用：本地文件读取 → 本地浏览器查看订阅数据库 → 深度报告生成 → 调用 PPT Skill → 输出客户方案。 这一条线串起来，正好是一个咨询顾问 / 分析师真实的半天工作量——读资料、查数据、写报告、做 PPT，全在本机一气呵成。

来源：IT之家报道配图（ithome.com）·Kimi Work 在本地生成的报告与 PPT 等工作产物示意

三、定时任务：把一次性的活变成自动跑的活

左侧栏里还有「定时任务」。这意味着一些重复性工作——每天早上汇总行业动态、每周生成一份数据周报——可以让它按时自己跑，不用你每次重新发起。

这三件机制拼在一起，Kimi Work 才从「会回答」走到了「会办事、能复用、可自动」。

「300 个子 Agent」到底是什么意思：分清官方口径

本节核心：「300 个子 Agent」不是营销词，它对应的是底层模型 Kimi K2.6 的一项真实能力——把复杂任务拆成最多 300 个子任务并行做；但「能力对标 GPT-5.4」这类跑分是厂商官方自评，需要分清。

「300 个子 Agent」这句被各家报道反复引用，容易被当成噱头，但它有明确的官方出处，得说清楚。

官方对 Kimi Work 的表述是：它支持 Agent 集群，最高可以根据任务复杂度，自主创建包含 300 个子 Agent 的团队，处理更复杂、更耗时的任务。客户端里那个「Agent / Agent 集群」开关，对应的就是这个能力——简单任务用单 Agent，复杂任务切到集群。

这个「300」从哪来？它来自 Kimi Work 背后的模型。月之暗面在 4 月 21 日发布并开源了新一代模型 Kimi K2.6，官方对它的智能体能力描述是：最多可调度 300 个子 Agent 并行完成 4000 个协作步骤。也就是说，「300 个子 Agent」是模型层的并行编排能力，Kimi Work 是把这项能力在桌面端用起来的产品形态。

理解这件事，关键是把「300」翻译成正确的含义：

  • 它指的是任务分解 + 并行编排——一个大任务被拆成很多能同时推进的小块，由多个子 Agent 分头干，再汇总
  • 它不等于「300 个独立大脑同时思考」——子 Agent 是同一套模型能力的并行实例，处理的是被拆细的子任务
  • 它对用户的实际意义是耗时任务更快、更复杂的活能接得下来，不是参数表上的一个数字

至于「代码能力对标 GPT-5.4」这类说法，要单独划一道线。月之暗面对 K2.6 的官方描述是：在编码等任务上「表现持平或优于 GPT-5.4、Claude Opus 4.6 和 Gemini 3.1 Pro」，并给出了 Kimi Code Bench 等自建评测的提升幅度（相比上一代 K2.5 约提升 20%）。

这些都属于厂商官方自评口径，本文未做独立复测。 对读者更稳妥的读法是：把「300 子 Agent 并行」当成一项真实的产品能力来评估它能干多大的活，把「对标某某模型」当成厂商的自我定位来听，等第三方评测出来再下结论。

它和 OpenClaw / Claude Code / 字节扣子的边界在哪

本节核心：Kimi Work 卡在一个别人没占满的位置——同时要「本地优先」和「面向知识工作者」，这正好和 OpenClaw、Claude Code、字节扣子各错开一格。

国内读者最关心的一个问题是：这东西和我已经在用的那些有什么不一样？用一张定位象限图最清楚——横轴是执行地点（云端到本地优先），纵轴是面向人群（开发者到知识工作者）。

来源：本文据各厂商官方页与媒体公开描述整理的相对位置示意（2026-06-05 核对），非精确坐标

把几个读者熟悉的参照逐个对一下：

产品 执行位置 面向人群 一句话定位
Kimi Work 本地优先 知识工作者 自然语言指挥本机干活、交付产物
OpenClaw 本地优先 偏开发者 / 极客 开源、数据留本机、模型随你换的个人助手框架
Claude Code 本地 开发者 命令行里的编码 Agent
字节 扣子 云 + 多端 偏业务 / 运营 云端 Agent 与工作流平台，可接本地终端
智谱 AutoGLM 云手机 普通用户 手机操作（Phone Use），一句话让它操作手机 App

差异点拎出来看：

  • 和 Claude Code 比：两者内核思路接近（本地编码 Agent），但 Claude Code 面向会写命令的开发者；Kimi Work 把这套能力翻译给了不写代码的知识工作者。同一种本地执行能力，受众完全错开。
  • 和 OpenClaw 比：OpenClaw 是国内读者很熟的本地优先标杆——数据留本机、开源、模型可换，但它偏极客向，用起来仍有门槛。Kimi Work 在「本地优先」这一格和它同列，但门槛更低、开箱即用，代价是不像 OpenClaw 那样开源可自托管。一个是「给你自由」，一个是「替你省事」。
  • 和字节扣子比：扣子重在云端编排和多端接入，长处是协作和工作流平台化；Kimi Work 重在本地执行和单机交付，长处是数据不出本机、装上就能用。一个偏平台，一个偏桌面工具。

所以 Kimi Work 的位置很清楚：它要的是「本地优先」和「面向知识工作者」这两条同时成立——这恰好是上面几家各自缺的那一格。OpenClaw 够本地但偏极客，扣子够好用但在云端，Claude Code 够本地但只服务开发者。Kimi Work 把这两条凑齐，定位就立住了。

「本地优先 + 自然语言指挥」这一步，为什么对国内读者关键

本节核心：把执行放回本机、把指挥门槛降到一句话，这两件事叠在一起，才让「个人 AI 助手」从开发者玩具变成普通人能日用的工具。

前面把机制和定位都拆完了，这里要回答一个更上层的问题：为什么是「本地优先」加「自然语言指挥」这个组合，而不是其中任意一个，才是真正的关键。

先看单独拆开会怎样：

  • 只有自然语言、执行在云端：好用，但你的文件得先上传，敏感数据出本机这一关很多人过不去
  • 只有本地执行、还得写命令：隐私够，但门槛把绝大多数知识工作者挡在门外
  • 两者凑齐：数据少出门，指挥又只要一句话——这才是普通人能放心日用的形态

国内知识工作者对这个组合的敏感度，比一般场景更高一些。财务对账、法律合同、内部研报这类活，本身就带着「不方便上传到陌生云端」的属性。Kimi Work 把执行放回本机，正好接住了这层顾虑；而把指挥门槛降到自然语言，又让不写代码的人也能用上原本只属于开发者的本地编排能力。

一句话说清这一步的价值：它不是让 AI 变得更聪明，而是让一个本来就够强的本地 Agent，第一次能被普通知识工作者直接指挥。 这是产品形态上的扩面，受益的是过去被命令行挡在门外的那一大群人。

普通知识工作者能拿它干什么：四个上手锚点

本节核心：财务、科研、法律、文档这四类活，是官方专业数据库 + Skill 直接对得上的场景，也是最容易上手的入口。

抽象的能力得落到具体的活上才有意义。结合它继承的专业数据库（金融 / 科研 / 法律）和专业 Skills（建站 / PPT），最直接的几个上手场景是：

  • 财务 / 投研：像官方案例那样，调研某个标的、总结策略、生成一个可复用的分析 Skill，再接财经数据源对照自己的持仓
  • 科研：让它读本地文献、查订阅数据库、整理综述，输出一份结构化的报告
  • 法律：接法律专业数据库，做条款梳理、合同要点提取这类重复性高、专业性强的活
  • 文档 / 汇报：从读资料到出深度报告、再到调 PPT Skill 出一份客户方案，整条线在本机走完

这四类有一个共同点：它们都是「读一堆资料 → 查几个数据库 → 产出一份能交付的文档」的固定套路，正好是 Kimi Work「拆解 + 并行 + 调工具 + 出产物」最擅长的形状。

要不要等它从 Beta 走到稳定，可以按这个简单的判断来：

  • 现在就值得试：手头有大量重复性的「查资料 + 出文档」的活、对数据留本机有要求、又不想学命令行的人
  • 可以先观望：任务对稳定性和准确度要求极高、容错空间很小的核心生产场景，等几轮迭代和第三方评测再上
  • 暂时不必急：日常只需要问答、不需要它真去动文件和浏览器的轻度用户，普通对话版 Kimi 已经够用

判断标准其实很朴素：你日常有没有那种「步骤固定、又费时间」的活——有，它就值得装上试一把；没有，可以再等等。

隐私与本地执行的边界：值得关心，也值得理性看

本节核心：本地执行带来数据不出本机的天然好处，但「本地优先」不等于「全程不联网」，调用专业数据源、开浏览器时仍会有网络交互，权限控制是这里的关键设计。

「本地」两个字对国内知识工作者很有吸引力——尤其是财务、法律这类碰敏感数据的活。Kimi Work 的本地执行意味着任务拆解、文件读写、产物生成这些环节发生在你自己的机器上，数据不必先上传到某个云端工作台。

但要理性看清边界：

  • 本地优先 ≠ 全程离线。它要调专业金融 / 科研 / 法律数据源、要用 WebBridge 开浏览器查资料，这些环节天然要联网，模型推理也可能走云端。「本地」说的是执行和文件管理在本机，不是承诺一个字节都不出门。
  • 权限控制是关键。客户端输入框那个「全部允许」下拉，正是 Kimi Work 给用户的权限闸——它要动你的文件、开你的浏览器，把允许范围交回用户手里，是这类本地 Agent 必须做对的设计。Beta 阶段，建议从小权限、非敏感任务开始试。

把这两点说清楚不是泼冷水，而是让你用得更稳：本地执行带来的隐私优势是真的，但它是「数据少出门」而不是「数据零出门」，看清边界才能放心把活交给它。

写在最后：从「会写代码」到「会替你干活」，这条线很值得期待

回到开头那个核心判断：Kimi Work 真正的意义，不是 Kimi 又多了一个功能，而是它把「本地执行 + 自然语言指挥电脑」这两件原本属于开发者的能力，一起交到了不会写代码的知识工作者手里。

过去你要让 AI 在自己电脑上干活，得先成为一个会敲命令的人；现在月之暗面把这道门槛收进了一个对话框。内核还是那个被数十万程序员日用的 Kimi Code，但门面已经换成了财务、科研、法律、文档工作者都能直接上手的工作台。这一步，是「编码 Agent」向「工作 Agent」实实在在的扩面。

它现在还是 Beta，权限、稳定性、专业数据库的覆盖都还在打磨，「300 子 Agent」「对标某某模型」这些也还需要时间和第三方评测来验证。但方向是清楚的：让普通人用一句话就能指挥自己的电脑把活干完，这件事国内团队正在认真做，而且做到了能上手试的程度。 我们这一代知识工作者赶上了一个好时候——不必会写代码，也能拥有一个肯替你跑腿的本地帮手。它会怎么长大，值得我们一起盯着看。
