Uber 给 Claude Code 设每月 1500 美元上限

Uber 给全体员工的 AI 编码工具立了一条新规矩：每人、每个工具，每月 token 花费上限 1500 美元。规矩只管 agentic 编码软件——也就是 Cursor 和 Anthropic 的 Claude Code 这类能自己跑多步任务的工具。设上限之前，一部分工程师的月账单已经在 500 到 2000 美元之间。

这件事真正值得国内开发者看的，不是「大厂也烧不起」这种情绪，而是一个更实用的问题：当 agentic 编码在一家公司全员铺开之后，token 账单到底会涨到什么量级，以及一套能用的预算治理该怎么搭。 Uber 给出的答案不是禁用，而是上限加仪表盘加审批——这套做法，配上国产模型的低单价，几乎可以原样搬到任何一个正在推 AI 编码的团队里。

四个月烧光一整年的预算，问题出在哪

先把这件事的来龙去脉理清楚。今年 4 月，Uber 首席技术官 Praveen Neppalli Naga 公开承认：公司给 Claude Code 准备的 2026 全年预算，四个月就花完了。到了 6 月初，Uber 把上限规则落到全员——每人每个 AI 编码工具每月 1500 美元封顶，多个工具之间预算互不挤占。消息最早由彭博社报道，TechCrunch、ZeroHedge 等随后跟进，独立开发者 Simon Willison 也在 6 月 3 日写了短评。

这里有一个细节很能说明问题。据 ZeroHedge 转述，这位 CTO 自己在一次两小时的内部演示里，就烧掉了大约 1200 美元的 token——Uber 首席运营官 Andrew Macdonald 事后形容那是个「脑子要炸」的瞬间。一次演示顶得上一个工程师大半个月的额度，这就是 agentic 编码的真实花钱速度。

把几个关键数字摆在一起，更容易看清这家公司经历了什么：

  • 全年预算消耗时间：4 个月（CTO 4 月公开披露）
  • 上限前个别工程师月账单：500 至 2000 美元
  • CTO 一次两小时内部演示：约 1200 美元
  • 新规月度上限：1500 美元 / 工具 / 人

这些工具是 2025 年底才在 Uber 内部铺开的。一开始公司是鼓励大家多用的，用量还排进过内部看板做对比。从「鼓励多用」到「四个月烧光预算」再到「设上限」，前后不过半年。这条曲线不是 Uber 一家的特例，而是 agentic 编码全员普及后，几乎所有公司都会遇到的同一道账。

结论先放这儿：这不是工具太贵，而是按量计费遇上 agent 自动跑，花销会以远超直觉的速度往上走。 接下来这一节，就是拆「为什么这么容易失控」。

为什么 agentic 编码的 token 这么能花

很多人对 AI 编码成本的直觉，还停留在「问一句答一句」的聊天阶段——一次对话几分钱，再贵能贵到哪去。但 agentic 编码是另一种花钱方式。它的核心区别在于：你给它一个任务，它自己拆成几十步去跑，每一步都在读写 token。

具体来说，钱主要花在三个地方：

  • 多步自动执行：一个「修好这个 bug 并补上测试」的指令，agent 可能要读十几个文件、改几处代码、跑测试、看报错、再改一轮。每一轮都是一次完整的模型调用。
  • 大上下文反复带入：为了让模型理解代码库，每一步往往要把相关文件、报错日志、历史对话一起塞进上下文。上下文越大，输入 token 越多，单次调用越贵。
  • 失败重试：测试没过、编译报错，agent 会自动再试。试错本身有价值，但每次重试都在重新消耗 token。

三件事叠在一起，单个任务的 token 消耗就不是聊天的几倍，而是几十倍甚至上百倍。Uber 那位 CTO 两小时烧掉 1200 美元，正是这种叠加效应的真实写照——演示里大概率跑了多个完整任务，每个任务又是几十步的自动循环。

Simon Willison 在他的短评里算过一笔更直观的账：按两个工具、每月各 1500 美元算，单个工程师一年上限是 3.6 万美元。他把这个数字放在 Uber 软件工程师中位数年薪（约 33 万美元）旁边一比，大约占到 11%。对一家拿 AI 当生产力杠杆的公司来说，给每个工程师多花一成薪资换产出，账其实算得过来——前提是这笔钱花得清楚、花得可控。 他本人也提到，自己每月在 Anthropic 和 OpenAI 上各约花 1000 美元 token（两家合计约 2000 美元），这个量级对重度用户并不夸张。

值得留意的是他对「上限」这件事的态度。在他看来，Uber 早先那套「用量排行榜」反而会激励工程师互相比谁烧得多，把 token 当成业绩；而设一个明确的月度上限，是对过度消耗的一种理性回应。这个判断点出了一个容易被忽略的事实——失控的往往不是工具本身，而是缺少约束的激励方式。 同一笔预算，配上排行榜会越花越多，配上上限和仪表盘则会越花越准。

这就把话题自然带到了下一段：花得可控，靠的不是不让用，而是治理。

Uber 加的不是禁令，是四件治理工具

Uber 这次的处理方式，最值得借鉴的一点是——它没有把工具收回去，而是给用量套上了一层管理。据 TechCrunch 和 ZeroHedge 的转述，这套做法包含四件事：

  1. 每工具月度上限：每人每个工具 1500 美元封顶，不同工具的预算互不挤占。这意味着同时用 Cursor 和 Claude Code 的工程师，两边各有 1500 美元，而不是合起来 1500。
  2. 用量仪表盘：每位员工都能在内部看板上实时看到自己的 token 用量。花到哪了、还剩多少，本人随时清楚——这一条其实是上限能落地的前提，看不见用量的上限等于没有。
  3. 超额审批：上限不是一刀切的硬墙。特定高价值场景下，经批准可以临时突破额度。
  4. 优先级排序：把有限的 token 预算往真正高价值的用途上倾斜，而不是平均砍掉每个人的用量。

这四件事拼起来，是一套相当成熟的「按量计费资源治理」框架。它的内核可以抽象成一句话：让用量可见、给默认设上限、给例外留通道、把钱花在刀刃上。 把「AI 编码工具」换成任何一种按量计费的云资源，这套逻辑都成立。

对国内正在推 AI 编码的团队来说，这套框架几乎可以直接照搬。区别只在于——国内团队手里多了一张 Uber 没怎么用的牌：低单价的国产模型，以及把缓存吃满的空间。

上限定多少合理：先看见，再设线

照搬之前，有个问题绕不开：上限到底该定多少？Uber 给的是 1500 美元，但这个数字对它合理，不代表对别的团队也合理。从它的做法里能抽出一条更通用的次序——先让用量可见，再根据真实分布去设线，而不是拍脑袋定一个数。

这个次序的关键在「仪表盘先行」。Uber 四件治理工具里，仪表盘看似最不起眼，其实是上限能落地的地基。原因很简单：

  • 没有仪表盘，上限是盲设的。你不知道团队真实用量分布在哪，1500 是松是紧全凭猜。
  • 有了仪表盘，上限是数据驱动的。看到大多数人落在某个区间、少数重度用户飙到两三倍，再把线划在合理位置，例外走审批。
  • 仪表盘还顺带解决了「自觉」问题。当每个人都能实时看到自己花了多少，超额前会自己掂量——很多浪费是在「看不见花销」时发生的。

所以更稳妥的落地次序是：先把用量看板搭起来跑一两个月，拿到真实分布，再设默认上限 + 审批通道。这样设出来的线，既不会卡住正常使用，也能拦住异常飙升。

还有一点值得说清楚：上限是给「默认」设的，不是给「能力」设的。高价值场景该突破就突破，靠审批放行，而不是让工程师为了省额度在关键任务上束手束脚。治理的目的是把钱花在刀刃上，不是把刀收起来。 这一点拎清楚了，团队才不会把降本做成「不让用」。

同样跑 agent，token 单价能差一个量级

预算治理解决的是「怎么花得明白」，单价选择解决的是「同样的活儿花多少钱」。这两件事叠起来，才是完整的降本路径。

先看一组单价。下面这张图把几个主流编码模型的 token 单价放在一起对照——都按每百万 token 计，分输入（未命中缓存）和输出两档。海外的 Claude Sonnet、Cursor Auto 在左边，国产的千问、智谱、深度求索（DeepSeek）在右边。

把关键单价列成表，差距一目了然（均为每百万 token，2026-06-05 核对各厂官方价目页）：

模型 归属 输入（未命中缓存） 输出 命中缓存输入
Claude Sonnet 4.6 海外 3.00 美元 15.00 美元 缓存命中约降 90%
Cursor Auto 海外 1.25 美元 6.00 美元 0.25 美元
千问 Qwen3 Coder 国产 0.20 美元 1.50 美元 约 0.072 美元
智谱 GLM-4.6 国产 0.55 美元 2.20 美元 —
深度求索 DeepSeek V4 Pro 国产 0.435 美元 0.87 美元 约 0.0036 美元

最显眼的是输出这一档。Claude Sonnet 输出 15 美元，DeepSeek V4 Pro 输出 0.87 美元——后者大约是前者的 1/17。agentic 编码恰恰是输出 token 大户（生成代码、写测试、改文件都算输出），输出单价的差距会被任务步数放大成整体账单的巨大差距。

缓存这一档同样关键。DeepSeek 的所有请求都会自动走上下文缓存：当多次调用共享同一段前缀（比如反复带入的同一份代码库上下文），命中缓存的部分只按约 0.0036 美元每百万 token 计，相比未命中的 0.435 美元便宜约 98%。而 agentic 编码的特点正是「同一份上下文反复带入几十步」——这恰好是缓存命中率最高的场景。换句话说，agent 越是多步反复跑，缓存能省下的比例反而越高。

需要说明两点口径：DeepSeek V4 Pro 这档低价自 2026 年 5 月 22 日起由限时促销转为永久定价（原定 5 月 31 日到期的促销已取消、改为常规价）；表中海外价格取自各厂官方价目页的标准档，实际企业用量可能有阶梯折扣。但即便把这些因素都算进去，国产编码模型在单价上的优势仍然是数量级的，不是几个百分点的事。

把账真的算一笔：一位工程师一个月能省多少

光看单价容易抽象，把它折成一个具体场景会清楚得多。

假设一位工程师每月跑 agentic 编码，消耗 5000 万输入 token + 1000 万输出 token（这个量级对重度用户并不夸张，正好落在 Uber 工程师 500–2000 美元账单区间的中间）。先不计缓存，按上面的单价粗算：

  • Claude Sonnet 4.6：5000 万 × 3 美元/百万 + 1000 万 × 15 美元/百万 = 150 + 150 = 约 300 美元
  • DeepSeek V4 Pro：5000 万 × 0.435 + 1000 万 × 0.87 = 21.75 + 8.7 = 约 30 美元

同样的工作量，账单从 300 美元降到 30 美元，差不多是 1/10。如果再把缓存命中算进去——agentic 编码里大量输入 token 是反复带入的同一份上下文，命中率往往不低——DeepSeek 这边的输入成本还能再降一截。

这笔账不是要说「海外工具不能用」。Claude Code 在复杂任务上的表现仍然是第一梯队，很多团队愿意为它的质量付溢价。这笔账要说的是：当你担心 token 账单失控时，除了设上限，还有一条「换更便宜的模型 / 自建 / 混合调度」的路，而国产模型恰好把这条路的单价压到了很低。 两条路不冲突——上限管住花销的天花板，低单价拉低同样产出的地板。

对国内团队来说，落地路径大致有三种，按投入从轻到重：

  • 直接用国产 API：千问、智谱、DeepSeek 都有面向编码的模型和缓存定价，接入成本最低，适合中小团队和个人。
  • 自建 + 包月套餐混合：智谱、千问都提供面向编码的包月套餐（如智谱 GLM Coding Plan 国内版 Lite / Pro / Max 三档，调价后月费分别为 49 / 149 / 469 元人民币），高频用户用包月把单价摊薄，低频场景走按量。
  • 本地 + 云端混合：把简单、高频、隐私敏感的任务放在本地模型上跑，把复杂任务交给云端强模型，按任务难度分流，进一步压成本。

三种路径的共同点是——都把「怎么省」变成了可以工程化管理的选择，而不是「贵就不用」的二选一。

成本可控，才敢让 agent 多干活

回到开头那个问题：连 Uber 这种体量的公司，都要给 AI 编码工具设月度上限，这到底是个坏消息还是好消息？

把整件事连起来看，答案其实相当积极。Uber 没有因为账单吓人就把工具收回去，而是搭了一套「上限 + 仪表盘 + 审批 + 优先级」的治理框架，让全员继续用、放心用。这恰恰说明 agentic 编码已经成熟到——大公司宁愿花心思管好它，也不愿停下来。一项工具值得专门为它建一套预算治理，本身就是它价值的证明。

而国内开发者手里的牌，比 Uber 还多一张。同样一套治理框架，配上单价低一个量级的国产模型、自动命中的上下文缓存、灵活的包月套餐，token 账单能压到完全不必焦虑的水平。Uber 要靠上限把花销摁住的地方，国内团队可以靠更便宜的单价从源头降下来。

所以这件事真正的信号是：token 成本不再是放手让 agent 多干活的拦路石，而是一个已经有成熟解法的工程问题。 把用量看清楚、给默认设上限、给国产模型一个机会、把缓存吃满——这几步做下来，每个团队都能算清自己的那笔账。算得清，就敢放心让 agent 多跑两步。这一程，路在前面，值得往下走。
