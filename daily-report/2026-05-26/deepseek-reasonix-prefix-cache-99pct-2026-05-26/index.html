DeepSeek Reasonix 99.82% 缓存命中：Claude Code 风格长会话从 61 美元跑到 12 美元，国内开发者直接受益

30 秒速览

中国独立开发者 esengine 写的一个终端编码 agent 叫 Reasonix，过去一天在 Hacker News 拿到 663 分、266 条评论冲上首页头部，GitHub 仓库已经 8.2k 星。它最反常的一个数字是缓存命中率 99.82%：一个普通的全职开发者一天发出 4.35 亿 token，如果按 DeepSeek 的标准价应该花约 61 美元，开了 prefix cache 之后实际付 12 美元。这是 5 倍降本。

它的工具形态和 Anthropic 家的 Claude Code 几乎完全一致：终端里跑、一行命令启动、能看代码能改代码能调工具。差别在两端：上游模型挂的是 DeepSeek V4 Pro / V4 Flash 而不是 Sonnet 4.6；架构上 Reasonix 全程围绕 prefix cache 字节级稳定性做减法，强行让上下文以追加形态增长，永不修改、永不重排——所以 99% 以上的 token 都能复用上一轮发过的，按 1/10 的价格收费。

对国内开发者来说更直接：DeepSeek API 国内能正常用、能用人民币付款、价格本来就比 Claude / GPT 便宜一个数量级，再叠加 Reasonix 这层缓存逻辑，等于把 "Claude Code 体验" 用一份饭钱的价格搬到自己终端里。Hacker News 评论区里 5 位陌生开发者各自用 OpenCode、Codex 桥、Reasonix 测了一圈，命中率全部在 95% 以上，不是孤证。

一个独立中国开发者怎么把 Hacker News 头版做下来

Reasonix 这个项目 5 月初首次在 GitHub 出现，没有公司背书，没有融资稿，作者 esengine 在 GitHub 上的个人页只挂了一句「今天是个好日子」（today is a good day），225 个关注者，主业看起来是 TypeScript 游戏框架——他另外几个仓库都是 ECS 游戏引擎、行为树 AI、WebAssembly 2D 引擎之类。Reasonix 是他第一次做大模型方向的工具。

但他选的切入角度非常聪明：不做模型、不做新协议、不做 IDE 插件，只做一件事——把 DeepSeek 已经存在了大半年的 prefix caching 特性吃到极致。HN 上的提交标题非常朴素，叫「DeepSeek 原生的编码 agent，缓存命中高、成本低」（DeepSeek native coding agent with high caching and low cost）。评论区一开始是质疑居多，几个老 harness 开发者上来就挑刺，说 "上下文稳定就为了缓存这是常识，不是创新"。后来真的有人贴出自己上周用 Reasonix 的账单截图，数据非常硬核：一位叫 stavros 的用户报了 4.72 亿命中 token / 1330 万未命中 token、命中率 97.27%；另一位 metalspot 在 V4 Flash 上测出 98.6% 的命中率；esengine 在 README 里贴的那个 99.82% 也不再像营销话术。

帖子热度从 200 分迅速爬到 663 分，挂在 HN 首页头部超过 24 小时。GitHub 星数同步从两千多冲到 8200+。一个独立中国开发者的开源项目用一个看似平凡的工程优化击中了海外重度 LLM 用户的真实痛点——他们手里那张每月 API 账单。

DeepSeek 的 prefix caching 到底特殊在哪里

要看明白 Reasonix 这次为什么能 5 倍降本，得先看懂 DeepSeek API 那套缓存价目。

DeepSeek 现在主推两款模型：V4 Flash 和 V4 Pro。两款的输入价目都分两档——cache-hit 输入和 cache-miss 输入。V4 Flash 命中价是每百万 token 0.0028 美元，未命中价是 0.14 美元，差了整整 50 倍。V4 Pro 在 5 月底前的促销价里命中是 0.003625 美元、未命中是 0.435 美元，差距 120 倍。促销结束后命中价回到 0.0145 美元、未命中价 1.74 美元，差距依然有 120 倍。

也就是说，DeepSeek 这套定价系统天然鼓励你把请求结构做得"前缀稳定"。每次发请求时 API 会自动检测：本次发送的 prompt 前 N 个 token 如果和你最近某一次请求的前 N 个 token 字节级完全一致，那这一段就按命中价收钱；从第 N+1 个 token 起的增量部分按未命中价收钱。

这套机制 OpenAI 也有，但 OpenAI 的缓存价折扣大概是 50%——你命中了，省下的也就是一半。DeepSeek 这一刀切到 1/50 ~ 1/120，相当于说："只要你愿意把上下文结构做得严谨，token 钱基本就免了。"

DeepSeek 在 4 月底的一次价格调整里把命中价直接降到了原来的 1/10，时间点是 2026 年 4 月 26 日。这次调整等于把缓存奖励放大到了离谱的程度——Reasonix 的诞生几乎可以算作对这次价格调整的直接回应。

99.82% 是怎么做到的：一个永远只追加的 agent loop

理论说完了，Reasonix 在代码层面是怎么把命中率推到 99.82% 的？看 README 和源代码注释，核心设计可以归纳成三条：

第一条：append-only loop。 大多数 coding agent 在多轮对话里会做"上下文压缩"——比如把前几轮的工具调用结果折叠成摘要、把超过窗口的部分裁掉、把代码块改成索引引用。这些操作每一次都会让缓存失效，因为前缀字节变了。Reasonix 反其道而行：能不动就不动，能追加就追加。哪怕代价是单次请求的 token 数膨胀到几十万，因为膨胀的部分 99% 是缓存里已有的，单价是 1/50 ~ 1/120，钱并不多花。

第二条：固定系统提示 + 固定索引 + 增量用户输入。 一个典型的 Reasonix 会话长这样：开头 30k token 是系统提示词加上整个仓库的语义索引，第一轮用户问"帮我重构 auth.ts"——这两部分加起来是首发，按未命中价付。第二轮用户继续问"再加上类型守卫"，Reasonix 把第一轮的全文（系统提示 + 索引 + 用户问 + 模型答）原封不动重发，只在结尾追加新的一句——前面 35k token 全部命中，新增的几百 token 按未命中价付。第三轮再追加，前面 40k token 全部命中。一直滚到几十轮以后，单轮请求里 99% 以上的 token 都是上一轮发过的字节完全一致的副本。

第三条：tool call 修复机制。 agent 跑工具调用时最容易把缓存毁掉——某个工具失败了、某个返回值格式变了、模型自己回退重试，每一次都会破坏前缀。Reasonix 在工具层做了一层 idempotent 重写：失败不删，错误不改，全部以"原本调用 + 修正记录"的形式追加在历史里，这样前缀永远是只增不减。

这套设计有一个反直觉的副作用：单次请求的 token 量越长，缓存命中比例越高，单 token 平均成本越低。普通 agent 跑 10 轮以后 token 越长越贵，Reasonix 跑 50 轮以后 token 越长反而越便宜。esengine 在 README 里管这叫「挂在那里跑就行」（leave it running）——你随便挂着跑一整天，不用心疼账单。

4.35 亿 token / 12 美元的账怎么算

把这套机制放回那个 12 美元 vs 61 美元的对比上：4.35 亿输入 token、单日、单开发者。

按 V4 Pro 促销价算：

  • 不开缓存：4.35 亿 × 0.435 美元 / 百万 = 189 美元。这是上限。
  • 真实场景里没有人完全不命中，正常的 agent 调用大概有 30% ~ 50% 命中率，预期总价大约 60 ~ 100 美元。esengine 公示的 61 美元就在这个区间。
  • 99.82% 命中（Reasonix 实测）：4.35 亿 × 99.82% × 0.003625 + 4.35 亿 × 0.18% × 0.435 = 1.57 + 0.34 ≈ 1.91 美元的输入成本。剩下的钱是输出 token：按 0.87 美元 / 百万估算，如果输出 1300 万 token（与输入相比的常见比例），是 11.3 美元。合计约 13 美元，和 12 美元的实测吻合。

对照看一下海外几款主流的同档工具，账面差距非常大：

Claude Code（API 直跑）：同样 4.35 亿 token 量级的负载，按 Sonnet 4.6 的 3 美元/百万输入、15 美元/百万输出来算，即便 Anthropic 自己也有 prompt caching（命中读取 0.5 美元/百万，相当于 1/6 的折扣），一天烧个 80 ~ 100 美元是常态。一线公司给 Claude Code 重度用户的预算大约是每天 13 ~ 30 美元、每月 150 ~ 250 美元，但那只是普通频率的开发者；想跑 esengine 公示的那种压力，月支出会很容易冲到 450 美元以上。

Cursor Pro（20 美元月费）：Cursor 自从 2025 年 6 月改成"月费 = 月度信用额度"以后，那 20 美元里能换的算力对应大约 1500 ~ 2500 万 token 的前沿模型用量。也就是说，Reasonix 用户一天的负载，Cursor Pro 用户得攒 20 ~ 30 天才能用完。超过额度有两个选择：升 Pro+（60 美元/月）或开按量付费。重度用户的实际月支出经常飙到 100 ~ 200 美元。

千问3-Coder 30B（本地）：本地路径完全不付 API 钱，但要付电费、显卡折旧、上下文窗口的硬约束。一台 RTX 4090 整机跑一整月，电费摊到一位全职开发者头上大约 75 美元/月——账面上比所有云方案便宜，代价是上下文长度受限、模型规模比 V4 Pro 小一个数量级，复杂任务上的"一次跑通率"会差不少。

把这四条放一张图上对比，Reasonix + DeepSeek 这条线大约在每月 360 美元的位置——比 Cursor 贵、比 Claude Code 便宜得多、比千问本地贵得多。它的位置非常微妙：你愿意付一点点钱换一个比本地大一个数量级的模型、比 Cursor 宽得多的上下文、比 Claude Code 便宜 5 倍以上的账单。

Claude Code 为什么没有同档的缓存

读到这里很多人会有一个疑问：既然 prompt cache 是 LLM API 都有的标配，为什么 Anthropic 不能在 Claude Code 里也做出 Reasonix 这种 5 倍降本？

答案在三个层面。

模型定价策略不同。 Anthropic 的 Sonnet 4.6 缓存读取价是 0.5 美元 / 百万、未命中价是 3 美元 / 百万——差距 6 倍。DeepSeek V4 Pro 是 0.003625 美元 / 百万 vs 0.435 美元 / 百万——差距 120 倍。同样命中 99%，DeepSeek 能省 99%、Anthropic 只能省 80% 左右。这是定价层的天花板。

模型自身决策。 HN 评论里有一位写了一年 harness 的 jbellis 提了个非常关键的角度：很多时候模型自己会主动打破缓存，因为它在内部测试中发现"重置上下文 / 重排提示词 / 强行刷新工具列表"能让输出质量更高。Anthropic 的 Sonnet 在很多场景下会主动拒绝复用缓存，宁愿多收点钱换准确度。这是一种工程取舍：是把成本压到极致，还是把"一次跑通"的概率拉满。Claude 选择后者。

生态绑定差异。 Claude Code 的目标用户是企业付费市场，月均 13 美元 / 活跃开发者 / 天的成本是被市场认可的，没有强动力做激进降本。Reasonix 服务的是个人开发者、独立工程师、初创团队——他们对每月 50 美元和每月 250 美元的差异极其敏感，愿意为了省钱接受更激进的工程取舍。

这是不同生态的选择，不存在绝对的好坏。但对一个国内独立开发者来说，今天能用 12 美元跑通 Claude Code 形态的体验，本身就是一个非常具体的好消息。

国内开发者怎么把它跑起来

Reasonix 当前支持三类终端：macOS、Linux、Windows 上的 PowerShell / Git Bash / Windows Terminal。安装两条命令搞定：

npm install -g reasonix
reasonix code my-project

或者用更短的别名 dsnix：npm install -g dsnix && dsnix code。一次性试用可以走 npx reasonix code 不装全局。

接下来要做的是配 DeepSeek API key。DeepSeek 官网用支付宝/微信都能充值人民币、按用量扣，国内常规网络环境就能用，不需要任何代理。新账户充 100 元人民币（约 14 美元）按 esengine 公示的命中率粗算够用 10 天左右。

Reasonix 的设计哲学是"挂在那儿跑"，所以推荐配合 tmux / WezTerm / Warp 之类支持长会话的终端，关掉自动 sleep。esengine 在 README 里贴了一张 hero 截图，是一个分屏的终端：左侧是 Reasonix 在跑长会话，右侧是 dashboard 实时显示 token 使用量和命中率。社区里有人吐槽这个项目的官网做得太花哨（动画排版、对比度差），但终端 UI 本身的可读性很高。

另一类玩法是把 Reasonix 当 MCP 服务端挂起来，让其他 IDE 通过 MCP 协议复用它的缓存。这是社区里一位开发者在 PR 里加进去的，作者收得很快。这意味着你的 VS Code、Zed、Sublime 都可以通过同一个 Reasonix 进程共享缓存，全公司 5 个开发者共享一个 cache 池——账单还能再压下去一个量级。

编辑说

Reasonix 这个项目本身工程量不大——按 esengine 自己提交记录看，主要代码是过去 30 天写完的，一两千行 TypeScript。但它代表的趋势比代码本身重要：当海外的主流 AI Coding 工具都在向上卷模型规模、卷推理质量、卷上下文长度时，一位中国独立开发者从下游 API 的定价缝隙里找到了一个 5 倍降本的工程位置。它不依赖国家政策、不依赖大厂资源、不依赖芯片自研，只依赖 DeepSeek 已经公开了大半年的一个 API 特性 + 一个肯把"前缀字节稳定"这件小事做到极致的工程意志。

它对国内开发者的直接价值很简单：一个 12 美元一天的 Claude Code 替代品，今天就能跑起来，账单付人民币。HN 头版的关注度只是一个外部确认信号，真正的故事是中国开发者社区里这种"把别人定的规则吃透到极限"的工程审美——它和今年 1 月 DeepSeek 自己用便宜 GPU 跑出 R1、和上海 AI Lab 用消费级显卡训出有竞争力的模型、和魔塔上一波又一波小规模高质量的开源工作——是同一种东西。

愿意把工程做扎实的人，永远不会因为算力不够而被困死。
