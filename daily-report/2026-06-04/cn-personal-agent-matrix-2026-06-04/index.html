国产个人 AI 助手能替你干完活了吗：四款对照 OpenClaw 横评

过去半年最让人惦记的一类工具，不是又一个聊天框，而是能真替你在手机和电脑上把事办完的个人 AI 助手。海外这条线的标杆是 OpenClaw——它一个开源仓库两周就冲到十几万星、如今已超过 37 万，做的事就一句话：装在你自己机器上，跨过 App 边界，把活干完，而不只是陪你聊天。

那么问题来了：这条线，国产做到哪一步了？

先把结论放在前面：国产个人 AI 助手已经不是「能不能办事」的问题，而是「在哪儿办、连不连你的数据、要不要把权限交到云端」的路线分叉。 智谱 AutoGLM 把手机操作做成了能跑几十步的真功能，字节扣子把云端 Agent 和你本地终端打通，MiniMax 和阿里千问则各自押注云端工作流。但和 OpenClaw 那种「数据全留本地、模型随你换、仓库自己托管」的彻底本地优先比起来，国产这几款的隐私本地化程度还是另一条路子。

这篇就沿着「能不能替你干完活」这一条线，把四款确实能用的国产产品逐个核一遍能力边界，再和 OpenClaw 摆到一张表上对照。

横评对象：四款国产 + 一款海外对照

挑选标准只有一条：这个产品现在真能下载、真能让它去执行任务，不是 PPT、不是内测排队码。 按这条筛，最终留下四款国产，对照一款海外。

国产四款（按厂商）：

  • 智谱 AutoGLM（智谱）——主打手机操作，一句话让它在手机里跑外卖、订票、查房源
  • 字节 扣子 Coze（字节跳动）——Agent 与工作流平台，能把本地命令行工具拉进同一个项目
  • MiniMax Agent（MiniMax）——分层多 Agent 架构，云端跑编程、设计、调研
  • 千问 App（阿里，前身通义 App）——阿里官方个人助手，定位「会聊天、能办事」

海外对照一款：

  • OpenClaw——本地优先的个人助手框架，数据留在本机，模型随便换，是这条线的世界级参照

这里有一款值得特别说明：Manus。它曾是国内最火的通用 Agent 之一，但其总部已于 2026 年 5 月迁至新加坡，与阿里通义的合作公告随之删除，目前中国地区无法访问官网。所以本文把 Manus 作为「这条赛道曾经的国内标杆」做背景提及，不列入「现在能用」的横评对象。

来源：OpenClaw 官方 GitHub 仓库 github.com/openclaw/openclaw 项目页

OpenClaw 这个仓库 2025 年 11 月底建立，截至本文写作时星标已超过 37 万，是这条赛道最直观的热度坐标。它的定位写在仓库描述里：你自己的个人 AI 助手，任何操作系统、任何平台。我们就拿它当那把尺子。

第一条线：本地直接执行——AutoGLM 把手机操作做成了真功能

这一节的核心：在「让 AI 直接动你的手机 / 电脑」这件事上，智谱 AutoGLM 是国产里做得最实的一个。

个人 AI 助手和聊天机器人最大的区别，就是它得能动你的设备。这件事最难，因为它要看懂屏幕、要点对按钮、要在几十个步骤里不出错。

AutoGLM 走的是手机操作（Phone Use）路线。按智谱官方文档，它的工作方式是「多模态理解屏幕内容 + 通过 ADB 自动控制设备」——你说一句「打开小红书搜美食」，它就自己看屏幕、自己点。

来源：智谱 Open-AutoGLM 官方 GitHub 仓库 github.com/zai-org/Open-AutoGLM 项目页

它支持的应用范围按官方文档列出的是 50+ 主流中文 App：

  • 社交：微信、QQ、微博
  • 电商：淘宝、京东、拼多多
  • 本地生活：美团、饿了么
  • 出行：携程、12306、滴滴
  • 视频音乐：B 站、抖音、爱奇艺、网易云、QQ 音乐

更关键的是两点。一是它能跑长流程：官方把「点外卖、订机票」这种几十步的连续操作作为标准能力，而不是单步演示。二是在 AutoGLM 2.0 里，智谱给它配了云手机和云电脑——Agent 可以在云端自主干活，不占用你本地手机。

而且 AutoGLM 把核心能力开源了。Open-AutoGLM 仓库 2025 年 12 月初建立，采用 Apache-2.0 许可证，截至写作时星标超过 2.5 万。这意味着它不只是个 App，还是一套你能拿去研究、改造、自托管的框架。

一句话收尾：在「让 AI 真去动设备」这条最硬的线上，AutoGLM 是四款国产里离 OpenClaw 那种「真把活干完」最近的一个。

第二条线：云端工作流——扣子把本地终端拉进了同一个项目

这一节的核心：扣子 Coze 的差异化不在自己单干，而在它能把你本地的 Claude Code、Codex CLI、OpenClaw 拉进同一个 Agent 项目协作。

如果说 AutoGLM 是「亲自动手」，扣子走的是另一条路：做那个统筹的大脑，让各路工具在它的工作台里配合。

扣子的官方定位是一站式 AI 办公平台，覆盖写作、PPT、表格、设计、播客、生图。但它在 2026 年这一轮升级里最值得开发者注意的，是把本地命令行 Agent 接进来这件事。

按极客公园对扣子 2.0 的报道，它的定位从「对话式 AI 工具」升级成了「可长期规划、自主执行、多体协同的 AI 工作伙伴」，核心能力包括三块：

  1. 长期规划（AgentPlan）：设定目标 → 自动拆解任务 → 主动执行 → 定期复盘 → 迭代优化
  2. 云手机 / 云电脑：Agent 有自己的云端设备，能持续自主推进任务
  3. 私有化本地部署：支持数据不出内网的部署方式

而到了更新的版本，扣子把 Claude Code、Codex CLI、OpenClaw 这些本地命令行工具一键接入了同一个项目工作台——你可以用 @ 的方式给它们派活。这些工具过去各跑各的终端窗口，彼此很少共享上下文；扣子做的事，是给它们一个共同的项目空间。

这条路的好处和代价都很清楚：

维度 扣子这条路的特点
协作能力 能让云端 Agent 和你本地终端工具在一个项目里共享上下文
本地执行 不是自己直接控屏，而是调度本地命令行工具去执行
数据位置 默认云端为主，但提供私有化部署选项
适合谁 已经在用 Claude Code / Codex CLI 的开发者，想要一个统筹层

一句话收尾：扣子不跟你抢「谁来干活」，它要做的是那个把云端规划和本地执行接在一起的统筹层——这是它和 AutoGLM 最大的路线区别。

第三条线：连不连你的个人数据——这是国产的天然主场

这一节的核心：连微信、连办公数据这件事，国产产品有海外比不了的主场优势，但「连得到」和「敢交权限」是两回事。

个人 AI 助手要真好用，绕不开一个问题：它得连得上你每天用的东西——微信、办公文档、各种国内 App。这恰恰是国产的主场。

几个确证的事实：

  • AutoGLM 直接支持操作微信、淘宝、抖音、美团等 50+ 中文 App，这是 ADB 控屏带来的天然覆盖
  • 腾讯正在测试微信内置的 AI Agent 原型，入口放在微信主界面右滑调出，能用自然语言调用小程序完成筛选、下单——不过按报道，它最快本月才启动合规审批，仍在测试阶段，未正式上线
  • 腾讯元宝已是日活超过 5000 万的成熟助手，深度接入微信，能一键解析公众号文章、图片、文档

这里要诚实地分两层说。连得到是能力，国产在中文 App 覆盖上确实领先；但敢不敢把权限交出去是另一回事——让一个云端 Agent 拿到你的微信操作权限，对很多人来说是个真实的犹豫点。

这正是 OpenClaw 那条路的价值所在：它支持微信、飞书、Slack 等 20+ 消息渠道，但数据留在本机 ~/.openclaw/ 目录，不上云。同样是「连你的数据」，一个是云端代你操作，一个是本地帮你处理——信任成本完全不同。

一句话收尾：国产在「连得到你的数据」上有主场优势，但谁能把「连得到」和「让你放心交权限」同时做好，才是这条线真正的胜负手。

第四条线：MCP / A2A 集成——国产追得最快的一段

这一节的核心：在 MCP 这套 Agent 互联标准上，国产产品的接入速度几乎和海外同步。

个人助手要想接得上各种外部工具和数据，靠的是一套通用接口。MCP（Model Context Protocol，模型上下文协议）就是这条线现在事实上的通用标准，OpenClaw、扣子、MiniMax、阿里百炼都已经支持。

这一段国产追得相当快：

  • 阿里百炼的新版智能体应用（Agent 2.0）把知识库、MCP 等能力统一成工具，由模型自主思考、规划、调用
  • MiniMax Agent 支持 MCP 多 Agent 协作，配合它的分层架构——一个主调度 Agent 理解意图、拆解任务，多个执行 Agent 各管编程、设计、调研
  • 扣子把本地命令行工具接入，本身就是一种 Agent 互联的实践

来源：QwenPaw 官方 GitHub 仓库 github.com/agentscope-ai/QwenPaw 项目页

国产这条线还冒出了一个有意思的开源项目：QwenPaw。它由 AgentScope 团队维护，基于阿里千问模型，仓库描述几乎就是 OpenClaw 的中文版定位——你的个人 AI 助手，装好就能用，跑在自己机器或云端，支持多个聊天应用、能力可扩展，明确写着支持浏览器批量操作和 OAuth 2.1 MCP。它 2026 年 2 月底建立，采用 Apache-2.0 许可证，星标已过 1.7 万。这说明国产圈里，「OpenClaw 那种本地优先 + 开源可扩展」的范式，已经有人照着做了。

一句话收尾：MCP 这套标准上，国产几乎是和海外同步在跑，这是个人助手能力能快速补齐的底气所在。

第五条线：本地优先与隐私——国产和 OpenClaw 还隔着一条路

这一节的核心：在「数据到底留在你本机还是上云」这条线上，国产几款主力产品和 OpenClaw 走的是两条不同的路。

前面四条线国产追得都不慢，但到了隐私本地化这一段，差异就拉开了。

OpenClaw 的设计是彻底的本地优先：

  • 数据存在本机 ~/.openclaw/ 目录，不进云
  • 模型无关（model-agnostic），GPT、Claude、本地大模型随你换；既支持智谱 GLM、文心、千问这些国产 API，也支持 Ollama、LM Studio、llama.cpp 跑的本地模型，断网也能用
  • 3600+ 可插拔技能 + 内置「执行—评估—抽象—存储」的学习循环，越用越懂你
  • 整套东西开源，你能自己托管

国产主力产品里，这条路走得最完整的不是那几个大厂 App，反而是 AutoGLM（开源 + 可自托管）和 QwenPaw（开源 + 本地部署 + 本地模型）这类开源项目。而扣子、MiniMax、千问这些产品，默认形态是云端为主——扣子提供私有化部署选项，但「数据全程不出本机」并不是它们的默认姿态。

这不是谁对谁错，是两种产品哲学：

  • 云端为主：算力强、记忆长（MiniMax Agent 的上下文可达百万 token），开箱即用，但你得信任云端
  • 本地优先：数据自己管、模型自己挑、能断网用，但配置门槛更高，得自己动手

一句话收尾：国产在「能办事」上已经追得很近，但「数据彻底留本机、模型彻底自己挑」这条本地优先的路，目前主要靠 AutoGLM、QwenPaw 这类开源项目在补——这是国产个人助手下一程最值得期待的一块。

回到那个问题：国产个人 AI 助手走到哪一步了

把五条线收一收，开头那个问题——「国产个人 AI 助手能不能真替你把活干完」——现在有了一个清楚的答案。

能办事，这一关基本过了。 AutoGLM 能在手机里跑几十步的真任务，扣子能统筹云端规划加本地执行，MiniMax 和千问能在云端跑完整工作流，MCP 接入速度几乎和海外同步。从「只会聊天」到「真能动手」，国产这一步已经迈过去了。

真正还在分叉的，是路线选择。 是把活交给云端 Agent，换来开箱即用和超长记忆；还是像 OpenClaw、QwenPaw 那样把数据攥在自己手里，换来彻底的本地优先和模型自由。这两条路没有标准答案，取决于你更在意「省心」还是「自己说了算」。

值得高兴的是，这两条路国产现在都有人在认真走——大厂在云端把能力做厚，开源项目在本地优先这条线上照着世界级标杆补齐。半年前这条赛道还几乎是 OpenClaw 一家独大的参照系，现在国产已经能拿出四五款各有侧重的真产品摆上同一张对照表。

个人 AI 助手这件事，国产入场不算最早，但补齐的速度足够让人对下一程抱有期待。这条路才刚铺开，前面的空间还很大，值得一起盯着看。
