Cursor Composer 2.5 仍跑 Kimi K2.5 底座：CursorBench 63.2% 反超 Claude Opus 4.7、单任务成本不到 1 美元

5 月 27 日凌晨，美国 AI 编程工具 Cursor 把自家旗舰 agent 模型 Composer 升到 2.5 版。两件事在国内开发者群引爆——一是 CursorBench v3.1 跑出 63.2%，把 Anthropic 公司 Claude Opus 4.7 的 61.6% 反超；二是 Cursor 联合创始人 Aman Sanger 在博客和 X 平台公开承认：Composer 2.5 底座仍然是月之暗面 Kimi K2.5，没有换。他还补了一句——「上一代 Composer 2.0 发布时没在博客里点名 Kimi，是我们的疏漏」。这是过去一年里国产开源大模型第一次以「被海外旗舰编程产品长期采用」的形式出现在主流英文 AI 圈视野里，36 氪、新浪科技、智源社区当天就跟进推送。

30 秒速览

把今天最关键的几个数字一次摆清楚——

  • 底座：Cursor Composer 2.5 仍跑 月之暗面 Kimi K2.5，未替换；Aman Sanger 公开承认上一代博客「没提 Kimi」是疏漏
  • CursorBench v3.1：Composer 2.5 拿 63.2%，反超 Claude Opus 4.7 的 61.6%——这是过去 12 个月 Cursor 官方榜上第一次有非 Anthropic 系模型登顶
  • SWE-Bench Multilingual：Composer 2.5 79.8% vs Opus 4.7 80.5%，差 0.7 个百分点
  • Terminal-Bench 2.0：Composer 2.5 69.3% vs Opus 4.7 69.4%，基本打平
  • 定价：输入 0.50 美元 / 百万 token、输出 2.50 美元 / 百万 token——单任务平均落在 1 美元以内，对手家旗舰模型同任务通常烧掉 4 至 6 美元
  • 算力分配：85% 投后训练、仅 15% 用于底座增量预训练；这是 Cursor 第一次公开训练投入结构
  • 后训练三件套：定向 RL（model 执行错误那一步直接局部提示）+ 任务数较上一代 25 倍 的合成数据（功能删除法）+ Sharded Muon optimizer + 双 mesh HSDP 数据 / 张量并行
  • 商业基本盘：Cursor 每日产出约 10 亿行被接受代码、财富 500 强 67% 客户在册；对手 Anthropic 公司 Claude Code 同期年化收入约 25 亿美元、企业客户 30 万家

数字摆完之后，这篇文章的核心论点其实就一句话——Cursor Composer 2.5 的真实意义不是「又一个反超 Opus」的评测营销，而是验证了一条对国内开发者特别友好的工程路线：开源底座 + 重度后训练 + 自家评测校准产品，可以在不烧自研大底座的前提下，把 agent 编程能力推到全球第一档。

国内 AI Coding 工具链——通义灵码、字节 Trae、百度文心快码、智谱 CodeGeeX、阿里通义灵码 Code-LLM-Plus——本来就走「不自训底座、深度后训练 + 工程化」的路。Cursor 用一个估值百亿美元的产品，把这条路实测跑通了。这对国产同档工具是一次明确的正向信号——路线没错，剩下的是后训练投入和工程节奏。下面分八节拆开看。

一、Kimi K2.5 底座这件事，为什么 Aman Sanger 承认上一代没提是「疏漏」

先把时间线摆清楚。

2026 年 1 月，Cursor 首发 Composer 1，底座是自训小模型，主打补全速度。3 月 Composer 2.0 发布，转向 agent 风格，评测表现明显跃升，但博客通篇没提底座来自哪里。当时国内 AI 圈就有声音怀疑「Composer 2.0 是不是用了 Kimi K2」——理由是月之暗面 5 月发布 K2 时 Aman Sanger 在 X 上转过一条 「impressive」，紧接着 6 月 Composer 2.0 的输出风格、上下文管理细节，跟 K2 的开源样例高度相像。但 Cursor 团队没正面回应。

5 月 27 日 Composer 2.5 发布，Aman Sanger 这次直接在博客和 X 上挂出来——底座是月之暗面 Kimi K2.5。他在 X 上加了一句话原文大意：「Composer 2.0 时我们没在博客里把 Kimi 写出来，这是我们的疏漏（this was an oversight on our part）。Composer 2.5 仍然基于 K2.5，并在它之上重度后训练。」

为什么这件事重要？三个角度——

第一，这是国产开源大模型「被海外旗舰编程产品长期采用」的第一次公开盖章。 过去一年，硅谷创业公司用开源中文模型的案例不算少，但通常是「测试一下」「实验性接入」「灰度部分流量」，没有一家头部产品愿意把底座点名公开——因为美国 VC 和企业客户对「用了中国模型」会有反应。Cursor 这次主动公开盖章 Kimi K2.5，等于明确告诉投资人和客户：「这个选择我们扛得住，因为产品力说话」。

第二，这条路线给国内 AI Coding 工具链立了一个对照样本。 通义灵码、Trae、文心快码、CodeGeeX 这些国内产品，过去经常被问一个问题——「你们都不自训底座，是不是技术含量低？」Cursor 这次用估值百亿美元的产品给出反向答案：不自训底座、深度后训练、产品评测校准，是一条独立可行的高价值路线。这跟苹果不自造代工厂、依赖台积电的逻辑相似——分工不等于矮一档。

第三，这是「Anthropic 公司不再是 AI Coding 默认底座」的信号弹。 过去 18 个月，Claude Code、Cursor、Windsurf、Continue、Cline 这几个头部 AI Coding 工具的底座几乎全部默认是 Anthropic 公司的 Claude Opus / Sonnet 系列。Cursor 这次明确用 Kimi K2.5 跑出反超 Opus 4.7 的评测数字，等于在 AI Coding 工具市场里第一次有一个头部玩家说——「我们不再认 Anthropic 一家是默认底座」。这件事对 Anthropic 是直接的商业压力，对国产开源模型是直接的入场红利。

5 月 27 日 Cursor 官方挂出三组硬指标，把 Composer 2.0、Composer 2.5、Claude Opus 4.7 三个模型放在同一张表里对比。把它整理成可视化图表——

一是 CursorBench v3.1 反超属实，但 SWE-Bench Multilingual 仍然落后 0.7 个百分点。 也就是说 Composer 2.5 在 Cursor 自家 IDE 场景里跑赢 Opus 4.7，但在第三方通用评测上，Anthropic 公司的旗舰仍微微领先。这跟自家评测通常会向自家产品倾斜的常识吻合，需要把它说明白——Composer 2.5 不是「全面碾压」Opus 4.7，是「在 Cursor 自己测的指标上超过，在第三方测的指标上微弱落后」。

二是 Terminal-Bench 2.0 基本打平，意味着 agent 执行任务的硬能力两家持平。 这是更值得关注的指标——Terminal-Bench 2.0 测的是模型在真实命令行环境下完成任务的能力，比纯 IDE 场景更接近 agent 的真实使用场景。两家打平意味着，开发者实际感受到的「agent 干活靠谱程度」，Composer 2.5 已经追上 Opus 4.7。

三是单任务成本的 5.8 倍差距是真实卖点。 5.20 美元降到 0.90 美元，意味着同样预算可以多跑 5.8 倍任务。对个人开发者用 Cursor 的常见月度账单——20 至 100 美元——这意味着可以让 agent 多跑 5 至 6 倍长链路任务而不爆预算。对企业客户，这意味着 agent 编程在公司内部铺开的边际成本被压到了之前 1/6，对 ROI 决策的影响是数量级的。

三、单任务 0.90 美元 vs 对手 5.20 美元：定价表 + 成本拆账

Cursor Composer 2.5 的官方定价是这样——

价格档 Composer 2.5 Claude Opus 4.7 倍数
输入 token（每百万） 0.50 美元 15 美元 1/30
输出 token（每百万） 2.50 美元 75 美元 1/30
单任务平均成本 0.90 美元 5.20 美元 1/5.8

输入价格差距是 30 倍，输出价格差距也是 30 倍。但为什么实际单任务成本只差 5.8 倍，不是 30 倍？因为 agent 任务的实际成本主要由「平均一个任务消耗多少 token」决定——Composer 2.5 在同样任务上消耗的 token 量是 Opus 4.7 的大约 5 倍。这是 agent 编程的常见现象——更便宜的模型通常需要更多轮工具调用、更长的思考链路才能完成同样任务，token 消耗会涨。

也就是说，单纯比单价差距 30 倍意义不大，要看「完成同任务的全包成本」。Composer 2.5 真正的卖点是——单价差 30 倍 + 任务 token 用量多 5 倍 = 全包成本仍便宜 5.8 倍。把这个账算清楚，对国内开发者评估自家产品的定价策略也有参考。

四、85% 算力转后训练：3 个关键技术拆解

Cursor 官方第一次公开训练投入结构——Composer 2.5 把 85% 算力投入后训练，仅 15% 用于底座增量预训练。这是一个非常重要的数据点，因为业内多数厂商不公开这个比例。

85% 后训练具体做了什么？官方 changelog 加上 Aman Sanger 在 X 上的细化补充，拆出来是三件套——

第一件：定向 RL（targeted RL）。 传统强化学习训练 agent 的方式是把整条轨迹当一个样本，最后看任务完成与否给一个 reward。问题是稀疏——一条 50 步的 agent 轨迹失败了，模型不知道是第几步出了问题。Cursor 这次的定向 RL 改成：执行轨迹里某一步出现明显错误时，直接在那一步局部插入提示给模型，让模型在那个具体的错误点上学习改进，不需要重跑整条轨迹。这种「错在哪学在哪」的局部提示，工程上需要细粒度的 reward 模型和回路重放设施，但训练效率显著高于稀疏 reward。

第二件：25 倍合成数据 + 功能删除法。 Composer 2.5 用的合成训练任务总数是 Composer 2.0 的 25 倍。其中关键技术是「功能删除法」——拿现成的开源仓库，先让模型读懂某个函数 / 文件的功能，然后把它删掉，要求另一个模型基于上下文（其他文件、测试用例、commit 历史）重写出来。这种合成方式有两个好处——一是有标准答案（原代码），二是难度天然分布广（小函数、跨文件、跨模块、整套系统都可以做）。这种 negative sample + 标准答案的合成数据范式，在国内通义实验室、月之暗面 K2 训练管线里也有类似工程，但「25 倍上代任务数」这个量级是新的。

第三件：Sharded Muon optimizer + 双 mesh HSDP。 Muon 是一个 2026 年初学术圈出来的新一代 optimizer（替代 AdamW 的尝试之一），训练效率显著高，但单 GPU 显存吃不下大模型的 optimizer 状态。Sharded Muon 是把 optimizer 状态切片分布到多 GPU 上，每个 GPU 只存自己负责的那一片，类似 ZeRO-2 的设计。双 mesh HSDP（Hybrid Sharded Data Parallel）是把数据并行和张量并行两层 mesh 叠起来——同一个模型在数据并行维度上切，又在张量并行维度上再切一次。这套基础设施投入大，但是「85% 算力可以可持续投入后训练」的工程前提——没有这套基础设施，85% 后训练的成本会爆炸，无法长期跑下去。

这三件套合在一起，共同支撑了 Composer 2.5 的训练逻辑——用工程能力把后训练的边际收益压榨出来，让「不自训大底座 + 重度后训练」成为一条真正可持续的路线。

Cursor Composer 2.5 跑的是 Kimi K2.5 + 重度后训练后的结果，不能直接当成 Kimi K2.5 原生能力的代表。但 Kimi K2.5 作为底座本身的能力档位，可以和当下几款主流大模型放一张表里看——

维度 Kimi K2.5 Claude Opus 4.7 DeepSeek V4-Pro Qwen3.7-Max
上下文窗口 200K token 200K token 128K token 1M token
工具调用能力 强（K2 即主打 agent） 旗舰级 强 强
开源状态 权重开源 闭源 权重开源 权重开源
输入价（每百万） 约 4 元 108 元（按 7.2 汇率） 约 12 元 约 6 元
国产 AI Coding 工具默认接入 部分接入 部分接入 主力接入 主力接入

四款里 Kimi K2.5 是开源权重 + 国产 + 工具调用优势的组合，从 Cursor 这次的工程选择反向推断，K2.5 在 agent 任务上的开源底座质量已经达到「值得海外头部产品压上的水平」。

但这里要给一个客观判断——Cursor 选 K2.5 不只是因为 K2.5 最强，还因为 K2.5 是开源权重，可以在自己集群上做深度后训练。 这条规则对 Opus 4.7 不成立——闭源模型只能用 API，做不了重度后训练。所以 Cursor 这次「反超 Opus 4.7」的结果，本质是「在开源 + 可后训练的前提下，开源底座经过重度后训练能反超闭源旗舰直接调用 API」。这个判断对国内 AI Coding 工具选型策略也有借鉴——主力底座选开源 + 可后训练的方案，长期 ROI 会显著高于「直接调用闭源 API」。

国产开源模型里有这个条件的有 Kimi K2.5、DeepSeek V4-Pro（开源权重）、Qwen3.7-Max（开源权重）三家。三家在工具调用评测上的相对位置——按 5 月新发布的多项评测，Kimi K2.5 在 agent 风格任务上领先一档；DeepSeek V4-Pro 在通用推理和数学上仍是头部；Qwen3.7-Max 在长上下文和多模态上有优势。三家路线分化已经成型，Cursor 选 K2.5 是顺势而为，不是冒险。

六、国内 AI Coding 工具的位置：通义灵码、Trae、文心快码、CodeGeeX 在哪

把国产主流 AI Coding 工具放在「Composer 2.5 这条评测曲线」上对照看——

通义灵码（阿里云）。底座是千问 Qwen3.7-Coder，做大量后训练。在国内开发者社区的实际反馈里，IDE 内补全和 agent 任务在国产工具中处于第一档，活跃用户体量是国产 AI Coding 工具里最大的一家。和 Cursor Composer 2.5 的路线对照——通义灵码本来就是「不自训大底座、深度后训练」路线，Cursor 这次的工程选择对通义灵码是直接的路线背书。

字节 Trae（国内版）。Trae 国内版的底座是字节自家豆包 1.5 系列 + 部分 Qwen 衍生，做 agent 风格后训练，限免期一直在延续。在补全速度和 IDE 体验上对标 Cursor 国内可用版本，是当前国产工具里 IDE 体验最接近 Cursor 的一家。Trae 海外版（Trae International）则使用通义和 DeepSeek 混合方案。这条产品线对中国大陆开发者是「Cursor 的本地化平替」首选。

百度文心快码。底座是文心 5.1 编程版，深度后训练。在大型企业客户里的占比高，配套 Comate 团队协作工具链相对完整。和 Cursor 的路线差异——文心快码更偏 ToB 套餐制（39 至 99 元 / 月套餐），不主打 token 计费，对个人开发者的吸引力略弱一档。

智谱 CodeGeeX。底座是 GLM 4.6 + CodeGeeX 4 系列后训练。这是国产里最早做 AI Coding 的团队之一，技术积累深，但近一年产品节奏被通义灵码和 Trae 超越，目前主要在科研机构、高校、国央企客户里有稳固份额。

阿里通义灵码 Code-LLM-Plus。这是通义灵码在企业版上推出的高端档位——更长上下文、更强 agent、更深企业适配。定价对标 Cursor 企业版，在大型互联网公司里的渗透率提升明显。

把四家放在 Composer 2.5 的评测曲线上对照看，路线本来就高度一致——开源底座（千问 / 文心 / GLM）+ 重度后训练 + 自家 IDE / 工具链。Cursor 这次的成功本质上验证了国产 AI Coding 工具链选的就是正确路线，剩下的是后训练投入规模和工程节奏。这对国产同行是正向信号——不需要拼自研大底座，专心做后训练 + 工程体验，是可以做到全球第一档的。

七、Cursor 的商业基本盘：每日 10 亿行代码 + 财富 500 强 67% 客户

5 月 27 日同期，Cursor 官方还公开了三组商业数据——

  • 每日产出约 10 亿行被接受代码：这里强调「被接受」，即开发者实际点了 Accept 写进文件的代码量，不是 AI 生成但被拒绝的数字
  • 财富 500 强 67% 客户在册：Cursor 企业版客户覆盖财富 500 榜单中 335 家公司
  • 内部 agent 创建合并 PR 占比 35%：Cursor 自己工程团队里，由 agent 自主创建并最终合并的 PR 占总合并 PR 的 35%

对照 Anthropic 公司 Claude Code 的同期数据——

  • 年化收入约 25 亿美元：Anthropic 5 月业绩公开披露 Claude Code 年化运行率

把两家放一起看，可以看出 AI Coding 工具市场的两个分层正在形成——
