Simon Willison 论断 Anthropic / OpenAI 已找到 PMF：从 200 美元订阅到 2180 美元 token 账单的 11 倍跃迁

30 秒速览

  • 顶帖事实：Simon Willison 5 月 27 日博文《I think Anthropic and OpenAI have found product-market fit》当晚冲上 Hacker News 首页第一，截稿时 209 分、231 条评论；提交人正是 Simon 本人（用户名 simonw）。
  • 核心论点：Simon 这位长期对 AI 商业模式持谨慎乐观的资深开发者，第一次明确说「他们找到了 PMF」——关键证据不是营收新闻稿，而是他自己的 30 天编程账单：支付 200 美元订阅，按官方 API 价折算的真实 token 价值是 2,180.16 美元（1,199.79 美元来自 Claude Code，980.37 美元来自 Codex CLI），订阅价相当于 API 价的 1/11。
  • 配套数字：Anthropic 据 The Information / Bloomberg 转述，2026 年第二季度年化营收冲到 109 亿美元（对比 2025 年 8 月披露的 40 亿美元，9 个月同比 170%），并「强烈传闻将迎来首个盈利季度」；SpaceX 与 Anthropic 签下 每月 12.5 亿美元、合约至 2029 年 5 月 的推理算力协议。
  • 企业定价两次跳档：Anthropic 2025 年 11 月把 Team 套餐改成 20 美元 / 座 / 月 + API 用量；OpenAI 2026 年 4 月跟进，取消按消息计费，所有产品统一按 token 口径计价（4 月 23 日全量）。
  • 国内对位：同期通义千问全年营收约 50 亿元人民币（中信证券估算），文心一言约 30 亿元人民币——加起来还不到 Anthropic 单季度的零头；不是国内做不出 coding agent，而是 消费者侧那一根「每月 200 美元长尾订阅」的轨道还没铺出来。
  • 本文核心论点：Simon 这条 209 分的顶帖之所以重要，不在于它「夸 Anthropic 厉害」，而在于它给出了 PMF 信号的可验证最小颗粒——一个真实开发者的 30 天账单。国内同行的下一步，是把同样级别的真实账单做出来，而不是再写一份对标 评测榜单 的 PPT。

一、Simon Willison 这条论断为什么 HN 209 分顶到头页

Simon Willison 是谁，HN 老用户基本都熟。Django 联合创作者之一、datasette 作者、长期写《Weeknotes》一年发上百篇博客的英国独立开发者，2023 年起把博客重心几乎全部转向 LLM 应用工程。他不是一个会随便说「PMF 找到了」的人——过去两年他的笔记里出现频次最高的词反而是 cautiously optimistic（谨慎乐观）。这次他直接拿掉了 cautiously。

帖子在 5 月 27 日北美下午由他本人提交（HN 用户 simonw），4 小时内冲到首页第一，截稿时 209 分、231 条评论。HN 是少数对「夸 AI」基本免疫的社区——前面几篇上头条的 AI 相关帖子普遍是反共识：Nolan Lawson《用 AI 更慢写更好的代码》1078 分、George Hotz《永恒的 9 月》800＋ 分。在这样一个集体气氛下，Simon 这条「我认为他们找到 PMF 了」反而被顶上去，说明社区对他给出的依据是认可的。

论据是什么？不是市场调研、不是 IDC 报告，是他自己的 Claude Code 和 Codex CLI 30 天用量。Simon 原文这一段写得平静而锋利：

我每月给 Claude Pro 付 100 美元、给 Codex Pro 付 100 美元，合计 200 美元。如果同样的使用量按 API 价单独结算，过去 30 天我的账单总额是 2,180.16 美元——Claude Code 部分 1,199.79 美元，Codex CLI 部分 980.37 美元。

也就是说，Anthropic 和 OpenAI 各自承担了 Simon 这个用户大约 1000 美元 / 月的真实算力成本，只收他 100 美元订阅费。Simon 在文章末段直接给出他的判断：这不是亏损补贴，是定价策略——他们在用月费档锁定高强度用户，然后用企业按量计费档把利润收回来。

HN 评论区第一条置顶来自用户 tptacek（Thomas Ptacek，Fly.io 创始团队、知名安全工程师）：「这是我读过的关于 AI 行业 PMF 最有说服力的一段，因为它的颗粒度是一个真实开发者一个月的真实账单，而不是 ARR 或者 monthly active users。」第二条来自 jasonwatkinspdx：「这个 11 倍倍数告诉我们：当前的 200 美元档不是 freemium 引流，是承诺保护价。Anthropic / OpenAI 在赌未来 12 个月单 token 推理成本继续下降 50% 以上，把价差吃回来。」

我把这条帖子排进今天的国外热轨头条，不是因为「Anthropic 又涨了」——而是因为 Simon 给出了国内同行可直接复用的 PMF 验证方法：拿你自己最重度用户的 30 天真实账单，按官方 API 价折算，看订阅价是 API 价的几分之一。这个倍数小于 3，说明定价合理但用户黏性弱；倍数大于 8，说明产品已经踩到「让人停不下来」的区间。倍数 11，意味着 Simon 的 200 美元订阅，他自己都觉得「赚到了，绝不会退订」。

二、200 美元 → 2180 美元 的 11 倍跃迁拆解：账单实账

先看 Simon 给的两条原始数字。Claude Pro 100 美元 / 月，包含 Claude Code 全功能不限次（仅有公平使用条款约束）；Codex Pro 100 美元 / 月，包含 Codex CLI 不限次。两个加起来 200 美元，覆盖他全部 AI 编程工作流。

然后是 30 天后回头折算：Claude Code 部分按 Claude 4.7 Sonnet 当前公开 API 价（输入 3 美元 / 百万 token，输出 15 美元 / 百万 token，缓存读 0.30 美元 / 百万 token）算下来 1,199.79 美元；Codex CLI 部分按 GPT-5.1 Codex API 价（输入 2.5 美元 / 百万 token，输出 10 美元 / 百万 token）算下来 980.37 美元。总计 2,180.16 美元。

这个 11 倍折扣意味着两件事，恰好对应消费侧和供给侧两条 PMF 信号。

第一件，消费侧：Simon 这种用户，每天 8 小时挂着 Claude Code 写代码，触发的真实推理消耗对应官方 API 标价 1000 美元 / 月。但他每月只愿意付 100 美元订阅。这中间的 900 美元差额，不是 Anthropic 在烧钱拉新——Simon 本来就是付费老用户。Anthropic 愿意吃下这 900 美元，因为他们判断 Simon 留下来比赚他这一个月差价更重要。这正是经典的「找到 PMF 之后压定价、用毛利率换增长曲线」打法。

第二件，供给侧：这种打法只在「单位推理成本快速下降」的曲线上才成立。Anthropic / OpenAI 现在敢锁 200 美元价格档，背后是他们已经看到自己未来 12-18 个月的推理成本可以再降一半。算力来源也已经备好：Anthropic 与 SpaceX 签的 每月 12.5 亿美元、至 2029 年 5 月 的 Colossus 推理算力长约（后面第四节细聊），就是这个判断的实锤。

国内同行经常会问：「为什么我们做不出这样的 200 美元订阅档？」答案不在产品功能，在两个先决条件：(a) 你能不能稳定看到未来 12 个月单 token 成本下降 50% 以上的供给路径；(b) 你愿不愿意吃下当下的毛利率换长期黏性。这两条任何一条不成立，都顶不住 11 倍的折扣承诺。

补充一句 Simon 文章里没强调但同样关键的事：这个 200 美元档是 individual developer 档，不是企业档。Anthropic 和 OpenAI 真正赚钱的位置在企业按 token 计价。Simon 给出的 11 倍折扣并不是企业账单的折扣——企业账单基本就是 API 标价。所以 PMF 在两个层面同时成立：个人开发者层面用低价订阅锁人，企业层面按 token 收钱。这是两段式的商业模型，不是单点优惠。

三、Anthropic 营收 40 亿 → 109 亿 9 个月内的 170% 同比是怎么涨上来的

Simon 的博文里只引用了一句「年化营收 109 亿美元」，没展开。我把背景补齐——这条数字源自 The Information 5 月 25 日报道、Bloomberg 同日跟进，对应的是 Anthropic 2026 年第二季度（4-6 月）年化口径的预测值。对比起点是 2025 年 8 月 Anthropic 内部沟通的 40 亿美元年化营收。9 个月时间，同比 170%。

整段增速我分成三段看：

第一段（2025-08 → 2025-11）：40 亿 → 65 亿美元。 Claude Code 在 1Password、Notion、Replit、Stripe、Vercel 等约 50 家头部客户跑通试点。月增约 6%，对应 Claude 4.5 Sonnet 上线后的口碑发酵期。这一段增长几乎全部来自 Claude Code 在已有大企业内部 50-500 工程师团队的渗透。

第二段（2025-11 → 2026-02）：65 亿 → 80 亿美元。 Cursor、Cline、Aider、Continue 等第三方 IDE 集成大规模接入 Anthropic API，带来上千家中小开发团队。月增降到约 4%，看似减速，实际是高基数效应——绝对增量反而更大。Cursor 这一家给 Anthropic 贡献的月消耗在 2026 年 2 月已经跨过 8000 万美元 / 月，仅次于 Anthropic 直接客户。

第三段（2026-02 → 2026-Q2 末）：80 亿 → 109 亿美元。 大客户深耕期。Uber 把 5000 名工程师全员铺开 Claude Code，2026 年 4 月单月就吃掉了 Anthropic 大约 1500 万美元；Stripe、Airbnb 等公司紧随其后。月增重回约 7%。这一段的关键不是新客户，而是单工程师月消耗从 500 美元抬到 2000-5000 美元——也就是把 Simon 这种用法从 1% 重度用户扩散到 50% 工程师的过程。

Simon 原文那句「Anthropic 强烈传闻将迎来首个盈利季度」（potentially even operating at a profit for the first time）在这里特别关键。AI 公司从 2023 年起就在反复被问「什么时候停亏」，三年过去答案一直是「还在烧」。Q2 2026 如果真的转正，那就是 LLM 商业化第一次明确兑现「PMF 之后毛利收得回来」的承诺。

横向对比 OpenAI：2026 年第一季度末公布年化营收约 130 亿美元，但仍然亏损（主要是 Stargate 数据中心折旧 + 持续训练大模型支出）。OpenAI 在体量上仍然领先，但 Anthropic 在「单位营收的成本控制」上有可能先一步走出补贴期。

放回到 2025 年 8 月那个 40 亿美元的起点：当时不少人（包括我自己）的判断是「AI Coding 是个不错的子市场，但天花板可能在 100 亿美元年化」。9 个月之后，单 Anthropic 就要碰天花板了。所以那个起点的判断错在哪？错在低估了「单工程师月消耗」的弹性。500 美元 / 月会让财务侧皱眉，2000 美元 / 月反而被工程团队接受——因为对应的是节省 10-20% 工程师工时，性价比成立。

四、SpaceX-Anthropic 12.5 亿美元 / 月推理协议背后的算力结算逻辑

第三段那个营收曲线之所以能继续涨，前提是 Anthropic 拿得到对应规模的推理算力。Simon 在博文最后一节提到一笔交易：SpaceX 与 Anthropic 签下每月 12.5 亿美元的 Colossus 算力长约，至 2029 年 5 月。

这是一个相当重的承诺。月付 12.5 亿、合约 36 个月，对应的总额 450 亿美元——是 Anthropic 自己当前年化营收的 4 倍多。背后逻辑不难理解：

第一，Anthropic 之前主要依赖 AWS Trainium / Google TPU，自建算力比例低。SpaceX 这一笔等于在英伟达 H100 / B100 之外又开了一条供给曲线。

第二，SpaceX 的 Colossus 数据中心专为推理优化（不是训练优化），单 token 推理边际成本据多个供应商交叉报价大约比通用云便宜 20-30%。把高频推理迁过去，能直接压低 Simon 文中那个 11 倍折扣的承担成本。

第三，合约到 2029 年 5 月，意味着 Anthropic 把未来 3 年的算力价格锁住了。哪怕英伟达后续涨价、电力涨价、模型变大，Anthropic 在 SpaceX 这部分都是定额。这对企业客户是强承诺信号——「我们不会因为成本失控而临时涨价」。

国内对应位置上，谁有可能签这样的长约？字节、阿里都有自建算力，但与外部签长约的公开记录目前只有 昆仑万维与华为昇腾的 1.7 万张昇腾 910C 合作（2026 年 5 月 Skyclaw-V1 训练同时官宣）、百度与中科曙光在内蒙古的 5 万张 K200 长约。规模上和 SpaceX-Anthropic 这笔差一个数量级，但方向是对的——锁价格、锁供给、把不确定性从季度账单挪到合约里。这是 PMF 之后必然出现的供给侧动作。

值得提醒一句：12.5 亿美元 / 月在国内读者听来是天文数字，但放回 Anthropic 109 亿美元年化营收语境里，单月支出占月营收约 14%（109 ÷ 12 ≈ 9.08 亿 / 月营收口径）。这个比例和云厂商基础设施支出占营收的 40-60% 比起来其实是低的——说明 Anthropic 仍然在用 AWS / GCP 兜底基础负载，SpaceX 是高频推理的尖端补充而非全量替换。

五、企业定价模型两个月内两次跳档：从座位制到 token 制说明什么

Simon 在博文中段花了一整节解释「为什么 Anthropic 和 OpenAI 在过去 6 个月几乎同时改了企业定价」。这件事看上去像是两家在比谁先动，实际上是 PMF 找到之后必然发生的定价升级。我把两次改动按时间顺序排一遍：

Anthropic（2025 年 11 月 14 日）：把 Team 套餐从「25 美元 / 座 / 月，无限制使用 Claude」改成 「20 美元 / 座 / 月 + API 用量按官方价单独计费」。表面看座位费降了 5 美元，实际上消耗大的工程师月支出从被压制的 25 美元拉到 500-2000 美元真实账单。Uber 这种重度用户公司，月账单从原来的 12.5 万美元（5000 工程师 × 25）一下子跨到 750 万美元（5000 × 平均 1500）量级。

OpenAI（2026 年 4 月 2 日宣布、4 月 23 日全量）：取消 ChatGPT Team 的「每月 N 条消息」上限，所有产品（ChatGPT Business / Enterprise / Codex CLI）统一按 token 计价。改前 Team 套餐 25 美元 / 座 / 月含一定消息数，超出阻塞；改后变成 20 美元 / 座 / 月 + token 用量，与 Anthropic 体感几乎一致。

两家几乎同步改这件事，技术解释是「按消息数计费让重度用户被限速」，商业解释是 PMF 找到之后从「拉新优先」切到「按价值收钱」。能这么做的前提是：你已经看到用户在重度用、舍不得退订，你才有资格把按量收费摆到台面。Simon 那个 11 倍折扣的折扣端，正是这次定价跳档之后的 个人开发者 锚点；企业端这一边，则是按 token 实价收。

国内同行最关心的一个问题是：国内有没有可能跑出类似的两段式定价？ 拿千问 Code 和 Cursor CN 当样本看，目前主流仍然是「企业按席位、个人按订阅」的旧模型。Cursor CN 2026 年 4 月底跟进了「20 元 / 座 + 按用量」的国内版（用千问 3-Coder + 部分 DeepSeek-V3.1 路由），但市场反馈是用户对「按 token 计费」陌生，企业财务担心月度账单波动。这种心理障碍要被攻破，可能需要 6-12 个月时间，让重度团队先吃到「按 token 透明对账」的甜头。

我倾向认为这是国内 AI Coding 商业化的下一个关键拐点。谁先在国内把 token 计价做到企业愿意签合同，谁就拿到了 Anthropic 在海外拿到的那张牌——按真实价值收钱，而不是按席位数收钱。

六、国内对位：通义千问 ~50 亿、文心一言 ~30 亿，coding agent 在国内为什么还未撑起规模化付费

把镜头转回国内。Anthropic 2026 年 Q2 年化 109 亿美元（约 783 亿元人民币）这个量级，对应国内是什么水平？

按公开信息和券商估算交叉看：

  • 阿里通义千问 2025 财年（截至 2026 年 3 月）AI 相关营收约 50 亿元人民币（中信证券估算，含模型 API + Qwen Code / Qwen Chat 订阅 + 阿里云通义生意参谋等垂直产品），其中 coding 相关约 8-12 亿元。
  • 百度文心一言 2025 自然年度 AI 相关营收约 30 亿元人民币（华泰证券估算），其中文心智能体平台 + Comate 编程助手合计约 5-8 亿元。
  • 字节豆包/Trae 2026 年 Q1 AI 相关营收没有公开拆分，市场估算 Trae 编程助手贡献年化约 6-10 亿元，主要来自字节内部 + 外部企业试点。

三家加起来年化营收差不多 90-100 亿元人民币（约 12-14 亿美元），不到 Anthropic 单季度的 1/3。差距不是「国内技术不行」——千问 3-Coder、DeepSeek-V3.1、文心 4.5 在 SWE-Bench / LiveCodeBench 这些 评测榜单 上已经追到全球第二梯队（参见 2026-05 公开榜单）。差距在消费者侧的 200 美元长尾订阅没建起来。

我把它拆成三个直接原因：

第一，国内开发者付费习惯还在迁徙。 个人开发者愿意月付 20 美元 ≈ 145 人民币的群体在国内大约 8-15 万人（综合 Cursor CN / Trae / Qwen Code 公开订阅数推算），而美国同档付费开发者群体在 80-120 万。10 倍量差，直接对应订阅收入 10 倍差。这个不是政策能解决的，是用户习惯和工程师人均可支配收入的复合差。

第二，企业财务对「按 token 计费」抗性强。 国内企业 IT 财务报销流程里，「席位订阅」是熟门熟路的科目，「按用量结算」常常会被合规和审计反复质询。即便产品好，采购流程也会卡半年。Anthropic 在美国能靠 Uber、Stripe 这种敢吃按量账单的客户先把规模拉起来，国内目前敢这么做的大型工业企业还在个位数。

第三，coding agent 这一类产品形态的本地化还在初期。 千问 3-Coder 在国内的 IDE 集成（VSCode 插件、Trae IDE 内置）2026 年 3 月才大规模铺开，时间上比 Claude Code 晚 8 个月。8 个月在 LLM 节奏里几乎是两个迭代周期。但好消息是：千问 3-Coder 现在已经支持 MCP、子代理调度、长上下文 200K，功能上和 Claude Code 同代不再有断层，差的只是用户基数。

我的判断是：国内年内会先在企业侧（金融、汽车、芯片设计）跑出第一批按 token 计费的标杆案例，再倒推到个人订阅档。顺序和海外相反——海外是 Claude Pro 个人档先火，再扩到企业；国内更可能是大客户先签合同，再用收入反哺把个人档定价压下来。这两条路径都能走通，但国内这一条会更艰难，需要更多耐心。

写给国内做 AI Coding 工具的同行：千问 3-Coder、DeepSeek 这些底模能力已经够用，真正卡住规模化的是定价模型和企业合规流程。把按 token 计费做成企业愿意签的标准合同，这件事本身的价值不输于把模型能力再涨 5%。

七、PMF 找到之后下一步：Anthropic / OpenAI 的服务质量风险

PMF 不是终点。Simon 在博文倒数第二节做了一段冷静的提醒：Anthropic 和 OpenAI 当前在高消耗用户上正面临服务质量争议。

具体几个公开事件：

  • 2026 年 3 月 Reddit r/ClaudeAI 大规模讨论：高频用户反馈 Claude Code 在订阅档下被悄悄限速——单会话超过 200K token 后响应明显变慢，Anthropic 官方一度沉默 11 天才回应。
  • 2026 年 4 月 Hacker News 顶帖（用户 jtreminio）：「I'm a $200/month Claude Pro user and Anthropic just throttled me without warning」——326 评论，最终 Anthropic 官方发声明承认存在「公平使用动态调控」。
  • 2026 年 5 月 Codex CLI 用户在 X 上集体抱怨：高 token 消耗用户被分流到 GPT-5.1 mini 而不是完整 GPT-5.1 Codex；OpenAI 5 月 20 日发改进说明，承诺给订阅用户默认走完整模型。

这些事件揭示了一个 PMF 之后必然出现的两难：承诺无限制使用，但成本不允许真的无限制。Anthropic / OpenAI 给出的折中是「订阅档承诺优先体验、但用量超过统计学高位时引入动态调控」。Simon 在文章里给的判断我赞同：

这个折中在当前 token 成本曲线下是合理的；如果 12 个月内推理成本再降一半，订阅档真正走向「无暗中限速」是可能的；如果成本曲线不达预期，订阅档会涨价或者重新引入显式限额。

从 PMF 角度看，这种「订阅档承诺 - 现实限速」的张力恰恰证明产品 太 受欢迎了。问题不是「用户不爱用」，而是「用户太爱用以至于他们的真实价值远远超过他们付的钱」。这是好问题，不是坏问题。

国内同行如果将来跑到同样位置，会面对一模一样的张力。提前想清楚两件事：(a) 个人档承诺什么、限速什么、写不写进用户协议；(b) 出现争议时多快公开回应。海外这一波翻车基本都败在「沉默 + 偷偷调整」，而不是「调整本身」。

八、编辑说：从 PMF 信号能给国内做 AI Coding 工具的团队什么参考

我不想给这篇专题加一份「国内开发者必须做的 5 件事」清单——那种姿态有顾问腔。我只想把 Simon 这条 209 分顶帖里最值得国内同行直接拿走的判断框架列出来，剩下交给各团队自己取舍。

第一，PMF 信号的最小颗粒不是 ARR、不是月活，是「真实用户 30 天账单的折扣倍数」。 拿你产品当前最重度的 10 个用户，按官方 API 价折算他们的实际消耗，对比他们付的订阅费。倍数小于 3，定价偏贵或者用户黏性弱；倍数 3-6，正常；倍数 6-10，正在接近 PMF；倍数大于 10，已经在 PMF 区间，下一步是企业按量计费扩张。

第二，定价模型决定可扩张范围。 单纯座位制天花板低；纯按 token 计费 friction 高（这个词中文可读作「采购摩擦」）；「低订阅 + 高用量按 token」的两段式模型 是 Anthropic / OpenAI 这一轮跑出来的真正赢家配方。国内同行如果要复用，先把企业财务对「按用量结算」的合规问题解决掉，这件事比模型能力还重要。

第三，PMF 之后供给侧要立刻动。 Anthropic 找到 PMF 之后立刻签 SpaceX 36 个月长约锁价；国内同行找到 PMF 之后该签的不一定是英伟达 / 昇腾长约，但一定要有一条「未来 18 个月单 token 推理成本下降 50% 以上」的路径。这条路径可以是自研推理芯片、可以是模型蒸馏、可以是 KV cache 优化——但它必须存在，且对企业客户讲得通。

第四，服务质量争议会比预想来得更早。 在用户基数 10 万级时就会出现，不是百万级。提前准备公开沟通话术、提前把动态调控规则写进用户协议、提前安排团队成员负责 Reddit / 知乎 / V2EX 的舆情响应。沉默 11 天是 Anthropic 这一波被骂得最重的环节，国内同行最不该重复的也是这一条。

第五，营收增长曲线分三段，不是一条直线。 Anthropic 40 亿 → 109 亿这 9 个月是「头部企业试点 → IDE 集成扩散 → 大客户深耕」三段叠加。国内同行如果只看第一段的高增速就以为可以一路涨，会在第二段（高基数减速期）误判 PMF 失效；只有等到第三段（单工程师月消耗从 500 抬到 2000）出现，才算真正进入扩张期。

最后说一句立场：我在国内做 AI 开发，跟绝大多数读到这里的同行一样，最希望看到的是中国厂商在这一轮里跑出对标 Anthropic / OpenAI 的体量。Simon 这条 209 分顶帖给的信号不是「美国人赢了」，而是「PMF 信号可被验证、路径可被复用」——而国内同行已经离这条路径很近，差的不是模型能力，是 200 美元订阅档背后的两段式商业逻辑。这件事难，但不远。
