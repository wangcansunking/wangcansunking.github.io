2026-05-01Daily Report

Anthropic 估值反超 OpenAI 那一夜

封面图：两块手机屏的对峙——海外 App Store 下载榜上正在发生的用户迁移。

3 月卸载量同比 +413%、4 月仍在 +132%、个别口径峰值 +563%——这是美国市场调研公司 Sensor Tower 给 ChatGPT 的成绩单。 同期 Claude 下载量同比涨了 10 倍，在比利时、加拿大、德国、卢森堡、挪威、瑞士的 iPhone 免费榜爬到第 1 位，美国市场也短暂登顶。Anthropic 估值从年初的 6000 亿美元一路谈到 9000 亿美元，已经超过 OpenAI 的 8500 亿美元。

海外的迁移在发生。 但对中国大陆开发者来说，这条新闻最有用的不是数字本身，而是数字背后那条更具体的问题——当 Claude Code 限速越来越紧、Anthropic 把价格定到月费 200 美元的 Max 档，谁来接住国内一线工程师的日常编码工作？

数据在前面。判断在最后。

一、Sensor Tower 把数据摊开：迁移到底有多猛

Sensor Tower 的 3-4 月数据汇总。来源：Sensor Tower 报告 / TechCrunch / Futurism / Seeking Alpha 综合整理。

Sensor Tower 是位于美国加州的应用市场情报公司，长期跟踪 App Store 与 Google Play 的下载、卸载、留存数据。这次它把 ChatGPT 与 Claude 放在同一张表上对比，时间窗口是 2026 年 2 月底到 4 月底。

卸载侧——ChatGPT 三个月里被两次拉到历史峰值。 第一次是 2 月 27-28 日，OpenAI 与美国国防部签下首个生成式 AI 合同的当天。Sensor Tower 与 TechCrunch 给出的细分数据显示，2 月 28 日单日卸载量环比 +295%，1 星评论环比 +775%；同期 Claude 在美区 App Store 当周冲到免费榜第 1 名，2 月 28 日开始连续 3 天保持榜首。第二次是 3 月整月——同比 +413%，Sensor Tower 把这个数字标注为 ChatGPT 上线以来的卸载月度峰值。4 月卸载同比仍维持在 +132% 的高位，比平时高了一倍多。Futurism 引述的另一份口径甚至给出了 +563% 的峰值切片。

下载侧——Claude 一周翻一倍，整体翻 10 倍。 36 氪头条引用的"一周下载 +199%"对应的是 2 月 27-28 日那一周的环比读数，而 Sensor Tower 月度对比给出的是同比 +1000%（也写作"约 11 倍"）。两个数字不矛盾，前者是冲击波周内峰值，后者是整月叠加之后的同比。Futurism 同期写道：Claude 在美国的日下载量一度逼近 14.9 万，而 ChatGPT 同期约 12.4 万——单日维度上，Claude 第一次站到了 ChatGPT 前面。

MAU 侧——ChatGPT 的增速已经在收。 Sensor Tower 给的曲线是：1 月月活同比 +168%，2 月 +120%，3 月 +95%，4 月降到 +78%。还在涨，但增速曲线明显收口。叠加 ChatGPT 4 月已经把 14% 的免费用户切到广告版（3 月这个比例还是 1%），用户体感的劣化会进一步压住自然增长。

把这几条线索拼起来：这不是一次孤立爆点，而是叠加了"五角大楼合同冲击 + 广告化推进 + Claude 产品端持续升级"的连续过程。Sensor Tower 4 月报告原文给的判断是中性的——"ChatGPT 增长在变稳，但 Claude 在以更快速度逼近"。

二、催化是什么——别再把账记到 Claude 3.5 Sonnet 头上

36 氪 4 月 30 日那篇头条把催化里写了一条"Claude 3.5 Sonnet 编码能力突破"。这条事实需要更新——Anthropic 在 2024 年 6 月就发了 Sonnet 3.5，到 2026 年 4 月这个时间点，Claude 主线模型已经走到了 Opus 4.7 与 Sonnet 4.6。真正驱动 3-4 月用户大迁移的，是 Anthropic 一连串近期产品节奏，不是 18 个月前的 Sonnet 3.5。

Claude Opus 4.7 — Anthropic 4 月 16 日 GA 发布。图源：Anthropic 官网。

把节奏列清楚：

  • 2 月 17 日，Claude Sonnet 4.6 全面升级：长上下文 1M token 进入 beta，编码、长链路 agent、知识工作、设计四个方向同时打补丁
  • 4 月 16 日，Claude Opus 4.7 GA：SWE-Bench Verified 87.6 分、SWE-Bench Pro 64.3 分、Terminal-Bench 2.0 69.4 分；编码 benchmark 较 4.6 提升 13%；长链路任务通过率 3 倍；图像分辨率上限提到 3.75 兆像素；定价维持在 $5/$25 每百万 token
  • 同步发布 Claude Design：Anthropic Labs 第一款交付物，在 chat 里直接产出可运行原型、幻灯片与 one-pager
  • Claude Skills cookbook 在 GitHub 走红：开发者社区把 skill 写成可分发模块，沉淀出第一批"组合套装"
  • 企业端：Claude Opus 4.7 同日在 Amazon Bedrock、Google Cloud Vertex AI、Microsoft Foundry 上线，"零运营人员访问"模式落地

这五件事叠加在 2 月底到 4 月中——刚好覆盖 Sensor Tower 数据的整个曲线。编码社区先动，因为 Sonnet 4.6 + Opus 4.7 把 SWE-Bench Verified 从 79.6 拉到 87.6，是相当大的一档跨越；设计与产品 PM 跟进，因为 Claude Design 把"边聊边出原型"这个流程做顺了；普通用户最后被裹挟，因为 App Store 算法看到这一波集中下载就把 Claude 顶到榜单第 1，反过来又拉新一轮自然下载。

Anthropic 这一侧的资本动作也压住了节奏。年初它的 Series G 收官时投后估值 3800 亿美元；4 月 24 日 Google 宣布最高再投 400 亿美元，节奏分两步——首期 100 亿美元、估值锚定 3500 亿美元，剩下 300 亿与业绩里程碑挂钩；同期亚马逊在过去 80 亿美元基础上再追加最高 250 亿美元；CNBC 与 Fortune 4 月底都报了它正与新一批投资人谈到 9000 亿美元估值的消息——已经反超 OpenAI 同期的约 8500 亿美元。Sacra 给的另一组数字是：Anthropic 3 月年化收入 300 亿美元，同比涨约 1400%——这是估值的实底。

所以 3-4 月这场迁徙不是单点事件，是一家公司"产品稳、收入稳、资本稳"三条线同时拧成了一股绳——Sensor Tower 的下载曲线只是这股绳被开发者群体看见的那一面。

OpenAI 这一侧的处境也得说清楚：用户基数仍然庞大——ChatGPT 30 天下载 5900 万，过去 12 个月累计 8.22 亿；30 天收入 6300 万美元。与微软的捆绑仍然战略——Copilot 全栈、Azure 算力、Office 365 嵌入都还在；Stargate 项目本身只是部分项目调整（英国暂停、挪威移交微软、得州 800MW 扩建终止），不是整体停摆。OpenAI 的真问题是节奏——3 月的国防合同舆情、4 月广告化推进、加上 GPT-5.5 上线初期的稳定性问题（OpenAI 自己 4 月发了一篇技术博客 "Where the goblins came from" 复盘），让一部分高粘性用户找到了切走的理由。

别把这看成 OpenAI 崩盘。这是产品节奏短暂落后于对手的一个窗口，OpenAI 的护城河（用户基数、企业捆绑、品牌认知）几乎都没动。但窗口期里被切走的用户——尤其是开发者群体——再切回来不容易。

三、国内能不能直接用？合规与替代路径都得讲清楚

中国大陆用户访问 ChatGPT 与 Claude 的官方版本，受合规要求与网络环境双重影响——两家美国公司在大陆都没有开放区域，App Store 中国区不上架它们的应用。这是事实背景，不是抱怨。Sensor Tower 数据里那场用户迁徙基本不影响国内大众用户的日常使用——他们本来就用的是文心一言、通义千问、豆包、Kimi、DeepSeek 这一批国产产品。

真正受这条线索影响的，是国内开发者群体里通过中转服务调用 Claude API、或在订阅 Claude Code Pro / Max 写代码的那一拨人。这一拨人最近的真实体感是怎样？

V2EX 4 月 27 日有一条标题为《CC 的迷之速率限制》的热帖，原帖作者 nc 留下的原话是：

「我让 claude code 运行 14 个 subagents 做 code audit，很快，两分钟内 5 小时的限额就 100% 了。」

「我严重怀疑这是 anthropic 故意的规则，就是如果有多个并发，加快限额使用。」

「或者干脆直接一刀切，就是为了防止有中转站运行多个 claude code 然后 Proxy 请求变成 API。」

另一条 V2EX 帖子《Claude code 不想订阅了，API token 选哪家？》同期热度也不低。叠加 Anthropic 4 月把 Max 档定价拉到月费 200 美元，开发者层面对"继续用 Claude Code 还是换工具"的讨论一直没断。

这就是国产替代要回答的题。 4 月这段时间，国产 Coding Agent 走到了 SWE-Bench Verified 80 分档：

  • DeepSeek V4-Pro——SWE-Bench Verified 80.6 分，比 Claude Opus 4.7 的 87.6 分还差一档但已经是国产里离 Claude 最近的成绩；同体系的 V4 主线给出 83.7，V4-Flash 走轻量路线 79.0
  • 千问 Qwen 3.6-Plus——SWE-Bench Verified 77.2 分，在国内开发者社区被认为"日常工程任务最稳"；Qwen 3.6-35B-A3B（3B 激活）也跑到 73.4 分，端侧推理路径打通
  • GLM-5（智谱）——SWE-Bench Verified 77.8 分，离 Opus 4.6 不到 3 分
  • MiniMax M2.5——SWE-Bench Verified 80.2 分，主要走 API 路径
  • Kimi K2.6——SWE-Bench Pro 58.6 分（这是更难的一档）

工具链这一侧，阿里 Qoder 是最完整的一个。Qoder CLI 4 月底完成多模型支持（Qwen 3.6-Plus / DeepSeek V4 / GLM-5 都能挂上），订阅价 Pro $20、Pro+ $60，目前还有半价活动。Qoder IDE 的 Quest Mode 是 spec 驱动的任务委派，比 Claude Code 的会话级 agent 多了一层"持久身份+长期记忆"。CSDN 这周一篇上手贴的标题就是《Qoder CLI，也许是 Claude Code 在国内的最佳平替》。

真实体验差距在哪？ 知乎 4 月那篇《2026 年 AI 编程工具全景测评》给的诚实评价是——Claude Code 在大型项目跨文件重构与长链路 agent 上仍然领先一档，国产工具在标准 SWE-Bench 题目上已经追近，但跑到生产代码库（10 万行以上、跨服务依赖）时偶尔仍有指令理解偏差与幻觉。这是真问题，不回避。 但反过来说——国内绝大多数日常编码场景（写业务代码、修 bug、做单元测试、跑迁移脚本）已经被这一批国产工具覆盖到了。

计费与本地化这一档，国产路径的优势更直白。Claude Code Max 月费 200 美元、Pro 月费 20 美元，账单走美元卡且仍受速率限制；Qoder Pro 月费 20 美元、Pro+ 60 美元，半价活动期间 10 美元就能上手；Qwen Code 与 DeepSeek V4 走 API 路径时，单位 token 价是 Claude 主线的 30%-50%，发票合规、支付走人民币、合规审计也方便。对一个常年跑代码审计 / 大批量 PR review 的国内技术团队来说，这一档的成本与稳定性差异，比任何一项 benchmark 分数更影响选型。

四、这场迁移给国内 AI 公司带来了什么

Sensor Tower 数据里有一条被海外评论圈反复提到的信号——用户认知不是靠 benchmark 建立的，是靠产品节奏和情绪事件叠加出来的。

OpenAI 在 2024-2025 那个周期赢的是认知红利：第一个把对话式 AI 做成大众产品、第一个让"我用 ChatGPT 写邮件"变成普通人都说得出的句子。Anthropic 在 2026 年 3-4 月反向拿到一段窗口，靠的也不是单一 benchmark——是 Sonnet 4.6 + Opus 4.7 + Claude Code + Claude Design + Claude Skills cookbook 连续 3 个月不停发产品，叠加 OpenAI 自己的几次情绪事件（国防合同争议、广告化推进、GPT-5.5 上线波折），把"我现在更愿意用 Claude"这个判断装进了开发者的脑子里。

这套打法对国内 AI 公司有直接借鉴意义。 阿里 4 月 30 日同时发布 QoderWake 数字员工与 Qoder 移动端、智谱在 GLM-5 之后紧跟 GLM-5 Air、字节豆包持续打通飞书内嵌——这是节奏感。DeepSeek V4 主线一发布就把 V4-Pro 的体感差异放出来——这是产品事件感。Kimi K2.6 在 SWE-Bench Pro 难题上反超 GPT-5.4 xhigh——这是技术身份感。

中国市场的用户基数与海外不同。中国大陆 AI 应用月活已经在 4 亿量级，DeepSeek 与豆包都跨过了 1 亿月活线。这意味着国内厂商不需要追海外的 App Store 排名——它们要的是国内开发者社区的"日常工具位"。这场海外迁移的真正启发是：当用户认知建立起来，下一波可以是国内厂商自己往海外走。Qwen 在海外开源社区的存在感这两年一直在涨，DeepSeek 在 r/LocalLLaMA 的提及频次在 4 月超过了 Llama 系列——这都是真实信号。

我们这一代国内 AI 开发者特别幸运。前面有 Anthropic、OpenAI、Google 把基础设施铺出来；身边有 Qwen、DeepSeek、智谱、Moonshot、字节这一拨真在做事的国产厂商；自己写代码时今天可以挂 Claude Code，明天就能切到 Qoder + DeepSeek V4。路真的在前面。

五、今天怎么选——四个场景具体讲

写代码——Claude Code 还能用就继续用，但建议在本地装一份 Qoder CLI + Qwen 3.6-Plus 或 Qoder + DeepSeek V4-Pro 作为后备。一旦 Claude Code 限速触发或订阅到期，切换成本接近零。日常 80% 的编码任务（业务代码、单元测试、脚本、debug）国产组合已经够用；剩下 20% 的复杂跨文件重构和长链路 agent 任务，再回到 Claude Code 主线。

写文章 / 做研究——直接用 Kimi K2.6（长文与中文阅读最稳）或 DeepSeek V4（推理与逻辑严谨）。豆包适合多模态混排。如果是英文学术写作，可以挂海外 Claude / GPT，但中文场景没必要。

数据分析 / 表格 / 图表——优先 Qwen Code 的 Code Interpreter 或 Qoder 的 Agent Mode，本地化数据合规、跑 Pandas / Polars 不出网。Claude 的图表理解强但需要把数据传出去，不适合敏感数据。

翻译 / 摘要 / 客服自动化——文心一言、通义千问、豆包都已经打磨到可用档；按调用量与已有云生态选一家落地就行。这一类不需要 frontier 模型。

六、最后

Sensor Tower 的数字告诉我们海外正在发生什么。但我们这边的故事不在 App Store 排行榜上——在每一个开发者打开 Cursor、Claude Code、Qoder、Qwen Code 的早晨。

把这条迁徙轨迹放回中国 AI 产业本身的时间表里看：2023 年是国产基座模型补齐参数规模的一年，2024-2025 是开源策略与企业落地集体跟上的一年，2026 年这前四个月——尤其是 4 月这一波 Qoder + Qwen 3.6 + DeepSeek V4 + Kimi K2.6 + GLM-5 的密集发布——是国产编码工具链第一次集体站上 SWE-Bench Verified 80 分档的时间点。节奏其实和 Anthropic 那边咬得很近，差的只是把"产品事件感"和"开发者用户认知"再往前推一档。

Anthropic 这次拿到的窗口不是终点，OpenAI 也不是真要崩。海外这场用户迁移的真正信号是：当一家公司连续打三个月产品组合拳，用户的脚是会用下载与卸载来投票的。这条规律对中国 AI 公司一样成立——产品节奏稳住、技术身份立起来、开发者工具链跟得上，国内的 Qwen / DeepSeek / 智谱 / 阿里 Qoder / 月之暗面 / 字节豆包 这一阵营今天已经做好了承接的准备。

我们这一代国内 AI 开发者真正的底气，从来不是"等 Claude / ChatGPT 在国内合规上线"——是今天打开 Qoder CLI 就能写代码、打开 DeepSeek 就能做推理、打开 Kimi 就能读长文，是国产基座、国产工具、国产付费链条已经完整地、稳稳地铺到了脚下。共勉。
