英伟达 RTX Spark：Windows 机本地跑 120B 模型

5 月 31 日，英伟达在 Computex 台北的主题演讲上发布了 RTX Spark：一颗把 Grace CPU 和 Blackwell GPU 封在一起的超级芯片，最高 128GB 统一内存，官方说本地能跑 120B 参数的大模型，外加 100 万词元的上下文。它要进的不是数据中心，而是普通人桌上的 Windows 笔记本和台式机。

更值得注意的是同台的合作方：微软。Surface Laptop Ultra 成了 RTX Spark 的首发机型之一，英伟达 CEO 黄仁勋和微软 CEO 纳德拉一起站台。

这件事的核心判断是：英伟达把过去只有近五千美元的专用开发机才装得下的「个人 AI 超算」，下放成了能进消费级 Windows 机器的芯片。但别被「本地跑 120B」唬住——它的内存带宽只有约 300 GB/s，不到一张 RTX 4090 的三分之一。它真正解决的是「装得下」，不是「跑得快」。

先看英伟达官方端出来的是什么。

RTX Spark 是什么：一颗芯片，把超算搬上桌

来源：英伟达官方新闻稿 RTX Spark 发布配图

按官方公布的规格，一颗 RTX Spark 超级芯片包含这几样：

  • 20 核 Grace CPU，搭配一颗有 6144 个 CUDA 核心的 Blackwell GPU，用英伟达自家的 NVLink-C2C 把两块芯片直连。
  • 最高 128GB 统一内存：CPU 和 GPU 共用同一块大内存，不用像传统显卡那样受限于独立显存的容量。
  • 1 petaflop（每秒千万亿次）的 AI 算力。

官方给的本地能力清单也很直白：本地跑 120B 大模型加 100 万词元上下文、用本地智能体干活、渲染 90GB 以上的 3D 场景、剪 12K 视频、生成 4K 的 AI 视频，顺带还能 1440p 分辨率、100 帧以上玩 3A 游戏。

配套还有一套本地智能体运行时和一组 Windows 安全机制，让智能体能在受控的沙箱里访问本地文件、邮件、日历、甚至实时屏幕内容——全部在设备上处理，不上云。黄仁勋的说法大意是 PC 正在被重新发明，他把 RTX Spark 形容为「把英伟达积累的 CUDA、RTX、AI 平台全装进一颗超级芯片」；纳德拉则说目标是「用 Windows 把不计量的智能送到每个家庭、每张桌子」。

最关键的一句宣传是「本地跑 120B 模型」。这正是过去几年本地跑模型的人最眼馋、又最难够到的能力——而难点恰恰不在算力。

「本地跑 120B」难在哪：不是算力，是内存

来源：Tom's Hardware 关于 RTX Spark 在 Computex 2026 发布的报道

想在本地跑一个 120B 的大模型，第一道也是最硬的门槛，是内存装不装得下。

一个 120B 模型，就算压到 4 位量化，权重也要约 60GB 以上，再加上对话上下文占用，没有 100GB 上下的内存根本放不进去。一张 RTX 4090 只有 24GB 显存，连模型的一个零头都装不下；想靠多卡拼显存，几张 4090 加主板电源机箱，价格和功耗都不是普通人扛得住的。

RTX Spark 的 128GB 统一内存，正是冲着这道门槛来的。CPU 和 GPU 共用一块大内存，128GB 足够把 120B 模型整个放进去——这是它最实在的价值：让一台桌面级机器第一次能「装得下」过去只能上服务器的大模型。

但「装得下」和「跑得快」是两回事。模型每生成一个字，都要把全部权重从内存里完整读一遍，能多快读完，取决于内存带宽。这一点上，RTX Spark 的短板就露出来了。

和已上市的机器摆在一起：带宽这一栏最说明问题

要判断 RTX Spark 跑大模型到底快不快，最好的参照不是它自己的宣传，而是已经上市、有真实成绩的同类机器。把四台摆在一起算账：

  • 英伟达 RTX Spark：约 128GB 统一内存，内存带宽约 300 GB/s（第三方报道口径，官方发布稿未给出），本地可跑约 120B，参考价位据报道约 1.8 到 2.5 万元（同样是第三方口径）。
  • 英伟达 DGX Spark（已上市的前身）：128GB，带宽 273 GB/s，跑 70B 到 120B，上市价 3999 美元、2026 年 2 月起因内存涨价上调到约 4699 美元。
  • 苹果 M5 Max：最高 128GB 统一内存，带宽 614 GB/s，70B 跑得比较顺，整机约 2 万元起。
  • RTX 4090：24GB 显存，但带宽高达 1008 GB/s，4 位量化能跑约 32B，约 1.3 万元。

把带宽这一栏单独拎出来读：RTX Spark 约 300 GB/s，只有 M5 Max 的一半都不到，不到 RTX 4090 的三分之一。

这意味着什么？4090 装不下 120B，但它能装下的模型跑得飞快；RTX Spark 装得下 120B，但跑起来会慢。 两者解决的是完全不同的问题——4090 是「小模型快」，RTX Spark 是「大模型能跑」。

它跑起来大概什么速度：看已上市的 DGX Spark

来源：英伟达 DGX Spark 官方产品页

RTX Spark 还没上市，但它的「前身」DGX Spark 已经卖了一阵，用的是同一套 Grace-Blackwell 思路、128GB 统一内存、273 GB/s 带宽，和 RTX Spark 约 300 GB/s 几乎在一个量级。所以 DGX Spark 的真实表现，基本就是 RTX Spark 跑大模型速度的预演。

社区对 DGX Spark 的实测口径相当一致：它确实能把几十 B 到上百 B 的模型装进去跑起来，但因为带宽只有 273 GB/s，生成速度被牢牢压住——跑大模型时每秒吐出的词元数，是个位数到十几的水平，远不如同价位多张消费级显卡拼出来的速度。它的卖点从来不是「快」，而是「一台安静的小盒子，能在本地完整跑通大模型、做开发和验证」。

把这个口径搬到 RTX Spark 上，结论就清楚了：它会是一台能在本地把 120B 模型完整跑通的 Windows 机器，适合做开发、验证、隐私优先的本地推理；但如果你要的是「字蹦得飞快」的日常体验，约 300 GB/s 的带宽决定了它给不了。

对国内开发者，这台机器排在哪

来源：Hacker News 首页，RTX Spark 与 Surface Laptop Ultra 同日登上讨论榜

抛开「快不快」，RTX Spark 真正的意义，是给本地跑大模型多了一种此前没有的形态。

过去想在本地跑大模型，国内开发者基本只有两条路：一条是堆消费级显卡（4090、5090），显存够大才能装下中等模型，但 120B 这种量级靠单卡装不下；另一条是上苹果的大内存机器（M5 Max、M2 Ultra），靠统一内存装得下，靠高带宽跑得还行，但软件全绑在苹果自家框架上。

RTX Spark 给的是第三种：Windows 上、英伟达 CUDA 工具链里、128GB 统一内存的桌面机器。对习惯了英伟达工具链、又想在本地跑大模型的人，它补上了「Windows + CUDA + 大内存」这个之前空着的位置。据报道，秋季上市时会有 30 多款笔记本、10 多款台式机，来自华硕、戴尔、惠普、联想、微软 Surface、微星等厂商。

当然，几个现实问题得摆明：这些是基于 Arm 架构的 Windows 机器，部分老软件的兼容性要等上市后才知道；约 300 GB/s 的带宽决定了它跑大模型是「能跑」而非「跑得快」；价格和国内上市节奏目前都还是第三方报道，官方发布稿没给。这些都要等秋季真机和真实评测出来才能下定论。

那它到底适合谁

把「装得下、但跑不快」这个底子摆清楚，适合谁就很好判断了。

  • 想在本地完整跑通大模型做开发、验证的人：这是 RTX Spark 最对口的人群。128GB 统一内存能把 120B 模型整个装下，又是熟悉的 Windows + 英伟达工具链，不用为了大内存去碰另一套框架。生成不算快，但做开发、调试、跑流程足够。
  • 看重隐私、要把数据留在本地的人：模型、文件、智能体全在设备上跑，不出本地这一点，对处理敏感数据的场景有实打实的价值。
  • 想要日常聊天、写代码「字蹦得飞快」的人：这台机器给不了。约 300 GB/s 的带宽决定了它生成偏慢，这类需求一张高带宽的消费级显卡反而更顺手。
  • 预算有限、只跑中小模型的人：跑 32B 以内的模型，一张 4090 又快又便宜，没必要为 128GB 大内存多花钱——大内存的价值只有在你真要跑 120B 这种量级时才兑现。

一句话：它是「想在本地把大模型完整跑起来」的人的机器，不是「想让模型跑得更快」的人的机器。

收个尾

回到开头那句判断：英伟达这次把 128GB 统一内存的个人 AI 超算，从近五千美元的专用开发机下放进了消费级 Windows 机器，本地装得下 120B；但它约 300 GB/s 的带宽不到 4090 的三分之一，解决的是「装得下」，不是「跑得快」。

这台机器最大的价值，不在「比谁快」，而在「让更多人第一次能在自己桌上的 Windows 机器里，把过去只能上服务器的大模型完整跑通」。配上微软 Surface 首发、30 多款机型铺货，本地跑大模型这件事，正从极客的折腾，变成摆在货架上、普通开发者也能买来就用的产品。

跑得快不快，秋季真机和社区实测会给答案。但「能装下大模型的桌面机，开始变成一个正经品类」这件事，本身就值得记一笔。
