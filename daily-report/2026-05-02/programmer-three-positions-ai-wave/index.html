Uber 烧光全年 AI 预算之后：程序员的三个站位

4 个月烧完 2026 全年 AI 预算。这不是哪家创业公司的小账本，是 Uber CTO Praveen Neppalli Naga 在 2026 年 4 月对 The Information 承认的事——「I'm back to the drawing board」。同一周，美团 LongCat-2.0 把 5 至 6 万张国产卡跑出一个万亿基座；Anthropic 估值踩过 9000 亿美元，反超 OpenAI；图灵奖得主 Mike Stonebraker 在 36kr / InfoQ 公开警告「AI Agent 全是数据库问题」。AI 浪潮第三年，水位没有退，水流方向在变。

程序员要做的不是学更多新东西，是看清自己在新水流里的站位。

一、行业：贵的不再是「写代码」

Uber 这件事最有信号意义的不是「烧完预算」四个字，是 CTO 的下一句话——「back to the drawing board」（回到画板）。意思是 2025 年底全员推 Claude Code 那一刀下去，工程效率账没算清——单位 token 单价 × 工程师调用频次 × 重试次数，叠加起来比月度预算线快 3 倍。Anthropic 给到的 Max 档月费是 200 美元每人，乘以 Uber 这种规模的工程团队，4 个月把全年额度耗光不夸张。

放在更大尺度看：

  • Anthropic 估值在 2026-04 谈到 9000 亿美元，反超 OpenAI 同期约 8500 亿；Sacra 给的另一组数字是 Anthropic 3 月年化收入 300 亿美元，同比涨约 1400%
  • 谷歌 4-24 宣布最高再投 Anthropic 400 亿美元，亚马逊在过去 80 亿基础上追加最高 250 亿美元
  • 国产 GPU 三家 Q1 2026 财报：寒武纪营收 28.85 亿、净利 10.13 亿；摩尔线程营收 7.38 亿、首次单季盈利 0.29 亿；沐曦营收 5.62 亿、亏损收窄到 0.99 亿

这些数字串起来传递的是同一件事——AI 工具的真实经济模型还在 calibration。卖方（Anthropic / OpenAI / 国产 GPU 厂商）在赌一条上行曲线，买方（Uber 这一档企业用户）在用真金白银做实测，结果发现单价低于预期不等于总账低于预期——使用频次和重试成本被严重低估。

对这一档真实账本最敏感的，恰恰是程序员自己。

贵的不再是「写代码」这个动作。 写代码这件事，2026 年的 SWE-Bench Verified 头部开源模型已经能拿到 80% 段位（DeepSeek V4-Pro 80.6%、Qwen 3.6-Plus 77.2%、Mistral Medium 3.5 77.6%）。Claude Code、Cursor、Codex CLI、Qoder、Aider 这一拨工具把工程师写代码的 wall-clock 时间普遍压缩了 30%-50%。

但贵的是「知道在什么场景该用什么、用到什么深度、ROI 怎么度量」。Uber 案例的核心教训不是「Claude Code 不好用」——它好用，是「工程团队没人在度量它真正花了多少」。这个度量能力是 2026 年最稀缺的工程岗位能力之一。

二、技术：工具链在转，不是被替代

Claude Code 限速越来越紧、Anthropic 4-30 关闭 Sonnet 4 / 4.5 的 1M context beta、Uber 烧完预算公开复盘——这些事件叠在一起，说明同一件事：单一工具栈的稳定性在下降，可替代性在上升。

国内开发者这一侧已经在用脚投票：

  • Qoder CLI 4 月底完成多模型支持（Qwen 3.6-Plus / DeepSeek V4-Pro / GLM-5 都能挂上），Pro 月费 20 美元、半价活动 10 美元
  • DeepSeek V4-Pro：SWE-Bench Verified 80.6%、输出 ¥24/M（约 $3.3/M）
  • Qwen 3.6-Plus：SWE-Bench Verified 77.2%
  • 智谱 GLM-5：4-30 在 Coding Agent 上撞见「降智」，公开复盘是 PD 分离架构里 KV Cache 跨节点复用竞态 + HiCache 分层缓存读取时序漏洞，5.1 版已修，方案被 SGLang 开源社区采纳

这一拨国产工具的成熟，把过去 3 年程序员对 Claude Code「越来越离不开」的依赖关系降到一个可逆程度。技术栈层面，工程师真正该练的能力不再是「学会用 Claude Code」，而是更上一层的判断力——工具链审计能力：

  1. 价格 / 质量 / 速度三角的实时评估——同一段需求在 Claude Code、Qoder、Cursor、Aider、自托管 vLLM 上跑出的输出，准确率、首字节时延、月度账单各自什么分布？这套评估不是一次跑通就够，是每月至少回归一次
  2. 可切换性建模——如果 Anthropic 明天再次限速 / 涨价 / 政策变动，多久能把团队迁到 Qoder CLI + DeepSeek？迁移成本是 1 周还是 1 月？这个数字必须在团队预案里
  3. 自托管的实测门槛——不是「能不能跑」，是「跑起来稳定吗」。Mistral Medium 3.5 官方说 4 GPU 即可自托管，HF 模型卡推荐 --tensor-parallel-size 8 部署，但 Q4 量化版在 AMD Strix Halo 平台 HN 用户实测只有 3 tokens/秒——硬件适配不是 OEM 报价单上能看出的
  4. 失败场景的认知——智谱 GLM-5 那次「降智」是国内大模型团队第一次把超大规模推理事故讲到工程颗粒。读懂那篇复盘文，比读 100 篇「最佳实践」更有价值——能预判自己这一头的推理服务在 40K-120K 长上下文区间会不会撞同样的坑

这套能力在简历上不容易写——它不是「精通 Python」、「熟悉 Kubernetes」那种关键词，更像「能在 24 小时内把团队从一个 LLM 供应商切到另一个，不丢生产数据、不破坏 CI/CD」。

Web 端这一侧的位移更微妙：Chrome 4 月把 Gemini Nano 装进 window.LanguageModel，Mozilla 在 W3C standards-positions 仓库 issue #1213 标 position: negative。一行 await LanguageModel.create() 就能调起浏览器里的端侧大模型——但 Firefox 用户跑不了。Mozilla 给出三条理由，最具体的一条是 Chrome 的 explainer 把模型耦合写得过死，Web 标准会绕着 Gemini 的脾气长。

前端工程师 2026 年要做的取舍变成了：要不要现在押注 Chrome only 的 API？押了能拿到端侧大模型的隐私优势 + 离线能力，不押就等 W3C 把这条路走完——可能是 2 年也可能是 5 年。这种押注本身就是一项工程判断能力。

三、岗位：工作内容在重组，不是消失

图灵奖得主 Mike Stonebraker 4-30 在 36kr / InfoQ 给出的那段话，把岗位变化的方向讲得非常具体：

「LLM 写 SQL 在公开 benchmark 上能到 80%+ 准确率。但接到真实生产仓库——表 schema 复杂、命名不规范、跨库 join、隐含外键约束——准确率掉到接近 0%。AI Agent 一旦走到 read-write 这一步，立刻面对事务、一致性、原子性这三件经典问题。」

对一线工程师的含义很具体：模型替代的从来不是某个职位 title，是这个职位每天产出的某些「工作产物的中间形态」。拆到具体岗位：

前端工程师：写组件这件事被部分替代——Cursor / v0 / bolt.new 给原型代码的速度比手敲快 5 倍。但理解产品需求里的隐含状态机、跟设计师就交互边界拍板、给后端定 API 契约、性能预算和无障碍审计这一套东西，没有 LLM 能替团队做。前端工程师 2026 年的差异化不在「会写 React」，在「能从一个 Figma 文件推断出未明说的 6 种状态、给后端开 5 条新接口、并知道哪一条要做缓存」。

后端工程师：写 CRUD 接口被替代了一半——OpenAI Codex CLI 跑过 Postgres + FastAPI 的样板代码再正常不过。但 Stonebraker 的警告恰好戳中后端的真实价值——事务边界、一致性级别、并发控制、容灾预案。Cursor 不会指出某条 SQL 在生产高峰期跑会锁住主表 30 秒，也不会替团队在 Kafka 重试逻辑里识别幂等性陷阱。

算法 / ML 工程师：训模型这件事被收敛到一小撮顶级团队——DeepSeek、千问、智谱、Mistral、Anthropic、OpenAI。其余 99% 的算法岗在做的不是「训」，是「调」——把开源基座套到自己业务的领域数据、做 SFT、做 RAG、做 evaluation 管线。这一档岗位的差异化已经从「会调参」转到「会做 evaluation」——OpenAI 哥布林事件那篇复盘讲的就是这个：reward 给偏一个标签，GPT-5.5 整代被刻进口癖；OpenAI 内部花 4 个月才定位到根因。对齐工程的硬度，是模型公司今天最缺的能力，也是高级算法岗最值钱的能力之一。

SRE / 平台工程师：Stonebraker 警告里那三件事——事务、一致性、原子性——AI Agent 走 read-write 路径时全部撞上。当 Coding Agent 自动开 PR、自动改 schema、自动跑 migration 的时候，平台工程师的工作不是被替代，是变重——给 AI Agent 设边界、做 audit log、做权限隔离、做 rollback 方案。Zig 项目把 LLM 写进 Code of Conduct 三条禁令的那个决定，本质就是开源治理这一档岗位率先发现：让 AI 提交的代码进入主干，对维护团队的总成本是负的。这个判断要靠平台工程师做出来。

架构师 / 技术 lead：变化最大。AI 工具把一线写代码的杠杆放大了 30%-50%，但放大的是初稿——架构决策的边界、跨服务的一致性约束、长期维护成本的取舍这些工作没有任何工具能替代。Uber CTO 说「back to the drawing board」，本质是在说这一波的 AI 工具上下来之后，架构层面的算账方法整个要重做——单位 token 价格、agent 调用频次、长任务并行度、retry 策略、模型切换成本，都要进入架构 spec。

没有被替代的不是某个岗位 title，是「不可替代的判断力」这个能力本身。 每个岗位上，最先被替代的是「写出来的中间产物」，最难被替代的是「为什么这么写、什么时候不该这么写」的判断。

四、收官：奋起的姿势

Anthropic 估值反超 OpenAI 那一夜，国产模型团队没闲着——美团 LongCat-2.0、DeepSeek V4-Pro、智谱 GLM-5.1、千问 Qwen 3.6-Plus 在 4 月最后一周集体出货万亿基座。中国 AI 工程师所处的工作面，比过去任何时候都完整。

但「工作面完整」不等于「站位稳」。Uber 烧完全年预算这件事在 2026 年不会是孤例，它是 AI 工具经济从「大量补贴 + 跑马圈地」过渡到「真实 ROI 度量」这个新阶段的开端。在这个阶段被冲走的，不是不会用 AI 的程序员，是只会用 AI 的程序员。

程序员的「奋起」从来不是学更多新东西，是从「代码生产者」转成三件事的合体——

  • 行业层的判断者：知道这一档工具的真实经济曲线在哪、ROI 该怎么算
  • 技术层的工具链审计师：能在 24 小时内把团队从一个供应商切到另一个，不丢生产数据
  • 岗位层不可替代的判断力：在前端 / 后端 / 算法 / 平台 / 架构每一个具体战场上，把「为什么这么做」讲清楚

水流方向已经变了。下一拨能在水面上的人，不是游得最快的，是看清水流的。
