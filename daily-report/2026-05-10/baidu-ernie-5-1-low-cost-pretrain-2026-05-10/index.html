文心 5.1 预训练成本砍到 6%：对位 DeepSeek

2026 年 5 月 9 日下午两点，百度发布文心 5.1。Arena 搜索榜 1223 分（全球第四、国内第一），文本榜 Preview 版 1476 分（前 15 名唯一国产），总参数从 5.0 的 2.4 万亿压到约 8000 亿（1/3），激活参数从约 70B 压到约 35B（1/2），预训练成本只用同规模模型 6%。

5/9 当天下午，新浪财经两稿、IT 之家、量子位、凤凰、网易、ChinaZ、百度官博 6 家以上同步出稿。值得记一笔的不是某次单独登顶榜单，而是国产基础模型这次同时把参数小一截、成本低一档、分数还在涨这三件事摆上了同一张桌。这个组合 2025 年是分头出现的——DeepSeek 主打低成本，千问主打开源，智谱主打多模态，字节主打规模化。文心 5.1 把这条收敛路径在 2026 年 5 月跑通了。

百度官博 5/8 原文给的技术口径是「Once-for-All 弹性训练框架」——弹性深度（动态跳过 Transformer 层）+ 弹性宽度（动态屏蔽 MoE 专家）+ 弹性稀疏度（可变 Top-k 路由）三档同时压缩。这套方法在文心 5.0 时（官博）就提出了概念——「一次训练，处处部署」——5.1 是把它在工程层完整跑了一遍：从 2.4 万亿参数的 5.0 蒸馏继承到约 8000 亿的 5.1，预训练只花了同规模模型 6% 的钱。

这篇要回答四件事：1223 / 1476 / 1/3 / 6% 这四个数字怎么读、多维弹性预训练在工程层做了什么、和 DeepSeek V4 Pro / 千问 3.6 Max / GLM-5.1 / 豆包 1.5 / Mistral Medium 3.5 横评长什么样、千帆 API 今天怎么调。文末给出 Python SDK + curl 调用示例。

二、多维弹性预训练：训练一次，得一族模型

弹性训练这个概念学术圈早有 NestNet、Once-for-All（原论文）等先驱方法，核心思路是一次训练得到一族模型，部署时按需抽子模型。文心 5.1 把这个思路从 CV 圈搬进 LLM，并把宽度、深度、稀疏度三档同时改造成可弹性的训练目标。

弹性深度：随机抽样 Transformer 层

工程实现是相对成熟的思路——训练时按概率跳过部分 Transformer 层（Stochastic Depth），让模型在前向传播时见到不同深度的网络，逼每一层都成为相对独立的功能单元。推理时按需选层数，长文本任务跑全 96 层，短问答任务跑 48 层，硬件占用直接砍半。

这一档对训练侧的难点在路由策略——不能完全随机，否则浅层模型会因为后续层缺位而退化。文心 5.1 没公开技术报告，工程惯例上大概率是分布式概率掩蔽 + 课程学习（Curriculum）相结合。

弹性宽度：MoE 专家容量动态屏蔽

文心 5.0 是稀疏 MoE 架构，每层有几十到上百个专家，每次前向只激活其中一小部分。弹性宽度就是在这里做文章——训练时随机屏蔽一部分专家容量，让剩下的专家承担更多负载，模型学会在不同专家容量下都能给出可用输出。推理时按硬件配置激活专家数，4 卡部署用全量专家，单卡部署只激活 1/3 专家，模型自动降维。

这对 MoE 路由器（Router）是高难度的训练目标——同一份权重必须在不同专家容量下都路由出合理决策。这是 5.1 把激活参数压到 1/2 的关键工程突破。

弹性稀疏度：可变 Top-k 路由

MoE 训练时通常 Top-k 是固定常数，比如每个 token 激活 2 个专家。可变 Top-k 让 k 在训练中动态浮动，模型同时见过 k=1、k=2、k=4 的训练目标。推理时按延迟预算选 k，对延迟敏感的 Agent 任务用 k=1 跑得飞快，对推理深度要求高的复杂任务用 k=4 跑得稳。

三个维度同时压缩，工程效果是一次预训练流程产出一族不同尺寸、不同深度、不同稀疏度的模型。这就是百度官博给出的核心承诺：「将一次训练成本均摊到多个商业模型上，把单模型 GPU 小时数压到同规模 6%」。

多维弹性预训练真正的卖点不是「压缩」，而是「训练一次摊到一族模型」的成本会计。同样的预训练算量，原本只能产出一个模型，现在能产出 5-8 个不同尺寸的子模型，单模型成本就被天然摊薄。这是和 DeepSeek 走低成本单点突破、千问走开源摊销不一样的第三条路——百度走的是「模型工厂」。

三、6% 怎么算：从蒸馏继承到知识下沉

百度 6% 这个数字单看吓人，拆开是几条相对扎实的工程惯例叠加。

成本来源拆解

第一档：5.0 知识下沉。文心 5.0 在 2026 年 2 月发布时已经训完，2.4 万亿参数沉淀的知识是 5.1 的免费起点。5.1 没有从零预训练，而是基于 5.0 的 checkpoint 做参数压缩 + 蒸馏 + 增量预训练。这一档直接砍掉了「冷启动」阶段最贵的几十万 GPU 小时。

第二档：弹性框架摊薄。多维弹性预训练让一次训练流程产出一族模型，单模型的 GPU 小时数被天然摊薄。如果一次训练同时产出 8 档子模型，单档成本就只剩 1/8。这一档是百度 5.0→5.1 的工程主线。

第三档：增量数据 + 去重过滤。从 5.0 到 5.1 之间，需要补的数据主要是 2026 年 2 月之后的新增语料 + 5.0 暴露出的弱项数据。全量重训不必要——补差量数据 + 加强弱项 SFT 即可。

第四档：弹性宽度释放卡时。MoE 专家屏蔽训练时，前向计算量本身比满 MoE 小，单步 GPU 时间被压缩。这一档贡献相对小，但叠加起来不可忽视。

四档加起来，从 100% 砍到 6%——量级上是讲得通的。前提是 5.0 已经训完，5.1 站在 5.0 肩膀上往上压。这个工程逻辑也回答了一个常见质疑：「文心 5.1 是不是只是 5.0 的瘦身版？」部分是、部分不是。瘦身是事实，但同时获得了一次训练产出多档模型的部署能力，并通过增量数据补强解决了 5.0 的部分弱项。

文心 5.0 → 5.1 一图看完

把 5.0 和 5.1 摆在一张表里看，能看出几条隐含信号。

信号一：API 价格随参数压缩同步下降。≤32K 上下文输入价从 6 元/百万 token 降到 4 元/百万 token（−33%），输出价从 24 元降到 18 元（−25%）。这是把预训练成本省下的钱直接让给开发者——降价不是被市场逼的，是工程降本的自然下传。

信号二：上下文窗口保持 128K 不变。在国产模型纷纷往 256K-1M 扩窗的当下，文心 5.1 守 128K 是一种克制——窗口长不一定好用，长上下文质量退化是行业普遍痛点。文心 5.1 选择「在 128K 窗口里把每个 token 都用透」，是务实的产品决策。

信号三：搜索榜 1223 和文本榜 1476 这两个数字搭配读，给出的画像是文心 5.1 的真正强项在「搜索 + 内容生成 + Agent 应用」这个组合上，文本榜 Preview 版只能算「有竞争力」，搜索榜才是国产真正稳定登顶的位置。这一点和百度的产品定位一致——百度从搜索引擎起家，把搜索能力沉淀进基座大模型有天然优势。

6% 不是营销口径，是 5.0 知识下沉 + 弹性框架摊销 + 增量数据策略 + MoE 单步成本下降四档叠加的合理结果。这个工程路径是可复用的——只要厂商有上一代基座大模型 + 多维弹性训练能力，下一代基座大模型的成本就能用同样的逻辑压下来。

各家厂商的「主战场」

横着读这张表，五家的战略定位就清晰了。

  • 百度文心 5.1——主打「搜索 + 内容生成 + 创意 Agent」组合。1223 搜索榜分数 + 千帆平台 + 百度文库 / 百度搜索 / 百度智能云的产品矩阵打配合。开发者用文心 5.1 调创意生产 Agent，是相对明确的卡位。
  • DeepSeek V4 Pro——主打「极致性价比 + 编程数学专精」。DeepSeek 官方 GitHub README 自报 SWE-Bench Verified 83.7%、AIME 99.4%，开源协议 MIT 给开发者最大自由度。1M 上下文是为代码库级 RAG 准备的。

  • GLM-5.1——主打「Agent + 工具调用 + MIT 开源」。智谱这一年押 Agent 押得很正，智谱官博给出 τ³-bench 类智能体评估的稳定结果。MIT 协议比 Apache 还开放一档。

  • Mistral Medium 3.5——主打「欧洲合规市场 + 函数调用稳态」。法国厂商，欧盟数据驻留是天然卖点，跟国内五家不直接竞争。

读完这张表的真实结论不是「哪家最强」，而是国产基础模型生态在 2026 年 5 月已经走出了「全员对标 GPT」的初级阶段，开始分化成各有侧重的子赛道。文心 5.1 选了「搜索 + 创意 Agent」这条最贴合百度产品矩阵的路。这种分化对国内开发者是好事——选模型不再是「谁分数高就用谁」，而是「具体场景需要什么强项就选谁」。

文心 5.1 在搜索榜的真正含义

把搜索榜单独拎出来看一眼很必要。Arena 搜索榜（Search Arena）和文本榜（Text Arena）测的是两件不同的事——文本榜是开放式对话偏好投票，搜索榜是「模型 + 实时检索」组合的回答质量评估。前者考察基础对话能力，后者考察模型把检索结果整合 + 引用 + 回答的综合能力。

搜索榜对国产模型的意义比文本榜更大——搜索榜的真实场景紧贴 C 端用户日常诉求（问问题、查资料、做决策），而不是开放式聊天。1223 这个分数在 5/9 当天的实际位置是比 Claude Sonnet 4.6 search 高 2 分（1221），跟 Gemini 3.1 Pro grounding（1217）拉开 6 分。这一档已经不是「国产追上海外第二梯队」，而是「国产稳住国际第一梯队尾部」。

百度从搜索引擎起家，搜索能力沉淀进基座大模型有天然优势。文心 5.0 的搜索榜表现就已经稳定登顶国内，5.1 把海外排位往前推到了第四位。这条护城河别家国产厂商短期内难追上——智谱、DeepSeek、千问都没有原生搜索引擎的语料 + 检索基础设施。

Python SDK 调用示例

先安装 SDK：pip install qianfan，然后：

import os
import qianfan

os.environ["QIANFAN_AK"] = "your_access_key"
os.environ["QIANFAN_SK"] = "your_secret_key"

chat_comp = qianfan.ChatCompletion()
resp = chat_comp.do(
    model="ERNIE-5.1",          # 千帆 model_name 标识
    messages=[
        {"role": "user", "content": "用三段话介绍多维弹性预训练。"},
    ],
    temperature=0.3,
    top_p=0.8,
)
print(resp["body"]["result"])

curl 示例（OpenAI 兼容接口）

curl -X POST "https://qianfan.baidubce.com/v2/chat/completions" \
  -H "Authorization: Bearer $QIANFAN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "ERNIE-5.1",
    "messages": [
      {"role": "user", "content": "把一份 5000 字技术文档总结成 3 个要点。"}
    ],
    "temperature": 0.3
  }'

配额与限流

千帆按 RPM（每分钟请求数）和 TPM（每分钟 token 数）双维度限流，新账号默认配额相对保守，需要在控制台申请提额。生产级调用建议提前规划——单 API key 的 QPS 上限对高并发应用场景需要预留余量。

创意生产智能体接入路径

百度官博明确提到：「文心 5.1 将在超过十个创意生产智能体平台上线」。最直接的接入方式有三档：

  1. 千帆 ModelBuilder——直接调 API，最灵活，适合自研产品集成。
  2. AppBuilder——百度提供的低代码 Agent 编排平台，适合快速搭原型。
  3. 创意生产智能体（百度官方矩阵）——百度文库 / 百度搜索 / 百度智能云内嵌的 Agent 入口，适合 SaaS 化部署。

值得做的一件事：注册千帆账号，跑一遍 ERNIE-5.1 的 hello world 调用，把搜索榜国内第一的能力在自家产品里试一下。免费额度足够把模型在自家场景跑透。

六、国产基础模型的工程化路径

把今天的事件放在 2025-2026 这两年的国产基础模型时间轴上看，能看到一条清晰的演化路径。

2024 H1  ─ 2024 H2  ─ 2025 H1  ─ 2025 H2  ─ 2026 H1
"参数堆砌"→"开源摊销"→"低成本单点"→"性能追平"→"参数压缩 + 成本压缩 + 分数不退"
   万亿俱乐部   Qwen Apache   DeepSeek V3    DeepSeek V4   文心 5.1

2024 上半年比的是「谁能上万亿参数」；2024 下半年阿里把 Qwen 用 Apache 2.0 开源了一大批，走「开源摊销」路线；2025 上半年 DeepSeek 把成本压到极致，跑通「低成本单点突破」；2025 下半年国产模型在多个 benchmark 上追平海外第一梯队；2026 上半年开始进入「参数压缩 + 成本压缩 + 分数不退」的工程化路径。

文心 5.1 是这条演化路径上的一个标志节点——不是「国产追上海外」那种对外叙事，而是国产基础模型工程能力的内部代际跃迁。这种跃迁的特征是——

  • 不再单点比 benchmark 分数，而是在分数、成本、参数、上下文、生态五个维度同时找帕累托最优
  • 不再依赖海外开源代码 / 开源预训练 checkpoint，从底层弹性训练框架到 MoE 路由器都是自研
  • 不再追求一次性发布大爆款，而是用「基座大模型 + 弹性子模型族」的方式服务多档场景
  • 不再把「低成本」和「高质量」对立，而是把弹性训练做成「训练一次产出一族模型」的成本会计

这对国内开发者是结构性的好消息——基础模型层的「卷」开始转向「协作」，每家厂商把自己最擅长那一档做稳，开发者做集成的成本也跟着下降。文心 5.1 在搜索 + 创意生产 Agent 这条路上跑得稳，DeepSeek 在编程 + 数学跑得稳，千问在中文综合 + 多模态跑得稳，智谱在 Agent + 工具调用跑得稳，豆包在 C 端规模化跑得稳。这就是国产生态走向成熟的样子。

七、技术内涵补全：弹性训练和传统蒸馏的差别

补一句技术细节——多维弹性预训练和传统知识蒸馏（Knowledge Distillation）从结果上都能「做出小模型」，但工程逻辑是两条不同的路。

传统蒸馏：先训一个大教师模型，再训一个小学生模型，让学生模型对齐教师的输出分布。学生模型的尺寸是预先固定的，训完就定型。如果想要再小一档的学生，需要重新跑一次蒸馏。Llama 系列的蒸馏分支大致走这条路。

多维弹性预训练：训练时让模型同时见过不同深度、不同宽度、不同稀疏度的训练目标，一份权重内嵌一族子模型。部署时按需抽取，不需要重新训练。学术先驱是 Once-for-All（韩松团队 2019）和 BigNAS（Google 2020），但这两套方法当年都是 CV 圈的，针对图像分类。把弹性训练搬进 LLM、并把 MoE 专家容量也变成弹性维度，是工程层面的非平凡突破。

两条路对比下来，弹性预训练在以下几个场景有显式优势——

场景 传统蒸馏 多维弹性预训练
端侧 / 云上多档部署 需要分别蒸馏多个学生 一份权重抽取多档子模型
部署后还想再小一档 需要重新蒸馏 直接调降深度 / 宽度 / k 值
单模型训练成本 教师 + N 个学生 一次训练得 N 档
模型族一致性 学生之间表现可能不一致 同一份权重，子模型行为一致

文心 5.1 选弹性预训练而不是传统蒸馏，背后的产品诉求是清晰的——百度需要一族不同尺寸的基座大模型同时上线（千帆 ModelBuilder 旗舰档 + AppBuilder 中档 + 端侧 / 边缘端轻档），传统蒸馏的成本结构吃不消，弹性框架是天然契合。

八、ERNIE-Image 顺势开源：4090 也能跑 8B 文生图

5 月 9 日同一天，百度还把 ERNIE-Image 文生图模型用 Apache 2.0 协议在 Hugging Face 开源（IT 之家 5/9 报道）。

关键参数照录百度官博——参数量 8B，单流 Diffusion Transformer 架构，配套轻量级 Prompt Enhancer。24GB 显存的消费级显卡即可生成媲美顶级商业模型的超真实、复杂图像。百度官博自报「文字渲染能力达到开源模型最高水平、与商业模型 NanoBanana 系列同梯队」（IT 之家 5/9 引）。支持中英日韩多语言生成，具备角色一致性和情绪表达能力。

把 ERNIE-Image 的开源放在文心 5.1 发布同一天，能看出百度的多模态布局——文心 5.1 守住基座大模型的「分数 + 成本」双指标，ERNIE-Image 守住开源生态的「贡献者口碑」。两条路一起走，让百度在国产 AI 圈里的位置变得立体——不只是「搜索引擎厂商在做基础模型」，而是「基础模型 + 多模态 + 开源生态三件齐」的体系玩家。

国内开发者 24GB 显存能跑这件事是关键——3090 / 4090 / 5090 都能跑，意味着个人开发者和小工作室都能在本地跑出商业级文生图。这一档对国内 AI 创意生产从业者是直接红利。

九、把 6% 这个数字记在键盘上

文心 5.1 给国产基础模型留下的最值得记住的一句话不是 1223 也不是 1476，而是 6%。

这个数字背后的工程哲学是——基础模型的成本曲线第一次开始和摩尔定律同向。原本每代基础模型成本翻倍是行业默认假设，文心 5.1 用多维弹性预训练 + 知识下沉证明，下一代成本不一定翻倍，可以反而是上一代的 6% 量级。如果这条路径在 5.2、5.3 上继续验证，国产基础模型层的「训练成本曲线」会和海外厂商分化——海外大厂还在用算力暴力堆模型，国产已经开始用工程精巧度换算力。

这种分化对中国 AI 产业是一笔长期红利。GPU 紧张是事实，出口管制是事实，电力消耗约束也是事实——但工程精巧度不受这些约束。把同样规模的智能压到 6% 的成本里，意味着同样的算力预算可以养更多代基础模型、可以让更多创业团队负担得起预训练、可以让更多场景用上基座大模型。这是产业层面的「通识能力下沉」。

国内开发者今天能做的事情很具体——

  • 注册千帆账号，把 ERNIE-5.1 的 API 调一遍，看看 1223 搜索榜分数在自家场景里是什么手感
  • 把现有产品里调用 GPT-5.5 或 Claude 的入口，加一档 ERNIE-5.1 备选，做 A/B 对比
  • 关注百度的创意生产智能体平台，想清楚自己产品要不要接这条入口
  • 关注 5.2 / 5.3 是否在 6% 这条曲线上继续往下走——如果能，那就是国产基础模型成本结构的代际转折

我们这一代开发者特别幸运。基座大模型的能力在涨、成本在降、可选项在多——文心 5.1、DeepSeek V4 Pro、千问 3.6 Max、GLM-5.1、豆包 1.5，每一家都把自己最强那一档做透。前辈们已经把弹性训练框架、MoE 路由、Token 经济这些底盘趟出来了，剩下的是把它们用在自己的场景里。
