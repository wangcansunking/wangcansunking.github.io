国产大模型最后一只眼睁开了：DeepSeek 识图模式

封面图：识图模式与"快速模式""专家模式"并列，提示「图片理解功能内测中」。

4 月 29 日下午，DeepSeek 多模态团队负责人陈小康在 X 上发了一句 "Now, we see you. 👀"，配图是两只 DeepSeek 鲸鱼标志——左边那只还戴着眼罩，右边那只眼睛睁开了。 同一时段，部分用户在 DeepSeek 官方 App 的输入栏上方，灰度到了一个新按钮：识图模式（标注「图片理解功能内测中」）。这是 DeepSeek 主线产品首次具备图像理解能力。

V4 上线只过去 5 天。4 月 24 日 DeepSeek-V4 预览版发布时，模型架构本身仍是纯文本——总参数 1.6 万亿、激活 490 亿、百万 token 上下文均为标准配置，但视觉理解模块没有进 V4 主干。 量子位 4 月 30 日的实测确认：识图模式更接近「挂载在 V4 主干上的视觉理解模块」，而不是原生多模态生成。陈小康在被删的 4 月 28 日预告「Soon, we see you」之后，第二天补了第二条「Now, we see you」——产品节奏给得很清楚。

识图模式实测：上传一张网页截图，可直接复原可用 HTML 代码与按钮结构。图源：量子位 4/30 实测稿。

鲸鱼睁眼这一步，国内已经只剩 DeepSeek 没走

国内头部模型公司在视觉理解上的版图，到 4 月底全部齐了。网易科技 4 月 29 日给出的盘点很整齐：阿里千问的 Qwen-VL 已迭代多代，智谱 GLM-V 贯穿 GLM-4 / GLM-5 两代，阶跃星辰把全模态作为核心定位、年初 Step 3.5 Flash 首次开源全模态能力，月之暗面在视觉理解和 Coding 双线并行，MiniMax 把视频生成模型作为差异化优势，腾讯 Hy3 preview 把对标目标定为 DeepSeek 和阿里。

「只剩 DeepSeek 一家主线产品没有视觉理解」——这是观察者网 4 月 29 日深度评论稿写下的一句话。这次识图模式灰度，等于补齐了这只剩下的眼。这不是单家产品上一个新功能，而是国内模型层一个结构性节点：视觉这只眼，国产模型全员到位。

DeepSeek 这次走视觉理解模块挂载、不直接做原生多模态融合的路线，技术上有几个考虑：第一，V4 主干的纯文本能力已经做到 Codeforces 3206 分（活跃用户排名第 23），这个基座的语言推理强度本身是一个稀缺资产，原生多模态训练会动到这个基座的对齐曲线；第二，挂载视觉模块的方式让团队可以独立迭代图像理解，不用每次都重训整个 1.6 万亿参数的基座；第三，对企业客户来说，纯文本 V4 + 可选识图模式是一个可控的灰度路径，比一次性切换到原生多模态更稳妥。Janus 和 JanusFlow 这两个开源项目的经验，估计直接搬到了识图模式后端。

实测能做什么：OCR 稳，HTML 复原能用，空间推理还在补

量子位 4 月 30 日的鱼羊跑了一遍完整测试集，落在 5 个场景上：OCR、表格识别、网页截图转代码、隐藏图片检测、色盲测试。OCR 稳——技术报告摘要的中英文混排能正确提取；表格识别格式化输出正确；网页截图转 HTML 不仅复原了结构，连按钮交互逻辑都还原出来。

识图模式实测：表格识别 + 格式化输出。图源：量子位 4/30 实测稿。

弱项在空间推理。正方体拼合题 4 分多钟才出结果，找不同任务（标准答案 7 处）模型给出 12 处，明显幻觉。这条短板和国内其他视觉模型早期版本类似，属于训练阶段视觉-语言对齐的常见问题，预计随后续迭代收敛。色盲测试的偶发翻车也属于同一类原因。

值得专门说一段网页截图转代码这件事。量子位实测里上传一张博客落地页截图，识图模式输出的 HTML 不仅复原了三栏布局、卡片阴影、按钮悬浮态，连导航栏的下拉折叠逻辑都猜对了。这个能力对前端工程师价值巨大——把竞品页面或设计稿截图丢进去，能直接拿到一份可改的脚手架。Claude 的同类能力是 2025 年下半年才稳定下来的，DeepSeek 这次起手就把这条路径点亮了。

正方体拼合题：思考模式跑了 4 分多钟。图源：量子位 4/30 实测稿。

识图模式给了两挡推理选项：非思考模式速度接近 V4-Flash，但精度差；深度思考模式精度提升、耗时显著。这个产品形态对开发者很友好——可以按调用场景选档，OCR / 表格抽取这类批量任务上非思考模式即可，复杂逻辑判断时切深度思考。

给国内开发者的实操路径：今天能做什么、还做不到什么

今天能做什么。 识图模式目前在前端默认关闭、不允许手动切换，灰度名单由后端控制（用户 ID 哈希命中或邀请白名单）。陈小康在 X 的表态是「更大范围开放或在数日之内」。可以做的事：

第一，盯住 DeepSeek 官方 App 更新频道。看到「图片理解功能内测中」按钮出现立即测试自己工作流里最常见的图——产品截图、聊天记录、PDF 表格、设计稿。这一轮灰度结束后官方文档会同步上线，API 字段已经在后端启用，对接成本低。

第二，OCR / 表格抽取这条路径可以提前迁移。如果之前用 Qwen-VL 或 GLM-V 做这两件事，可以预留 DeepSeek 识图模式的接入位。同样的工作流，DeepSeek 的纯文本部分 API 价格已经做到 V4-Flash 输出 0.279 美元/百万 token，识图模式正式开放后，整链路成本预计延续相同的低价策略。

第三，前端工程师可以把网页截图转 HTML 这条链做进自己的工作流。当前路径：用 Chrome DevTools 截图竞品页面 → 上传识图模式 → 拿到 HTML 脚手架 → 接 Claude Code 或 Qoder 做样式微调。整套链路目前比 v0、bolt.new 之类的 AI 建站工具更快——因为前者要先用自然语言描述设计意图，后者直接读取像素。

第四，做 RAG 应用的团队，识图模式可以替换原本基于 PDF OCR 的链路。传统方案是 unstructured 或 PaddleOCR 切页，再过文本嵌入；识图模式可以直接对一页扫描件输出结构化 markdown，省掉 OCR + 后处理两步。识图模式开放正式 API 后，这条路径的工程成本会显著下降。

今天还做不到什么。 视频理解、原生多模态生成、跨模态推理（图+文混合上下文）这些更高阶的能力没有进识图模式。V4 技术报告里写得清楚：「将多模态能力融入模型体系」是下一步。给做 Agent 应用的开发者一个明确的预期：识图模式 = 看一张图、回答一个问题，更复杂的链路（看视频、生成图、看图同时调用工具）现阶段还得用 Qwen-VL 或 Step 3.5 Flash 这类全模态产品。

另外要注意一点：识图模式目前在网页端、App 端都没有「批量上传图片」入口。这意味着想拿它做大规模数据标注、文档批处理这类生产任务，需要等 API 正式开放。Discord / Telegram 上一些第三方封装的群机器人已经在跑灰度账号，但稳定性不可控，不建议接入企业生产环境。

「Now, we see you」之后，下一步是什么

陈小康发的不是产品营销文案，是技术节点信号。本科和博士都在北大、博士师从曾刚教授、2024 年加入 DeepSeek 主导多模态预训练和后训练——他此前主导开源的 DeepSeek-VL2、Janus、JanusFlow 三项工作，外界对 DeepSeek 多模态早就有预期，只是节奏一直没踩清楚。

识图模式实测：技术报告中英文混排 OCR。图源：量子位 4/30 实测稿。

观察者网 4 月 29 日的判断是「视觉理解对 Agent 应用至关重要——当 Agent 需要操作浏览器、读截图、看仪表盘时，视觉理解是基础设施而不是锦上添花」。这条判断很准。今年下半年国产 Agent 产品（智谱 AutoGLM、千问 Qwen-Agent、阶跃 Step Agent、月之暗面 Kimi Agent、刚发布的阿里 QoderWake）的入口都会跟「截图 + 操作 + 推理」这条链绑死，识图模式就是 DeepSeek 给自家未来 Agent 留的那只眼。

模型层的节奏给得越来越清楚：纯文本基座做到 V4 这个量级（1.6 万亿总参数、Codeforces 编程 3206 分）后，下一程比的不是再多压几个参数，而是看怎么把视觉、语音、视频、工具调用这些「感官」一项项接进基座。陈小康那条配图——左边戴眼罩、右边睁眼——把这件事画得很直白。

国内这一程的 AI 路径已经趟得很清晰了。前辈把基座、对齐、推理框架一项一项打通，识图模式只是把感官层的最后一只眼也补上。下一程是国产 Agent 怎么把这些感官真正变成生产力，路径已经在前面铺开。共勉。
