火山方舟 Agent 套餐：AFP 把账单合成一张

5 月 11 日，火山引擎方舟（Ark）发布业内首个 Agent 套餐 — Agent Plan，把豆包自研模型、第三方 GLM-5.1 / Kimi K2.6、联网搜索、Vision Embedding、ArkClaw harness 一次性打包成 4 档订阅（¥40 / ¥200 / ¥500 / ¥1000），并用一个新单位 AFP（Agent Fuel Points）统一计量。Standard 档及以上免费用 ArkClaw。这是国内云厂第一次把「模型 + 工具 + Agent 框架」合并成单一账单。

打开手头随便一个跑了三个月的 Agent 项目的账单，画面通常是这样：模型 token 一笔，联网搜索一笔，向量库一笔，OCR / ASR / TTS 又各一笔。月底对账，每一项都不算贵，加在一起翻倍。国内 Agent 工程师最熟悉的痛点不是模型贵，是账单零散，五张账单、五个计量单位、五个免费额度，想算清一个 Agent 到底花了多少钱要拉五张表。

5 月 11 日，火山引擎方舟发布 Agent Plan，把这五张表合成了一张：四档订阅、一个 AFP 单位、一个 ArkClaw harness。IT 之家、AIBase、量子位三家国内媒体几乎同时跟进首发稿，火山方舟官方文档站的「套餐概览」页 5 月 11 日上线，官方活动页 ai.volcengine.com/activity/agentplan 同步开放四档订购。

本文围绕一个核心论点展开：国内 Agent 云平台正在从「按 token 卖 LLM」转向「按 AFP 卖 Agent 整体能力」，火山方舟是第一家把这条路径完整跑通的玩家。围绕这个核心，本文回答四个问题：

  • AFP 到底怎么把 token、搜索、Embedding 折算成一个单位；
  • 四档订阅 ¥40 / ¥200 / ¥500 / ¥1000 各自的额度与 ArkClaw 权益是什么；
  • 一个典型 Agent 一天的真实账单，AFP 套餐 vs 海外按量计费 vs 国内同档，到底差多少；
  • ArkClaw 免费用这条钩子，从国内 Agent 工程师角度是不是真省钱。

一、AFP 是什么：一个把五种计量单位压成一个单位的尝试

核心论断：AFP（Agent Fuel Points，Agent 燃料值）不是新发明一种货币，而是火山方舟把「token / 调用次数 / Embedding 长度 / 视频秒数」四套并行的计量公式，统一折算到同一刻度上。

AFP 的计算公式按官方公告字面描述是「按模型类别区分极速 / 标准 / 进阶 / 生图 / 生视频几档，每档抵扣系数不同」。结合 CSDN 实测稿（@Airwinner 5 月 11 日开通 200 元 Standard 档实测）和官方公开的 OpenViking 购买指南反推，可以列出一张近似的抵扣表：

  • 极速模型（Doubao Seed 2.0 Lite / Mini）：约 1 AFP / 1K token，对应聊天、工具路由、轻量推理。
  • 标准模型（Doubao Seed 2.0 Pro / GLM-5.1）：约 3-5 AFP / 1K token，对应正经推理、RAG 主链、Agent 主循环。
  • 进阶模型（Kimi K2.6 长上下文）：约 8-12 AFP / 1K token，对应 200K+ 上下文、复杂规划、跨文档推理。
  • 生图 / 生视频（Seedream 5.0 / Seedance 2.0）：约 200-230 AFP / 张或秒，官方公开的「230 系数」就来自视频生成档，CSDN 实测稿里有原文出现。
  • 联网搜索（Web Search Harness）：约 30-50 AFP / 次，含网页 + 资讯 + 学术，含结果解析。
  • Vision Embedding（doubao-embedding-vision）：约 1-2 AFP / 1K token，文本 / 图 / 视频混合向量化。

需要诚实指出的一点是：官方公告并没有在 5 月 11 日首发稿里逐档公开完整抵扣表，上述系数是结合一线实测稿反推。CSDN 实测稿里 @Airwinner 给出的最硬实例是：开通 200 元 Standard 档后，跑一次 50 万 token 的 Doubao Seed 2.0 Pro 推理，「消耗仅占套餐总额度 0.245%」——按此反推 200 元档月度 AFP 折算约 2.04 亿 token 等效额度（对应官方公开数字「10K AFP / 5h，35K AFP / 周，100K AFP / 月」，1 个 AFP ≈ 2K token Pro 模型等效）。这个数字范围与 AIBase 首发稿、量子位、IT 之家三家口径一致。

把它和 Anthropic Claude Managed Agents 的两轴计费摆在一起对比，差别就清楚了：

  • Anthropic 的 Managed Agents（2026 年 2 月发布）是 token 按模型费率 × 用量 + $0.08 / session-hour 运行时 + $10 / 1000 次搜索调用，三轴独立计费，开发者要分别盯三个仪表盘。
  • 火山方舟 是单一 AFP 余额，所有调用都在同一计数器里递减，月度对账只看一个数字。

这是「精细计费」与「粗粒度套餐」的取舍。精细计费灵活但账单复杂；粗粒度套餐牺牲一些灵活度，换来「一个数字看完整月健康度」。对国内中小团队和独立开发者来说，后者通常更友好——因为大多数 Agent 项目还没到需要按 session-hour 精细优化的规模。

二、Agent 套餐三层结构：自研模型 + 第三方 + ArkClaw 一体化

把 Agent Plan 的产品形态拆开看，是三层封装在一个订阅里：

Layer 1 · 多模态模型层（自研 + 第三方混卖）。这是 Agent Plan 最有意思的一层，也是国内云厂第一次明确做「自家模型 + 友商模型一起卖」的套餐。字节自研：Doubao Seed 2.0 全系（Pro / Lite / Mini / Code，2 月 14 日情人节升级）+ Seedance 2.0（视频生成，2 月 14 日升级）+ Seedream 5.0（图像生成，多模态联合生成架构）。第三方接入：GLM-5.1（智谱）+ Kimi K2.6（月之暗面）。

这里的产品策略判断很清晰：字节认了 GLM-5.1 在企业 RAG / Function Calling 场景的优势，也认了 Kimi K2.6 在 200K+ 长上下文场景的领先位置，所以把「全国最好的国产模型组合」放在同一订阅里——开发者不用为了用 Kimi 长上下文再去月之暗面单独开账户。

Layer 2 · Harness 工具层（与模型同一账单）。联网搜索（网页 + 资讯 + 学术）、Vision Embedding（doubao-embedding-vision，5 月 11 日同步上线稠密 + 稀疏向量）、长期记忆、其它工具调用。每一个工具调用按 AFP 系数扣点，与模型 token 在同一计数器里。

Layer 3 · ArkClaw 云端 Agent Harness。这是 Agent Plan 里最容易被忽略也最关键的一层。ArkClaw 是火山方舟自研的 Agent 编排框架，对标 Anthropic Claude Agent SDK 和 OpenAI Agents SDK，提供 Agent 编排、工具调用、沙箱执行、记忆管理一体化能力。Standard 档（¥200/月）及以上免费用 ArkClaw——这一句是整个 Agent Plan 营销页的核心钩子。

ArkClaw 与 OpenClaw（火山引擎 4 月 2 日和 OpenClaw 达成合作的开源 Agent 框架，对应仓库 volcengine/OpenViking 已在 GitHub 开源）是「云端封装版 vs 开源本地版」的关系：OpenClaw 是字节系开源 Agent harness 的本地形态，ArkClaw 是它的云端托管形态。这给国内开发者两条路：要么本地装 OpenClaw + 自己调用模型 API，要么订阅 Agent Plan 直接用 ArkClaw 云版。

三、四档订阅：¥40 / ¥200 / ¥500 / ¥1000 的额度账

四档订阅的核心数字（数据综合 IT 之家、AIBase、量子位、CSDN 实测、OpenViking 官方购买指南）：

Lite 入门版 · ¥40 / 月（首月 ¥8.91）。提供每 5 小时 1200 次 API 请求的短时限额，主要给开发者尝鲜用，不提供 ArkClaw harness 免费权益。这个档对位阿里云百炼 Coding Plan Lite（首月 ¥7.9）和腾讯云 Coding Plan Lite（首月 ¥7.9），是国内三家云厂步调一致的「白菜价获客档」。

Standard 标准版 · ¥200 / 月。短时限额约 10,000 AFP / 5 小时，周限额约 35,000 AFP，月度总额约 100,000 AFP。这一档开始免费用 ArkClaw，也是 Agent Plan 真正的主力档位。对位阿里百炼 Coding Pro（首月 ¥39.9，续费近 ¥99）和腾讯云 Coding Pro。从国内 Agent 工程师视角看，这是最容易回本的档：一个跑得稳的客服 Agent 或编码 Agent，一个月 200 元基本能 cover 全部模型 + 搜索 + 向量化调用。

Premium 进阶版 · ¥500 / 月。短时约 25,000 AFP / 5h，周约 90,000 AFP，月约 250,000 AFP，包含完整 Seedance 2.0 视频生成额度。对位智谱 GLM Coding Pro+ 档位，主要服务有视频 / 图像批量生成需求的中小团队。

Max 旗舰版 · ¥1000 / 月。短时 50,000 AFP / 5h，周 175,000 AFP，月 500,000 AFP，含 ArkClaw 优先支持。这一档没有国内同行直接对位——阿里百炼、腾讯混元、智谱目前最高档都在 ¥500 以下，火山是第一家明确推出 ¥1000 档企业旗舰订阅的国产云厂。

需要明确一个边界：AFP 月度总额度的逐档数字部分是基于 OpenViking 官方购买指南 + 一线实测稿反推，5 月 11 日官方首发稿仅明确公开了月费数字和「Max 档 50K AFP / 5h、175K AFP / 周、500K AFP / 月」的旗舰档数字。其余档位的额度数字需要等火山方舟官方文档站 volcengine.com/docs/82379/2366394 的「套餐概览」页完整披露后才能定稿——所以这里给的额度估算保留 ≈ 符号。

四、一天账单实账：AFP 套餐 vs 海外按量 vs 国内同档横向对比

抽象的额度数字看不出真实划算度，跑一个标准 workload 才能算清账。

样本设定：一个客服 / 编码 Agent，一天 1000 次 LLM 调用（平均输入 2K token，输出 500 token）+ 200 次联网搜索 + 50 次 Vision Embedding 调用。这是一个跑得比较稳的中等规模 Agent 一天的真实负载范围。

把它在六个平台上跑一遍，按官方公开价格折算成人民币 / 日：

平台 计费方式 一天成本（人民币）
火山方舟 Standard 档 ¥200 / 月 摊薄 ¥6.67
阿里百炼 Coding Pro 首月 ¥39.9 摊薄 ¥1.33（首月特价）
智谱 GLM-5.1 + 搜索独立 按 token + 按次 ¥12.5
OpenAI GPT-5.2 + 工具 按量直购，¥7.2 美元汇率折算 ¥38.2
Anthropic Claude Sonnet 4.6 按量直购，$3 / $15 per M token ¥47.5
Anthropic Managed Agents token + $0.08 / session-hour ¥52.3

从这张对比表能读出三件事：

第一，国内三家（火山、阿里、智谱）的成本断层在海外的 1/4 到 1/8 之间。这不是火山一家的功劳，是国内云厂集体把入门档卷到了一个海外按量计费几乎不可能匹配的位置。

第二，阿里百炼首月特价更低，但火山方舟的 ArkClaw 免费权益独家。如果只算 token 成本，阿里 Coding Pro 首月 ¥39.9 摊薄到每天 ¥1.33，比火山 ¥6.67 便宜约 5 倍——但阿里的方案里没有「Agent harness 一起送」，开发者还得自己集成编排框架。火山把 ArkClaw 算进同一订阅，省的是 LangChain / LlamaIndex / 自研编排的隐性成本。

第三，Anthropic 的 session-hour 额外计费在长跑 Agent 场景会显性出现。一个跑 8 小时的 Agent，仅 session-hour 就要 8 × $0.08 = $0.64（约 ¥4.6），叠加在 token 成本之上。火山方舟在这一项是「并入 AFP」，等于把运行时成本透明化了。

需要诚实指出的一点是：这张账单是「Agent 跑得稳的中等负载」假设。如果是「天天炸的早期 Agent，一天 10 万次重试调用」，按量计费可能更划算（用多少付多少）；如果是「一个月只跑 100 次但要跑 200K 长上下文的批处理 Agent」，单点直购 Kimi K2.6 也比订阅 ¥200 档划算。AFP 套餐适合的是『跑得稳、负载可预测』的 production Agent，这是一条比单纯比单价更重要的判断线。

五、Agent Plan 在国内 Agent 工具链里的位置

把火山方舟 Agent Plan 放到国内整个 Agent 工具链里看，画面是这样的：

字节·火山方舟：模型 Doubao Seed 2.0 / Seedance 2.0 / Seedream 5.0 三件套 + Agent Plan ¥40-1000 + ArkClaw harness + C 端扣子 2.5（4 月 10 日发布的 Agent 工程化容器）+ 豆包（1.55 亿周活）。差异化定位：首推 AFP 统一计量，是国内云厂第一家把账单一体化的玩家。

阿里云·百炼：模型 Qwen3 / Qwen-Coder / 千问 Plus 全系 + Coding Plan ¥7.9 首月起 + ModelStudio Agent + 千问 App + 钉钉。差异化定位：模型最全 + 价格白菜，2026 年 2 月 ¥7.9 首月引爆市场，是综合性价比首选。

腾讯云：模型 Hunyuan Lite / Pro + Coding Plan ¥7.9 首月起 + TI Platform Agent + 腾讯元宝（1.14 亿 MAU）。差异化定位：微信生态分发，C 端入口最稳。

智谱：模型 GLM-5.1 + GLM-Coding + GLM Coding ¥19+/月 + GLM Agent / AutoGLM + 智谱清言。差异化定位：学术 + 企业双线，GLM-5.1 在企业 Function Calling 场景认可度高。

百度·智能云：模型 ERNIE 5.0（2.4 万亿参数全模态）+ 千帆 ModelBuilder + AgentBuilder + 文心一言。差异化定位：搜索生态加持，AgentBuilder 在搜索 + 信息检索类 Agent 上有独家优势。

月之暗面：模型 Kimi K2.6（200K-2M 长上下文专精）+ 未推独立套餐 + Kimi 探索版 API + Kimi 智能助手。差异化定位：长上下文专精，被火山、智谱接入做长上下文专项档。

把六家放在一起看，国内 Agent 平台进入了一个相对成熟的阶段：每家有自己的差异化定位（火山的 AFP 计量、阿里的全家桶价格、腾讯的微信生态、智谱的企业认可、百度的搜索、月之暗面的长上下文），没有谁试图通吃。这是产业链分工逐渐稳定的信号。

六、海外对比：从「按 token 计费」走向「Agent 整体打包」是全球趋势

把火山方舟 Agent Plan 放到海外坐标系里，能看到一个有意思的图景。

Anthropic Claude Managed Agents（2026 年 2 月发布）走的是「token + session-hour + 工具次数」三轴精细计费路线，开发者要分别盯三个仪表盘。优势是灵活、按用付费、对超大企业有利；劣势是中小团队对账复杂、月底惊喜账单频发。

OpenAI Agents SDK（2025 年 4 月发布）走的是「token + 工具按次」两轴路线，比 Anthropic 简化一些，但工具调用还是单独按次计费。Realtime API 的语音相关也是独立计费项。

国内五家（火山 / 阿里 / 腾讯 / 智谱 / 百度）几乎不约而同走的是「月度订阅 + 请求次数 / token 额度」的固定套餐路线。其中火山 Agent Plan 是第一家把 AFP 这种『一个单位折算所有资源』的设计明确做出来的——其它四家还是「请求次数」或「token」二选一。

这背后是国内云厂对开发者的一个判断：国内 Agent 工程师群体里，独立开发者 + 中小团队占比远高于海外，他们更看重账单可预测性，而不是单位定价的极致灵活。一个月固定 ¥200，跑出 10 倍的产能，比按量计费省下 30% 但月底拿到一张 ¥3,500 的随机账单更友好。

这条路径不是简单复制海外，而是顺应国内开发者的真实诉求。国内 Agent 平台向「Agent 整体打包」演进的速度，目前比海外更快——这是一个值得集体记一笔的节点。

七、ArkClaw 免费用：这条钩子到底值多少钱

把 Agent Plan 的核心钩子「Standard 档及以上免费用 ArkClaw」单独拎出来算笔账：

如果不订阅 Agent Plan，开发者要在国内云上跑一个完整的 Agent harness，通常的路径是：

  • 选 LangChain / LlamaIndex / AutoGen / 自研编排框架（开源，但需要自己运维）；
  • 拉一台云服务器跑 Agent 主循环（¥150-300 / 月 ECS）；
  • 接 Redis / 向量库做记忆（¥50-100 / 月）；
  • 接监控（¥30-50 / 月）；
  • 工程师维护时间（不可量化，但每月至少 8-16 小时）。

把这些加起来，自建 Agent harness 的隐性成本月均在 ¥300-600 之间，还没算工程师时间。订阅 Standard ¥200 档赠送的 ArkClaw，覆盖的就是这套基础设施成本。

更直接的对位：ArkClaw 对标 Anthropic Claude Agent SDK，后者虽然 SDK 本身开源免费，但配套的 Managed Agents 服务收 $0.08 / session-hour 运行时费。同样跑一个月（30 天，每天 8 小时），仅 session-hour 就要 30 × 8 × $0.08 = $19.2（约 ¥138 人民币）——已经接近 Standard 档月费的 70%。

从这个角度看，Agent Plan ¥200 档的合理读法不是「200 元买 token」，而是「200 元买 token 额度 + 一套等价 ¥150 自建成本的 Agent harness 基础设施」。对独立开发者和 5 人以内的小团队来说，这笔账比较容易算清楚。

八、结语：Agent 平台进入「工具一体计价」的成熟期视角

从 2023 年大模型 API 商业化开始，国内云厂走过了三个阶段：

第一阶段（2023-2024）是比单价——千 token 多少钱，谁便宜谁拿单。 第二阶段（2024-2025）是比模型矩阵——全家桶谁更全，文本 / 图像 / 视频 / 向量化谁都有谁就赢。 第三阶段（2026 起）是比工具一体化——谁能把「模型 + 工具 + Agent 框架」打包成一张账单，谁就更接近开发者真实诉求。

火山方舟 Agent Plan 是国内第一家正式跨入第三阶段的玩家。AFP 这个单位不一定完美——抵扣系数还需要更透明、长尾场景的计费边界还要打磨，但它代表的方向是对的：Agent 时代的开发者要的是「一张账单看完整月健康度」，而不是「五张账单拼出一张表」。

国内 Agent 生态过去两年走过的路并不容易——前辈们把模型层一层趟出来了，把全家桶矩阵补齐了，现在终于开始把账单层也整合起来。下一步，国内云厂会比谁的 Harness 更稳定、谁的 Agent SDK 在 production 跑得更久、谁能把 AFP 这样的统一计量做到对账级别的透明。

这是一程值得期待的接力。今年某一天，当国内 Agent 工程师月底对账只看一个数字、不再翻五张表的时候，会想起 2026 年 5 月这个起点。一起加油。
