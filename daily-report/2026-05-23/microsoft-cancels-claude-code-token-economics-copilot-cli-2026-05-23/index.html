微软暂别 Claude Code：国内自建路反而走通了

5 月 15 日前后，The Verge、Windows Central 等多家海外科技媒体相继披露：微软 Experiences + Devices 事业部高级副总裁 Rajesh Jha 发出一份内部备忘录，要求该事业部工程师在 6 月 30 日（微软 2026 财年结束日）之前，把大部分 Claude Code（Anthropic 官方 AI 编程命令行工具）的内部许可逐步交回，统一迁回 GitHub Copilot CLI（GitHub 官方 AI 编程命令行工具）。这则消息在 36 氪、X 平台、Reddit 同步发酵。表面上是工具切换，底子里是 Token 按量计费的账单——同一时段，Uber CTO 公开承认 2026 年全年 AI 预算「四个月就烧光了」，主要原因是工程师对 Claude Code 的重度使用。

这件事如果只从「微软抠门、Anthropic 委屈」这条线去读，会错过它真正有价值的部分。它把过去半年悄悄发酵的一个事实摊开了：按 Token 计费的 AI 编程工具，在重度使用场景下，账单已经超过了不少大厂能稳定接住的体量。换个角度看，对国内开发者反而是一个意外的好消息——DeepSeek（深度求索）V4 Pro 的缓存命中价做到 0.003625 美元/百万 token、本地千问 / Kimi 自建 + Cursor / Copilot CLI 混合的路径，在过去半年悄悄走通了一条比纯按量便宜一个量级的方案。

国内同行刷到这条新闻的第一反应，大多是「微软都用不起 Claude Code，那我那点机器学习预算还能撑住吗」。这种焦虑不必有。这件事真正的关键词，不是「谁打败了谁」，而是 Token 经济学——一台旗舰模型在重度 agent 工作流下，每天烧掉的 Token 数量已经走到了让所有按量付费方案都要重新算账的临界点。微软这次的选择，本质是把账单从「外部按量」切回「内部包月 + 自家议价」，逻辑跟国内团队过去半年逐步把高频任务迁到 DeepSeek 缓存命中 + 本地千问自建上是一回事。

一、事件时间线：从 12 月放开到 6 月 30 日切回

把这次切换的关键节点按时间排开，能看清楚它不是一次突发决策。

  • 2025 年 12 月：微软首次对内开放 Claude Code 使用许可。员工拿到一份 Anthropic 的企业许可，可以在内部代码库上跑 agent 工作流。
  • 2026 年 2-3 月：The Verge 据微软内部人士披露，Claude Code 的使用量在六个月内快速攀升，部分团队反馈它在多步骤 agent 任务上比当时的 GitHub Copilot CLI 更顺手。
  • 2026 年 5 月 15 日：Rajesh Jha 发出内部备忘录，The Verge 资深记者 Tom Warren 拿到原文。备忘录原话节选据 The Verge / Windows Central 转述为：「我们最初同时提供 Copilot CLI 和 Claude Code，目的是在真实工程流程中做对比、学到东西。Claude Code 是这个学习过程的重要一环……Copilot CLI 给了我们另一样特别重要的东西：一款可以和 GitHub 直接打磨，匹配微软自家代码库、工作流、安全要求和工程需求的产品。」
  • 2026 年 6 月 30 日：微软 2026 财年结束日，也是大部分 Claude Code 许可的回收截止日。Experiences + Devices（覆盖 Windows、Microsoft 365、Outlook、Teams、Surface 五条产品线）作为首批团队，要在这一天前完成迁移。
  • 后续：据公开报道，Claude 模型本身并未从微软内部消失——它会以 GitHub Copilot CLI 后端选项的形式继续提供。换句话说，模型还在，只是计费入口从 Anthropic 切回了微软自家的 Copilot 平台。

需要明确标注的几点单源待官方确认事项：「91% 工程团队使用 GitHub Copilot」是微软去年公开口径，过去六个月被 Claude Code 部分蚕食的具体新数字，截至本文截稿微软尚未更新；备忘录原文目前以 The Verge 转述为主，Anthropic 也未对此次内部决策做正式回应。

二、Token 经济学：Uber 四个月烧光全年预算的算法

如果只看微软这一家，故事还不够完整。同一时段最直接的旁证，是 Uber。

据 Yahoo Finance、AI Magazine、Storyboard18、briefs.co 等多家媒体汇总的公开口径，Uber 在 2025 年 12 月把 Claude Code 推广到工程团队，到 2026 年 2 月使用量翻倍——开发者发现它在多步骤 agent 任务上效率显著。结果是：Uber 2026 年全年 AI 预算，在四个月之内烧光，CTO 公开承认要「重新做预算」，盘算公司能否在规模化场景下持续负担这种生产力提升的成本。

具体到单人账单，Uber 公开的口径是：平均每位工程师每月 AI 相关支出 150–250 美元，重度用户 500–2000 美元。把这个数字换算成人民币，按 1 美元 ≈ 7 元的口径，相当于一名重度开发者每月在 AI 编程工具上花 ￥3,500–￥14,000。Uber 工程团队规模庞大，乘起来就是上亿美元级别的年度账单。同期 Uber 还有另一组数据值得记下来：95% 工程师每月使用 AI 工具，70% 的提交代码源自 AI，84% 已经被归类为 agentic coding 用户。

这就是 Token 按量计费的双刃剑。生产力是真的，账单也是真的。

把五种主流路径放在同一张图上看，账单层次差距一目了然。

工具 计费方式 单人月度成本（人民币口径） 主要适用场景
Claude Code Anthropic 按 token ￥3,500 – ￥14,000 重度 agent 工作流 / 长链路代码生成
GitHub Copilot CLI 微软包月 + Credits ￥280 – ￥1,400 中度日常 / 已在 GitHub 生态团队
Cursor 包月分级 ￥140 – ￥1,400 个人 / 中小团队 IDE 重度用户
DeepSeek V4 Pro API 按 token + 缓存命中折扣 ￥60 – ￥320 高复用 prompt / 长上下文重复调用

几个口径需要说在前面：Claude Code 区间是 Uber 公开口径折算结果，包含完整 agent 工作流的 token 消耗；DeepSeek 区间按 200 万 token / 月 + 80% 缓存命中估算；Cursor / Copilot CLI 包月本身便宜，但重度用户都会撞到包月上限触发用量阶梯，所以高位区间也不低。Antigravity 当前免费是预览策略，未来定价待 Google 正式公布。

三、横评五家：从纯按量到包月 + Credits 到本地自建

把账单先放一边，光从工具形态看，五条路其实代表了五种不同的 Token 治理思路。

Claude Code：Anthropic 官方 AI 编程命令行，主打长链路 agent 工作流。它的强项是模型本身（Claude Opus / Sonnet 系列）在多步骤推理上的表现稳定，缺点是 token 消耗直接走 Anthropic 计费表，没有包月封顶。对小团队和独立开发者，它在「质量优先、不计成本」场景仍然好用；对大厂，它的问题不是工具好不好，而是预算不可预测。

GitHub Copilot CLI：GitHub 官方 AI 编程命令行，微软在 6 月 1 日把整条 Copilot 产品线切到了用量计费——Pro 档 10 美元 / 月含 10 个 AI Credits，Pro+ 档 39 美元含 39 个 AI Credits，Business 档 19 美元 / 用户 / 月含 19 个 Credits，Enterprise 档 39 美元 / 用户 / 月含 39 个 Credits。1 个 Credit = 0.01 美元。表面上还是包月，本质是包月含一个起跳额度，超出按量再付费。微软这次把 Experiences + Devices 切回 Copilot CLI，背后还有一层：用同样的预算，微软对自家平台的议价权远大于对 Anthropic。

Cursor：AnySphere 公司的 IDE，Pro 档 20 美元 / 月、Ultra 档 200 美元 / 月，包月分级。它在国内有大量个人开发者用户，强项是 IDE 内一体化体验。对中度用户，包月体验比纯 API 友好，但 Ultra 档对应的是重度用户加买额度的妥协价。

DeepSeek V4 Pro API：深度求索的旗舰模型 API，5 月 22 日当下的口径是：缓存命中输入 0.003625 美元 / 百万 token、缓存未命中输入 0.435 美元 / 百万 token、输出 0.87 美元 / 百万 token，并附带一个 5 月 31 日截止的 75% 折扣。缓存命中价是发布价的 1/10——对那些 prompt 前缀高度复用的 agent 工作流（系统提示 + 项目上下文常驻），缓存命中率轻松上 80%，整体单价就被拉到了一个国外按量方案完全无法对比的水平。这条路过去半年在国内独立开发者圈子里走得很顺。

Antigravity：Google 推出的 agent 工作流工具，当前处于预览期免费阶段。它适合用来跟最新形态、做对照实验，但商用定价还没到摊牌的时候。

需要补一句客观陈述：Cursor 这种 IDE 一体化体验、Copilot CLI 这种与 GitHub 仓库深度耦合的产品，本身的产品价值不该被「token 单价」一个维度抹掉。微软这次切回 Copilot CLI，部分原因正是它能跟微软自家代码库的安全审查、合规要求、CI/CD 工作流直接打磨。

四、国内同行的三条降本路径 · 对比与实操

把视角拉回国内。重度 AI 编程工具用户在过去半年其实已经悄悄分出了三条主流路径。把它们摆在一起，比单看一个数字更有信心。

路径一：DeepSeek V4 Pro API + 缓存命中。适合独立开发者和小团队。组合是 Cursor 或本地 IDE 接 DeepSeek API，靠系统提示 + 项目上下文长期复用拉高缓存命中率。月度成本控制在 ￥60–￥300，质量在中文场景接近第一梯队。5 月 31 日前还有 75% 折扣可以吃。

路径二：本地千问 / Kimi K2 自建。适合对数据合规敏感的中型团队。组合是 vLLM 或 SGLang 把千问 Qwen3-Coder 30B 系列、Kimi K2 等模型起在内部 GPU 上，比如双 4090、H20 或 Blackwell B200。硬件一次性投入之后，推理边际成本接近 0，长期高频场景非常划算。代价是初期工程投入和模型选型试错。

路径三：混合 · 包月 + 按量备用。适合已经习惯 Cursor / Copilot CLI 的团队。日常走包月免心智负担，重活和长上下文重复调用走 DeepSeek API 缓存命中兜底。月度成本约 ￥280–￥800，体验顺畅，账单可预测。

这三条路有一个共同点：都没有把全部账单交给一个国外按量厂商。微软、Uber 这次撞上的问题，国内重度开发者过去半年用「DeepSeek 缓存命中 + 本地自建 + 包月 IDE」的组合悄悄绕过去了。这不是凑巧——是 Token 单价被压到 0.003625 美元 / 百万的国产基础设施，加上千问 / Kimi 这类开源旗舰本地可部署的现实，给了国内一条独特的解法。

五、客观陈述：这件事的边界在哪里

最后一节留给客观陈述。

第一，这是 Experiences + Devices 一个事业部的内部决策，目前公开报道并未确认覆盖微软全部工程团队。其他事业部（Azure、Office、安全、GitHub 本身）的内部许可状态，截至发稿没有更新消息。

第二，Claude 模型本身没有从微软消失。它依然以 GitHub Copilot CLI 后端选项的形式继续提供。这件事的本质是计费入口切换，不是模型替换。

第三，91% 这个数字是微软去年公开口径，过去六个月被 Claude Code 部分蚕食的新数字尚未公开。在引用时应当明确「微软去年公开口径」这个时段限定。

第四，Anthropic 截至本文截稿未对此次切换做正式回应。Boris Cherny（@bcherny，Claude Code 项目负责人）的公开账号也未就此事发声。这部分继续观察，等待官方表态。

第五，AI 软件价格过去一年「涨 20%–37%」这个数字，是 36 氪原文的一句行业陈述，没有列出具体来源。Gartner 2026 AI 市场报告反而给出另一种数据——平均 AI 软件价格自 2024 年以来下降 15%，主要因为竞争和效率提升。两组数据指向同一件事：按量计费让「平均单价」失去意义，真正贵起来的是「重度用量场景下的总账单」。所以涨没涨百分之几不重要，重要的是有没有把 prompt 缓存命中率和包月封顶用好。

六、读完之后

如果只能记一件事，那就是：Token 按量计费的账单，已经走到让大厂工程预算都要重新做的临界点。这不是哪家工具「不行了」，是整个按量模式在重度场景下的财务压力被显性化了。

对国内开发者，这是一个意外的好消息。过去半年，国内 AI 编程基础设施在 Token 单价这件事上做了非常扎实的活——DeepSeek V4 Pro 把缓存命中价压到 0.003625 美元 / 百万 token，千问 / Kimi 系列把本地可部署的开源旗舰拉到第一梯队，Cursor / Trae / Qoder / Antigravity 这些 IDE 工具又把日常体验做到顺畅。把这三层组合起来，国内重度开发者其实已经走通了一条比纯按量便宜一个量级、又不牺牲质量的路。

工具会换、计费会变、单价会涨会跌。真正能沉淀下来的是：手上有一份能跑得起重度 agent 工作流的本地 / 国产组合，并且账单可以预测。微软这次切换给所有人提了个醒，但对照着自家路径看，国内同行可以踏实一些。
