微软 Project Solara：没有 app，智能体直接接管设备

6 月 2 日的 Build 2026 上，微软发布了 Project Solara。台上演示的两台设备，都没有一个 app 图标：一枚挂在胸前的智能工牌，一块放在桌上的智能屏。你对它说话，它直接把事情办了。

这容易被当成「又一个语音助手」一笔带过。但 Solara 不是一个功能，是一套操作系统级的平台——微软瞄准的是下一次平台跃迁。

微软认为，继 PC 的窗口、手机的 app 之后，下一个交互范式是『智能体』：你不再点开一个个 app 去操作，而是把任务交给设备上的智能体直接完成。Project Solara 就是给这一类『没有 app』的新设备做的操作系统。它先不进手机，而是从企业场景的工牌和屏幕切入。 这篇讲清楚 Solara 是什么、现场那两台设备怎么干活、以及微软为什么从企业而不是手机起步。

先看 Solara 到底是什么。

Solara 是什么：给「没有 app」的设备做的操作系统

来源：Engadget 关于微软 Project Solara 发布的报道 engadget.com

Solara 的核心定位，是一套面向「智能体优先」设备的操作系统平台。它的关键设计有三点：

  • 用智能体取代 app：传统设备上，每个功能对应一个 app，你要自己点开、操作；Solara 上没有 app 这一层，你把意图说给设备，设备上的智能体调动数据和服务直接执行。
  • 建在 Android 之上：技术根基是微软的 MDEP（Microsoft Device Ecosystem Platform，微软设备生态平台），这是微软为企业设备做的一个 Android 分支。也就是说，Solara 不是从零写的新内核，而是站在 Android 这套成熟系统上。
  • 面向一整类新设备，不是单一产品：微软把 Solara 定位成「平台」，让接下来一波智能体优先的硬件都能跑在它上面，而不是只配一台机器。

一句话：Solara 之于智能体设备，相当于 Windows 之于 PC、Android 之于手机——它要做那一层让智能体能在设备上干活的底层系统。 而这套系统好不好用，得看跑在上面的设备。Build 现场，微软端出了两台。

现场两台设备：一枚工牌、一块屏，都替你把事办了

来源：TechRadar 关于 Project Solara 概念设备的报道 techradar.com

微软没有用一台手机来演示 Solara，而是用了两台形态完全不同的设备，正好说明它「面向一类设备」的野心。

设备 形态 现场演示的活
智能工牌 挂在胸前的可穿戴徽章 面向医护：扫患者二维码、录音并转写整次问诊、记录生命体征、发起开处方
智能屏 放在桌上的台面屏 接入 Microsoft 365：显示 Outlook 即将到来的日程、读取 Excel 数据、接受语音指令、按指令替你执行任务

这两台设备最值得注意的是那枚医护工牌：它把一线医护最耗时的杂活——记录、转写、录生命体征、开处方——交给了胸前这枚徽章上的智能体。医生只要正常问诊，工牌在旁边把病历的活儿默默做完。这正是「没有 app」最直观的样子：医生不需要在工牌上点开「录音 app」「病历 app」「处方 app」，他只管看病，智能体接管了流程。

智能屏则是同一套思路的桌面版：它不是让你在屏幕上戳一堆图标，而是让你开口，它去 Microsoft 365 里把日程、数据、任务办好。

两台设备形态天差地别，底层都是同一个 Solara——这恰恰是微软想证明的：这套操作系统不绑死在某一种硬件上。 那么问题来了：为什么是工牌和屏，不是一台 Solara 手机？

为什么从企业切入，不做手机

来源：TechBuzz 关于微软发布 Project Solara 智能体设备操作系统的报道 techbuzz.ai

微软选择从企业场景起步，这是一个想清楚了的选择，原因有两层。

第一，企业场景的任务边界清楚，智能体最容易做对。 一个医护工牌要干的活是确定的：扫码、转写、记录、开处方。任务越具体，智能体越不容易出错，也越容易证明价值。相比之下，消费手机上的需求五花八门，智能体一旦做错代价更大。

第二，已经有真实客户排队试点。 微软列出的试点企业是实打实的零售和医疗大户：Target、CVS Health、百思买（Best Buy）、Levi's。可穿戴工牌和桌面伴侣这两台概念设备，正在这几家落地试用。有真实场景、真实客户，比发一台面向所有人的手机更稳。

微软给 Solara 的定位也点出了它的野心：把 AI 从 PC 里放出来，走进真实的工作场景——它想做的是「下一台计算设备」，而这台设备的第一站是企业的货架前和诊室里，不是你的口袋。

理解了这一步，就能把 Solara 放进一个更大的图景里看。

放进大图景：从 app 到智能体，不止微软一家

Solara 不是孤立的一步。把最近这段时间几条线放在一起，会看到同一个方向——设备正在从「app 的集合」变成「智能体的入口」：

  • 微软自己：Build 2026 上，除了 Solara，还有把 Windows 推向智能体操作系统的一系列动作，方向一致——让智能体在系统层面替你干活。
  • OpenAI：据分析师报道，正联合高通、联发科做一款「用智能体取代 app」的手机，不过仍停留在供应链阶段，量产要到更晚。
  • 国内：已经有把系统级智能体装进真实手机的尝试，让助手听一句话就跨多个服务把事办了。

这些路线形态不同——有的是工牌，有的是屏，有的是手机——但指向的是同一件事：app 这一层会越来越薄，智能体会越来越厚。 Solara 的特别之处，在于微软没有等手机，而是先用企业设备把「没有 app」这件事跑起来给你看。

收尾：先把「没有 app」做成的人，看得更远

把 Project Solara 看完，可以收成一句：它不是一台炫技的概念机，而是微软给『从 app 到智能体』这次平台跃迁铺的一个基础平台——用一枚工牌和一块屏，先在企业场景里把『没有 app、智能体直接干活』这件事做成。

「设备没有 app」听起来还有点超前，但方向已经清楚：交互正在从「你操作工具」变成「你交代任务」。微软选择从边界清晰、客户明确的企业场景起步，是一条稳的路。这一类设备离普通人还有距离，但能在今天就看懂它在铺什么的人，等智能体真的接管更多设备时，心里早有一张地图。这是一个新平台正在打开的时刻，值得我们带着好奇一路看下去。
