Grok Build 入场：五个编程 agent 趋同，竞争转向价格

把今天市面上的五个主流编程 agent 摆在一起——Claude Code、Cursor、Codex、Antigravity，再加上刚入场的 Grok Build——你会发现一件有意思的事：它们越长越像。

都是命令行，加一个交互终端界面，加把任务拆给多个并行子 agent 同时做，加接入同一套开放的 agent 通信协议。半年前大家还在争论「编程 agent 到底该长什么样」，到 2026 年 6 月，这个问题基本已经吵完了——四个定义了这个品类的产品，在过去几个月里悄悄达成了一致。

就在这个节点，xAI 带着 Grok Build 入场。它最晚到，却开了五家里最高的标价：每月 300 美元。

所以这篇文章想说清楚一件事：当五个编程 agent 的形态趋同，竞争的轴线就从「谁的形态对」彻底转向了价格和你的使用习惯。 而对天天在用 Claude Code、Cursor 的人来说，这其实是个好消息——形态趋同意味着你学会一个就会用一片，还能自由地换；价格开打，受益的是你。下面把这条新战线上的几笔账摆开看。

它们到底长得有多像

先说这个「趋同」到底趋到了什么程度。

这四个产品收敛到的，是同一套范式，主要是三件事：

  • 命令行加交互终端界面。 编程 agent 不再是某个编辑器里的侧边栏插件，而是一个能独立跑在终端里、也能无界面脚本化调用的工具。
  • 并行子 agent。 一个大任务拆成若干小任务，分给多个子 agent 同时做，再把结果汇总——Claude Code 几周前刚把这套并行拆分做成了它的研究预览功能。
  • 接入开放协议。 越来越多的产品支持同一套 agent 通信协议，意思是同一个 agent「内核」可以接到不同的外壳上，工具之间不再是各说各话的孤岛。

当三件最核心的事都一样了，产品之间的差别就从「能不能做」缩小到了「贵不贵、顺不顺手」。这就是为什么说战线变了。

Grok Build 带着什么入场

Grok Build 是 xAI 的编程 agent 加命令行工具，2026 年 5 月中旬以早期测试版的形式上线，6 月初又把一个叫 Composer 2.5 的新模型放进了它，主打长任务和复杂指令的执行。

它最值得说的，不是又一个能写代码的模型，而是一个在工程设计上真有新意的选择——用 Git 工作树来隔离并行子 agent。

这话拆开讲：Claude Code 的并行子 agent，是在同一个工作目录里一起干活的，几个 agent 改同一批文件，容易互相踩脚。Grok Build 换了个思路，让每个子 agent 在自己独立的分支里实验，跑完再合并回来。对经常让多个 agent 同时改一个工程的人，这是个实打实解决问题的设计，也是目前几家里少见的差异化。

它的接口（API）能力也摆出来了：底层模型每百万 token 输入 1 美元、输出 2 美元，上下文窗口 256K，支持文字和图片输入，出字速度每秒 100 token 以上，还提供交互终端、无界面脚本和开放协议三种用法。

参数都不差。问题在价格那一栏。

价格这条新战线

把五个编程 agent 的入门月费排出来，差距一眼就看出来了：

  • Antigravity AI Pro、Claude Code Pro、Codex（含在 ChatGPT Plus 里）： 都在每月 20 美元上下。
  • Cursor 标准席位： 每月 32 美元（按年付的单席价）。
  • Grok Build： 促销价每月 99 美元，而且这只是 6 个月的限时优惠，标价是每月 300 美元。

也就是说，前四家挤在 20 到 32 美元这一档贴身肉搏，Grok Build 直接单开了一档，标价是隔壁的将近十倍。

再看这三家的成熟度和定位差异：

  • Claude Code 是多数团队已经在用的那一个，最成熟、文档全、在持续集成里跑得稳，跑的是 Anthropic 当前的旗舰模型。
  • Codex CLI 同样是正式版、成熟，背后是 GPT-5.5，而且已经被打包进了 ChatGPT 的各档订阅里，等于「顺手就有」。
  • Grok Build 还是早期测试版，公开跑分目前也偏低，没有便宜的入门档。

把这两张图叠在一起，Grok Build 的处境就清楚了：它带来了一个真创新（工作树隔离子 agent），但要价最高、成熟度最低、还没有跑分优势。它赌的不是「现在就赢」，是 xAI 的模型迭代速度和那个差异化设计能不能在半年内把这些短板补上。

团队的账更要算清楚

个人订阅是一回事，一个十人小组一年要花多少，是另一笔更现实的账：

同样十个人，一年的账能差出一倍多：GitHub Copilot 商业版约 2280 美元，Antigravity 约 2400 美元，Cursor 团队标准版约 3840 美元，Windsurf（现已并入 Devin 桌面版）团队版约 4800 美元。

这还只是订阅的固定费用，没算上按量计费的那部分。换句话说，当形态趋同、能力咬得很紧的时候，选哪一个越来越像一道采购题——要把单价、团队规模、按量成本和锁定风险一起摆上桌，而不是看谁的功能列表更花哨。

趋同对你意味着什么

形态趋同听起来像是「大家都变得没差别了」，但站在使用者这边，它其实是利好，原因有两个。

一是学习成本被摊薄了。 当五家都是命令行加并行子 agent 加开放协议，你在 Claude Code 上练出来的那套工作习惯——怎么拆任务、怎么写指令、怎么管上下文——换到 Cursor、Codex 上基本通用。你押注的是一种工作方式，而不是某一个会被淘汰的工具。

二是你不容易被锁死。 开放的 agent 通信协议越来越普及，意味着同一个 agent 内核能接到不同外壳上。哪天某家涨价、某家掉队，你迁走的成本比过去低得多。这种「随时能换」的底气，恰恰是价格战能打起来的前提——也是你作为用户最大的筹码。

所以 Grok Build 把标价开到 300 美元这件事，真正的看点不是「它好贵」，而是它在赌自己能用差异化和迭代速度撑起这个价。而隔壁四家被迫继续在 20 到 32 美元这一档拼价格、拼顺手程度。无论哪种，账单往下走、体验往上走的，都是用户。

写在最后

五个编程 agent 长成了一个样，这不是终点，是品类成熟的标志。

当「该长什么样」这个问题吵完了，竞争就落到了两件你最能直接感受到的事上——价格，和顺不顺手。 Grok Build 用一个真有新意的工作树隔离设计，加一个偏高的价格，给这条新战线添了一个变量；而你手里那套已经练熟的 agent 工作习惯，正在变得越来越值钱、也越来越不挑工具。

挑工具的时候，与其纠结谁的功能截图更漂亮，不如把单价、团队人数、锁定风险摆到一张表上算清楚。形态都差不多的时候，算得清账的人最从容。
