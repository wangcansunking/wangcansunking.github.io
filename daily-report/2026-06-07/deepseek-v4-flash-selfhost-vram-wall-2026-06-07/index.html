DeepSeek V4 自己部署：两张 4090 塞不下，官方接口更省

DeepSeek V4-Flash 有 2840 亿参数，是一个 MoE 架构的开源旗舰，权重公开、许可证宽松，谁都能下载。于是很多人下意识地以为：开源就等于「我自己机器也能跑」。

但真把它请回家，第一道门槛不是会不会配环境，而是一道纯粹的显存账。官方推荐的 FP4+FP8 精度要 170GB 显存才装得下；就算用社区最激进、明显掉点的 INT4 量化压一压，也得 90-100GB。两张 RTX 4090 加起来 48GB，连「质量有风险」的极限 Q2 量化都塞不下。

而同一个模型，官方接口的定价是每百万 token 输入 0.14 美元、输出 0.28 美元——便宜到你很难给「自己买卡跑」算出一笔划算的账。

所以这篇文章只回答一件事：DeepSeek V4-Flash 这个国产开源旗舰，你确实能下载到本地，但你的桌子放不下它，而且绝大多数情况下也没必要自己扛。 把显存账和接口账都算清楚，你反而会更有把握——知道什么时候该调接口、什么时候该在本地换一个小模型。所有显存数字来自三份公开显存表交叉核对，接口定价是官方现价，成本测算的假设我都写在图里，你可以照着自己的用量重算。

先看它到底有多大：284B 全都要进显存

理解这道显存墙，得先搞清楚 V4-Flash 的体量。它的关键参数是这样的：

  • 总参数 2840 亿，每个 token 只激活 130 亿。 这是 MoE（混合专家）的典型结构：算的时候确实只动用 130 亿参数，所以推理的计算量和一个中型模型差不多。
  • 上下文窗口 100 万 token。 这是它的卖点之一，但长上下文意味着巨大的 KV 缓存开销，后面算显存时这是一笔不能省的账。
  • 架构上用了一套压缩注意力的混合设计，目标是在保住百万上下文能力的同时，把每个 token 的推理计算量和 KV 缓存都压下去——这也是它「计算量像中型模型」的由来。

这里有个最容易被忽略、却决定成败的点：MoE 模型「只激活 130 亿」省的是算力，不是显存。 推理时虽然每个 token 只走 130 亿参数那条路，但你永远不知道下一个 token 会被路由到哪些专家，所以全部 2840 亿参数必须同时驻留在显存里待命。换句话说，显存占用是按 2840 亿这个总数算的，不是按 130 亿。

这就是为什么一个「计算量像中型模型」的东西，显存胃口却是旗舰级的。

各精度的最低显存如下：

  • FP4+FP8 官方推荐精度：约 170-175GB。 这是 DeepSeek 自己发布的检查点精度，质量无损。需要两张 H200，或者四张 A100。
  • Q5 / Q6 量化：约 128-160GB。 质量损失小，但显存还是专业级集群的量级。
  • Q4 / INT4（社区 AWQ 方案，约掉 5% 分数）：约 90-100GB。 这是「还能接受」的最激进档，要四张 4090（96GB）勉强够，且要牺牲一部分上下文长度和精度。
  • Q3 量化：约 80-96GB。
  • Q2 极限量化（有明显质量风险）：约 48-64GB。 哪怕压到这个会掉智商的程度，最低也要 48GB 起步，刚好踩在两张 4090 的红线上甚至越过去。

结论很干脆：两张 4090（48GB）这个不少人心里的「本地旗舰配置」，跑 V4-Flash 连门都摸不到。 真要用消费级卡硬上，最现实的也是四张 4090 跑 INT4，而那已经是「接受掉点 + 砍上下文」的妥协方案了。

那到底要什么机器才跑得动

换个问法：手里有多少显存，能对 V4-Flash 做什么？另一份独立整理的显存对照表回答得很清楚：

把它翻译成大白话：

  • 24GB（一张 4090 / 一张 3090）： 适合跑小模型或当接口客户端，别想 V4-Flash。
  • 80GB（一张 H100 / 四张 4090 凑）： 「理论上可以试试 V4-Flash 的 Q2/Q3，或者大量内存换显存式的卸载」——注意用词是「理论上」「试试」。
  • 128GB： V4-Flash 的 Q4 才开始变得现实，Q5/Q6 仍然紧张。
  • 192-256GB： V4-Flash 的 FP8 才算住得舒服。

这张表里专门有一句点题的话：DeepSeek V4 和那些 26B、31B 的中型开源模型「根本不在一个本地部署的量级」——中型模型还能讨论怎么塞进 24GB、32GB 的卡，V4 这种庞大的 MoE，完整本地部署很快就进入「多卡工作站乃至服务器」的领域。

所以现实的自托管最低配置，是两张 H200，或者两张 RTX Pro 6000 Blackwell 这类专业卡。引擎层面，社区和官方都推荐用 vLLM 或 SGLang——它们支持 MoE 的专家并行和这套混合注意力的核，能把 100 万上下文的 KV 缓存管好；而 Ollama 这类工具虽然上手简单，跑这种 MoE 会丢掉专家路由的效率，只适合原型验证，不适合正经服务。

社区的共识：别自己扛这堵墙

这件事最有说服力的，不是某张显存表，而是真正天天在本地折腾模型的那群人的态度。在 r/LocalLLaMA 这类聚集了大量本地部署玩家的社区里，关于 V4 自托管的共识相当一致，而且很冷静：

  • 不要试图完整自托管 DeepSeek V4。 需要 DeepSeek V4 这一档能力的时候，就直接调官方接口。
  • 手里只有 24GB 到 48GB 显存做本地私有部署， 更实际的选择是 7B、14B、32B、70B 这些量级的量化模型——Qwen、Gemma、Llama 系列里都有称手的。
  • 一句被反复提起的话是：如果你的目标是在个人电脑上跑一个模型，DeepSeek V4 就不是那个目标。

这不是泼冷水。它其实把选择讲清楚了：本地小模型解决「随时能用、数据不出门」的需求，旗舰能力交给接口，两条路各司其职。真正会折腾的人，早就不纠结「什么都要本地跑」了。

接口便宜到什么程度：算一笔账

既然反复说「接口更省」，那就把账算到底。DeepSeek V4-Flash 的官方接口定价是这样的：

  • 输入：每百万 token 0.14 美元（命中缓存的部分只要 0.0028 美元，便宜到几乎可以忽略）。
  • 输出：每百万 token 0.28 美元。
  • 输入输出混合下来，一个粗略的均价大约是每百万 token 0.21 美元。

这是什么概念？按每天 5000 万 token 的不小用量算，一天的接口费用大约 7 到 14 美元。一个月撑死几百美元。

再看自托管这边的账。要稳定跑 V4-Flash，前面说了，得是两张 H200 这一档。把硬件按三年摊销，加上一天 24 小时不停机的电费，每月固定成本大约 1800 美元——而且这是机器闲着也要付的钱。我把两条成本线随用量的变化画在一起，交点很说明问题：

按每百万 token 0.21 美元的接口均价、每月 1800 美元的自托管固定成本算，两者的持平点在每月约 86 亿 token——折算下来要每天近 3 亿 token、而且机器接近满载连轴转，自托管才开始比接口便宜。

这个量级是什么水平？一个中等规模的产品，全天候不间断地把这台机器喂满，才勉强够到这条线。绝大多数个人、团队、甚至不少创业公司，全月的真实用量离这个数字都差着一两个数量级。也就是说，对大多数人，接口不只是省心，是实打实地省钱。

那什么时候才轮到自托管

把账算这么清楚，不是说自托管一无是处。它在三种情况下依然成立，而且这三条要同时满足才划算。下面这份自托管指南给的三个理由，和这里要说的几乎一一对应——它也直接点明，每天用量没到几十亿 token，接口几乎总是更省：

具体到三条：

条件 说明
数据绝对不能出门 合规、隐私或行业监管要求推理全程留在自己机房，这时候省不省钱是次要的
已经有多卡专业机器 手里本来就有 H200 / A100 这类卡在做别的事，边际成本才低
用量大且稳定满载 接近前面算的每天数亿 token、且常年跑满，摊销才摊得动

只要这三条缺一条，结论基本都会倒向接口。尤其是「我想买两张 4090 在家自己跑 V4」这个最常见的念头——从显存这关就已经走不通了。

值得一提的是，这并不是 DeepSeek 做得不好，恰恰相反。一个国内团队能把 284B 的开源旗舰做出来、还把接口价格压到每百万 token 一毛多美元，本身是件了不起的事。它真正打破的，是「开源 = 人人都能自托管」这个想当然的等式：开源给了你下载和审视权重的自由，但能不能跑在你自己的机器上，是另一道由显存决定的物理题。

写在最后：把卡用在该用的地方

回到开头那句话——DeepSeek V4-Flash 你能下载，但你的桌子放不下它。

这件事其实让人踏实：你不用纠结要不要为了一个 284B 的模型去攒一台多卡工作站。更聪明的配置是两条腿走路——需要顶尖能力的时候调官方接口，按量付费、便宜到离谱；需要数据不出门、随时离线可用的时候，在本地跑 7B 到 70B 的量化小模型，一张 4090 甚至一台 16GB 的笔记本就够。各自的账都算得清，各自的活也都干得漂亮。

显存墙不是坏消息，它只是帮你把「什么该本地、什么该上接口」这条线划清楚了。把这条线划清楚的人，手里的每一张卡都用在了刀刃上。
